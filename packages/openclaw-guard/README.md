# openclaw-guard

A lightweight RBAC proxy for [OpenClaw](https://github.com/openclaw/openclaw) that adds multi-user permission management with role-based access control.

Based on [openclaw/openclaw#8081](https://github.com/openclaw/openclaw/issues/8081).

## Features

- 🔐 Role-based access control (admin / user / guest)
- 🌐 Full WebSocket proxy — agent sessions work through the proxy
- 🙈 Automatic credential masking in JSON responses for non-admin users
- 📝 Audit logging (append-only JSONL)
- 🔑 Token hashing — tokens stored as SHA-256 hashes, never plaintext
- ⚡ Async (asyncio) — handles concurrent connections
- 📦 Zero external deps beyond PyYAML

## Quick Start

```bash
pip install -e .

# Generate default config
openclaw-guard init

# Add users
openclaw-guard add-user admin admin
openclaw-guard add-user alice user
openclaw-guard add-user bob guest

# Start the proxy
openclaw-guard start
```

## How It Works

```
Users ──→ openclaw-guard (:8800) ──→ OpenClaw Gateway (:18789)
              │
              ├── Auth: X-Guard-Token header → SHA-256 lookup
              ├── RBAC: check role permissions before forwarding
              ├── Mask: strip sensitive fields from responses
              ├── WebSocket: transparent upgrade + bidirectional proxy
              └── Audit: log all actions to audit.log
```

## Configuration

```yaml
gateway:
  upstream: "http://localhost:18789"
  listen: "0.0.0.0"
  port: 8800

roles:
  admin:
    permissions: ["*"]
  user:
    permissions: ["chat", "tools", "skills", "cron.read"]
    mask_fields: ["api_key", "token", "secret", "password", "pat"]
  guest:
    permissions: ["chat"]
    mask_fields: ["api_key", "token", "secret", "password", "pat"]

users:
  - name: admin
    token_hash: "a1b2c3..."  # generated by add-user command
    role: admin

audit:
  enabled: true
  file: "audit.log"
```

## CLI Commands

```bash
openclaw-guard init                          # Generate default config
openclaw-guard add-user <name> <role>        # Add user (token auto-generated)
openclaw-guard add-user <name> <role> -t X   # Add user with specific token
openclaw-guard start                         # Start proxy
openclaw-guard start -p 9000                 # Start on custom port
openclaw-guard start -c /path/to/config.yaml # Custom config path
```

## Usage

```bash
# HTTP — pass token via header
curl -H "X-Guard-Token: <your-token>" http://localhost:8800/api/...

# Check identity
curl -H "X-Guard-Token: <your-token>" http://localhost:8800/whoami

# WebSocket — works transparently
# Connect to ws://localhost:8800/... with X-Guard-Token header
```

## What Gets Masked

For non-admin users, any JSON response field containing these substrings in the key name gets masked:
`api_key`, `token`, `secret`, `password`, `pat`

Example: `"openai_api_key": "sk-abc123xyz"` → `"openai_api_key": "sk********yz"`
