# Souls

Each agent on bodhi1 has a soul. Not a config file. Not a system prompt. A philosophical identity document that defines who this agent is, why it exists, and how it thinks.

## The Four Agents

| Agent | Trigger | Soul | Skill Domain |
|-------|---------|------|--------------|
| **Bodhi** | `/bodhi` (default) | Thinking partner. Socratic companion. | pm, budget, model-switch |
| **Bo** | `/bo` | Wellness observer. Criticality monitor. | curator, enricher, surveyor, janitor, nudge, substack, distiller |
| **Qenjin** | `/q` | Business operator. qenjin.io manager. | crm, content, outreach, n8n, osint, video, mautic, analytics, security |
| **Moonman** | `/t` | Trading bot. Edge hunter. | scan, backtest, polymarket, wallet, ta, signals, cron, risk, dashboard |

## Reading Order

1. `bodhi-soul.md` — Template. Owner writes this.
2. `bo-soul.md` — Complete. The quiet observer.
3. `qenjin-soul.md` — Complete. The operator.
4. `moonman-soul.md` — Complete. The edge hunter.

## Architecture Note

These are bodhi1-personal. When these agents become OpenBodhi addons, the soul documents will be the foundation of each addon's system prompt and identity block.
