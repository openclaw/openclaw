## 四、P0 缺口详细分析

### P0-1: fin-market-data → `src/market-data/tools.ts`

**问题**: 用户无法通过 AI 查询实时价格、orderbook、市场概览。datahub 覆盖了 OHLCV/ticker，但缺 orderbook + exchange-level overview。

**4 个工具**:

- `fin_market_price` — 实时价格 + OHLCV K线（部分被 datahub 替代）
- `fin_market_overview` — 交易所 top movers/volume leaders
- `fin_orderbook` — 盘口深度（datahub 不提供）
- `fin_ticker_info` — 24h 详细 ticker

**迁移方案**: 从 git 恢复 333 LOC → `src/market-data/tools.ts`，改用 trader-plugin 内部的 ExchangeRegistry 引用（不再跨 service 查找）。

### P0-2: fin-portfolio → `src/portfolio/tools.ts`

**问题**: AI 无法回答"我的总资产是多少"、"所有交易所的持仓"。这是用户最基本的交互需求。

**5 个工具**:

- `fin_portfolio_view` — 跨交易所余额聚合 + 总资产
- `fin_portfolio_history` — 组合历史价值快照
- `fin_exchange_balance` — 单交易所余额详情
- `fin_positions` — 跨交易所实时持仓
- `fin_order_history` — 历史订单查询

**迁移方案**: 从 git 恢复 507 LOC → `src/portfolio/tools.ts`。可复用 `gatherLiveTradingData()` 中已有的余额/持仓聚合逻辑。

### P0-3: fin-monitoring → `src/monitoring/`

**问题**: dashboard 的 `routes-alerts.ts` 有 3 个 HTTP 端点 + `data-gathering.ts` 有 3 处调用 `fin-alert-engine`，但 AlertEngine 实体不存在 → 功能空转。

**4 个工具 + 2 个 services**:

- `fin_set_alert` — 创建价格/P&L/技术指标告警
- `fin_list_alerts` — 列出所有告警
- `fin_remove_alert` — 删除告警
- `fin_monitor_run_checks` — 手动触发告警检查
- `fin-alert-engine` service — AlertEngine + AlertStore (SQLite)
- `fin-monitoring-scheduler` service — 后台定时轮询

**迁移方案**: 从 git 恢复 559 LOC → `src/monitoring/alert-engine.ts` + `alert-store.ts` + `tools.ts`。在 index.ts 注册 2 个 services，修复 dashboard 空转。
