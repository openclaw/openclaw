## OpenClaw 愿景

OpenClaw 是真正能做事的 AI。
它在您的设备上运行，在您的频道中运行，遵循您的规则。

本文档解释了项目的当前状态和方向。
我们仍处于早期阶段，因此迭代速度很快。
项目概述和开发者文档：[`README.md`](README.md)
贡献指南：[`CONTRIBUTING.md`](CONTRIBUTING.md)

OpenClaw 最初是一个个人 playground，用于学习 AI 并构建真正有用的东西：
一个可以在真实计算机上运行真实任务的助手。
它通过几个名称和外壳演变：Warelay -> Clawdbot -> Moltbot -> OpenClaw。

目标：一个易于使用、支持广泛平台、尊重隐私和安全的个人助手。

当前的重点是：

优先级：

- 安全和安全默认值
- 错误修复和稳定性
- 设置可靠性和首次运行用户体验

下一个优先级：

- 支持所有主要模型提供商
- 改进对主要消息频道的支持（并添加一些高需求的频道）
- 性能和测试基础设施
- 更好的计算机使用和代理 harness 能力
- CLI 和 web 前端的人体工程学
- 在 macOS、iOS、Android、Windows 和 Linux 上的配套应用程序

贡献规则：

- 一个 PR = 一个问题/主题。不要捆绑多个无关的修复/功能。
- 超过约 5,000 行更改的 PR 仅在特殊情况下才会被审查。
- 不要一次打开大量小 PR；每个 PR 都有审查成本。
- 对于非常小的相关修复，鼓励将它们分组到一个集中的 PR 中。

## 安全

OpenClaw 中的安全是一个深思熟虑的权衡：强大的默认值而不牺牲能力。
目标是在保持实际工作强大的同时，使风险路径明确且由操作员控制。

规范的安全政策和报告：

- [`SECURITY.md`](SECURITY.md)

我们优先考虑安全默认值，但也为受信任的高功率工作流程提供清晰的旋钮。

## 插件和记忆

OpenClaw 有一个广泛的插件 API。
核心保持精简；可选功能通常应该作为插件发布。

首选插件路径是 npm 包分发加上用于开发的本地扩展加载。
如果您构建插件，请在自己的存储库中托管和维护它。
将可选插件添加到核心的标准故意很高。
插件文档：[`docs/tools/plugin.md`](docs/tools/plugin.md)
社区插件列表 + PR 标准：https://docs.openclaw.ai/plugins/community

记忆是一个特殊的插件槽，一次只能激活一个记忆插件。
今天我们提供多个记忆选项；随着时间的推移，我们计划收敛到一个推荐的默认路径。

### 技能

我们仍然提供一些捆绑技能以获得基线用户体验。
新技能应该首先发布到 ClawHub (`clawhub.ai`)，而不是默认添加到核心。
核心技能添加应该很少见，并且需要强有力的产品或安全理由。

### MCP 支持

OpenClaw 通过 `mcporter` 支持 MCP：https://github.com/steipete/mcporter

这保持 MCP 集成灵活且与核心运行时解耦：

- 添加或更改 MCP 服务器而无需重启网关
- 保持核心工具/上下文表面精简
- 减少 MCP 变动对核心稳定性和安全性的影响

目前，我们更喜欢这种桥接模型，而不是在核心中构建一流的 MCP 运行时。
如果有 `mcporter` 尚未支持的 MCP 服务器或功能，请在那里打开一个问题。

### 设置

OpenClaw 目前在设计上以终端为先。
这使设置明确：用户可以看到文档、身份验证、权限和安全状态。

长期来看，我们希望随着加固成熟，提供更轻松的入职流程。
我们不希望方便的包装器隐藏用户的关键安全决策。

### 为什么选择 TypeScript？

OpenClaw 主要是一个编排系统：提示、工具、协议和集成。
选择 TypeScript 是为了保持 OpenClaw 默认可黑客攻击。
它广为人知，迭代速度快，易于阅读、修改和扩展。

## 我们不会合并的内容（暂时）

- 当它们可以在 ClawHub 上生活时的新核心技能
- 所有文档的完整文档翻译集（推迟；我们计划稍后进行 AI 生成的翻译）
- 不符合模型提供商类别的商业服务集成
- 围绕已经支持的频道的包装器频道，没有明确的能力或安全差距
- 当 `mcporter` 已经提供集成路径时，核心中的一流 MCP 运行时
- 作为默认架构的代理层次结构框架（管理器的管理器 / 嵌套规划器树）
- 复制现有代理和工具基础设施的重型编排层

这个列表是路线图的护栏，不是物理定律。
强烈的用户需求和强有力的技术理由可以改变它。
