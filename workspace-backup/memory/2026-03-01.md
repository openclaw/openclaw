### 经验教训

**1. Polymarket Proxy Wallet**
- 用户钱包是通过 Polymarket.com 创建的代理钱包
- 需要使用 `signature_type=1`（POLY_PROXY）
- 不能使用 `signature_type=0`（EOA）

**2. API Key 权限**
- 用户提供的 API Key 可能只有只读权限
- 可以通过私钥派生具有完整权限的 API Key
- 派生方法：`client.create_or_derive_api_creds()`

**3. 订单价格策略**
- 固定价格（90%）可能低于市场价
- 应该实时获取订单簿，按市场价挂单
- 或者使用略低于最低卖价的价格

**4. 最小交易量**
- Polymarket 最小订单金额：5 USDC
- 小于 5 USDC 的订单会被拒绝
