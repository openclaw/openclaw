## 📋 World Monitor 核心架构

### 1. 分层推理架构

```
用户请求
    ↓
【本地 LLM 优先】Ollama (免费)
    ↓ 失败
【云端 LLM】Groq (快)
    ↓ 失败
【备用 LLM】OpenRouter (多模型)
    ↓ 失败
【浏览器推理】Transformers.js (T5)
```

**启发**：
- 多层容错机制，确保稳定性
- 成本优先级：本地免费 → 云端付费
- 性能优先级：Groq > OpenRouter > 浏览器

---

### 2. 多源数据聚合

**数据源**：
- 150+ RSS feeds（地缘政治、防御、能源、科技、金融）
- 18+ 实时视频流
- 22 个 live webcams
- 40+ 数据层（冲突、军事基地、核设施、海底电缆）

**应用场景**：
- 实时事件检测
- 地理位置映射
- 时间基线异常检测

---

### 3. AI 驱动分析

**本地 Embeddings**：
- 模型：all-MiniLM-L6-v2 (ONNX)
- 用途：向量相似度计算
- 优势：无需 API 密钥，数据不离开本地

**向量搜索**：
- 算法：余弦相似度
- 用途：事件相关性匹配

**LLM 分析**：
- 本地优先 → 云端备用
- 用途：事件分类、摘要生成

---

### 4. 智能缓存与去重

**缓存策略**：
- Redis 缓存
- 15 分钟聚合缓存
- 避免重复分析

**去重算法**：
- Jaccard 相似度 > 0.6 → 重复
- 基于内容相似度判断

---

### 5. 信号融合机制

**多事件聚合**：
- 相同主题的事件聚合
- 计算聚合置信度
- 地理区域汇聚

**异常检测**：
- 时间基线对比
- 突发事件识别

## 🎯 可借鉴到 Polymarket 狙击系统

### 1. 多源数据聚合

**当前问题**：
- 事件源单一（仅硬编码伊朗事件）
- 无法实时获取新闻

**优化方案**：
```
事件源
    ↓
【免费 RSS】→ Google News, CryptoPanic (RSS)
    ↓
【免费 API】→ NewsAPI (免费额度), Hacker News
    ↓
【付费 API】→ Brave Search, CryptoPanic (Pro)
    ↓
【Twitter/X】→ 官方账号动态（需绕过反爬）
    ↓
数据聚合层
```

**实施步骤**：
1. 整合免费 RSS feeds（优先）
2. 接入 NewsAPI（100 次/天免费）
3. 评估 Brave API 成本（$0.0018/次）

---

### 2. 向量相似度匹配

**当前问题**：
- 关键词匹配粗糙（"伊朗" → 33% 相关性）
- 无法理解语义相似度

**优化方案**：
- 本地 ONNX Embeddings（all-MiniLM-L6-v2）
- 计算事件描述与市场标题的向量相似度
- 替代关键词匹配

**优势**：
- 精度更高（语义理解）
- 零成本（本地推理）
- 数据不离开本地

---

### 3. 多事件聚合置信度

**当前问题**：
- 单事件决策，置信度不稳定
- 无法综合判断

**优化方案**：
```
同一主题的多个事件
    ↓
聚合分析
    ├─ 事件数量加权
    ├─ 时间衰减（新事件权重高）
    ├─ 来源可信度加权
    └─ 向量相似度聚合
    ↓
综合置信度（0-100%）
```

**示例**：
- 事件1：伊朗海军霍尔木兹活动（来源 A，权重 0.8）
- 事件2：美军增兵中东（来源 B，权重 0.9）
- 事件3：伊朗发布强硬声明（来源 C，权重 0.7）

聚合置信度 = (0.8 + 0.9 + 0.7) / 3 × 向量相似度因子

---

### 4. 分层推理架构

**当前问题**：
- 依赖单一 LLM（GLM-4.7）
- 速率限制影响运行

**优化方案**：
```
事件分析
    ↓
【本地 LLM】Qwen3.5-27B (免费)
    ↓ 失败/超时
【云端 LLM】GLM-5 (主要)
    ↓ 失败/速率限制
【云端 LLM】GLM-4.7 (备用)
```

**优势**：
- 本地优先，降低成本
- 多层容错，提高稳定性
- 避免单点依赖

---

### 5. 智能缓存与去重

**当前问题**：
- 每次都重新分析相同事件
- 浪费 token

**优化方案**：
- Redis 缓存事件分析结果（15分钟）
- Jaccard 相似度去重（>0.6 视为重复）
- 避免重复分析
