# LLM 模型配置优化

## 当前状态

- **默认模型**：glm-4.5-air（已修复）
- **问题**：GLM-4.7-flash 返回推理文本，置信度过低
- **速率限制**：429 错误（需要添加缓存）

## 推荐方案

### 方案 1：GLM-4.5-air（推荐）

**优势**：
- ✅ 标准 JSON 输出
- ✅ 速度快（2-5 秒）
- ✅ 成本适中（~0.007 元/次）
- ✅ 准确性高

**配置**：
```bash
export GLM_MODEL=glm-4.5-air
```

### 方案 2：Claude 3.5 Sonnet（最准确）

**优势**：
- ✅ 准确性最高（90%+）
- ✅ 原生 JSON 输出
- ✅ 推理能力强

**成本**：~0.07 元/次

**配置**：
```bash
export LLM_PROVIDER=claude
export ANTHROPIC_API_KEY=sk-xxx
```

### 方案 3：DeepSeek-V3（最便宜）

**优势**：
- ✅ 极低成本（~0.0007 元/次）
- ✅ 中文友好
- ✅ 速度快

### 方案 4：多模型轮换（推荐）

**策略**：
- 50% 流量：glm-4.5-air
- 30% 流量：deepseek-v3
- 20% 流量：claude-3.5-sonnet（高价值事件）
