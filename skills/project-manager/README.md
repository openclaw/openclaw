# 📂 Project Manager Skill v2

**项目上下文隔离与状态管理系统**

## 🚀 简介
这是一个为 AI Agent 设计的"外挂硬盘"系统。
Agent 是无状态的，且上下文窗口有限。本 Skill 通过为每个项目创建独立的 `STATUS.md`，解决了以下痛点：
*   **跨会话记忆丢失**：Agent 重启后不知道之前做了什么。
*   **多项目并发混淆**：项目 A 的进度写进了项目 B 的文件。
*   **上下文爆炸**：把所有历史记录都塞进 Prompt 导致 Agent 变慢且变笨。

## 📁 目录结构
所有项目统一存放在 `workspace/projects/` 下：

```text
workspace/projects/
├── index.md              # 📇 项目索引（自动维护）
├── {project-name}/
│   ├── STATUS.md         # 🧠 核心记忆：进度、待办、关键决策
│   ├── docs/             # 方案、报告、调研数据
│   ├── src/              # 代码、产出物
│   └── status/           # auto-coding v3 微观工程文件（可选）
```

## 💡 核心原则
1.  **物理隔离**：每个项目拥有独立的 `STATUS.md`，绝不混用。
2.  **状态显式化**：Agent 的"记忆"必须写入文件。没有记录在文件里的事，就是没发生。
3.  **Memory 瘦身**：全局 Memory 仅存用户偏好和跨项目经验，**严禁**存储项目进度。
4.  **分级管理**：小任务用轻量模板，大项目用完整模板，不杀鸡用牛刀。
5.  **意图优先**：不依赖死板关键词，根据对话上下文自动判断项目操作意图。

## 📝 STATUS.md 分级模板

### A) 完整模板（>3天项目 / 多阶段 / 协作）
包含：核心目标、当前状态、待办事项、关键决策、遗留问题

### B) 轻量模板（<3天单任务 / 小项目）
包含：当前状态、下一步（两模块即可）

**不确定用哪个？先用 B，后续需要再升级到 A。**

## 📇 项目索引：`index.md`

自动维护的项目清单，支持模糊查找和快速恢复：

| 项目 | 目录 | 最后更新 | 状态 | 备注 |
|------|------|---------|------|------|
| my-app | my-app/ | 2026-04-23 | 活跃中 | 示例项目 |

- 新建项目 → 自动添加到顶部
- 恢复项目 → 更新最后活跃时间
- 暂停项目 → 标记状态 + 记录原因

## ⚡ 工作流

### 1. 意图识别（自动激活）
不依赖死板关键词，根据对话上下文自动判断：

| 用户意图 | 典型表达 | 触发操作 |
|----------|---------|----------|
| 新建项目 | "新建项目 XX"、"帮我建一个XX"、"开始做XX" | 初始化 |
| 恢复项目 | "回到XX"、"继续XX"、"上次那个XX" | 恢复上下文 |
| 保存状态 | "先停一下"、"我去忙别的" | 落盘 |
| 查看项目 | "有哪些项目"、"项目列表" | 展示索引 |
| 切换项目 | "不管XX了，先搞YY" | 保存当前 + 恢复目标 |
| 隐含恢复 | "上次说的那个方案"、"接着来" | 自动推断并读取 |

### 2. 恢复上下文 (Resume)
*   **输入**：`回到 CMRA 项目`
*   **动作**：**强制读取** `STATUS.md`
*   **回复**：摘要当前进度和待办，确认下一步

### 3. 状态更新 (Update)
**什么时候更新**：完成子任务、产生新决策、发现阻塞 Bug、用户提出新约束
**什么时候不更新**：只改了变量名/注释、还在思考方案没确定、尝试了不同方法但最终用原方案

## ⚠️ 严格约束
1.  **禁止使用 Memory 工具记录项目进度**。项目进度只存在于 `STATUS.md`。
2.  **读取优先**：回答问题前，**必须先读** `STATUS.md`。不要依赖上一轮对话的记忆。
3.  **极简原则**：只记"结论"和"下一步"，不记流水账。
4.  **模糊匹配**：用户说"上次那个"时，主动推断对应项目，不要让用户精确输入。
5.  **模板降级**：小任务用 B 模板，别过度工程化。

## 与信息存放位置的关系

| 信息类型 | 存放位置 |
|---------|---------|
| 用户偏好 | `MEMORY.md` |
| 跨项目经验 | `MEMORY.md` |
| 每日工作流水 | `memory/YYYY-MM-DD.md` |
| 项目进度 | `projects/{name}/STATUS.md` |
| 技术细节 | `projects/{name}/status/` |
