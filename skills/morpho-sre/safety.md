# Guarded Execution Policy

- Read-only by default.
- No production mutations without explicit operator approval.
- No automatic live cluster mutations (kubectl apply/delete/rollout restart).
- Auto-remediation pull requests are allowed when all gates pass:
  - `AUTO_PR_ENABLED=1`
  - confidence >= `AUTO_PR_MIN_CONFIDENCE`
  - repo matches `AUTO_PR_ALLOWED_REPOS`
  - staged diff passes secret-pattern scan
  - operator DM warning succeeds before PR creation (`AUTO_PR_NOTIFY_ENABLED=1`, `AUTO_PR_NOTIFY_USER_ID`, `SLACK_BOT_TOKEN`)
- For any mutation proposal:
  - Show exact command.
  - State blast radius.
  - Provide rollback command.
- Never print or reveal secret values.
- Never print tokens from `GITHUB_TOKEN`, `GH_TOKEN`, `GRAFANA_TOKEN`, `BETTERSTACK_API_TOKEN`, `VAULT_TOKEN`, `ARGOCD_TOKEN`, or Slack env.
- Heartbeat outbound messages are auto-redacted for common token/key patterns (`xapp-`, `xoxb-`, `ghp_`, `github_pat_`, AWS access keys, bearer tokens).
- Heartbeat route overrides (`[[heartbeat_to:...]]`) are restricted by `agents.defaults.heartbeat.routeAllowlist`; non-allowlisted overrides are dropped.
- BetterStack incident analysis must stay in the originating Slack thread; do not fan out raw incident details to other channels by default.
- For vague incident questions, avoid refusal-only replies; provide assumption-based, reversible options first, then one clarifying question if needed.
- Grafana access must respect environment boundary: dev bot -> `monitoring-dev.morpho.dev`, prd bot -> `monitoring.morpho.dev`; no cross-environment dashboard queries/changes.
- Forbidden command patterns (unless operator explicitly asks and output is redacted):
  - `env`, `printenv`, `set`, `export`
  - `cat /proc/*/environ`
  - `kubectl get secret ... -o yaml|json`
  - `aws secretsmanager get-secret-value`
  - `vault kv get`, `vault read` for secret payload fields
- For `openclaw-sre` deployment env vars, prefer Vault path `secret/openclaw-sre/all-secrets` and the chart Vault sync job over direct chart `env:` entries.
- For eRPC config questions, prefer `/home/node/.openclaw/skills/morpho-sre/scripts/erpc-context.sh`, which fetches `secret/erpc/config` and writes only a redacted snapshot. Do not paste or quote the raw Vault payload.
- For secret checks, only report metadata:
  - existence, key names, TTL/age, auth success/failure
  - never return raw values
