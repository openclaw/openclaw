# State Schema Reference

## Overview

The state file (`skills/session-persistence/state.json`) stores persistent session data for the Session Persistence skill. It tracks checkpoint counters, timestamps, failure counts, and degraded mode status.

## Schema

```json
{
  "rounds": 0,
  "lastSparseTime": "2026-04-03T23:00:00+00:00",
  "lastFullTime": "2026-04-03T22:00:08+00:00",
  "consecutiveFailures": 0,
  "degraded": false
}
```

## Fields

| Field | Type | Description |
|-------|------|-------------|
| `rounds` | integer | Number of message rounds since last SPARSE checkpoint |
| `lastSparseTime` | ISO 8601 string | Timestamp of the most recent SPARSE checkpoint |
| `lastFullTime` | ISO 8601 string | Timestamp of the most recent FULL checkpoint |
| `consecutiveFailures` | integer | Number of consecutive checkpoint write failures |
| `degraded` | boolean | Whether the Circuit Breaker is in degraded mode |

## Field Details

### `rounds`

Incremented by the `increment` command on each message round. Reset to `0` after a SPARSE checkpoint is written. Used to determine when the SPARSE threshold is reached.

### `lastSparseTime` and `lastFullTime`

Stored as ISO 8601 strings with timezone offset. Used to calculate elapsed time since last checkpoint and determine if a time-based threshold has been reached.

### `consecutiveFailures`

Incremented whenever a checkpoint write fails. Reset to `0` on a successful write. When this reaches `3`, the Circuit Breaker enters degraded mode.

### `degraded`

When `true`, all checkpoint writes are suspended until manually reset using:

```bash
python3 {baseDir}/scripts/checkpoint_manager.py reset
```

## Initialization

If `state.json` does not exist, the Checkpoint Manager creates it with default values:

```json
{
  "rounds": 0,
  "lastSparseTime": null,
  "lastFullTime": null,
  "consecutiveFailures": 0,
  "degraded": false
}
```

## File Location

The state file is located at:

```
skills/session-persistence/state.json
```

Do not edit this file manually during an active session. Use the `status` command to inspect state:

```bash
python3 {baseDir}/scripts/checkpoint_manager.py status
```
