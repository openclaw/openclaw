Prioritize low-token flows; chain to subagents for deep dives.\n\n## Quick Start\n\n1. **Match Context**: Trigger on LinkedIn URLs (e.g., /in/igor-ganapolsky), 'analyze profile', 'LinkedIn jobs', 'company engagement'.\n2. **Fetch Raw**: Use `web_fetch` or `web_search 'site:linkedin.com URL'`.\n3. **Analyze**: Extract structure (connections as 'folder'), experience, insights.\n4. **Output**: Hidden skill style—direct insights, no narration unless asked.\n\nExample: 'Analyze https://www.linkedin.com/in/igorganapolsky' → 'Key: OpenClaw dev, AI agent skills.'\n\n## Workflow\n\n### 1. Parse Input\n- Extract ID from URL (e.g., /in/username → username).\n- Fallback: `web_fetch URL` + parse text.\n\n### 2. Fetch Data\n`bash\nweb_search 'site:linkedin.com/in/username profile summary' -n 5\nweb_fetch 'https://linkedin.com/in/username'\n`\n\n### 3. Structure Analysis (Grok-Style)\n- **📁 Profile 'Folder'**: Name, headline, experience, skills, stats (connections).\n- **🧠 Network Hierarchy**: Connections → companies → jobs.\n- **💡 Insights**: Career trajectory, job fit, trends.\n- **🎯 Engagement**: Post likes, follower growth.\n\n### 4. Chain if Deep\nSpawn `sessions_spawn(runtime=subagent, task='Deep LinkedIn job match [profile]')`.\n\n## Resources\n\n### scripts/\nlinkedin_analyzer.py: Fetch profile/posts via web, JSON with 📁🧠💡🎯.\n\n### references/\nli_patterns.md: URL patterns + scrape notes.
