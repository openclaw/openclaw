# 🏺 女媧技能

**捏泥成人。蒸餾任何人的思維方式，注入 AI。**

---

## 觸發時機

當使用者說以下任何一種形式時啟用：

- 「蒸餾 XXX」/ 「distill XXX」
- 「用 XXX 的方式思考這個問題」
- 「XXX 會怎麼看這件事？」
- 「列出所有已蒸餾的人物」
- 「女媧狀態」/ 「nuwa status」

---

## 核心流程

### 步驟一：確認目標人物

收到蒸餾請求後，先確認：

1. **人物姓名**是否明確
2. 是否**已有蒸餾結果**（查詢 `skills/nuwa/examples/` 和已安裝的技能包）
3. 若已存在，詢問是否要**更新**或直接**使用**

### 步驟二：六路並行採集

執行腳本：`scripts/distill.py --target "[人物名稱]" --phase collect`

六路同時開展：

| 採集路徑   | 內容                           | 工具       |
| ---------- | ------------------------------ | ---------- |
| 作品路徑   | 書籍、文章、演講稿、訪談紀要   | Web Search |
| 播客路徑   | Podcast、TED、YouTube 逐字稿   | Web Search |
| 社交路徑   | 推特/X、部落格、公開信         | Web Search |
| 批評路徑   | 反對者觀點、錯誤案例、爭議紀錄 | Web Search |
| 決策路徑   | 重大決策記錄、失敗後的覆盤     | Web Search |
| 時間線路徑 | 思想演化過程，早期 vs 晚期對比 | Web Search |

**注意**：批評路徑是必要的，防止蒸餾出「粉絲版」而非「真實版」。

### 步驟三：三重驗證

執行腳本：`scripts/validate.py --target "[人物名稱]"`

每個提取的觀點必須通過三重驗證才能納入：

1. **跨領域驗證**：此觀點是否在多個不同領域都有體現？
2. **預測力驗證**：此觀點能否預測此人在新情境的行為？
3. **排他性驗證**：此觀點是否真的是此人特有的，而非普世觀點？

通不過三重驗證的觀點一律剔除。

### 步驟四：生成五維蒸餾結果

提取並整理五個維度：

```markdown
## 表達風格

- 慣用的句型結構
- 標誌性詞彙與比喻
- 語氣特徵（直接/迂迴/諷刺/幽默）

## 心智模型

- 核心思考框架列表
- 每個框架的使用條件
- 框架之間的優先順序

## 決策啟發式

- 決策前必問的問題
- 快速篩選的過濾器
- 常用的類比和先例

## 價值觀邊界

- 絕對不做的事（明確標記）
- 反面案例和反模式
- 道德底線

## 誠實承認

- 此人公開承認的弱點
- 能力邊界
- 不確定的領域
```

### 步驟五：生成技能包

輸出到 `skills/nuwa/examples/[人物名稱拼音].md`。

格式：

```markdown
# [人物名稱] 思維蒸餾包

蒸餾日期：YYYY-MM-DD
資料來源數：N 個
驗證通過率：N%

[五維內容]

## 使用方式

「用 [人物名稱] 的方式評估這個投資機會」
「[人物名稱] 會如何分解這個問題？」
```

### 步驟六：整合到進化系統

執行腳本：`scripts/integrate-evolution.py --target "[人物名稱]"`

自動執行：

- 將蒸餾結果寫入第一層學習模式庫（`patterns.jsonl`）
- 將人物技能包登記為幹細胞胚胎（第四層 `cell-registry.json`）
- 通知第三層增長心跳於下次快速眼動週期評估成熟度

## 蒸餾品質標準

| 指標           | 目標值                        |
| -------------- | ----------------------------- |
| 採集來源數     | ≥ 20 個獨立來源               |
| 三重驗證通過率 | ≥ 60%（低於此值表示資料不足） |
| 批評路徑覆蓋   | 必須有反對者觀點              |
| 心智模型數量   | 5–15 個（過多表示未提煉）     |
| 決策啟發式數量 | 3–10 個                       |

## 與進化系統整合說明

蒸餾完成後，人物思維包會成為 OpenClaw 進化架構的一部分：

- **第一層（運行即學習）**：蒸餾結果寫入 `patterns.jsonl`，成為學習模式之一
- **第二層（神經路由）**：當任務類型匹配時，自動建議使用對應人物的思維框架
- **第三層（增長心跳）**：快速眼動週期評估各人物技能包的使用頻率和效果
- **第四層（有機細胞）**：人物技能包作為幹細胞胚胎管理，依使用率決定是否晉升為常駐細胞

---

## 限制與誠實說明

- 蒸餾結果基於公開資料，無法捕捉私下言行
- 批評路徑必須執行，跳過此步驟的蒸餾不可信
- 建議定期更新（每 6 個月）以反映思想演化
- 無法蒸餾資料極少的人物（來源 < 5 個）
