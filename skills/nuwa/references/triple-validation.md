---

## 一、為什麼需要三重驗證

蒸餾過程中，候選觀點可能來自以下問題來源：

| 問題來源     | 症狀                 | 對應驗證              |
| ------------ | -------------------- | --------------------- |
| 粉絲濾鏡     | 只收正面資料，無批評 | 批評路徑 + 排他性驗證 |
| 普世觀點混入 | 任何人都說得出的道理 | 排他性驗證            |
| 領域特定知識 | 只在一個領域適用     | 跨域驗證              |
| 觀點不精確   | 無法預測新情境       | 預測力驗證            |

三重驗證是**質量守門人**，不通過的觀點一律剔除。

---

## 二、驗證一：跨域驗證

### 定義

此候選觀點，是否在此人的**至少兩個不同領域**中都有明確體現？

### 判斷流程

```
候選觀點
    │
    ▼
找到第一個領域的證據（A 領域）
    │
    ▼
在 B 領域尋找同樣的結構
    │
    ├─── 找到 → ✅ 通過跨域驗證
    │
    └─── 未找到 → ❓ 可能是領域特定知識
                        │
                        ▼
                   嘗試第三個領域
                        │
                   ├─── 找到 → ✅ 通過
                   └─── 未找到 → ❌ 失敗，剔除
```

### 常見失敗案例

**案例**：「巴菲特說只買低估值的股票」  
**問題**：這只是投資領域的描述，無法在其他領域找到對應框架  
**判斷**：❌ 剔除（這是具體策略，不是跨域思維框架）

**案例**：「芒格說要從失敗中學習」  
**問題**：他在投資覆盤說過，在個人發展也說過，但這是普世觀點  
**判斷**：❌ 剔除（進入排他性驗證後會失敗）

**成功案例**：「芒格的逆向思考」  
**投資領域**：先想「哪些情況會讓我輸光」  
**管理領域**：先設計「哪些行為會讓公司失敗」的清單  
**個人生活**：先想「什麼習慣會讓人生毀掉」  
**判斷**：✅ 通過（三個領域都有明確體現）

## 三、驗證二：預測力驗證

### 定義

用此觀點，對此人**從未公開評論過的新情境**進行預測，預測結果是否明確？

### 判斷流程

```
候選觀點
    │
    ▼
設定新情境（此人沒有公開評論過的）
    │
    ▼
用觀點推導預測
    │
    ├─── 預測結果明確（A 而非 B）→ ✅ 通過預測力驗證
    │
    ├─── 預測含糊（「可能 A 或 B」）→ ❌ 觀點不夠精確，需細化
    │
    └─── 預測矛盾（A 和 B 都可以）→ ❌ 剔除
```

### 新情境設計原則

新情境應滿足：

1. **真實可能發生**：非假設性場景
2. **此人未公開表態**：不能用已知答案作弊
3. **明顯需要使用候選觀點**：能直接套用

### 案例演示

**候選觀點**：「芒格的激勵機制優先原則」  
**新情境**：某家私募股權公司邀請芒格加入顧問委員會，每次交易成功收取 0.5% 的分成費  
**用觀點推導**：

- 芒格的激勵機制優先原則：先問「這個激勵會讓顧問說什麼？」
- 0.5% 分成意味著顧問每次推薦成交都有利可圖 → 可能推薦不適合的交易
- 芒格會立刻拒絕，因為這個激勵結構會扭曲他的建議品質

**預測**：✅ 明確，芒格會拒絕  
**通過**：✅

---

**候選觀點**：「芒格喜歡中國市場」（弱版本）  
**新情境**：2025 年中國某電商公司 IPO，估值合理，護城河清晰  
**用觀點推導**：

- 芒格確實在晚年投資了阿里巴巴，表示對中國市場有信心
- 但他也承認自己在中國文化背景下的直覺不可靠
- 預測：他可能買，也可能不買，取決於「這是他能理解的商業模式嗎」

**預測**：❌ 含糊，無法明確預測  
**結論**：這個觀點太寬泛，需要細化或剔除

## 四、驗證三：排他性驗證

### 定義

此觀點是否**真的是此人特有的**，而非任何成功人士都會說的普世觀點？

### 普世觀點黑名單（自動剔除）

以下觀點無論以什麼形式出現，都必須剔除：

```
誠實是最重要的
努力工作
持續學習
長期思考
保持好奇心
永不放棄
團隊合作
以客為尊
謙遜
專注
```

### 判斷測試

**放入任何人名測試**：把候選觀點放入 "任何成功人士都說 [觀點]"，若這句話成立，說明此觀點缺乏排他性。

**例子**：

- "任何成功人士都說要誠實" → 成立 → ❌ 剔除
- "任何成功人士都說要建立多元思維模型格柵" → 不成立，這是芒格特有的術語 → ✅ 通過

### 深度排他性

有些觀點表面上聽起來特別，但深入看也是普世觀點的包裝：

**表面特別**：「投資要有安全邊際」  
**本質**：「不要冒不必要的風險」 = 任何謹慎的人都這樣說  
**判斷**：排他性弱，需要更具體的版本（如：芒格如何計算安全邊際？他的標準是什麼？）

## 五、三重驗證矩陣

每個候選觀點在驗證後填入以下矩陣：

| 觀點     | 跨域驗證 | 預測力驗證 | 排他性驗證 | 最終結果 |
| -------- | -------- | ---------- | ---------- | -------- |
| 逆向思考 | ✅       | ✅         | ✅         | 納入     |
| 長期投資 | ✅       | ✅         | ❌         | 剔除     |
| 能力圈   | ✅       | ✅         | ✅         | 納入     |
| 持續學習 | ✅       | ❌         | ❌         | 剔除     |

**通過標準**：三重全通過才能納入

## 六、邊緣案例處理

### 情況 A：觀點通過 2/3 驗證

**建議**：不納入主要清單，但標記為「待確認」（存入附錄），在下次更新時重新驗證。

### 情況 B：觀點只在一個領域但極其深刻

**建議**：仍然剔除，但可以在「蒸餾說明」區段中備註「此人在 X 領域有特殊深度，但尚未發現跨域遷移」。

### 情況 C：批評者提出的觀點（批評路徑）

**特殊規則**：批評者對此人的批評，如果能通過跨域驗證（批評者發現的模式在多個領域都有體現），也可以納入「誠實承認」或「反面模式」區段。

## 七、驗證腳本說明

`scripts/validate.py` 實作以下自動化檢查：

| 自動化檢查                 | 對應的三重驗證           |
| -------------------------- | ------------------------ |
| 普世觀點黑名單比對         | 排他性驗證（部分自動化） |
| 必要區段完整性檢查         | 結構性前提               |
| 心智模型數量範圍（5-15）   | 提煉充分性               |
| 決策啟發式數量範圍（3-10） | 提煉充分性               |
| 骨架佔位符偵測             | 完整性前提               |
| 來源數量檢查（≥20）        | 資料充分性前提           |

**注意**：跨域驗證和預測力驗證目前需要人工執行，腳本無法自動化這兩項。

---

## 八、人工驗證工作流程

1. 使用 `scripts/distill.py --phase plan` 生成採集計劃
2. 執行六路採集，收集候選觀點
3. 對每個候選觀點，手動填寫三重驗證矩陣
4. 只把三重全通過的觀點寫入蒸餾文件
5. 執行 `scripts/validate.py --target "[人物名稱]"` 進行自動化結構驗證
6. 通過後執行 `scripts/integrate-evolution.py --target "[人物名稱]"` 整合到進化系統
