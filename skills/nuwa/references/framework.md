> 版本：v1.0  
> 制定日期：2026-05-14

## 一、核心理念

### 蒸餾 vs 模仿

女媧蒸餾**不是模仿**，而是提取可遷移的**認知結構**。

| 維度   | 模仿         | 蒸餾             |
| ------ | ------------ | ---------------- |
| 目標   | 複製表面行為 | 提取底層結構     |
| 結果   | 角色扮演     | 思維框架         |
| 可靠性 | 低（易失真） | 高（跨域可驗證） |
| 實用性 | 表演性       | 工具性           |

**蒸餾定義**：從大量公開資料中，提取此人**獨特的、可操作的、跨域可驗證的**認知模式。

### 品質三角

```
        排他性
       /      \
      /        \
跨域性 ———— 預測力
```

- **排他性**：此觀點是此人特有的，而非任何人都會說的普世觀點
- **跨域性**：此觀點在此人的多個不同領域中都有體現
- **預測力**：此觀點能預測此人在新情境（未見過的場景）中的行為

三角缺一，蒸餾即失敗。

## 二、六路並行採集框架

### 設計原則

單一資料來源容易產生確認偏誤。六路採集確保：

1. **縱深覆蓋**：同一人物的不同媒介表達
2. **批評平衡**：必須包含反對者觀點，防止「粉絲版」蒸餾
3. **時間維度**：早期思想 vs 晚期思想的對比，捕捉思想演化

### 六路說明

#### 路徑一：作品路徑（Works）

**採集內容**：書籍、文章、演講稿、訪談紀要、公開報告

**採集邏輯**：此人親筆或親口的一手資料，是蒸餾的最高品質來源。

**搜尋策略**：

```
"{人物名稱}" 書籍 著作 思想
"{人物名稱}" 演講 keynote speech
"{人物名稱}" 訪談 interview 觀點
"{人物名稱}" site:medium.com OR site:substack.com
```

**品質標準**：

- 優先一手引述（直接引語優於轉述）
- 注意翻譯可能帶來的語義失真
- 記錄來源出版日期（用於時間線分析）

---

#### 路徑二：播客路徑（Podcast）

**採集內容**：Podcast 逐字稿、TED 演講字幕、YouTube 訪談字幕

**採集邏輯**：口語表達往往比書面更真實——更少修飾、更多即興反應。

**搜尋策略**：

```
"{人物名稱}" podcast transcript
"{人物名稱}" TED talk
"{人物名稱}" YouTube interview transcript
```

**品質標準**：

- 優先完整逐字稿而非摘要
- 注意訪談者的引導可能扭曲回答
- 長篇訪談比短訪談更有價值（>30 分鐘）

---

#### 路徑三：社交路徑（Social）

**採集內容**：推特/X 推文集、部落格文章、公開信、社群貼文

**採集邏輯**：社交媒體是最即時、最未經過濾的思維快照。

**搜尋策略**：

```
"{人物名稱}" twitter tweets notable
"{人物名稱}" blog post essay
"{人物名稱}" open letter public statement
```

**品質標準**：

- 優先高互動貼文（更可能是此人的核心觀點）
- 注意情緒化時刻的表達可能非代表性
- 多年時間跨度的社交資料比單點更有價值

---

#### 路徑四：批評路徑（Criticism）⚠️ 必要

**採集內容**：批評者觀點、失敗案例、爭議紀錄、公開辯論

**採集邏輯**：

> 「只收讚美資料，蒸餾出的是公關稿，不是真實思維。」

批評路徑是防止確認偏誤的關鍵。它揭示：

- 此人思維框架的**邊界條件**（在哪裡失效？）
- **對立者的反駁邏輯**（哪些觀點被反對？為什麼？）
- **此人的盲點**（連批評者都發現的問題）

**搜尋策略**：

```
"{人物名稱}" criticism critique wrong mistake
"{人物名稱}" controversy failure
"{人物名稱}" disagree counterargument
```

**品質標準**：

- 優先有充分論據的批評（而非人身攻擊）
- 優先此人**親自回應過**的批評（可以看到他如何處理異見）
- 記錄批評者的背景（領域相關性很重要）

**警告**：若批評路徑資料少於 3 個獨立來源，整個蒸餾結果應標記為「⚠️ 可信度待確認」。

---

#### 路徑五：決策路徑（Decisions）

**採集內容**：重大決策記錄、投資/商業決策覆盤、失敗後的公開反思

**採集邏輯**：決策記錄是思維框架的**活體標本**。理論人人會說，決策才真實。

**搜尋策略**：

```
"{人物名稱}" decision making process
"{人物名稱}" investment decision business decision
"{人物名稱}" retrospective lessons learned
```

**品質標準**：

- 優先有明確結果的決策（而非設想性決策）
- 失敗決策的覆盤比成功案例更有揭示性
- 記錄決策時間點（用於時間線分析）

---

#### 路徑六：時間線路徑（Timeline）

**採集內容**：思想演化過程、早期 vs 晚期觀點對比、「改變心意」的記錄

**採集邏輯**：人的思維框架會演化。捕捉演化過程才能理解**此時此刻**的版本。

**搜尋策略**：

```
"{人物名稱}" early career vs later thinking evolution
"{人物名稱}" changed mind updated belief
"{人物名稱}" intellectual journey
```

**品質標準**：

- 記錄「改變心意」的具體事件和原因
- 對比早期（事業初期）和晚期（成熟期）的同一議題
- 注意：蒸餾結果應基於**最近期的思維版本**

---

### 採集數量標準

| 路徑       | 最少來源數 | 建議來源數 | 必要性   |
| ---------- | ---------- | ---------- | -------- |
| 作品路徑   | 5          | 10+        | 高度建議 |
| 播客路徑   | 3          | 8+         | 建議     |
| 社交路徑   | 3          | 10+        | 建議     |
| 批評路徑   | 3          | 5+         | **必要** |
| 決策路徑   | 2          | 5+         | 建議     |
| 時間線路徑 | 2          | 5+         | 建議     |
| **合計**   | **18**     | **43+**    | —        |

> 總來源數 ≥ 20 為最低標準，低於此值品質不可靠。

## 三、三重驗證規則

每個候選觀點都必須通過三重驗證才能納入最終蒸餾結果。

### 驗證一：跨域驗證（Cross-Domain）

**問題**：此觀點是否在此人的多個不同領域中都有體現？

**操作方式**：

1. 確認此觀點的表現領域（例：投資決策）
2. 在其他領域中尋找同樣的結構（例：管理決策、個人生活）
3. 若只在一個領域出現 → 可能是領域特定知識，非思維框架

**舉例（芒格）**：

- 「逆向思考」 — 投資中用「先想為什麼不買」，管理中用「先想什麼會讓公司失敗」
- ✅ 跨域：通過

**失敗案例**：

- 「買可口可樂股票」 — 只是具體決策，不是思維框架
- ❌ 不跨域：剔除

---

### 驗證二：預測力驗證（Predictive Power）

**問題**：若此人面對一個新情境（他從未公開評論過的），此觀點能否預測他的反應？

**操作方式**：

1. 設定一個此人從未明確表態的新情境
2. 用候選觀點推導預測反應
3. 若預測是明確的（A 而非 B）→ 通過
4. 若預測含糊（「可能A也可能B」）→ 觀點不夠精確，需細化或剔除

**舉例（芒格）**：

- 觀點：「避開激勵機制扭曲的環境」
- 新情境：某投行提議高佣金結構
- 預測：芒格會拒絕，因為佣金激勵可能扭曲建議品質
- ✅ 預測明確：通過

---

### 驗證三：排他性驗證（Exclusivity）

**問題**：此觀點是否真的是此人特有的，而非普世觀點？

**普世觀點黑名單**（自動剔除）：

- 誠實是最重要的
- 努力工作
- 持續學習
- 長期思考
- 保持好奇心
- 永不放棄
- 團隊合作
- 以客為尊

**判斷原則**：若這個觀點放在任何成功人士身上都適用 → 排他性不足 → 剔除

**舉例（芒格）**：

- 「多元思維模型格柵」 — 這是芒格特有的術語和系統
- ✅ 有排他性：通過

- 「長期投資勝過短期」 — 任何價值投資者都這樣說
- ❌ 無排他性：剔除

---

### 驗證通過率標準

| 通過率  | 解讀                       | 處置           |
| ------- | -------------------------- | -------------- |
| ≥ 80%   | 優秀（資料充分，觀點精準） | 直接使用       |
| 60%–79% | 合格（可接受）             | 標記信心度為中 |
| 40%–59% | 不足（需要更多資料）       | 回到採集階段   |
| < 40%   | 失敗（資料或框架有問題）   | 重新開始       |

## 四、五維蒸餾輸出結構

### 維度一：表達風格（Expression DNA）

**目的**：讓 AI 能以此人的**說話方式**輸出，而非只是輸出內容

**應包含**：

- **語氣特徵**：直接/迂迴/諷刺/幽默/嚴肅（選最符合的）
- **標誌詞彙**：此人慣用的 3–5 個特有詞彙或術語
- **句型結構**：慣用的句型模式（先結論後論據 or 先背景後結論？）
- **比喻偏好**：此人喜歡從哪個領域取比喻（科學/歷史/運動/商業？）
- **節奏感**：快速犀利 or 緩慢深思？短句 or 長句？

**範例格式**：

```markdown
## 表達風格

- **語氣**：直接、不留情面，有時帶諷刺
- **標誌詞彙**：「格柵」、「逆向思考」、「能力圈」、「護城河」、「誰是傻瓜」
- **句型結構**：先給結論，再給兩三個支撐例子
- **比喻偏好**：生物學（基因、演化）、物理學（臨界點）、歷史案例
- **節奏感**：節奏緩慢，善用停頓和「反問」
```

---

### 維度二：心智模型（Mental Models）

**目的**：提取此人**慣用的認知框架**，這是蒸餾的核心

**格式規範**：

```markdown
### [編號]. [心智模型名稱]

- **描述**：用一句話說明這個框架是什麼
- **使用條件**：此人何時會啟用這個框架（觸發情境）
- **運作方式**：這個框架如何指導思考和行動
- **代表語錄**：直接引用此人原話（若有）
- **跨域例子**：此框架在不同領域的體現
```

**數量標準**：5–15 個（< 5 表示提取不足；> 15 表示未提煉）

---

### 維度三：決策啟發式（Decision Heuristics）

**目的**：提取此人做決策時的**快速過濾器**，可立即操作

**與心智模型的區別**：

- 心智模型是認知框架（如何看世界）
- 決策啟發式是行動規則（如何做決定）

**格式規範**：

```markdown
1. **[啟發式名稱]**：[一句話描述這個規則，必須是可操作的動詞句]
   - 例子：[具體決策場景]
   - 反例：[違反此規則的情形]
```

**數量標準**：3–10 個（太少無法指導決策；太多說明未提煉）

---

### 維度四：價值觀邊界（Value Boundaries）

**目的**：明確**此人絕對不做的事**，以及**他批評什麼行為**

**這比「此人相信什麼」更有用**，因為：

1. 邊界更難偽裝，更真實
2. 知道邊界才能模擬「此人會拒絕什麼」
3. 批評模式揭示深層價值觀

**格式規範**：

```markdown
### 絕對不做（Red Lines）

- [邊界 1]：[說明為什麼這是紅線]
- [邊界 2]：...

### 反面模式（Anti-Patterns）

此人公開批評的行為：

- [行為 1]：[此人的批評理由]
- [行為 2]：...
```

---

### 維度五：誠實承認（Honest Acknowledgments）

**目的**：記錄此人**公開承認的弱點和不確定領域**

**重要性**：這是蒸餾的「誠信標記」，防止把此人塑造成全知全能

**格式規範**：

```markdown
- **承認的弱點**：[具體描述，附來源]
- **能力邊界**：[此人認為自己不擅長的領域]
- **改變心意的案例**：[此人承認曾經錯誤的具體案例]
- **不確定的領域**：[此人公開表示不確定的事]
```

## 五、蒸餾品質評估標準

### 量化標準

| 指標           | 最低標準 | 優秀標準 |
| -------------- | -------- | -------- |
| 採集來源總數   | ≥ 20     | ≥ 50     |
| 批評路徑來源數 | ≥ 3      | ≥ 8      |
| 三重驗證通過率 | ≥ 60%    | ≥ 80%    |
| 心智模型數量   | 5–15     | 7–12     |
| 決策啟發式數量 | 3–10     | 5–8      |
| 資料時間跨度   | ≥ 3 年   | ≥ 10 年  |

### 質化標準

**通過標準**：

- ✅ 所有五個維度都已填寫（無骨架佔位符）
- ✅ 心智模型都有跨域例子
- ✅ 決策啟發式都是可操作的動詞句
- ✅ 包含批評路徑資料
- ✅ 至少一個「誠實承認」來自批評路徑

**失敗警示**：

- ❌ 所有觀點都是正面的（缺乏批評路徑）
- ❌ 心智模型過於抽象無法操作
- ❌ 含有普世觀點（「努力工作」、「持續學習」等）
- ❌ 任何骨架佔位符未填寫
- ❌ 來源數量低於 20

## 六、更新與維護策略

### 更新週期建議

| 情境             | 建議更新週期                       |
| ---------------- | ---------------------------------- |
| 活躍中的公眾人物 | 每 6 個月                          |
| 退休/不活躍人物  | 每 2 年                            |
| 歷史人物         | 每 5 年（或有新史料時）            |
| 重大事件後       | 立即更新（如：醜聞、公開轉變立場） |

### 版本管理

蒸餾文件應記錄版本號（v1.0, v1.1, v2.0...）：

- 小版本更新（v1.0 → v1.1）：新增資料，無結構調整
- 大版本更新（v1.0 → v2.0）：此人思想有重大演化，需重新驗證所有觀點

## 七、常見蒸餾錯誤

### 錯誤一：粉絲版蒸餾

**症狀**：所有觀點都是正面的，沒有弱點和失敗
**原因**：跳過批評路徑
**修正**：強制執行批評路徑，以不少於 3 個批評來源為門檻

### 錯誤二：普世觀點混入

**症狀**：「努力工作」、「誠實是最重要的」出現在心智模型中
**原因**：未執行排他性驗證
**修正**：對照普世觀點黑名單，逐一過濾

### 錯誤三：領域混淆

**症狀**：心智模型只在一個領域有例子
**原因**：未執行跨域驗證
**修正**：要求每個心智模型必須有至少兩個不同領域的例子

### 錯誤四：行為預測失敗

**症狀**：用此觀點推導新情境時，預測含糊或可正可反
**原因**：觀點不夠精確，仍停留在表面層次
**修正**：繼續細化，直到預測變得明確

### 錯誤五：時間點混亂

**症狀**：混用早期和晚期觀點，未標記時間
**原因**：未執行時間線路徑
**修正**：所有觀點都應標記大約時間，最終蒸餾優先使用最近期版本

## 八、與女媧進化架構的整合

蒸餾完成的技能包會進入四層進化架構：

```
蒸餾完成
    │
    ▼
第一層（運行即學習）
patterns.jsonl ← 蒸餾結果作為學習模式
    │
    ▼
第二層（神經路由）
任務匹配時自動建議 → 「這個問題適合用芒格框架？」
    │
    ▼
第三層（增長心跳）
REM 週期評估使用頻率和效果
    │
    ▼
第四層（有機細胞）
幹細胞胚胎 embryo
    │ 成熟度 > 0.8
    ▼
常駐細胞（installed）
```

### 整合欄位對應

| 蒸餾資料     | 整合目標                     | 用途       |
| ------------ | ---------------------------- | ---------- |
| 心智模型清單 | patterns.jsonl.mentalModels  | 模式匹配   |
| 技能包路徑   | cell-registry.json.skillPath | 快速載入   |
| 來源數量     | patterns.jsonl.sourceCount   | 信心度加權 |
| 驗證通過率   | patterns.jsonl.confidence    | 初始信心度 |
