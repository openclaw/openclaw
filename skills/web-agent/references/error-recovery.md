# 에러 복구 패턴

> 웹 자동화 중 발생하는 공통 에러에 대한 복구 전략

## 1. 팝업/모달 처리

### 감지

- `snapshot`에서 모달/overlay/dialog role 감지
- 배경이 어두워진 경우 (dim overlay)
- "쿠키 동의", "뉴스레터 구독", "앱 다운로드" 등 일반적 팝업

### 복구 순서

1. 닫기 버튼 찾아 클릭 — `X`, `Close`, `닫기`, `×`, `Đóng`
2. `act: press → key: "Escape"` 시도
3. 모달 외부 영역 클릭 시도
4. 3가지 모두 실패 → 페이지 새로고침 후 재시도

```
snapshot → 모달 감지?
  → Yes: 닫기 버튼 클릭 → 실패? → ESC → 실패? → 배경 클릭 → 실패? → 새로고침
  → No: 정상 진행
```

## 2. 로딩 대기

### 전략

- `act: wait` — `textGone` (로딩 텍스트 사라짐 대기) 또는 특정 요소 출현 대기
- 최대 타임아웃: **10초**
- 타임아웃 시: 페이지 새로고침 1회 → 재시도

### 일반적 로딩 패턴

| 사이트 | 로딩 표시    | 대기 조건        |
| ------ | ------------ | ---------------- |
| Shopee | 스켈레톤 UI  | 제품 카드 출현   |
| TikTok | 스피너       | 댓글 텍스트 출현 |
| Zalo   | 로딩 바      | 채팅 목록 출현   |
| NVD    | "Loading..." | 테이블 행 출현   |

## 3. 요소 못 찾음 (Element Not Found)

### 복구 순서

1. 스크롤 다운 → `act: press → key: "PageDown"` → 재 snapshot
2. iframe 내부 확인 → `snapshot` with `frame` 파라미터
3. 다른 탐색 전략 시도 (role 대신 텍스트 기반)
4. **3회 실패 시** → 작업 중단 + 사용자에게 Discord DM 보고

```
시도 1: snapshot → 요소 탐색
시도 2: 스크롤 + snapshot
시도 3: iframe 확인 + snapshot
→ 모두 실패: 사용자 알림
```

## 4. CAPTCHA/인증

### 감지

- snapshot에서 "captcha", "I'm not a robot", "Verify", "인증" 텍스트
- reCAPTCHA iframe 감지
- hCaptcha 위젯 감지

### 복구

1. 작업 즉시 일시중지
2. Discord DM 알림:
   > "⚠️ CAPTCHA가 감지되었습니다. 수동 해결 후 '계속' 이라고 알려주세요."
   > URL: [현재 페이지 URL]
3. 사용자 응답 대기
4. 사용자 해결 확인 후 → snapshot으로 CAPTCHA 사라짐 확인 → 작업 재개

### ⚠️ 절대 금지

- CAPTCHA 자동 우회 시도 금지
- CAPTCHA 해결 서비스(2Captcha 등) 사용 금지

## 5. 세션 만료/로그인 필요

### 감지

- URL이 `/login`, `/signin`, `/auth` 등으로 리다이렉트
- snapshot에서 "로그인", "Sign in", "Đăng nhập" 텍스트 + 입력 폼

### 복구

1. 로그인 페이지 리다이렉트 감지
2. 사용자에게 Discord DM 알림:
   > "⚠️ [사이트명] 세션이 만료되었습니다. 재로그인이 필요합니다."
3. QR/수동 로그인 → 사용자 완료 알림 대기
4. 로그인 확인 후 원래 작업 URL로 복귀

### ⚠️ 보안 원칙

- 비밀번호 자동 입력 금지 (사용자 명시적 요청 제외)
- 크레덴셜 파일 저장 금지
- OAuth/QR 로그인 우선

## 6. Rate Limiting

### 감지

- HTTP 429 응답 (콘솔에서 확인)
- "Too many requests" 페이지
- "접근이 차단되었습니다" / "Access denied" 페이지
- Cloudflare 차단 페이지

### 복구 (지수 백오프)

```
1차: 30초 대기 → 재시도
2차: 1분 대기 → 재시도
3차: 2분 대기 → 재시도
4차: 작업 중단 + 사용자 알림
```

### IP 차단 감지 시

- 즉시 작업 중단
- Discord DM 알림:
  > "⚠️ [사이트명]에서 IP 차단이 감지되었습니다. 작업을 중단합니다."
- 해당 사이트 24시간 접근 자제

## 7. 네트워크 에러

### 감지

- 페이지 로딩 실패 (빈 페이지, ERR_CONNECTION_REFUSED 등)
- snapshot 실패

### 복구

```
1차: 10초 대기 → 재시도
2차: 30초 대기 → 재시도
3차: 작업 중단 + 사용자 알림
```

### DNS/SSL 에러

- DNS 실패: 사이트 다운 가능성 → 사용자 알림
- SSL 에러: 무시하지 않음 → 사용자 알림

## 공통 원칙

1. **최대 재시도 3회** — 무한 루프 방지
2. **사용자 알림은 Discord DM** (채널: 지니님 DM)
3. **보안 우선** — 의심스러운 상황에서는 중단
4. **로그 기록** — 에러 발생 시 URL, 시간, 에러 유형 기록
5. **graceful degradation** — 부분 데이터라도 저장 후 실패 보고
