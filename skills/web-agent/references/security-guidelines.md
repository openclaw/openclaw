# 보안 가이드라인

## 크레덴셜 관리

- 로그인 정보는 절대 스킬 파일에 하드코딩하지 않음
- 환경변수 또는 .env 파일 참조
- memory에도 비밀번호 저장 금지

## 접근 범위

- 공개 페이지 스크래핑: 자유롭게
- 로그인 필요 페이지: 사용자 사전 동의 필요
- 결제/주문 페이지: 절대 자동 조작 금지
- 개인정보 수집: PDPA/GDPR 준수

## Rate Limiting

- 같은 사이트 요청 간 최소 2초 간격
- 하루 최대 100회/사이트
- 차단 감지 시 즉시 중단

## 데이터 보안

- 수집 데이터 중 개인정보 마스킹
- 공개 리포트에 민감 정보 제외
- memory 파일 .gitignore에 포함되어 있는지 확인
