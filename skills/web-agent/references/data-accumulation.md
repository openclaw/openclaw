# 데이터 축적 전략

## 저장 계층

### Layer 1: 실시간 알림 (Discord DM)

- 가격 5% 이상 변동
- 경쟁사 신규 프로모션
- 에러/차단 발생
- 형식: 간결한 1~3줄

### Layer 2: 데일리 노트 (옵시디언)

- 경로: 00.DAILY/{날짜}\_{카테고리}.md
- 당일 전체 수집 데이터
- 요약 + 상세 테이블
- 태그: web-agent, 카테고리

### Layer 3: 누적 메모리 (memory/\*.md)

- memory/price-monitor.md — 가격 시계열
- memory/competitor-watch.md — 경쟁사 동향
- memory/web-agent-log.md — 실행 로그/에러
- 최신 N건만 유지, 오래된 건 아카이브

### Layer 4: 프로젝트 docs (장기 분석)

- 주간/월간 트렌드 분석
- 인사이트 리포트
- 프로젝트별 docs/intelligence/ 폴더

## 데이터 정리 주기

- 매일: 데일리 노트 자동 생성
- 매주: memory/\*.md 정리 (50건 초과 시 아카이브)
- 매월: 월간 트렌드 리포트 → 프로젝트 docs/

## 시계열 분석

누적된 가격 데이터를 활용:

- 가격 추이 분석 (상승/하락 트렌드)
- 프로모션 주기 패턴 발견
- 경쟁사 가격 전략 분석
- 최적 가격 제안
