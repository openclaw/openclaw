# 蜂巢复活系统 (Hive Resurrection System)

**技能版本**: v1.0  
**创建时间**: 2026-03-10  
**技能类型**: 生存能力/集群守护  

---

## 技能描述

多节点集群互相守护系统，实现任意节点故障时的自动复活。

**核心思路**:
```
每个节点运行两个进程：
1. openclaw 主进程（干活的）
2. watchdog 守护进程（看门狗，负责重启 + 接受远程指令）

watchdog 永远不会挂（systemd 保证）
其他节点发现 openclaw 挂了 → 通知该机器的 watchdog → watchdog 重启 openclaw
```

---

## 架构设计

```
节点 A                          节点 B
┌──────────────┐              ┌──────────────┐
│  openclaw ✗  │  ←─检测到挂─  │  openclaw ✓  │
│  watchdog ✓  │  ←─发指令──→  │  watchdog ✓  │
│    ↓重启     │              │              │
│  openclaw ✓  │              │              │
└──────────────┘              └──────────────┘
```

## 核心组件

| 组件 | 功能 | 运行位置 |
|------|------|---------|
| watchdog.js | 看门狗守护进程 | 每台机器 |
| monitor.js | 集群监控进程 | 每台机器 |
| config.json | 集群配置 | 每台机器 |
| install.sh | 一键安装脚本 | 部署机器 |

---

## 关键特性

1. **去中心化** - 无单点故障，每个节点独立
2. **自动选举** - 集群选举避免重复操作
3. **安全认证** - HMAC 签名防止恶意指令
4. **频率限制** - 防止疯狂重启循环
5. **健康检查** - 主动检测 + 被动通知
6. **远程可控** - 支持远程重启/停止/查询

## 部署条件

- Node.js >= 18
- Linux systemd (或 Windows 服务)
- 节点间网络互通
- 每台机器独立配置

---

## 使用场景

1. **多机器部署** - 2+ 台机器运行 OpenClaw
2. **高可用需求** - 需要 7x24 小时不间断服务
3. **无人值守** - 无法及时人工介入的场景
4. **边缘节点** - 分布式部署的边缘计算节点
