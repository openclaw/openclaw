# Model Router — Detailed Routing Logic

## Sonnet 판단 기준 (빠르고 효율적)

- "~해줘", "~확인해", "~보여줘" 같은 단순 요청
- 파일 1~2개 수정
- 단일 프로젝트 내 작업
- 정보 조회/검색
- 하트비트/크론 실행
- git 커밋, push
- 상태 보고

## Opus 판단 기준 (깊고 정확)

- "설계해줘", "분석해줘", "왜 그런지", "최선의 방법" 같은 추론 요청
- 여러 파일/프로젝트에 걸친 작업
- 에러 근본 원인 분석
- 새 프로젝트/기능 아키텍처
- 브라우저 자동화 (ASC, 복잡한 웹 플로우)
- 사업 전략/방향성 논의
- MAI Universe 크로스 프로젝트 시너지 분석
- "이 이슈 분석이 되나요?" 같은 메타 질문

## 실전 예시

```
지니: "비트코인 분석해줘"
→ Sonnet (단순 스크립트 실행)

지니: "MAIBOTALKS App Store 거부 원인 분석하고 코드 수정해줘"
→ Opus (멀티스텝 분석 + 코드 수정 + 브라우저 자동화)

지니: "고생했습니다"
→ Sonnet (일상 대화)

하트비트 폴
→ Sonnet (상태 체크)
```

## 전환하지 않는 경우

- 이미 올바른 모델이면 불필요한 전환 금지 (API 호출 낭비)
- 현재 작업 중간에 전환 금지 (작업 완료 후 다음 턴에서 전환)
- 지니님이 `/model`로 명시 지정한 경우 라우터 비활성
