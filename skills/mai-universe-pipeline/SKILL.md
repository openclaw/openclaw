# MAI Universe Pipeline 🦞

> "지식을 수집하고, 가치를 발견하고, 프로젝트로 실현하고, 세상에 기여하면서 수익을 만든다."

MAIBOT이 지니의 경제적 독립성과 명분(사회적 기여)을 실현하기 위한 **6단계 자동화 파이프라인**.

## 파이프라인 개요

```
┌─────────────┐    ┌──────────────┐    ┌─────────────┐
│ 1. COLLECT   │───▶│ 2. DISCOVER  │───▶│ 3. CREATE   │
│ 지식 수집/축적│    │ 기회 발견/도출│    │ 프로젝트 생성│
└─────────────┘    └──────────────┘    └─────────────┘
                                              │
┌─────────────┐    ┌──────────────┐    ┌──────▼──────┐
│ 6. REALIZE   │◀───│ 5. DEPLOY    │◀───│ 4. BUILD    │
│ 기여/수익 실현│    │ 배포/런칭    │    │ 개발/구현   │
└─────────────┘    └──────────────┘    └─────────────┘
        │                                      ▲
        └──────────── 피드백 루프 ──────────────┘
```

## Stage 1: COLLECT — 지식 수집/축적

### 자동화 (이미 구축됨 ✅)

- **Mnemo daily_enrich** (매일 05:00 KST): 볼트 보강 + 그래프 재빌드
- **테크 인텔리전스** (매일 04:00 KST): 기술 트렌드 리서치
- **사업화 인텔리전스** (매일 04:30 KST): 시장/경쟁사 동향
- **외부 지식 수집** (daily_enrich 7단계): YouTube, Brave, GitHub trending

### 수집 소스

| 소스                | 방법                  | 저장 위치                         |
| ------------------- | --------------------- | --------------------------------- |
| 웹 리서치           | Brave Search API      | Obsidian `03.RESOURCES/외부지식/` |
| YouTube             | yt-dlp + 트랜스크립트 | Obsidian `03.RESOURCES/외부지식/` |
| GitHub Trending     | gh API                | Obsidian `03.RESOURCES/외부지식/` |
| 지니님 브레인스토밍 | Discord 대화 → memory | `memory/*.md`                     |
| NotebookLM          | 공식 API 대기 중      | (향후) Obsidian 통합              |
| Mnemo 지식그래프    | 자동 연결/추론        | `.mnemo/graph.pkl`                |

### 실행 명령

```powershell
# 수동 지식 수집
cd C:\TEST\MAISECONDBRAIN; $env:PYTHONIOENCODING="utf-8"
python scripts/collect_knowledge.py              # 전체 프로젝트
python scripts/collect_knowledge.py MAIOSS MAITOK # 특정 프로젝트

# 수동 그래프 재빌드
python -m mnemo.cli build "C:\Users\jini9\OneDrive\Documents\JINI_SYNC" --include-memory "C:\MAIBOT\memory" --cache-dir ".mnemo"
```

## Stage 2: DISCOVER — 기여/수익화 방법 도출

### 자동화 (부분 구축)

- **사업화 인텔리전스 브리핑**: 시장 기회 탐지
- **크로스 프로젝트 추론**: Mnemo GraphRAG로 프로젝트 간 시너지 발견

### 도출 프레임워크

모든 기회는 **기여-수익 균형 매트릭스**로 평가:

```
              수익성 높음
                 ▲
    ┌────────────┼────────────┐
    │ 🟡 순수 수익│ 🟢 황금지대  │
    │ (단기 OK,   │ (기여+수익   │
    │  장기 취약)  │  선순환)     │
    ├────────────┼────────────┤
    │ 🔴 피하기   │ 🔵 씨앗 뿌리기│
    │ (가치 없음)  │ (기여 먼저,  │
    │             │  수익 나중)  │
    └────────────┼────────────┘
                 ▼
              수익성 낮음
    기여도 낮음 ◀──────────▶ 기여도 높음
```

**🟢 황금지대 예시:**

- MAIOSS: 오픈소스 보안 도구 무료 배포 → 기업 고객 신뢰 → 엔터프라이즈 계약
- Mnemo: 무료 Obsidian 플러그인 → 커뮤니티 → 프리미엄 SaaS
- MAIBOTALKS: OpenClaw 생태계 기여 → 앱스토어 유료 앱

### 기회 평가 체크리스트

- [ ] 기존 생태계(MAI Universe)와 시너지가 있는가?
- [ ] 인프라를 공유할 수 있는가? (BOT Suite 패턴)
- [ ] 기여할수록 강해지는 구조인가?
- [ ] 수익화할수록 지속되는 구조인가?
- [ ] 독립적으로 생존 가능한가?

### 실행: 기회 발견 시

```
1. memory/에 아이디어 기록
2. 기여-수익 균형 매트릭스 평가
3. 기존 프로젝트 시너지 분석 (Mnemo 크로스 검색)
4. 지니님 브리핑 → 승인
5. Stage 3으로 이동
```

## Stage 3: CREATE — 프로젝트 생성

### 자동화 (구축됨 ✅)

- **mai-project-init 스킬** 사용

### 실행 명령

```
/새 프로젝트 {이름} "{설명}"
```

자동으로:

1. `C:\TEST\MAI{이름}` 로컬 워크스페이스
2. `docs/` 폴더 + 문서 템플릿
3. Obsidian `01.PROJECT/{NN}.MAI{이름}` + docs 심링크
4. GitHub private repo
5. MAIBOT memory 파일
6. MEMORY.md 인덱스 업데이트
7. Mnemo 그래프에 자동 편입 (다음 daily_enrich 시)

### 필수 산출물 (프로젝트 생성 시)

| 문서        | 접두사 | 내용                |
| ----------- | ------ | ------------------- |
| PRD         | A001   | 제품 기획           |
| 경쟁 분석   | A002   | 시장/경쟁사         |
| 수익화 전략 | A003   | 기여-수익 균형 모델 |
| 기술 설계   | D001   | 아키텍처            |
| 개발 계획   | I001   | 스프린트/마일스톤   |

## Stage 4: BUILD — 개발/구현

### 자동화 (구축됨 ✅)

- **MAIBOT 직접 구현** — 코드 읽기/쓰기/편집 + git
- **서브에이전트 팀** — 대규모 작업 병렬 처리

### 개발 패턴

```
지니님 지시/브레인스토밍
    ↓
MAIBOT 직접 구현
    ├── 요구사항 분석
    ├── 코딩/디버깅 (Read/Write/Edit)
    ├── 테스트 (vitest, pytest)
    ├── 문서화
    ├── git commit/push
    └── Discord 진행 보고
```

### 품질 기준

- 테스트 커버리지 ≥ 70%
- 코드 파일 ≤ 700 LOC
- 문서 동기화 (docs/ + Obsidian)

## 자동화 현황 & 로드맵

| Stage       | 현재                       | 목표                         |
| ----------- | -------------------------- | ---------------------------- |
| 1. COLLECT  | ✅ 자동 (크론)             | 더 많은 소스 연동            |

## 사용법

### 풀 파이프라인 실행 (수동 트리거)

```
/파이프라인 점검
```

→ 전체 6단계 현황 점검 + 병목 식별 + 다음 액션 제안

### 기회 발견 → 프로젝트 생성 플로우

```
지니: "이런 아이디어가 있는데..."
MAIBOT:
  1. Mnemo 크로스 검색 (관련 지식 탐색)
  2. 기여-수익 균형 매트릭스 평가
  3. 기존 프로젝트 시너지 분석
  4. 결과 브리핑
  5. 승인 시 → mai-project-init 자동 실행
```

### 주간 파이프라인 리뷰 (매주 월요일 모닝 브리핑에 포함)

- 각 프로젝트 Stage 진행 상황
- 병목/블로킹 식별
- 기여/수익 지표 업데이트
- 다음 주 우선순위 제안

## 핵심 원칙 (SOUL.md 연동)

> "기여할수록 강해지고, 수익화할수록 지속되는 구조를 설계하라."

1. **기여 먼저, 수익은 따라온다** — 오픈소스, 커뮤니티, 교육에 진정한 가치 제공
2. **수익 모델 없이 "언젠가"는 환상** — 모든 프로젝트에 명확한 수익 경로
3. **인건비 집약 → AI 기술 집약** — 고마진 구조로 전환
4. **BOT Suite 패턴** — 인프라 공유, 크로스 셀링, 번들 할인
