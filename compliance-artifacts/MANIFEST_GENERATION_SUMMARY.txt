  Key Design Decisions
================================================================================

NARROWLY SCOPED CLAIMS:
  ✓ "~127 gated contract points across 12 stages"
  ✓ NOT "universal mediation of all agent operations"
  ✓ NOT "100% of operations are gated"
  
  This prevents reviewers from asking: "What about X operation?"
  Answer: "X is outside the 12 gated stages. See manifest for what IS gated."

FAIL-CLOSED DEFAULTS:
  ✓ Router outage → block (no retry, no fallback)
  ✓ Pack missing → block (ABSTAIN_CLARIFY)
  ✓ Empty allowlist → block (no contracts to route)
  ✓ Mismatch detected → block (for TOOL_DISPATCH_GATE)

ATOMIC COMMIT DISCIPLINE:
  ✓ 4 high-impact stages commit decision before executing side effect
  ✓ If commit fails, operation is blocked (not retried)
  ✓ Prevents "router said yes, but we can't record that" scenarios

CAPABILITY-BASED FILTERING:
  ✓ Per-contract capability requirements
  ✓ Contracts filtered before routing (not after)
  ✓ Allows fine-grained permission models
  ✓ Example: Can allow NETWORK_GET_PUBLIC but deny NETWORK_RAW_SOCKET
