Fixes #45080

## Summary

`openclaw logs --follow` occasionally fails with gateway connect errors while the gateway is running. Server logs show **handshake timeout** / **closed before connect** (~3s), often when the CLI is still loading plugins or the gateway is busy.

## Root cause

Short window between WebSocket accept and first `connect` frame; slow CLI startup or loaded gateway can exceed that window intermittently.

## Change (this PR)

- Retry **only the first** `logs.tail` when **`--follow`**: up to **3** attempts, **1.5s** apart.
- Retry on transient errors (e.g. `gateway not connected`, `gateway connect failed`, `closed before connect`, `handshake timeout`, `ECONNREFUSED`, …).
- No retry for one-shot `logs` or for follow polling after the first successful tail.
- Unit tests for error classification and retry behavior.

## Verify

```bash
openclaw logs --follow
```

## Workaround (before fix)

```bash
tail -f /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log
```
