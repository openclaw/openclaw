# Task: Fix Discord missing Intent crashing entire Gateway (Issue #27002)

## Problem

When a Discord bot account is added WITHOUT enabling "Message Content Intent" in the
Discord Developer Portal, the entire Gateway crashes — taking down ALL channels
(Telegram, WhatsApp, other Discord bots, etc.).

A single misconfigured Discord bot account should never crash the whole system.

## Expected behavior

1. Catch the intent/auth error for the specific Discord account
2. Log a clear warning identifying which account failed and why
3. Disable ONLY the affected account
4. All other channels continue running normally

## Key files to investigate

- `extensions/discord/src/monitor/provider.ts` — Discord connection/lifecycle
- `extensions/discord/src/channel.ts` — channel startup
- `extensions/discord/src/probe.ts` — probe/health check
- Look for where unhandled Discord connection errors propagate to crash the gateway

## Instructions

1. Read key files to trace the crash path
2. Plan the fix — wrap Discord account connection in try/catch
3. On connection error: log warning with account name + error reason, mark account as disabled
4. Implement with minimal diff
5. Run formatter: `npx oxfmt` on changed files
6. Run tests: `npx vitest run --reporter=verbose` on changed test files
7. Show output — only commit if ALL tests pass
8. Do NOT use `String(err)` — use `err instanceof Error ? err.message : "unknown error"`
9. Do NOT run npm/pnpm install
