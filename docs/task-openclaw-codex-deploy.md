# ТЗ: Развертывание OpenClaw с использованием Codex-подписки

1. Подготовить инфраструктуру: поднять/проверить `docker-compose.yml`, `.env`, сетевые порты и DNS для gateway.
2. Настроить модельный провайдер `openai-codex:default` и подключить рабочий OAuth/API ключ Codex в конфиге OpenClaw.
3. Прописать Telegram Bot API токен и нужные переменные окружения в `.env` (без секретов в репозитории).
4. Запустить контейнер OpenClaw, проверить `openclaw model status --json --agent main` и отсутствие ошибок авторизации в логах.
5. Проверить рабочий сценарий: отправить тестовое сообщение в Telegram и убедиться в наличии ответа от модели через OpenAI/Codex.
6. Зафиксировать шаги диагностики и обновлений в `README.md` или отдельной `docs/`-заметке для воспроизводимого развёртывания.
