# OpenClaw Design Deliverables

This folder contains the contribution-oriented audit package for understanding OpenClaw and turning that understanding into a high-leverage open-source contribution.

## Deliverables

- [Repo audit and contribution plan](/design/openclaw-repo-audit-contribution-plan)
  Full audit covering system overview, architecture, execution flow, core modules, product/features, top problems, opportunities, and contribution proposals.
- [System inventory](/design/openclaw-system-inventory)
  A more operational breakdown of subsystems, ownership boundaries, hotspots, and where to start by area.
- [Contribution shortlist](/design/openclaw-contribution-shortlist)
  PR-shaped contribution ideas with scope, impact, modules, and suggested first slices.
- [Memory diagnostics PR spec](/design/openclaw-memory-diagnostics-pr-spec)
  A contribution-ready spec for the recommended first PR, including scope, files, tests, and draft PR framing.

## Companion file

- [`/ROADMAP.md`](../../ROADMAP.md)
  Phase-based roadmap for turning the audit into an actual contribution path.

## Suggested reading order

1. Read the audit report.
2. Use the system inventory to decide which subsystem matches your strengths.
3. Pick one idea from the contribution shortlist.
4. Use `ROADMAP.md` to turn that into a discussion or PR sequence.
