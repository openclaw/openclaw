## 一、投资人视角：我为什么投（或不投）

### 1.1 我看到的市场机会

**中国油气管网的数字化窗口正在打开：**

```
规模：
  · 国家管网管理里程 > 10 万公里（2025）
  · 压气站/输气站 > 800 座
  · 阀室/计量站 > 15,000 个
  · 每年新增 2,000–3,000 公里管道

政策催化：
  · GB 32167-2015（管道完整性管理）强制执行，罚款机制明确
  · 「十四五」工业互联网专项支持工业数字化
  · 无人值守/少人值守成为国家管网明确方向（节约人力成本 60%+）

财务动机（老板级）：
  · 1 次计划外停输（天然气主干管）损失 ¥100-500 万
  · 1 座压气站年运维成本 ¥3,000-8,000 万
  · 数字化可降低运维成本 15-25%（麦肯锡数据）
  · ROI 通常 < 18 个月（在高输量场站）
```

**为什么现在是时机：**

```
2020 年之前：AI 太贵，边缘算力不够，大模型不存在
2023 年：GPT-4 出现，但无本地部署，合规风险
2026 年（现在）：Qwen3.6-35B 本地部署，1×H100 就够
               MOIRAI 2.0 零样本时序，Babylon.js 8 WebGPU 光追
               Eclipse Ditto 稳定生产级，GraphRAG 知识图谱成熟
→ 第一次所有技术同时就绪的窗口
```

### 1.2 我担心的风险（投资人会问的问题）

**风险 1：「你们怎么进门？」（销售难度）**

```
现实：
  · 国家管网/中石油/中石化的 IT 采购周期 12-24 个月
  · 必须通过集团级信息化部审批
  · 关键词：「国产化」「信创」「等保三级」

如何破局：
  ✅ 从「城燃/地方管网」切入（采购周期 3-6 个月，更灵活）
  ✅ 集成商渠道（找已有甲方关系的工程公司合作）
  ✅ 先免费 POC（2 周上线 → 出数据 → 让数据说话）
  ✅ 「信创合规版」：全部用国产（Qwen/飞书/海光 DCU 可替代 H100）
```

**风险 2：「OT 系统不连网，怎么接数据？」**

```
现实：
  · 大多数生产控制网和管理网物理隔离
  · 工控安全等保 2.5 级，严禁外部连接

如何破局：
  ✅ 本地部署（所有 11 个 Docker 服务在客户机房）
  ✅ 单向数据二极管（只出不进，硬件隔离设备，市场有成熟品）
  ✅ Phase 1 不接 OT，只接办公网 SCADA 历史数据库（SQL/CSV 导入）
  ✅ 数据不出厂区（符合等保要求）
  → 解耦 OT 接入，先跑 AI，再打通数据
```

**风险 3：「大公司（华为/西门子）会不会进来把你们挤掉？」**

```
华为的路径：卖算力 + 云 → 卖工业 AI 平台（类 AWS + SageMaker）
  · 他们做水平平台，不做垂直场景
  · 不会给你写「SDV-001 阀门的 ISO 6D 合规检查建议」

西门子 SIMATIC IT / MindSphere：
  · 适配西门子自己的设备，中国厂家设备支持很差
  · 价格是我们的 10 倍以上，主要服务央企大客户

圆晖科技（最直接的竞争者）：
  · 强于 3D 数字样机渲染，偏向工程设计阶段
  · 弱于实时运营 AI（没有 RAG/预测/工单闭环）

结论：没有人在做「中国油气场站运营 AI + 本地部署 + 飞书原生 + 开源技术栈」
      这个精确交叉点当前是空白
```

**风险 4：「核心团队依赖，你走了怎么办？」**

```
应对：
  · 知识沉淀在系统里（L0-L3 知识库本身就是资产，不依赖个人）
  · 文档驱动开发（所有决策有 AGENTS.md 追溯）
  · OpenClaw 开放生态，有社区支撑
```

## 二、用户视角：我是油气场站工程师，这是我真正需要的

### 2.1 我是谁（真实用户画像）

```
角色 1：场站值班员（最终用户，24h 轮班）
  · 年龄：25-45 岁
  · 技能：操作 SCADA，但不会写代码
  · 痛点：凌晨接到报警电话，要在黑暗中判断要不要停机
  · 他要的：「告诉我现在应该做什么，我不想自己查手册」
  · 手机：一定是微信/飞书，不会去开 PC 查系统

角色 2：设备工程师（核心用户，日常运维决策者）
  · 年龄：30-50 岁
  · 技能：懂设备，不懂 IT
  · 痛点：查压缩机手册要翻 300 页，找到对应故障要 1 小时
  · 他要的：「给我答案，给我依据，让我可以签字」
  · 关键：他的签字意味着他负责，所以 AI 必须给 citations

角色 3：场站主管（审批者，买单者）
  · 年龄：40-55 岁
  · 痛点：人员不够，经验流失（老师傅退休），出了事要负责
  · 他要的：「证明这个系统能减少我的风险，数字说话」
  · 买单逻辑：1 次预防性维修 vs 1 次停输事故

角色 4：公司 IT 部门（拦路虎）
  · 痛点：网络安全审查，等保合规，不想再维护新系统
  · 他要的：「给我一键部署，数据不出厂，有等保报告」
```

### 2.2 用户的真实痛点（不是我们猜的，是行业调研结果）

```
痛点 1：「知识在老师傅脑子里，他一退休我就完了」（知识流失）
  → 我们的答案：L3 知识库 = 把老师傅的脑子数字化
  → 每次工单验证 = 老师傅的判断变成 AI 永久记忆

痛点 2：「报警了，但不知道严不严重，要不要停机」（决策辅助）
  → 我们的答案：MOIRAI + 物理代理模型 + citations 给置信度
  → 不是「可能有问题」，是「87% 置信度，72h 内失效概率 38%」

痛点 3：「做个维修计划要查 3 个系统，写 2 小时报告」（效率）
  → 我们的答案：AI 30 秒生成工单草案，工程师审批 12 秒
  → 整个过程从 3 小时 → 42 秒

痛点 4：「年轻人不愿意来，招不到有经验的」（人才稀缺）
  → 我们的答案：AI 把经验门槛从「10 年」降到「会用手机」
  → 新员工跟 AI 学，3 个月达到老员工 80% 的判断能力

痛点 5：「检查记录全是纸，管理局来检查全靠翻档案」（合规）
  → 我们的答案：数字护照 + 工单链 + 自动生成 GB 32167 合规报告
  → 每次 AI 建议都有 citations，天然是合规证据

痛点 6：「我们场站太偏远，网络不稳定，云端的用不了」（边缘部署）
  → 我们的答案：本地 Docker 部署，完全离网运行
  → Phase 3：7B 边缘模型，RTX 4090 本地，无需公网
```

### 2.3 用户的购买决策链

```
谁发现问题：值班员/工程师（感知到痛点）
谁提出需求：设备工程师 / 生产主管（内部 champion）
谁做技术评估：IT 部门（合规/安全/接口）
谁审批预算：副总裁/总工程师（ROI 判断）
谁签合同：公司分管副总（最终决策）

→ 我们需要分别影响每一层：
  值班员：让他 1 周内觉得「有 AI 帮忙轻松多了」
  工程师：让他有一次「AI 比我先发现问题」的体验
  IT：给他等保报告模板 + 本地部署文档
  副总：给他 ROI 计算器（防损 ¥ vs 采购 ¥）
  总工：给他行业对标报告（哪些同行已在用）
```

### 2.4 用户最在乎的三件事（按重要性排序）

```
第一：AI 说的话，可信吗？
  · 每条建议必须有来源（citations）
  · 必须可以说「为什么 AI 这么说」
  · 错了谁负责？→ AI 是建议，人审批后执行，人负责
  → 我们的 citations 强制机制 + HITL 是正确答案

第二：接了之后，我要维护多少？
  · 工程师不懂 IT，不想管服务器
  · 最好「装了就用，用了就有」
  → 一键 Docker Compose 部署 + 自动更新 + 运维文档
  → 提供远程运维支持服务（¥2 万/年）

第三：我的数据安全吗？
  · 管道运营数据属于「重要数据」，受数据安全法保护
  · 不能传到境外，最好不出厂区
  → 本地部署是核心卖点，不是妥协
```

## 三、借力业界创新公司（不重复造轮子）

### 3.1 对标创新公司矩阵

**Palantir AIP（最重要的对标）**

```
Palantir 做对了什么：
  · 「Ontology」概念——业务对象图谱作为 AI 决策的上下文
    （就是我们的 equipment_id 主线 + GraphRAG 知识图谱）
  · AI 不直接执行，只「建议动作」，人在决策节点
    （就是我们的 TaskFlow HITL）
  · 每个 AI 操作可以追溯到数据源
    （就是我们的 citations 强制机制）

  Palantir 最贵的东西：$8000/用户/年，主要服务政府/国防

我们的策略：
  复制 Palantir 的「AI Action」理念，聚焦油气场站垂直场景，
  价格做到 1/10，国产化合规，本地部署
  这是 Palantir 在中国工业场景的「平替」+「垂直深化」版
```

**Samsara（IoT + 移动体验的标杆）**

```
Samsara 做对了什么：
  · IoT 设备 → 云 → 漂亮移动 App 的完整闭环
  · 极其简单的用户体验（司机/操作员也能用）
  · 数据驱动的安全评分（可量化的价值）

我们借鉴：
  · 移动端（飞书）体验要做到 Samsara 级别的简洁
  · 场站运营「健康分」（0-100，每日更新，可分享给管理层）
  · 「AI 日报」像 Samsara 的「行程报告」一样简洁直观
```

**SparkCognition（工业 AI 预测维护专家）**

```
SparkCognition 做对了什么：
  · 专注压缩机/涡轮机的预测性维护 AI
  · Darwin AI：AutoML 为工业场景生成预测模型
  · 已证明：预测准确率 > 90%，客户停机减少 70%

我们借鉴：
  · MOIRAI 2.0 覆盖他们的时序分析功能（开源，免费）
  · 聚焦相同的设备类型（压缩机/阀门/仪表）
  · 建立「设备健康预测」标准化输出（RUL + 置信度 + 根因）
```

**Scale AI（数据标注即护城河）**

```
Scale AI 做对了什么：
  · 把「数据标注」做成核心竞争力和商业模式
  · RLHF 数据集成为 AI 公司的必需品
  · 军事/自动驾驶领域的数据资产估值超过软件产品本身

我们借鉴（最重要）：
  ★ 把「工业工单标注」当做核心战略资产来运营
  · 每个客户场站的工单 = 标注好的工业 AI 训练数据
  · 建立标准化的「工单标注协议」（与 Scale AI 的标注规范同级）
  · 未来：领域模型可以独立销售/授权
  · 第一方标注数据 → 第三方 AI 公司愿意付钱买
```

**Uptake / SparkCognition / Seeq（工业 AI 分析三巨头）**

```
共同的经验教训：
  · 纯分析工具没有「执行闭环」，用户觉得「看了但没用」
  · 必须要有「建议 → 批准 → 执行 → 验证」的完整闭环
  · 我们已经通过 TaskFlow HITL 解决了这个问题 ✅

Seeq（时序分析专家）的开源生态：
  · seeq-python：时序数据操作库，可直接用
  · 我们可以用 Seeq 的概念设计「工况分析」页面
```

**Anduril（AI 原生自主系统，2029 的路线图）**

```
Anduril 的 Lattice AI 平台：
  · 传感器融合 → 态势感知 → AI 决策 → 自主执行
  · 这就是我们的「工业大脑」的国防版

对我们的启发：
  · 现在建立的 equipment_id 主线 + 数字孪生 =
    未来机器人/无人机的「作战地图」
  · Spot 机器人巡检：需要知道场站每个设备的位置和状态
  · 我们的 Ditto 数字孪生就是这张地图
  · 今天做好基础，2029 年接入机器人时无缝对接
```

**Weights & Biases（MLOps 数据飞轮）**

```
W&B 做对了什么：
  · 让 AI 模型的训练过程可观测、可追溯、可复现
  · 实验跟踪 → 版本管理 → 团队协作

我们直接用：
  · W&B 开源版（Wandb）跟踪 Qwen3.6 LoRA 微调实验
  · 记录每次微调的训练数据量 + 评估指标 + 模型版本
  · 向客户展示「AI 成长曲线」（数据量 vs 准确率）
```

## 七、批判性审视：这个方案最可能在哪里失败

### 7.1 三个致命弱点和对策

**弱点 1：「AI 说错了怎么办？」（信任问题）**

```
风险：AI 给了错误建议 → 工程师执行 → 发生事故
      → 客户起诉，公司声誉崩塌

对策：
  ① 所有 AI 输出必须有 citations（可追溯）
  ② HITL 强制人审批（AI 永远是「建议」，人负最终责任）
  ③ 风险分级：低风险可直接执行，高风险必须二次确认
  ④ AI 免责声明 + 法务条款（学习 Palantir 的合同范本）
  ⑤ 置信度低于 70% → 不推送，转「请人工评估」

但关键是：SCADA 也会给出错误读数，人工判断也会失误
          我们的系统让错误「可审计」，而不是「更多错误」
```

**弱点 2：「我们公司只有 5 个工程师，撑不起 10 个客户」（人力瓶颈）**

```
风险：每个客户上线需要大量手工录入（台账/知识库/3D）
      规模化时服务成本超过收入

对策：
  ① 自动化台账录入（P&ID OCR + Qwen3.6-VL，人工只做审核）
  ② 知识库自动摄入（PDF 批量处理，LlamaIndex 流水线）
  ③ 3D 场景参数化生成（设备类型库 → 自动布局）
  ④ 远程实施（不需要现场，全程飞书/Teams 远程）

  目标：单客户上线成本从 3 人/月 → 1 人/10 天
```

**弱点 3：「客户不续费，因为感受不到 AI 价值」（留存问题）**

```
风险：AI 没有产生明显 ROI → 第二年不续费

对策（ROI 可视化）：
  ① 系统首页「AI 价值记录」：防损总金额 / 节省工时 / 成功预警次数
  ② 每季度「AI 价值报告」（自动生成，发给客户管理层）
  ③ AI 准确率趋势图（让客户看到 AI 在成长）
  ④ 每次成功预警 → 立即推送「本次防损 ¥XX 万」

  关键：让「看不见的价值」变成「看得见的数字」
```

## 九、最终判断

```
这个项目值得做吗？

投资人视角：✅ 值得
  · 市场大，窗口开，竞争空白，技术就绪
  · 护城河是「数据 + 知识图谱」，不是代码（代码可以复制，数据不行）
  · 估值逻辑清晰：ARR × 10-20 + 领域数据资产

用户视角：✅ 愿意买单
  · 解决的痛点真实存在（停机损失可量化）
  · 体验足够简单（飞书 + 12 秒审批）
  · 数据安全（本地部署，数据不出厂）
  · ROI 说得清楚（1 次防损 = 全年软件费）

最关键的一句话：
  这不是在和西门子竞争工业软件市场，
  而是在建立中国油气场站领域的「AI 原生知识基础设施」。

  软件会过时，数据不会。

  每一个场站上线，数据飞轮就快一点，
  每一条被验证的工单，护城河就深一点，
  每一次准确预警，客户粘性就强一点。

  这是一个越做越值钱的生意。
```
