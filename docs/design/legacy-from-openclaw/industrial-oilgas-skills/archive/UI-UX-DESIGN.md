# ClawTwin UI/UX 设计规范

## 对标 Palantir Foundry + AIP 的工业智能决策界面

> 适用范围：ClawTwin Studio（Web/Tauri）、ClawTwin Command（大屏）  
> 参照系：Palantir Gotham（情报地图）、Palantir Foundry（对象平台）、Palantir AIP（AI 行动）  
> 原则：**对象即中心 · 空间即情报 · AI 即行动 · 决策留痕**

---

## 一、设计哲学：从「仪表盘」到「决策指挥系统」

### 1.1 Palantir 给我们的核心启示

Palantir 的 Gotham 系统最根本的创新不是"好看的地图"，而是建立了一套**操作员认知框架**：

```
传统工业软件的认知模型：
  「我去哪个菜单找我需要的数据？」
  数据分散在 SCADA / ERP / CMMS / 报表 → 操作员在多系统间切换

Palantir Gotham 的认知模型：
  「我关心的那个目标/对象现在怎样了？」
  点击对象 → 对象的全部情报自动汇聚 → 决策立即可达

ClawTwin 的认知模型（工业场景）：
  「这台设备现在怎样了？我该怎么做？」
  点击设备 → 实时状态 + 历史趋势 + AI 诊断 + 相关工单全部到场
  AI 建议行动 → 一键审批 → 闭环记录 → 经验沉淀
```

### 1.2 五个设计铁律

```
铁律 1：3D/地图是唯一的信息入口
  所有告警、工单、AI 建议，都锚定到 3D 空间中的物理设备
  操作员永远从「看到什么」开始，而不是从「去哪里找」

铁律 2：AI 是上下文的，不是对话框的
  AI 解读随「选中设备」自动触发，不需要用户手动提问
  工单草稿在用户点击「建工单」时已经填好，不是空表单
  AI 建议附带证据（citations），不是裸判断

铁律 3：行动是一键的，决策是有痕的
  从「发现问题」到「发起工单」不超过 3 次点击
  每个行动都有完整的操作人/时间/AI建议/审批人记录

铁律 4：界面按角色收缩，不是按功能展开
  操作员看到的是：当前设备状态 + 我的待办
  主管看到的是：场站全局 + 待审批队列
  工程师看到的是：设备详细数据 + 知识库 + 趋势分析

铁律 5：时间是第五维度
  底部时间轴让操作员能"穿越"到历史任意时刻查看状态
  所有数据显示当前还是历史，始终清晰标注
```

```
  · 每台设备是一个「发光球体/方块」，颜色 = 状态（PBR材质，HDRI光照）
```

## 四、时间轴（底部 Timeline）

这是 Palantir 的「时序维度」——让操作员能"穿越"到历史时刻：

```
底部时间轴（40px 高度，全宽）：

┌─────────────────────────────────────────────────────────────────────┐
│ ◀ │ ████████████████████████████████░░░░░░░░░░░░░ │ 14:30 │ ▶ 实时 │
│   │ ↑↑                              ↑              │        │        │
│   │ 事件标记（红色尖峰=告警/橙色=P2） 拖拽游标       │        │        │
│   │                                                │        │        │
│ [1h] [6h] [24h] [7d] [30d]          [跳转时间点]   │        │ 回到实时│
└─────────────────────────────────────────────────────────────────────┘

时间轴行为：
  · 拖动游标 → 3D 场景中所有设备显示历史状态（颜色/数值都是历史的）
  · 红色/橙色竖线 = 历史告警发生点，点击高亮该事件
  · 右侧「▶ 实时」按钮 = 跳回当前实时状态
  · 历史模式下，顶栏显示「[历史模式: 2025-05-06 14:30]」提示条
  · Phase A 实现：滑动时间轴，显示历史分钟级数据
```

### 6.2 告警响应决策树

```
告警触发时，ClawTwin 的 AI 决策树（不是全自动，是辅助人决策）：

P1 告警（振动 > 5.0 mm/s）：
  ① Platform 立即推送飞书卡片给值班主管
  ② Studio R 区：AI 解读自动生成（不等用户点）
  ③ 工单草稿自动创建（DRAFT 状态）
  ④ 操作员看到：情报 + 草稿 + 一键确认
  → 人决策：确认工单 → 提交 → 主管审批

P2 告警（温度 78-84°C）：
  ① Studio R 区：黄色告警卡片
  ② AI 解读：「当前趋势，预计 X 小时达到 P1」
  ③ 不自动建工单，由操作员判断
  → 人决策：继续观察 或 手动建工单

MOIRAI 预测告警（预测 6h 内超阈值）：
  ① Studio R 区：「预测提醒」蓝色卡片
  ② AI 解读：「基于 MOIRAI 预测，置信度 XX%」
  ③ 建议：「是否提前安排预防性检修？」[建工单]
  → 人决策：是否建预防工单（此为核心价值：事前而非事后）
```

```
  · 悬浮 Tooltip：过去 1 小时趋势迷你图
```

### 9.2 AIInsightCard（AI 情报卡片）

```
结构：
┌─────────────────────────────────────┐
│ ✦ AI 情报解读    [展开] [复制]       │  ← 头部（紫色标识）
├─────────────────────────────────────┤
│ 「振动从 3 天前持续上升...」          │  ← 主判断（流式输出）
│                                     │
│ 证据链：                             │
│ • 数据: 振动 3.2→4.8 mm/s (3天)    │  ← 数据 citation
│ • 预测: MOIRAI 置信度 87%           │  ← AI 模型来源
│ • 案例: WO-031 (2024-03-15)        │  ← 历史 citation（可点击）
│ • 规程: GB/T 7777 §4.3             │  ← 知识 citation（可点击）
└─────────────────────────────────────┘

关键：citation 必须可点击，点击跳转到原始来源。
这是 Grounding 原则的 UI 落实。
```

### 9.3 ActionButton（行动按钮层级）

```
主要行动（Primary）：   实心蓝   「✅ 用 AI 草稿建工单」
确认行动（Confirm）：   实心绿   「审批通过」
危险行动（Danger）：    实心红   「紧急停机（仅主管）」
次要行动（Secondary）： 边框蓝   「保存草稿」「查看历史」
文字行动（Ghost）：     纯文字蓝 「展开完整分析」「相关文档」

行动按钮原则：
  · 每个情报卡片最多 2 个 Primary 按钮
  · 危险操作必须有二次确认 Modal
  · AI 建议的行动用 ✅ 前缀，区别于用户主动发起
```

## 十二、错误状态、空状态、加载状态设计（架构师视角）

> 这些"边界状态"决定系统的专业感和可信度。工业场景用户对系统稳定性极度敏感。

### 12.1 加载状态（数据未到达时）

```
原则：不显示空白或 Loading 转圈，使用骨架屏（Skeleton），让用户感知到「数据在路上」

设备列表骨架屏（L 区）：
  ░░░░░░░░  ◯◯◯  ─────────────
  ░░░░░░░░  ◯◯◯  ─────────────
  ░░░░░░░░  ◯◯◯  ─────────────

3D 场景加载状态：
  · 底部加载进度条（细线，Primary 蓝色）
  · 文字：「正在加载场景... 设备模型 3/16」
  · 不要全屏 loading 遮罩（太粗暴）

AI 情报生成中：
  · AIInsightCard 显示「✦ AI 正在分析...」
  · 有打字机光标效果（表明 AI 在思考）
  · 同时，MetricBar 已经显示（AI 不阻塞实时数据）

实时数据首次加载超过 3 秒：
  · 显示「数据加载中...（Platform 响应时间 Xs）」
  · 不要让用户看到空白数字
```

### 12.2 错误状态

```
错误类型分级：

E1 - 数据源超时（Platform 无响应）：
  · 顶栏：「Platform ● 连接中断 | 显示最后更新数据（14:30）」
  · 所有指标：数值显示为灰色，附小标「⚠ 数据 15 分钟前」
  · 不自动刷新死循环，30 秒后提示「重新连接」按钮
  · 不推送告警（数据来源不可信时，不发假告警）

E2 - AI 推理超时（vLLM 无响应）：
  · AIInsightCard：「AI 分析暂时不可用（响应超时）」
  · 降级方案：显示规则引擎的简单判断「振动超预警阈值」（无 citations）
  · 实时指标、工单、历史数据不受影响（AI 是可选增强，不是必须）
  · 标注「数据正常，AI 服务临时中断」

E3 - 单台设备数据丢失（OPC-UA 断点）：
  · 该设备状态变为「⚫ 离线/数据中断」
  · L 区该设备显示灰色，移到列表底部
  · R 区情报面板：「C-001 数据采集中断（从 14:45 起）」
  · 不推送告警（避免采集故障误报）
  · 飞书推送：「C-001 数据采集中断，请检查 opcua-bridge」（推给 IT 运维群）

E4 - 表单提交失败（工单提交失败）：
  · 不丢失用户填写的内容（草稿保存在 localStorage）
  · 提示：「工单提交失败（网络异常），草稿已保存，请稍后重试」
  · [重试] 按钮明显，不需要重新填写
```

### 12.3 空状态（首次使用 / 没有数据）

```
场站刚接入，设备数据为零：
  3D 区域：显示场站建筑轮廓（灰色），设备位置用虚线框标记
  文字：「场站设备正在接入，期望看到 16 台设备」
  操作引导：[导入设备清单] [查看接入文档]

知识库为空（L0/L1 未初始化）：
  知识库页面：空状态图 + 「还没有行业知识，AI 推理将缺乏依据」
  操作引导：[使用行业知识包（推荐）] [手动上传文档]
  行业知识包：一键导入，包含 200+ 油气行业标准文档

工单列表为空：
  工单看板：「今日没有工单，设备运行正常 🟢」
  不显示空状态图，简洁清晰即可

AI 无法分析（知识库和历史数据不足）：
  AIInsightCard：「当前知识和历史数据不足以给出建议。
                  设备运行 7 天后 MOIRAI 基线才能建立。
                  知识库建议先导入行业标准（L0/L1 层）。」
  → 不给出无依据的判断，宁可说「不知道」
```

### 12.4 乐观更新与确认态（减少等待感）

```
工单状态更新的乐观更新策略：

操作员点「标记完成」后：
  ① 立即：工单卡片变灰，显示「正在提交...」（乐观更新，不等服务端）
  ② 1-2 秒后：服务端确认 → 卡片更新为「✓ 已完成 · 11:32」
  ③ 若失败：恢复原状态，显示「提交失败，请重试」

原则：
  · 读操作：优先显示缓存数据，后台刷新
  · 写操作：乐观更新 + 失败回滚，不让用户等空白
  · 状态变更（审批/拒绝）：立即禁用按钮，不可重复点击
```

### 16.3 P&ID 技术实现规范

**数据格式**：DEXPI XML（ISO 15926-2 兼容）作为底层数据格式

```json
// P&ID 设备节点 JSON（react-flow node）
{
  "id": "C-001",
  "type": "compressor",
  "position": { "x": 400, "y": 200 },
  "data": {
    "label": "压缩机 C-001",
    "equipment_id": "C-001",
    "design": { "pressure": "6.0 MPa", "flow": "150 km³/d" },
    "realtime": { "pressure": 5.8, "flow": 142.3 },
    "status": "warn", // normal | warn | alarm | offline
    "alerts": ["PIC-001 偏高 5.2MPa"]
  }
}
```

**交互行为**：

```
点击设备符号 → 右侧 R 区自动切换为该设备情报页（与 3D Twin 视图联动）

Ctrl+F → 搜索设备号（自动定位并高亮）
```

**与实时数据的集成**：

```
每 10 秒：Platform API /v1/readings/latest?equipment_ids=... 批量拉取
实时值叠加在仪表符号旁（小标签）
状态着色：green(normal) / yellow(warn) / red(alarm) / gray(offline)
闪烁动画：告警状态的设备符号边框以 1Hz 闪烁（吸引注意）
```

```typescript
export function UrgencyCountdown({ predicted_breach_minutes, threshold_label, confidence }: UrgencyCountdownProps) {
```

### 20.3 振动频谱可视化组件（默会知识外显化）

基于 2025 学术研究（HORAI 论文）：振动频谱比原始时序值携带更多故障类型信息。

```typescript
// src/components/intelligence/SpectrogramView.tsx

import { useEffect, useRef } from "react";
import { api } from "@/lib/api";

export function SpectrogramView({ equipmentId }: { equipmentId: string }) {
  const canvasRef = useRef<HTMLCanvasElement>(null);
  const [spectrumData, setSpectrumData] = useState<{ freq: number; amplitude: number }[]>([]);
  const [aiInterpretation, setAiInterpretation] = useState<string>("");

  useEffect(() => {
    api.get(`/v1/equipment/${equipmentId}/spectrum?window=60s`).then(r => {
      setSpectrumData(r.data.spectrum);
      setAiInterpretation(r.data.ai_interpretation);
    });
  }, [equipmentId]);

  // 绘制频谱图（Canvas）
  useEffect(() => {
    if (!canvasRef.current || spectrumData.length === 0) return;
    const ctx = canvasRef.current.getContext("2d")!;
    const { width, height } = canvasRef.current;

    ctx.clearRect(0, 0, width, height);
    ctx.fillStyle = "#0D1117";
    ctx.fillRect(0, 0, width, height);

    const maxAmp = Math.max(...spectrumData.map(d => d.amplitude));
    const maxFreq = Math.max(...spectrumData.map(d => d.freq));

    // 绘制频谱柱
    spectrumData.forEach((d, i) => {
      const x = (d.freq / maxFreq) * (width - 40) + 20;
      const barH = (d.amplitude / maxAmp) * (height - 40);

      // 高亮异常频率（超过均值 2σ 的峰值）
      const mean = spectrumData.reduce((s, p) => s + p.amplitude, 0) / spectrumData.length;
      const std = Math.sqrt(spectrumData.reduce((s, p) => s + (p.amplitude - mean) ** 2, 0) / spectrumData.length);
      const isAnomalous = d.amplitude > mean + 2 * std;

      ctx.fillStyle = isAnomalous ? "#EF4444" : "#1F6FEB";
      ctx.fillRect(x - 1, height - 20 - barH, 2, barH);
    });
  }, [spectrumData]);

  return (
    <div className="border border-[#30363D] rounded-lg overflow-hidden text-xs">
      <div className="flex items-center justify-between px-3 py-1.5 bg-[#161B22] border-b border-[#21262D]">
        <span className="text-[#8B949E]">📊 振动频谱分析</span>
        <span className="text-[#6B7280]">实时 · 60s 窗口</span>
      </div>
      <canvas ref={canvasRef} width={280} height={80} className="w-full bg-[#0D1117]" />
      {aiInterpretation && (
        <div className="px-3 py-2 bg-[#0D1117] border-t border-[#21262D]">
          <span className="text-[#8B5CF6]">🤖 </span>
          <span className="text-[#C9D1D9]">{aiInterpretation}</span>
        </div>
      )}
    </div>
  );
}
```

### 22.5 全局搜索（Command Palette）— Palantir 标志性交互

```typescript
// src/components/layout/CommandPalette.tsx
// 触发：Cmd+K（Mac）/ Ctrl+K（Win）

import { useState, useEffect, useRef } from "react";
import { useTwinStore } from "@/stores/twin.store";

export function CommandPalette() {
  const [open, setOpen] = useState(false);
  const [query, setQuery] = useState("");
  const [results, setResults] = useState<SearchResult[]>([]);
  const inputRef = useRef<HTMLInputElement>(null);
  const { setSelectedEquipmentId, setActiveCenterView } = useTwinStore();

  // Cmd+K 快捷键
  useEffect(() => {
    const handler = (e: KeyboardEvent) => {
      if ((e.metaKey || e.ctrlKey) && e.key === "k") {
        e.preventDefault();
        setOpen(o => !o);
      }
      if (e.key === "Escape") setOpen(false);
    };
    window.addEventListener("keydown", handler);
    return () => window.removeEventListener("keydown", handler);
  }, []);

  // 打开时聚焦
  useEffect(() => {
    if (open) setTimeout(() => inputRef.current?.focus(), 50);
  }, [open]);

  // 实时搜索（防抖 200ms）
  useEffect(() => {
    if (!query.trim()) { setResults([]); return; }
    const t = setTimeout(() => {
      api.get("/v1/search", { params: { q: query, limit: 8 } })
        .then(r => setResults(r.data.results));

    return () => clearTimeout(t);

    if (result.type === "equipment") {

      setActiveCenterView("twin");
    } else if (result.type === "workorder") {
      setActiveCenterView("kanban");
    } else if (result.type === "alarm") {
      setSelectedEquipmentId(result.equipment_id);
    }
    setOpen(false);
    setQuery("");

  if (!open) return null;

  return (

          <span className="text-[#8B949E]">🔍</span>
          <input
            ref={inputRef}
            value={query}
            onChange={e => setQuery(e.target.value)}
            placeholder="搜索设备、工单、告警、知识…"
            className="flex-1 bg-transparent text-[#E6EDF3] text-sm outline-none

          />
          <kbd className="text-[10px] text-[#6B7280] bg-[#21262D] px-1.5 py-0.5 rounded">
            ESC
          </kbd>
        </div>

        {/* 搜索结果 */}
        <div className="max-h-80 overflow-y-auto">
          {results.length === 0 && query && (
            <div className="p-4 text-sm text-[#6B7280] text-center">无结果</div>
          )}
          {results.length === 0 && !query && (
            <div className="px-4 py-3">
              <p className="text-[10px] text-[#6B7280] uppercase tracking-wide mb-2">快捷跳转</p>

                  setOpen(false);
                }} />
              ))}
            </div>
          )}
          {results.map(result => (
            <SearchResultRow key={result.id} result={result} onSelect={() => handleSelect(result)} />

        </div>
      </div>
```

### 22.9 响应式断点规范（ClawTwin 只支持桌面 + 飞书移动）

> ClawTwin Studio 不做通用移动端，仅做：
>
> 1. 大屏（≥ 1440px）：全五区布局
> 2. 普通桌面（1024-1440px）：NavRail 窄化为 200px
> 3. 飞书移动端（< 768px）：只用飞书 App，不用 Studio Web

```
breakpoint    NavRail    CenterPanel    IntelPanel
≥ 1440px      240px      flex-1         360px
1280-1440px   200px      flex-1         320px
1024-1280px   180px      flex-1         300px（自动折叠）
< 1024px      禁止访问，引导至飞书移动端
```

**实现**（`StudioShell.tsx` 断点检测）：

```typescript
// 小屏检测 + 引导（< 1024px 用户）
function MobileGuard({ children }: { children: React.ReactNode }) {
  const isSmall = useMediaQuery("(max-width: 1023px)");
  if (isSmall) {
    return (
      <div className="h-screen flex flex-col items-center justify-center bg-[#0D1117] p-8 text-center">
        <p className="text-4xl mb-4">📱</p>
        <h2 className="text-[#E6EDF3] text-lg font-semibold mb-2">请使用桌面端</h2>
        <p className="text-[#8B949E] text-sm">
          ClawTwin Studio 为桌面大屏优化设计。<br/>
          在手机上请使用飞书 App 访问 ClawTwin Bot。
        </p>
      </div>
    );
  }
  return <>{children}</>;
}
```
