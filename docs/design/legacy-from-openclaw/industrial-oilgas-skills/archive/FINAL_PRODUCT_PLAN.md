**版本**：4.0 · 2026-05-08（基于真实模型版本修订）  
**GPU**：200B 以下任意可用  
**主模型**：Qwen3.6 家族（文字 + 图像原生多模态）

**战略目标**：2026 出货 → 2027 数据飞轮 → 2029 领域大模型

## 一、模型选型定稿（精简正确版）

### 1.1 Qwen3.6 家族（主力栈）

Qwen3.6 是 2026 年 4 月发布的最新一代，**原生支持文字 + 图像**，不需要单独的视觉模型：

| 模型                | 参数 / 激活          | 能力                                    | 工业用途                                   | GPU 需求（FP16）            |
| ------------------- | -------------------- | --------------------------------------- | ------------------------------------------ | --------------------------- |
| **Qwen3.6-35B-A3B** | 35B 总 / **3B 激活** | MoE 稀疏；文字+图像；Agent 编码；推理快 | **日常主力**：工单/规程/P&ID 解析/铭牌识别 | 2×A100 40G（INT4 单卡可跑） |
| **Qwen3.6-27B**     | 27B dense            | 深度推理；无图像                        | **重型推理**：合规审查/多步根因/报告生成   | 2×H100 80G（INT4 单 H100）  |
| **Qwen3.5-Omni**    | ~7B                  | 文字+图像+音频+视频；流式语音           | 备用全模态（如飞书语音处理不足时）         | 1×A100 40G                  |

**核心判断**：

- Qwen3.6-35B-A3B 推理时只用 3B 激活参数 → **速度接近 3B 模型，能力接近 35B**
- 图像能力内置，P&ID 解析、铭牌 OCR、设备照片识别 → **不需要单独的视觉模型**
- Apache 2.0 许可，可商用私有部署

### 1.2 时序大模型（传感器异常检测）

| 模型           | 版本                   | 特性                                            | 用途                        |
| -------------- | ---------------------- | ----------------------------------------------- | --------------------------- |
| **MOIRAI 2.0** | 2025-08 / 论文 2026-02 | 比 1.0 快 44%、小 96%；解码器架构；GIFT-Eval #1 | OPC UA 传感器零样本异常检测 |

MOIRAI 2.0 参数极小，**CPU 即可运行**，不占 GPU 资源。

### 1.3 图像分割（摄像头场景可选）

| 模型                | 许可       | 用途                                               |
| ------------------- | ---------- | -------------------------------------------------- |
| **SAM 2.1**（Meta） | Apache 2.0 | 摄像头画面设备区域分割（Phase 3 引入，当前不需要） |

### 1.4 声音交互：完全依托飞书

```
飞书原生能力（无需自建 ASR/TTS）：
  · 飞书语音消息 → 飞书服务端自动转文字 → Webhook 传给 OpenClaw
  · 飞书语音通话 → 实时字幕（飞书 AI）→ 用户提问文字传入
  · 飞书 AI 助手（飞书自带）→ 可作为轻量问答入口
  · AI 回答 → 富文本卡片（非语音播报，减少复杂度）

不需要部署：Whisper / FunASR / CosyVoice
结论：飞书语音能力足够覆盖现场工程师场景，
      专注 OpenClaw + 飞书卡片的工作流设计即可
```

### 1.5 GPU 资源分配（最优方案）

```
配置：4×H100 80G SXM（或等效，总 320G VRAM）

  GPU 0-1（160G）：Qwen3.6-35B-A3B FP16 × 2 实例
                   → 24/7 在线；主力推理；图像+文字
  GPU 2（80G）：   Qwen3.6-27B INT4
                   → 按需启动；重型推理任务
  GPU 3（80G）：   训练专用（LoRA fine-tuning，阶段性使用）
                   → 平时空闲时运行 MOIRAI 评测 / 新模型测试

  CPU 服务器：     MOIRAI 2.0（极小模型，CPU 足够）
                   Jena Fuseki / Milvus 数据库服务

  备注：若仅有 2×H100，优先 Qwen3.6-35B-A3B INT4（单 H100 可行）
        INT4 量化精度损失 <1.5%，推理速度提升 2-3x
```

## 三、Phase 1（2026 Q2，12 周）：飞书 AI 助手 MVP

### 目标：1 个客户 · 30 台关键设备 · 飞书每天用

**只做这 4 件事**，其他全部推迟：

```
交付物 1：Qwen3.6-35B-A3B 私有部署
  · Docker + vLLM 部署包（客户内网一键启动）
  · 连接 OpenClaw Gateway
  · 基础工业系统提示词（角色：场站 AI 助手；安全边界；引用要求）

交付物 2：飞书 AI 晨报（每天 6:00 自动推送）
  · LangGraph 定时 Agent，每天凌晨聚合数据
  · 内容：昨日异常汇总 · 今日待处理事项 · 设备健康摘要
  · 格式：飞书富文本卡片（有按钮：「批准」「推迟」「查看详情」）
  · 数据来源：手动维护的 station-data.json（不需要真实 OPC UA）

交付物 3：工单 AI 助手（飞书对话触发）
  · 用户飞书消息：「SDV-001 需要 PST 测试」
  · AI：检索规程（RAG）→ 生成工单草稿 → 发飞书审批卡片
  · 审批人点「批准」→ 工单记录写入（文件存储，不需要 CMMS）
  · 审批链路：提交人 → 技术负责人 → 归档

交付物 4：设备问答（飞书对话）
  · 用户：「告诉我 PRV-030 的维修历史」
  · AI：查 station-data.json + RAG 规程库 → 回答 + 引用来源
  · 支持中文语音消息（飞书自动转文字后送入 OpenClaw）
```

**12 周计划**：

```
Week 1-2：环境搭建
  · Qwen3.6-35B-A3B vLLM 部署（Docker Compose）
  · OpenClaw Gateway + 飞书桥接验证
  · Milvus + Jena Fuseki 启动

Week 3-4：数据建立
  · station-data.json（30 台设备手工录入，或 P&ID 照片 → Qwen3.6 提取）
  · L0 知识库：相关 GB/SY 标准 3-5 份 → 分块向量化
  · L3 知识库：客户提供的设备手册 / 操作规程 5-10 份

Week 5-8：核心功能开发
  · LangGraph 三 Agent（晨报 / 问答 / 工单）
  · 飞书卡片模板（晨报 / 告警 / 工单审批）
  · RAG 检索服务（citations 强制）
  · 工单 HITL 流程（提交 → 审批 → 归档）

Week 9-10：客户内网部署
  · Docker Compose 一键安装（目标：1 天完成部署）
  · 数据迁移脚本（客户提供 Excel 台账 → station-data.json）
  · 用户培训（1 次，半天）

Week 11-12：上线验证
  · 飞书晨报连续推送 7 天
  · 工单完整走通 3 条（从 AI 草稿到归档）
  · 用户反馈收集 → 快速修复
  · 签署年度续费合同
```

**最小技术栈（Phase 1 只需要这些）**：

```
推理：       Qwen3.6-35B-A3B（vLLM）
知识：       Milvus（向量）+ Jena Fuseki（本体）+ MinIO（文档）
Agent 编排： LangGraph 0.2+
飞书集成：   OpenClaw Gateway + 飞书桥接（已有）
数据：       station-data.json + SQLite（工单记录）
部署：       Docker Compose（6 个服务，单机可跑）
监控：       Grafana（基础日志，非实时工况）
```

**Phase 1 成功标准**：

```
✅ 飞书晨报连续推送 ≥ 14 天无中断
✅ 工单 AI 草稿准确率 ≥ 60%（客户填写满意度问卷）
✅ 规程查询命中相关段落 ≥ 70%
✅ 客户签署年度合同（¥15-30 万）
```
