# 一、建设定位

---

## 要解决的三个问题

- **多入口**：状态与工单口径易散
- **规程难证**：口头多、难打开依据
- **闭环难追**：告警—处置—审批断链

→ **不换核心生产系统**，增加可治理的协同层

---

## 建设在企业中的位置

| 客户已有             | 本期建设                            |
| -------------------- | ----------------------------------- |
| IMS、SCADA、IM、身份 | **语义与协同中枢** + 指挥中心 Web   |
| 工艺与安全责任不变   | 助手与 Web 只处理**授权内业务对象** |

---

## 产品构成（能力包）

| 层       | 交付内容                                            |
| -------- | --------------------------------------------------- |
| **终端** | 指挥中心 Web · **IM + 卡片** · 企业 AI 助手（可选） |
| **中枢** | 语义与对象服务 · 工单协同 · 规程与知识 · **连接器** |
| **横切** | 场站/角色权限 · **审计** · 与统一身份对接           |

## 建设框架 · 三条业务链路（逻辑核对）

| 链路                       | 怎么走（用户可见）                                         | 与设计文档一致要点                                |
| -------------------------- | ---------------------------------------------------------- | ------------------------------------------------- |
| **① 人在 IM 里「问、办」** | 员工对话进 **企业 AI** → AI 经 **MCP** 调平台工具取数/建单 | **会话不进**平台做语义解析；平台 **只认工具调用** |
| **② 人在 IM 里「点卡片」** | 按钮/表单 → **直达平台**（签名回调）                       | 对应 `POST /v1/feishu/events`；**与①分流**        |
| **③ 人在指挥 Web**         | 浏览器登录 → 平台 **JWT** → `/v1/*`                        | Studio **不走** Agent 绕路                        |

**硬边界**：**不向 DCS/PLC 下控制**；OT 进入平台的仅是 **信息侧** 汇聚（只读/策略由贵司网信定）。

## 建设与集成关系（提要）

| 对接                | 说明                                                |
| ------------------- | --------------------------------------------------- |
| **IMS ↔ 平台**      | 设备/工单等 **映射或同步**，权威以 **SoT 书面**为准 |
| **OT → 平台**       | 过程量与告警源 **单向只读**（网络策略由贵司定）     |
| **IM ↔ 用户**       | **① 对话**走 AI；**② 卡片**回调平台（见上表）       |
| **AI ↔ 平台**       | Service Token → **MCP**，与 REST **同一对象模型**   |
| **指挥 Web ↔ 平台** | 用户 **JWT** → **`/v1/*`**                          |

| **可信** | 答复 **可点开规程/记录**，非空口「模型口述」    |

## 三场必选（D1 · D2 · D3）

| 编号   | 演什么              | 证明点             |

| **D1** | 告警→规程→工单→结案 | **闭环**可追溯     |

---

## 三场演示 · 一眼对齐

|            | **D1 闭环** | **D2 有据**            | **D3 锁住**        |
| ---------- | ----------- | ---------------------- | ------------------ |
| **金标准** | 人对上号    | IM 与指挥台 **无矛盾** | 越权 **全拒+留痕** |
| **最怕**   | 状态对不上  | **无引用**瞎答         | 能看见他站数据     |

---

## D1｜告警 → 规程 → 工单 → 结案

**观众**：站长、值班长、工程师、安监

1. 告警**绑定设备**出现
2. 值班员 **确认**（人、时进审计）
3. IM 问规程 → **可点开引用**
4. **工单** → 主管 **审批** → 执行 **回填** → **关闭**

**算过**：状态变化与责任人对上号

---

## D2｜问设备一站式

**观众**：一线、工程师

- 只问 **约定装置号**：状态摘要、未结工单、近期已确认告警
- 再问「上次类似怎么修」→ **历史/知识 + 引用**

**算过**：与指挥中心**同一口径**；无引用胡编 = 失败

---

## D3｜权限与场站

**观众**：网信、安监

- A 仅 **S1 站**；用 **B 站装置/工单** 试探 → **拒绝**
- **有日志可查**

**算过**：试探用例 **全拦截**

## MCP 工具（Agent 可调 · 与 REST 同源模型）

`get_equipment_context` · `get_station_overview` · `list_equipment` · `get_equipment_readings` · `get_active_alarms` · `search_knowledge_base` · `create_work_order` · `get_work_order`

## 集成与数据主权（SoT）

| 数据/动作              | 典型权威（由企业定）            |
| ---------------------- | ------------------------------- |
| 设备主数据、正式工单号 | 多为 **IMS**                    |
| 过程量、告警源         | **工控/采集域**（只读进协同层） |

## 常见疑问（简答）

| 问                       | 答                                                              |
| ------------------------ | --------------------------------------------------------------- |
| **会动工艺 / DCS 吗？**  | **不会**。建设在信息链与协同，**不下发控制指令**。              |
