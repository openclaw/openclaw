## 一、三个入口，各有分工

作为运营人员，你会与 ClawTwin 通过三个入口交互：

```
┌──────────────────────────────────────────────────────┐
│   入口                用途                 打开方式     │
├──────────────────────────────────────────────────────┤
│  飞书 / 钉钉          日常快速处理          手机/PC 聊天  │
│  ClawTwin Studio     深度调查 / 配置        浏览器        │
│  OpenClaw 对话        自然语言查询/指令      聊天窗口      │
└──────────────────────────────────────────────────────┘
```

**核心原则**：90% 的日常操作在飞书里完成；Studio 用于复杂分析和配置；OpenClaw 用于你不知道从哪里开始的情况。

## 二、最重要的旅程：报警处理与 HITL 审批

### 场景：C-001 泵压异常

**第一步**：你在飞书收到一张交互卡片（不是文字）

```
┌─────────────────────────────────────────────┐
│ 🔴  运营审批请求 — 压力报警自动处理          │
├─────────────────────────────────────────────┤
│ AI 分析摘要                                  │
│ 泵 C-001 轴向振动 9.2 mm/s（阈值 7.0），    │
│ 疑似密封圈磨损。历史案例：2023-08 同型号泵   │
│ 相同症状更换后运行至今。                     │
│                                              │
│ 置信度：**84%**  📍 XX 站场                  │
│                                              │
│ 请根据 AI 建议决策。批准后将自动执行。       │
├─────────────────────────────────────────────┤
│  [✓ 批准执行]           [✗ 拒绝]            │
└─────────────────────────────────────────────┘
```

**第二步**：你直接点击「✓ 批准执行」

系统弹出确认框 → 你点确认 → 飞书显示「已批准，正在执行后续步骤」

**第三步**：Playbook 自动继续

- 创建维修工单 WO-2024-0089
- 通知值班工程师李工
- 你收到工单创建确认通知

**整个过程：30 秒，不需要离开飞书，不需要登录 Studio。**

---

### 如果你需要更多信息再决策

方法 A：问 OpenClaw

```
你：C-001 泵最近一个月振动趋势怎样？
OpenClaw：[调用 get_station_health] 过去 30 天...
          第 23 天开始振动值持续上升，当前 9.2 mm/s...
你：好，批准
```

然后回飞书卡片点批准，或者直接告诉 OpenClaw：

```
你：帮我批准 run-id abc123
OpenClaw：[调用 approve_playbook_run] 已批准 ✓
```

方法 B：在 Studio 查看完整上下文再批准

- 飞书卡片 → 点「在 Studio 中查看」链接（告警通知卡片有此按钮）
- Studio 审批页面有完整的设备历史、传感器曲线、CBR 推荐案例

## 三、日常运营快速命令（通过 OpenClaw）

### 查询类（只需说出来）

```
"今天有多少报警？"          → 调用 get_alarm_summary
"有没有需要我审批的？"       → 调用 list_pending_hitl
"XX 站场现在状态怎样？"      → 调用 get_station_health(station_id=XX)
"C-001 设备当前状态？"       → 调用 get_equipment_status(id=C-001)
"最近一周哪台设备报警最多？" → 调用 list_active_alarms + 分析
```

**示例对话**：

```
你：早上好，今天有什么需要我关注的？
OpenClaw：
  📊 今日状态（截至 08:30）：
  - 待审批：2 条（XX 站 P1×1，YY 站 P2×1）
  - 活跃报警：5 条（P1×1, P2×2, P3×2）
  - 昨夜自动处理：3 条，全部成功 ✓

  建议先处理 XX 站的 P1 审批，需要我拉出详情吗？
```

### 操作类（OpenClaw 帮你执行）

```
"帮 C-001 泵创建巡检工单"  → create_action(type=inspection_work_order)
"把 A-007 报警降级为 P3"   → create_action(type=alarm_severity_adjust)
"触发 XX 站的日常巡检 Playbook" → trigger_playbook(id=daily_inspection)
```

## 六、不同自主级别的体验（L0 - L5）

ClawTwin Playbook 支持 6 个自主级别，你会感受到不同的交互：

| 级别 | 你看到什么                         | 你需要做什么             |
| ---- | ---------------------------------- | ------------------------ |
| L0   | 飞书通知：「设备报警，请人工处理」 | 手动判断 + 手动创建工单  |
| L1   | 飞书通知 + AI 诊断建议             | 看建议，手动执行         |
| L2   | 飞书交互卡片（本次重点）           | 看卡片，一键批准/拒绝    |
| L3   | 飞书通知 + 自动执行结果            | 确认执行结果是否符合预期 |
| L4   | 飞书通知：「已自动处理，点击查看」 | 事后复查                 |
| L5   | 无通知（全自动）                   | 定期查看 Studio 执行日志 |

**建议**：从 L2 开始，积累 1-2 个月 OutcomeEvent 后，P2/P3 报警逐步提升到 L3-L4。

## 七、常见问题

**Q：飞书卡片按钮点击后没反应？**
→ 检查 `CLAWTWIN_FEISHU_CARD_SECRET` 是否正确配置
→ 卡片有效期 10 分钟，超时后需在 Studio 手动审批

**Q：OpenClaw 说「不知道有几个待审批」？**
→ 确认 ClawTwin MCP Server 已正确注册到 OpenClaw
→ 在 OpenClaw 配置页检查 `clawtwin` MCP 连接状态

**Q：我发现 AI 建议错误，应该如何反馈？**
→ 拒绝卡片，在 Studio 中记录 OutcomeEvent（标注真实原因）
→ CBR 系统会将你的决策纳入学习，下次相同情况建议会更准确

**Q：想让某类报警完全自动处理（不用审批）？**
→ Studio > Playbook 编辑器 > 设置 `hitl_auto_approve_if: confidence > 0.90`
→ 建议至少积累 10 次人工确认后再开启自动批准
