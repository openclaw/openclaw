# IBM Z / s390x (LinuxONE)

OpenClaw supports IBM Z systems running the **s390x** architecture, including deployments on IBM LinuxONE Community Cloud. This guide covers the installation workaround and known limitations for this platform.

## What you need

- An IBM Z or LinuxONE system (physical or cloud VM)
- Ubuntu 22.04 LTS (s390x) or a comparable s390x-capable Linux distribution
- Node.js v20 or newer (v24 recommended)
- About 10–15 minutes

## Why the standard install fails

The default `npm install -g openclaw` fails on s390x with an `EBADPLATFORM` error:

```
EBADPLATFORM Unsupported platform for @lancedb/lancedb@latest:
  wanted {"os":"linux","cpu":"s390x"} (current: {"os":"linux","cpu":"x86_64"})
```

This happens because `@lancedb/lancedb` — OpenClaw's optional vector store dependency — does not publish prebuilt binaries for the s390x architecture. OpenClaw core and all non-LanceDB features work normally.

## Install OpenClaw

<Steps>
  <Step title="Verify your architecture">
    Confirm you are running on s390x:

    ```bash
    uname -m
    # Expected output: s390x
    ```

  </Step>

  <Step title="Install or verify Node.js">
    If Node.js is not already installed:

    ```bash
    curl -fsSL https://deb.nodesource.com/setup_24.x | sudo -E bash -
    sudo apt-get install -y nodejs

    node --version   # Should show v24.x.x or v22.x.x
    npm --version    # Should show 10.x.x or higher
    ```

  </Step>

  <Step title="Install OpenClaw with platform bypass">
    Use `--force` to bypass the LanceDB platform check:

    ```bash
    sudo npm install -g openclaw --force
    ```

    The `--force` flag allows installation to proceed even though LanceDB prebuilt binaries are unavailable. npm will show warnings about `@lancedb/lancedb` — these are harmless and can be safely ignored.

  </Step>

  <Step title="Verify the installation">
    ```bash
    openclaw --version
    openclaw gateway status
    ```

    If both commands succeed, OpenClaw is installed and running correctly.

  </Step>

  <Step title="Start the Gateway (optional)">
    To run the Gateway persistently in the background:

    ```bash
    openclaw gateway start
    openclaw onboard --install-daemon
    ```

    For configuring messaging channels such as Telegram or Discord, see [Channels](/channels).

  </Step>
</Steps>

## Known limitations

Because `@lancedb/lancedb` is unavailable on s390x, the following features are affected:

| Feature | Status on s390x | Workaround |
|---------|-----------------|------------|
| Core agent functionality | ✅ Fully working | — |
| Built-in short-term memory | ✅ Fully working | Uses SQLite by default |
| Dreaming (memory promotion) | ✅ Fully working | No LanceDB dependency |
| here.now publishing | ✅ Fully working | No architecture dependency |
| AgentMail | ✅ Fully working | Cloud API, no local dependency |
| LanceDB vector store | ⚠️ Unavailable | Use SQLite fallback or cloud vector services |
| Large-scale knowledge base search | ⚠️ Reduced | Consider OpenAI or Anthropic hosted search |

> **Impact:** For most daily use cases the limitation is transparent. Only users relying heavily on LanceDB-powered large-scale vector search need to seek alternatives.

## Troubleshooting

**"command not found: openclaw" after install**

Check that npm's global bin directory is in your PATH:

```bash
export PATH="$(npm prefix -g)/bin:$PATH"
openclaw --version
```

Add the line to `~/.bashrc` or `~/.profile` to persist it.

**Permission errors (EACCES)**

If you hit permission errors:

```bash
sudo npm install -g openclaw --force
```

Alternatively, configure npm to use a Node version manager (nvm, fnm) to avoid global sudo installs.

**Node.js version too old**

If you see engine version warnings:

```bash
node --version   # Verify this shows v20 or newer
curl -fsSL https://deb.nodesource.com/setup_24.x | sudo -E bash -
sudo apt-get install -y nodejs
```

## IBM LinuxONE Community Cloud

Using the IBM LinuxONE Community Cloud free tier:

1. Spin up an **Ubuntu 22.04 LTS** instance (s390x)
2. Connect via SSH
3. Follow Steps 1–5 above
4. Run `openclaw gateway start` to start the daemon

Your OpenClaw instance works identically to x86_64 deployments for all core features.
