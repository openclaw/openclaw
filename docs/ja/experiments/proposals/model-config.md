# モデル設定（探索）

このドキュメントは、将来のモデル設定に関する **アイデア** をまとめたものです。  
出荷される仕様ではありません。現在の挙動については、以下を参照してください。
配送仕様ではありません。 現在の動作については、次を参照してください。

- [Models](/concepts/models)
- [Model failover](/concepts/model-failover)

## 動機

運用者が求めていることは次のとおりです。

- プロバイダごとの複数の認証プロファイル（個人対仕事）。
- 予測可能なフォールバックを伴う、シンプルな `/model` の選択。
- テキストモデルと画像対応モデルの明確な分離。

## 可能な方向性（高レベル）

- モデル選択はシンプルに保つ：オプションのエイリアスを伴う `provider/model`。
- プロバイダーに明示的な順序を持つ複数の認証プロファイルを持たせる。
- すべてのセッションで一貫してフェイルオーバーするよう、グローバルなフォールバックリストを使用する。
- 明示的に設定された場合にのみ、画像ルーティングを上書きする。
