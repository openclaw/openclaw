# 모델 설정 (탐색)

이 문서는 향후 모델 구성에 대한 **아이디어**를 정리합니다. 이는
출시되는 사양이 아닙니다. 현재 동작에 대해서는 다음을 참고하십시오:

- [Models](/concepts/models)
- [Model failover](/concepts/model-failover)
- [OAuth + profiles](/concepts/oauth)

## 동기

운영자는 다음을 원합니다:

- 프로바이더별로 여러 인증 프로필 (개인용 vs 업무용).
- 예측 가능한 폴백을 갖춘 간단한 `/model` 선택.
- 텍스트 모델과 이미지 기능을 갖춘 모델 간의 명확한 분리.

## 가능한 방향 (개요)

- 모델 선택을 단순하게 유지: 선택적 별칭을 포함한 `provider/model`.
- 프로바이더가 명시적인 순서를 갖는 여러 인증 프로필을 가질 수 있도록 합니다.
- 모든 세션이 일관되게 페일오버되도록 전역 폴백 목록을 사용합니다.
- 명시적으로 구성된 경우에만 이미지 라우팅을 재정의합니다.

## 열린 질문

- 프로필 로테이션은 프로바이더 단위여야 합니까, 모델 단위여야 합니까?
- UI 에서 세션의 프로필 선택을 어떻게 노출해야 합니까?
- 레거시 설정 키에서의 가장 안전한 마이그레이션 경로는 무엇입니까?
