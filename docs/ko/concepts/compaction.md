# 컨텍스트 창 및 압축

모든 모델에는 **컨텍스트 창** (볼 수 있는 최대 토큰) 장기 실행 채팅은 메시지와 도구 결과를 축적합니다. 창문이 꽉 닫히면 OpenClaw **콤팩트** 오래된 기록은 한도 내에서 유지됩니다.

## 압축이란 무엇입니까?

압축 **오래된 대화를 요약합니다** 요약 항목으로 압축하고 최근 메시지를 그대로 유지합니다. 요약은 세션 기록에 저장되므로 향후 요청에서는 다음을 사용합니다.

- 압축 요약
- 압축 지점 이후의 최근 메시지

압축 **지속된다** 세션의 JSONL 기록에 있습니다.

## 구성

보다 [압축 구성 및 모드](/concepts/compaction) 에 대한 `agents.defaults.compaction` 설정.

## 자동 압축(기본값은 켜짐)

세션이 모델의 컨텍스트 창에 가까워지거나 이를 초과하면 OpenClaw는 자동 압축을 트리거하고 압축된 컨텍스트를 사용하여 원래 요청을 다시 시도할 수 있습니다.

다음 내용이 표시됩니다.

- `🧹 Auto-compaction complete` 상세 모드에서
- `/status` 전시 `🧹 Compactions: <count>`

압축하기 전에 OpenClaw는 다음을 실행할 수 있습니다. **자동 메모리 플러시** 매장으로 향하다
디스크에 내구성 있는 메모를 남깁니다. 보다 [메모리](/concepts/memory) 자세한 내용과 구성을 확인하세요.

## 수동 압축

사용 `/compact` (선택적으로 지침과 함께) 압축 패스를 강제 실행하려면 다음을 수행합니다.

```
/compact Focus on decisions and open questions
```

## 컨텍스트 창 소스

컨텍스트 창은 모델별로 다릅니다. OpenClaw는 구성된 공급자 카탈로그의 모델 정의를 사용하여 제한을 결정합니다.

## 압축과 가지치기

- **압축 **: 요약하고 **지속된다** JSONL에서.
- **세션 가지치기**: 낡은 것을 다듬는다 **도구 결과** 오직, **메모리 내**, 요청에 따라.

보다 [/개념/세션 가지치기](/concepts/session-pruning) 자세한 내용은 가지치기를 참조하세요.

## 팁

- 사용 `/compact` 세션이 진부하다고 느껴지거나 맥락이 부풀어오르는 경우.
- 큰 도구 출력은 이미 잘렸습니다. 가지치기를 하면 도구 결과 축적을 더욱 줄일 수 있습니다.
- 새로운 슬레이트가 필요하다면, `/new` 또는 `/reset` 새 세션 ID를 시작합니다.
