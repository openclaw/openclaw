# 시스템 프롬프트

OpenClaw는 모든 에이전트 실행에 대해 사용자 정의 시스템 프롬프트를 구축합니다. 프롬프트는 **OpenClaw 소유** p-coding-agent 기본 프롬프트를 사용하지 않습니다.

프롬프트는 OpenClaw에 의해 수집되어 각 에이전트 실행에 주입됩니다.

## 구조

프롬프트는 의도적으로 간결하며 고정 섹션을 사용합니다.

- **압형**: 현재 도구 목록 + 간단한 설명.
- **안전**: 권력을 추구하는 행동이나 감독 우회를 방지하기 위한 짧은 가드레일 알림입니다.
- **기술** (사용 가능한 경우): 요청 시 기술 지침을 로드하는 방법을 모델에 알려줍니다.
- **OpenClaw 자체 업데이트**: 달리는 방법 `config.apply` 그리고 `update.run`.
- **작업공간**: 작업 디렉토리(`agents.defaults.workspace`).
- **선적 서류 비치**: OpenClaw 문서(repo 또는 npm 패키지)에 대한 로컬 경로 및 이를 읽는 시기.
- **작업공간 파일(삽입됨)**: 아래에 부트스트랩 파일이 포함되어 있음을 나타냅니다.
- **모래 상자** (활성화된 경우): 샌드박스 런타임, 샌드박스 경로 및 승격된 exec 사용 가능 여부를 나타냅니다.
- **현재 날짜 및 시간**: 사용자 현지 시간, 시간대 및 시간 형식입니다.
- **답장 태그**: 지원되는 공급자에 대한 선택적 응답 태그 구문입니다.
- **심장 박동**: 하트비트 프롬프트 및 응답 동작.
- **실행 시간**: 호스트, OS, 노드, 모델, 저장소 루트(감지된 경우), 사고 수준(한 줄).
- **추리**: 현재 가시성 수준 + /reasoning 토글 힌트.

시스템 프롬프트의 안전 가드레일은 권고 사항입니다. 모델 동작을 안내하지만 정책을 시행하지는 않습니다. 강력한 시행을 위해 도구 정책, 실행 승인, 샌드박스 및 채널 허용 목록을 사용합니다. 운영자는 의도적으로 이를 비활성화할 수 있습니다.

## 프롬프트 모드

OpenClaw는 하위 에이전트에 대해 더 작은 시스템 프롬프트를 렌더링할 수 있습니다. 런타임은 다음을 설정합니다.
`promptMode` 각 실행에 대해(사용자 측 구성 아님):

- `full` (기본값): 위의 모든 섹션을 포함합니다.
- `minimal`: 하위 에이전트에 사용됩니다. 생략하다 **기술**, **메모리 리콜**, **OpenClaw 자체 업데이트**, **모델 별칭**, **사용자 신원**, **답장 태그**, 
  **메시징**, **무음 답장**, 그리고 **심장 박동**. 압형, **안전**,
  작업 공간, 샌드박스, 현재 날짜 및 시간(알려진 경우), 런타임 및 삽입
  컨텍스트 유지가 가능합니다.
- `none`: 기본 식별 라인만 반환합니다.

언제 `promptMode=minimal`, 추가로 삽입된 프롬프트에는 레이블이 지정됩니다. **하위 에이전트
맥락** 대신에 **그룹 채팅 컨텍스트**.

## 작업공간 부트스트랩 주입

부트스트랩 파일은 잘려서 아래에 추가됩니다. **프로젝트 컨텍스트** 따라서 모델은 명시적인 읽기 없이도 ID 및 프로필 컨텍스트를 볼 수 있습니다.

- `AGENTS.md`
- `SOUL.md`
- `TOOLS.md`
- `IDENTITY.md`
- `USER.md`
- `HEARTBEAT.md`
- `BOOTSTRAP.md` (새로운 작업공간에만 해당)

큰 파일은 마커로 잘립니다. 파일당 최대 크기는 다음에 의해 제어됩니다.
`agents.defaults.bootstrapMaxChars` (기본값: 20000) 누락된 파일은 다음을 삽입합니다.
짧은 누락 파일 마커.

내부 후크는 다음을 통해 이 단계를 가로챌 수 있습니다. `agent:bootstrap` 돌연변이를 일으키거나 대체하다
삽입된 부트스트랩 파일(예: 스와핑 `SOUL.md` 대체 페르소나의 경우).

삽입된 각 파일이 얼마나 기여하는지(원시 파일과 삽입 파일, 잘림, 도구 스키마 오버헤드)를 검사하려면 다음을 사용하세요. `/context list` 또는 `/context detail`. 보다 [문맥](/concepts/context).

## 시간 처리

시스템 프롬프트에는 전용 메시지가 포함되어 있습니다. **현재 날짜 및 시간** 섹션
사용자 시간대가 알려져 있습니다. 프롬프트를 캐시에 안정적으로 유지하기 위해 이제 프롬프트에만 포함됩니다.
는 **시간대** (동적 시계 또는 시간 형식 없음)

사용 `session_status` 에이전트가 현재 시간을 필요로 할 때; 상태 카드
타임스탬프 라인을 포함합니다.

다음으로 구성:

- `agents.defaults.userTimezone`
- `agents.defaults.timeFormat` (`auto` | `12` | `24`)

보다 [날짜 및 시간](/date-time) 전체 동작 세부정보를 확인하세요.

## 기술

적격 기술이 존재하는 경우 OpenClaw는 컴팩트를 주입합니다. **사용 가능한 스킬 목록**
 (`formatSkillsForPrompt`)에는 **파일 경로** 스킬마다. 는
프롬프트는 모델에게 다음을 사용하도록 지시합니다. `read` 나열된 위치에 SKILL.md를 로드하려면
위치(작업공간, 관리형 또는 번들형). 자격을 갖춘 기술이 없는 경우,
스킬 부분은 생략합니다.

```
<available_skills>
  <skill>
    <name>...</name>
    <description>...</description>
    <location>...</location>
  </skill>
</available_skills>
```

이렇게 하면 기본 프롬프트를 작게 유지하면서 타겟 스킬 사용을 계속 활성화할 수 있습니다.

## 선적 서류 비치

사용 가능한 경우 시스템 프롬프트에 다음이 포함됩니다. **선적 서류 비치** 다음을 가리키는 섹션
로컬 OpenClaw 문서 디렉토리(둘 중 하나) `docs/` repo 작업공간 또는 번들 npm에서
패키지 문서) 및 공개 미러, 소스 저장소, 커뮤니티 Discord 및
클로허브([https://clawhub.com](https://clawhub.com)) 기술 발견을 위해. 프롬프트는 모델이 먼저 로컬 문서를 참조하도록 지시합니다.
OpenClaw 동작, 명령, 구성 또는 아키텍처 및 실행
`openclaw status` 가능한 경우 자체적으로 수행합니다(액세스가 부족한 경우에만 사용자에게 요청).
