원클릭 템플릿을 사용하여 철도에 OpenClaw를 배포하고 브라우저에서 설정을 완료하세요.
이것은 가장 쉬운 "서버에 터미널 없음" 경로입니다. Railway가 사용자를 위해 게이트웨이를 실행합니다.
그리고 당신은 다음을 통해 모든 것을 구성합니다 `/setup` 웹 마법사.

## 빠른 체크리스트(신규 사용자)

1. 딸깍 하는 소리 **철도에 배포** (아래에).
2. 추가 **용량** 에 장착됨 `/data`.
3. 필수 설정 **변수** (적어도 `SETUP_PASSWORD`).
4. 할 수 있게 하다 **HTTP 프록시** 항구에 `8080`.
5. 열려 있는 `https://<your-railway-domain>/setup` 마법사를 완료하세요.

## 원클릭 배포

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  철도에 배포
</a>

배포 후 다음에서 공개 URL을 찾으세요. **철도 → 서비스 → 설정 → 도메인**.

철도는 다음 중 하나를 수행합니다.

- 생성된 도메인을 제공합니다(종종 `https://<something>.up.railway.app`), 또는
- 사용자 정의 도메인을 연결한 경우 사용자 정의 도메인을 사용하세요.

그런 다음 다음을 엽니다.

- `https://<your-railway-domain>/setup` — 설정 마법사(비밀번호로 보호됨)
- `https://<your-railway-domain>/openclaw` — 컨트롤 UI

## 당신이 얻는 것

- 호스팅된 OpenClaw 게이트웨이 + 제어 UI
- 웹 설정 마법사 `/setup` (터미널 명령 없음)
- 철도 볼륨을 통한 영구 저장(`/data`) 따라서 구성/자격 증명/작업 공간은 재배포 후에도 유지됩니다.
- 백업 내보내기 시간 `/setup/export` 나중에 철도에서 이주하기 위해

## 필수 철도 설정

### 공용 네트워킹

할 수 있게 하다 **HTTP 프록시** 서비스를 위해.

- 포트: `8080`

### 볼륨 (필수)

다음 위치에 마운트된 볼륨을 연결합니다.

- `/data`

### 변수

서비스에서 다음 변수를 설정합니다.

- `SETUP_PASSWORD` (필수의)
- `PORT=8080` (필수 - 공용 네트워킹의 포트와 일치해야 함)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (권장)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (권장)
- `OPENCLAW_GATEWAY_TOKEN` (권장, 관리자 비밀번호로 취급)

## 설정 흐름

1. 방문하다 `https://<your-railway-domain>/setup` 그리고 당신의 `SETUP_PASSWORD`.
2. 모델/인증 제공업체를 선택하고 키를 붙여넣으세요.
3. (선택사항) 텔레그램/Discord/Slack 토큰을 추가합니다.
4. 딸깍 하는 소리 **설정 실행**.

Telegram DM이 페어링으로 설정된 경우 설정 마법사가 페어링 코드를 승인할 수 있습니다.

## 채팅 토큰 받기

### 텔레그램 봇 토큰

1. 메시지 `@BotFather` 텔레그램에서
2. 달리다 `/newbot`
3. 토큰을 복사하세요(다음과 같습니다. `123456789:AA...`)
4. 에 붙여넣으세요 `/setup`

### 디스코드 봇 토큰

1. 이동 [https://discord.com/developers/applications](https://discord.com/developers/applications)
2. **새로운 애플리케이션** → 이름을 선택하세요
3. **봇** → **봇 추가**
4. **메시지 콘텐츠 의도 활성화** Bot → Privileged Gateway Intents(필수, 그렇지 않으면 봇이 시작 시 충돌함)
5. 복사 **봇 토큰** 그리고 붙여넣기 `/setup`
6. 서버에 봇을 초대합니다(OAuth2 URL 생성기, 범위: `bot`, `applications.commands`)

## 백업 및 마이그레이션

다음 위치에서 백업을 다운로드하세요.

- `https://<your-railway-domain>/setup/export`

그러면 OpenClaw 상태와 작업 공간이 내보내지므로 구성이나 메모리 손실 없이 다른 호스트로 마이그레이션할 수 있습니다.
