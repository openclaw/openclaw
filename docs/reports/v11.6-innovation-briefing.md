# Innovation Briefing v11.6: Universal Research Engine & Knowledge Synthesis

_Дата: 2026-03-28 · Автор: OpenClaw Research Pipeline_

## 1. Executive Summary

В рамках директивы v11.6 реализован **UniversalParser** — единый мультиисточниковый движок исследований, объединяющий 6 источников данных через паттерн адаптеров. Три старых парсера (habr.py, github.py, reddit.py) удалены и заменены одним модулем `src/parsers/universal.py` (~480 LOC).

**Ключевые цифры:**

- **6** адаптеров (3 мигрированных + 3 новых)
- **~105 KB** собранных исследовательских данных (5 источников)
- **100** уникальных материалов (Top-20 × 5 источников)
- **20+** платформ оценено, 6 выбрано по критерию доступности API

## 2. Архитектура UniversalParser

### 2.1 Паттерн адаптеров

```
SourceAdapter (ABC)
├── HabrAdapter       — RSS/XML парсинг
├── GitHubAdapter     — REST API + GITHUB_TOKEN
├── RedditAdapter     — JSON API с rate-limit respect
├── SemanticScholarAdapter — REST API + опциональный S2_API_KEY
├── ArxivAdapter      — Atom XML API + certifi SSL
└── OpenAlexAdapter   — REST API, без аутентификации
```

**UniversalParser** — оркестратор, запускающий адаптеры конкурентно через `asyncio.gather()`.

### 2.2 Ключевые компоненты

| Компонент                       | Назначение                                                                                            |
| ------------------------------- | ----------------------------------------------------------------------------------------------------- |
| `ResearchItem`                  | Универсальный датакласс с полями: title, url, source, summary, authors, tags, score, published, extra |
| `SourceAdapter`                 | ABC с методом `fetch(query, limit, timeout) → List[ResearchItem]`                                     |
| `UniversalParser.search()`      | Конкурентный поиск по всем/выбранным источникам → `dict[source_name, list]`                           |
| `UniversalParser.search_flat()` | Плоский список с дедупликацией по URL и сортировкой по score                                          |

### 2.3 Обработка ошибок

- Rate-limiting: адаптеры возвращают `[]` при 429/403 (не бросают исключения)
- SSL: ArxivAdapter использует `certifi` для корректного CA bundle
- Timeout: настраиваемый (по умолчанию 20 сек на источник)
- `asyncio.gather(return_exceptions=True)` — один упавший адаптер не блокирует остальные

## 5. Технические решения и исправления

### 5.1 SSL Fix для arXiv

**Проблема:** `export.arxiv.org:443` отклонял SSL-подключения из-за не найденного CA certificate на Windows.  
**Решение:** Использование `certifi` для предоставления корректного CA bundle:

```python
ssl_ctx = ssl.create_default_context(cafile=certifi.where())
connector = aiohttp.TCPConnector(ssl=ssl_ctx)
```

### 5.2 arXiv Query Format

**Проблема:** Одиночный `all:query+string` разбивался на OR-токены.  
**Решение:** Явное AND-соединение:

```python
words = query.split()
search_query = "+AND+".join(f"all:{quote_plus(w)}" for w in words)
```

### 5.3 Semantic Scholar Rate Limiting

**Проблема:** 429 Too Many Requests без API-ключа (лимит 100 req/5 min).  
**Решение:** Для продакшена — установить `S2_API_KEY` через env. Адаптер автоматически подхватывает заголовок `x-api-key`.

## 7. Рекомендации для OpenClaw

### Немедленные действия

1. **Получить S2 API-ключ** — зарегистрироваться на [semanticscholar.org/product/api](https://www.semanticscholar.org/product/api), установить `S2_API_KEY` в конфиг
2. **Интегрировать ArxivAdapter в пайплайн** — свежие препринты дают сигнал раннего обнаружения трендов
3. **Настроить OPENALEX_EMAIL** — для доступа к polite pool (приоритетная обработка запросов)

### Среднесрочные улучшения

4. **Cross-Source Enrichment** — при нахождении статьи на arXiv, автоматический pullback цитирований через S2 или OpenAlex
5. **Trend Detection** — сравнение Top-20 между запусками для обнаружения восходящих тем
6. **Scheduled Pipeline** — еженедельный/ежедневный запуск deep research с diff-отчётом
7. **Кэширование** — Redis/SQLite кэш результатов с TTL для снижения нагрузки на API

### Потенциальные новые источники

8. **HuggingFace Papers API** — если появится стабильный endpoint (текущий PapersWithCode редиректит)
9. **DBLP** — хороший API для поиска по авторам/конференциям
10. **Crossref** — метаданные DOI, может дополнить OpenAlex
