# 头号助手

> 最后更新：2026-03-28
> 状态：Phase 1-3 完成，Phase 4+ 规划中

## 核心价值

**AI 全方位接管你的信息流，你只做"想和决定"。**

不只是聊天助手——头号助手接管消息、日程、健康、通知、文件，让你从信息洪流中解放出来。AI 负责看、筛、找、转、记、提醒，你只需要做决策。

---

## 系统架构

### 设计原则

1. **OpenClaw 做中枢** — 连接所有渠道和数据源，统一出入口
2. **双模型分层调度** — 日常任务走 DeepSeek（低成本），复杂决策走 Claude CLI（Max 订阅零成本）
3. **MCP 扩展数据源** — 通过 MCP Server 接入本地数据（微信、健康、日历等）
4. **隐私第一** — 所有数据本地处理，不用于训练，不给第三方

## 场景规划

### 已实现场景

#### 场景一：消息流转
三大渠道消息统一处理，自动分类，紧急消息跨渠道转发到 iMessage。
- 公众号/广告自动忽略
- 紧急消息转发到 iMessage
- 微信历史搜索（MCP）

#### 场景二：设备协同
手机发消息操作电脑文件。file-ops skill 已就绪。

#### 场景三：远程开发
手机指挥电脑写代码。dev-proxy skill + Claude CLI。

#### 场景四：深度学习
对任何主题深度研究。deep-research skill + Claude。

### 规划中场景

#### 场景五：日程管家

**目标**：接管日历，每天告诉你该做什么、什么时候出发。

**用户故事**：
- 早上推送："今天有3个会议，最早10点飞书群会。微信有2条待回复。"
- "明天下午3点在南山开会" → 自动创建日历事件
- "帮我把周五的会议推迟到下周一" → 修改日历
- 飞书消息提到"明天开会" → 自动提醒并建议创建日历

**接入方式**：Google Calendar MCP connector（claude.ai 已内置，需 OAuth 认证）

**优先级**：P0 — 价值极高，接入成本最低

#### 场景六：健康顾问

**目标**：整合健康和运动数据，主动提醒和建议。

**用户故事**：
- "这周运动量怎么样" → 分析步数、运动时长趋势
- 久坐超过2小时 → 主动提醒"该起来活动了"
- 睡眠数据异常 → "最近连续3天睡不到6小时，建议调整作息"
- 每周健康周报推送

**接入方式**：
- Apple Health / 华为健康数据导出为 CSV/JSON → MCP 解析
- 微信运动步数（通过微信 MCP 读取）
- 智能手环数据同步

**优先级**：P1 — 高价值，但需要手机端配合导出数据

#### 场景七：通知聚合中心

**目标**：所有 App 通知统一收口，AI 过滤噪音后只推送重要的。

**用户故事**：
- 手机上 50 条 App 通知 → 助手筛选后只给你 3 条重要的
- "外卖到了" → 推送
- "XX App 有新版本" → 忽略
- 快递状态更新 → "你有2个包裹明天到"

**接入方式**：
- Android：Tasker / MacroDroid → 通知转发到飞书/微信
- iOS：快捷指令 → 转发到 iMessage
- 或通过 KDE Connect 将手机通知同步到电脑

**优先级**：P1 — 高价值，但需要手机端配置

#### 场景八：财务管家

**目标**：自动分析消费，提醒预算。

**用户故事**：
- "这个月花了多少" → 分析支付宝/微信账单
- "和上个月比怎么样" → 消费趋势对比
- 月底超预算 → 主动提醒

**接入方式**：
- 微信/支付宝账单导出 CSV → MCP 解析
- 银行 App 账单截图 → Claude 视觉分析

**优先级**：P2 — 有价值，但数据获取依赖手动导出

#### 场景九：资讯过滤器

**目标**：不被信息流轰炸，只看值得看的。

**用户故事**：
- 订阅 RSS 源 → AI 按兴趣筛选，每天推送 5 条精选
- "最近AI领域有什么大事" → 从多个源综合分析
- 屏蔽标题党和重复报道

**接入方式**：
- RSS 订阅 + MCP 解析
- 微信公众号文章提取（通过微信 MCP）
- WebSearch 补充

**优先级**：P2 — 有价值，但需要定义兴趣模型

#### 场景十：剪贴板同步

**目标**：电脑和手机之间无缝传递文本和链接。

**用户故事**：
- 电脑上复制了一段文字，手机问"刚才复制了什么" → 返回内容
- 手机上看到一个链接，发给助手 → 助手在电脑上打开
- "把这段话复制到我的剪贴板" → 写入系统剪贴板

**接入方式**：自建 Clipboard MCP Server（Python，10 行代码）

**优先级**：P0 — 简单且实用

## 已知限制与风险

| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| DeepSeek 无法可靠调用 CLI | 复杂路由失败 | 用 OpenClaw 原生 cliBackends + @model 切换 |
| 微信 MCP 依赖 PC 在线 | 电脑关机时无法读取微信数据 | 短期接受；长期考虑服务器部署 |
| 微信仅支持1对1 | 无法处理群聊消息 | 平台限制，暂不解决 |
| Claude OAuth token 过期 | CLI backend 静默失败 | 日常使用 Claude Code 保持 token 刷新 |
| 健康数据需手动导出 | 不能实时分析 | 探索自动同步方案 |
| 手机通知转发需配置 | 初始设置复杂 | 写一键配置脚本 |
