# تاریخ اور وقت

50. provider timestamps محفوظ رکھے جاتے ہیں تاکہ ٹولز اپنی نیٹو semantics برقرار رکھیں (موجودہ وقت `session_status` کے ذریعے دستیاب ہے)۔
    Provider timestamps are preserved so tools keep their native semantics (current time is available via `session_status`).

## پیغام لفافے (بطورِ طے شدہ لوکل)

آنے والے پیغامات کو ایک ٹائم اسٹیمپ کے ساتھ لپیٹا جاتا ہے (منٹ کی درستی کے ساتھ):

```
[Provider ... 2026-01-05 16:26 PST] message text
```

یہ لفافہ ٹائم اسٹیمپ **بطورِ طے شدہ ہوسٹ-لوکل** ہوتا ہے، فراہم کنندہ کے ٹائم زون سے قطع نظر۔

آپ اس رویے کو تبدیل کر سکتے ہیں:

```json5
{
  agents: {
    defaults: {
      envelopeTimezone: "local", // "utc" | "local" | "user" | IANA timezone
      envelopeTimestamp: "on", // "on" | "off"
      envelopeElapsed: "on", // "on" | "off"
    },
  },
}
```

- `envelopeTimezone: "utc"` UTC استعمال کرتا ہے۔
- `envelopeTimezone: "local"` ہوسٹ ٹائم زون استعمال کرتا ہے۔
- `envelopeTimezone: "user"` `agents.defaults.userTimezone` استعمال کرتا ہے (ہوسٹ ٹائم زون پر واپس آتا ہے)۔
- ایک واضح IANA ٹائم زون استعمال کریں (مثلاً `"America/Chicago"`) تاکہ ایک مقررہ زون ہو۔
- `envelopeTimestamp: "off"` لفافہ ہیڈرز سے مطلق ٹائم اسٹیمپس ہٹا دیتا ہے۔
- `envelopeElapsed: "off"` گزرا ہوا وقت دکھانے والے لاحقے ہٹا دیتا ہے ( `+2m` انداز)۔

### مثالیں

**لوکل (بطورِ طے شدہ):**

```
[WhatsApp +1555 2026-01-18 00:19 PST] hello
```

**صارف کا ٹائم زون:**

```
[WhatsApp +1555 2026-01-18 00:19 CST] hello
```

**گزرا ہوا وقت فعال:**

```
[WhatsApp +1555 +30s 2026-01-18T05:19Z] follow-up
```

## سسٹم پرامپٹ: موجودہ تاریخ اور وقت

اگر صارف کا ٹائم زون معلوم ہو تو سسٹم پرامپٹ میں ایک مخصوص
**موجودہ تاریخ اور وقت** کا سیکشن شامل کیا جاتا ہے جس میں **صرف ٹائم زون** ہوتا ہے (کوئی گھڑی/وقت کی فارمیٹنگ نہیں)
تاکہ پرامپٹ کی کیشنگ مستحکم رہے:

```
Time zone: America/Chicago
```

جب ایجنٹ کو موجودہ وقت درکار ہو تو `session_status` ٹول استعمال کریں؛ اس کی اسٹیٹس
کارڈ میں ایک ٹائم اسٹیمپ لائن شامل ہوتی ہے۔

## سسٹم ایونٹ لائنز (بطورِ طے شدہ لوکل)

ایجنٹ سیاق میں شامل کی گئی قطار بند سسٹم ایونٹس کے آغاز میں ایک ٹائم اسٹیمپ لگایا جاتا ہے، جو
پیغام لفافوں کے لیے منتخب کردہ اسی ٹائم زون کو استعمال کرتا ہے (بطورِ طے شدہ: ہوسٹ-لوکل)۔

```
System: [2026-01-12 12:19:17 PST] Model switched.
```

### صارف کے ٹائم زون + فارمیٹ کی کنفیگریشن

```json5
{
  agents: {
    defaults: {
      userTimezone: "America/Chicago",
      timeFormat: "auto", // auto | 12 | 24
    },
  },
}
```

- `userTimezone` پرامپٹ سیاق کے لیے **صارف-لوکل ٹائم زون** سیٹ کرتا ہے۔
- `timeFormat` پرامپٹ میں **12h/24h ڈسپلے** کو کنٹرول کرتا ہے۔ `auto` OS کی ترجیحات کی پیروی کرتا ہے۔

## وقت کے فارمیٹ کی شناخت (خودکار)

جب `timeFormat: "auto"` ہو، تو OpenClaw OS کی ترجیح (macOS/Windows) کا معائنہ کرتا ہے
اور پھر لوکیل فارمیٹنگ پر واپس آ جاتا ہے۔ معلوم کی گئی ویلیو **فی پروسیس کیش** کی جاتی ہے
تاکہ بار بار سسٹم کالز سے بچا جا سکے۔

## ٹول پے لوڈز + کنیکٹرز (خام فراہم کنندہ وقت + نارملائزڈ فیلڈز)

چینل ٹولز **فراہم کنندہ کے مقامی ٹائم اسٹیمپس** واپس کرتے ہیں اور یکسانیت کے لیے نارملائزڈ فیلڈز شامل کرتے ہیں:

- `timestampMs`: ایپوک ملی سیکنڈز (UTC)
- `timestampUtc`: ISO 8601 UTC اسٹرنگ

خام فراہم کنندہ فیلڈز محفوظ رہتے ہیں تاکہ کوئی معلومات ضائع نہ ہوں۔

- Slack: API سے ایپوک نما اسٹرنگز
- Discord: UTC ISO ٹائم اسٹیمپس
- Telegram/WhatsApp: فراہم کنندہ کے مخصوص عددی/ISO ٹائم اسٹیمپس

اگر آپ کو لوکل وقت درکار ہو تو معلوم ٹائم زون استعمال کرتے ہوئے اسے بعد کے مرحلے میں تبدیل کریں۔
