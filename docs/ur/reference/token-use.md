# ٹوکن کا استعمال اور لاگت

21. OpenClaw حروف کے بجائے **ٹوکنز** کو ٹریک کرتا ہے۔ 22. ٹوکن ماڈل کے لحاظ سے مختلف ہوتے ہیں، لیکن زیادہ تر
    OpenAI طرز کے ماڈلز انگریزی متن کے لیے اوسطاً ہر ٹوکن میں تقریباً 4 حروف رکھتے ہیں۔

## سسٹم پرامپٹ کیسے بنایا جاتا ہے

OpenClaw assembles its own system prompt on every run. 24. اس میں شامل ہوتا ہے:

- ٹولز کی فہرست + مختصر وضاحتیں
- Skills کی فہرست (صرف میٹاڈیٹا؛ ہدایات ضرورت کے مطابق `read` کے ساتھ لوڈ ہوتی ہیں)
- خودکار اپڈیٹ کی ہدایات
- Workspace + bootstrap files (`AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md` when new). 26. بڑی فائلز کو `agents.defaults.bootstrapMaxChars` (ڈیفالٹ: 20000) کے ذریعے ٹرنکیٹ کیا جاتا ہے۔
- وقت (UTC + صارف کا ٹائم زون)
- جواب کے ٹیگز + ہارٹ بیٹ کا رویّہ
- رَن ٹائم میٹاڈیٹا (ہوسٹ/OS/ماڈل/سوچ)

مکمل تفصیل کے لیے [System Prompt](/concepts/system-prompt) دیکھیں۔

## کانٹیکسٹ ونڈو میں کیا شامل شمار ہوتا ہے

ماڈل کو موصول ہونے والی ہر چیز کانٹیکسٹ حد میں شمار ہوتی ہے:

- سسٹم پرامپٹ (اوپر درج تمام حصے)
- گفتگو کی تاریخ (صارف + اسسٹنٹ پیغامات)
- ٹول کالز اور ٹول نتائج
- منسلکات/ٹرانسکرپٹس (تصاویر، آڈیو، فائلیں)
- کمپیکشن کے خلاصے اور پروننگ کے آثار
- فراہم کنندہ کے ریپرز یا حفاظتی ہیڈرز (نظر نہیں آتے، مگر پھر بھی شمار ہوتے ہیں)

27. عملی تفصیل کے لیے (ہر انجیکٹ کی گئی فائل، ٹولز، اسکلز، اور سسٹم پرامپٹ سائز کے حساب سے)، `/context list` یا `/context detail` استعمال کریں۔ 28. دیکھیں [Context](/concepts/context)۔

## موجودہ ٹوکن استعمال کیسے دیکھیں

چیٹ میں یہ استعمال کریں:

- `/status` → **ایموجی سے بھرپور اسٹیٹس کارڈ** جس میں سیشن ماڈل، کانٹیکسٹ استعمال،
  آخری جواب کے اِن پُٹ/آؤٹ پُٹ ٹوکنز، اور **تخمینی لاگت** (صرف API کلید) شامل ہوتی ہے۔
- `/usage off|tokens|full` → ہر جواب کے ساتھ **فی‑جواب استعمال کا فوٹر** شامل کرتا ہے۔
  - ہر سیشن کے لیے برقرار رہتا ہے (بطور `responseUsage` محفوظ ہوتا ہے)۔
  - OAuth تصدیق **لاگت چھپا دیتی ہے** (صرف ٹوکنز دکھتے ہیں)۔
- `/usage cost` → OpenClaw سیشن لاگز سے مقامی لاگت کا خلاصہ دکھاتا ہے۔

دیگر سطحیں:

- **TUI/Web TUI:** `/status` + `/usage` سپورٹڈ ہیں۔
- **CLI:** `openclaw status --usage` اور `openclaw channels list` فراہم کنندہ کے کوٹہ ونڈوز دکھاتے ہیں
  (فی‑جواب لاگت نہیں)۔

## لاگت کا تخمینہ (جب دکھایا جائے)

لاگت آپ کے ماڈل پرائسنگ کنفیگ سے تخمینہ کی جاتی ہے:

```
models.providers.<provider>.models[].cost
```

29. یہ **USD فی 1M ٹوکنز** ہیں `input`, `output`, `cacheRead`, اور
    `cacheWrite` کے لیے۔ If pricing is missing, OpenClaw shows tokens only. 31. OAuth ٹوکنز
    کبھی بھی ڈالر لاگت نہیں دکھاتے۔

## کیش TTL اور پروننگ کا اثر

32. فراہم کنندہ کی پرامپٹ کیشنگ صرف کیش TTL ونڈو کے اندر لاگو ہوتی ہے۔ 33. OpenClaw اختیاری طور پر **cache-ttl pruning** چلا سکتا ہے: یہ سیشن کو اس وقت prune کرتا ہے جب کیش TTL ختم ہو چکا ہو، پھر کیش ونڈو کو ری سیٹ کرتا ہے تاکہ بعد کی درخواستیں پوری ہسٹری کو دوبارہ کیش کرنے کے بجائے تازہ کیش شدہ کانٹیکسٹ کو دوبارہ استعمال کر سکیں۔ 34. اس سے کیش
    رائٹ لاگت کم رہتی ہے جب کوئی سیشن TTL کے بعد غیر فعال ہو جائے۔

اسے [Gateway configuration](/gateway/configuration) میں کنفیگر کریں اور رویّے کی تفصیلات
[Session pruning](/concepts/session-pruning) میں دیکھیں۔

35. Heartbeat غیر فعال وقفوں کے دوران کیش کو **گرم** رکھ سکتا ہے۔ 36. اگر آپ کے ماڈل کی کیش TTL
    `1h` ہے، تو ہارٹ بیٹ انٹروَل کو اس سے ذرا کم (مثلاً `55m`) پر سیٹ کرنا مکمل پرامپٹ کو دوبارہ کیش کرنے سے بچا سکتا ہے، جس سے کیش رائٹ لاگت کم ہوتی ہے۔

37. Anthropic API کی قیمتوں میں، کیش ریڈز ان پٹ ٹوکنز کے مقابلے میں نمایاں طور پر سستے ہوتے ہیں، جبکہ کیش رائٹس زیادہ ملٹی پلائر پر بل کیے جاتے ہیں۔ 38. تازہ ترین ریٹس اور TTL ملٹی پلائرز کے لیے Anthropic کی
    پرامپٹ کیشنگ پرائسنگ دیکھیں:
    [https://docs.anthropic.com/docs/build-with-claude/prompt-caching](https://docs.anthropic.com/docs/build-with-claude/prompt-caching)

### مثال: ہارٹ بیٹ کے ساتھ 1 گھنٹے کی کیش گرم رکھیں

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long"
    heartbeat:
      every: "55m"
```

## ٹوکن دباؤ کم کرنے کے مشورے

- طویل سیشنز کا خلاصہ بنانے کے لیے `/compact` استعمال کریں۔
- اپنے ورک فلو میں بڑے ٹول آؤٹ پُٹس کو مختصر کریں۔
- اسکل کی وضاحتیں مختصر رکھیں (اسکل فہرست پرامپٹ میں انجیکٹ ہوتی ہے)۔
- طویل اور تحقیقی کام کے لیے چھوٹے ماڈلز کو ترجیح دیں۔

اسکل فہرست کے درست اوورہیڈ فارمولے کے لیے [Skills](/tools/skills) دیکھیں۔
