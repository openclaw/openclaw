# การเริ่มต้นใช้งาน (แอปmacOS)

เอกสารนี้อธิบายโฟลว์การเริ่มต้นใช้งานครั้งแรกในสถานะ **ปัจจุบัน** เป้าหมายคือประสบการณ์ “วันแรก” ที่ราบรื่น: เลือกตำแหน่งที่ Gateway ทำงาน เชื่อมต่อการยืนยันตัวตน รันวิซาร์ด และปล่อยให้เอเจนต์บูตสแตรปตัวเอง

**Gateway** ทำงานอยู่ที่ใด?

- **Mac เครื่องนี้ (เฉพาะภายในเครื่อง):** การเริ่มต้นใช้งานสามารถรันโฟลว์ OAuth และเขียนข้อมูลรับรองลงในเครื่องได้
- **ระยะไกล (ผ่าน SSH/Tailnet):** การเริ่มต้นใช้งานจะ **ไม่** รัน OAuth ในเครื่อง; ต้องมีข้อมูลรับรองอยู่บนโฮสต์Gateway
- **ตั้งค่าภายหลัง:** ข้ามการตั้งค่าและปล่อยให้แอปยังไม่ถูกกำหนดค่า

<Tip>
**เคล็ดลับการยืนยันตัวตนของ Gateway:**
- วิซาร์ดจะสร้าง **โทเคน** ให้แม้ในกรณี loopback ดังนั้นไคลเอนต์ WS ภายในเครื่องต้องยืนยันตัวตน
- หากปิดการยืนยันตัวตน กระบวนการใดๆในเครื่องสามารถเชื่อมต่อได้ ควรใช้เฉพาะบนเครื่องที่เชื่อถือได้อย่างสมบูรณ์
- ใช้ **โทเคน** สำหรับการเข้าถึงหลายเครื่องหรือการ bind ที่ไม่ใช่ loopback
</Tip>
</Step>

การเริ่มต้นใช้งานจะขอสิทธิ์ TCC ที่จำเป็นสำหรับ:

- การทำงานอัตโนมัติ (AppleScript)
- การแจ้งเตือน
- การช่วยการเข้าถึง
- การบันทึกหน้าจอ
- ไมโครโฟน
- การรู้จำเสียงพูด
- กล้อง
- ตำแหน่งที่ตั้ง

</Step>
<Step title="CLI">
  <Info>ขั้นตอนนี้ไม่บังคับ</Info>
  แอปสามารถติดตั้ง `openclaw` CLI แบบส่วนกลางผ่าน npm/pnpm เพื่อให้เวิร์กโฟลว์ในเทอร์มินัลและงาน launchd ใช้งานได้ทันที
</Step>
<Step title="แชตการเริ่มต้นใช้งาน (เซสชันเฉพาะ)">
  หลังจากตั้งค่าเสร็จ แอปจะเปิดเซสชันแชตการเริ่มต้นใช้งานแบบเฉพาะ เพื่อให้เอเจนต์แนะนำตัวและชี้แนะขั้นตอนถัดไป วิธีนี้ช่วยแยกคำแนะนำในครั้งแรกออกจากการสนทนาปกติของคุณ ดู [Bootstrapping](/start/bootstrapping) สำหรับรายละเอียดสิ่งที่เกิดขึ้นบนโฮสต์Gateway ระหว่างการรันเอเจนต์ครั้งแรก
</Step>
