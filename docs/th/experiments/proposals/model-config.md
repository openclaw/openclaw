# คอนฟิกโมเดล(การสำรวจ)

เอกสารนี้รวบรวม **แนวคิด** สำหรับการกำหนดค่าโมเดลในอนาคต ยังไม่ใช่สเปกที่พร้อมส่งมอบ สำหรับพฤติกรรมปัจจุบัน โปรดดูที่: It is not a
shipping spec. For current behavior, see:

- [Models](/concepts/models)
- [Model failover](/concepts/model-failover)
- [OAuth + profiles](/concepts/oauth)

## แรงจูงใจ

Operators want:

- โปรไฟล์การยืนยันตัวตนหลายโปรไฟล์ต่อผู้ให้บริการ(ส่วนตัวเทียบกับงาน)
- การเลือก `/model` ที่เรียบง่ายพร้อมกลไกสำรองที่คาดเดาได้
- การแยกที่ชัดเจนระหว่างโมเดลข้อความและโมเดลที่รองรับภาพ

## แนวทางที่เป็นไปได้(ระดับสูง)

- ทำให้การเลือกโมเดลเรียบง่าย: `provider/model` พร้อมนามแฝงที่เป็นทางเลือก
- ให้ผู้ให้บริการมีโปรไฟล์การยืนยันตัวตนหลายโปรไฟล์ โดยมีลำดับที่ชัดเจน
- ใช้รายการสำรองส่วนกลางเพื่อให้ทุกเซสชันสลับไปใช้ได้อย่างสม่ำเสมอ
- ปรับการกำหนดเส้นทางภาพเฉพาะเมื่อมีการตั้งค่าอย่างชัดเจนเท่านั้น

## คำถามที่ยังเปิดอยู่

- การหมุนเวียนโปรไฟล์ควรเป็นรายผู้ให้บริการหรือรายโมเดล?
- UI ควรแสดงการเลือกโปรไฟล์สำหรับเซสชันอย่างไร?
- เส้นทางการย้ายจากคีย์คอนฟิกเดิมที่ปลอดภัยที่สุดคืออะไร?
