# การเข้าสู่ระบบเบราว์เซอร์+การโพสต์ไปยังX/Twitter

## การเข้าสู่ระบบด้วยตนเอง(แนะนำ)

เมื่อเว็บไซต์ต้องการการเข้าสู่ระบบ ให้ **ลงชื่อเข้าใช้ด้วยตนเอง** ในโปรไฟล์เบราว์เซอร์ของ **โฮสต์** (เบราว์เซอร์OpenClaw)

**อย่า** ให้ข้อมูลประจำตัวของคุณกับโมเดล **อย่า**มอบข้อมูลรับรองของคุณให้กับโมเดล การเข้าสู่ระบบแบบอัตโนมัติมักจะกระตุ้นระบบป้องกันบอตและอาจทำให้บัญชีถูกล็อก

กลับไปยังเอกสารเบราว์เซอร์หลัก: [Browser](/tools/browser)

## ใช้โปรไฟล์Chromeใด?

OpenClawควบคุม **โปรไฟล์Chromeเฉพาะ** (ชื่อ `openclaw` อินเทอร์เฟซโทนสีส้ม) ซึ่งแยกจากโปรไฟล์เบราว์เซอร์ที่คุณใช้ประจำวัน สิ่งนี้แยกจากโปรไฟล์เบราว์เซอร์ที่คุณใช้ประจำ

มีสองวิธีง่ายๆในการเข้าถึง:

1. **ขอให้เอเจนต์เปิดเบราว์เซอร์** แล้วคุณจึงเข้าสู่ระบบด้วยตนเอง
2. **เปิดผ่านCLI**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

หากคุณมีหลายโปรไฟล์ ให้ระบุ `--browser-profile <name>` (ค่าเริ่มต้นคือ `openclaw`)

## X/Twitter: โฟลว์ที่แนะนำ

- **อ่าน/ค้นหา/เธรด:** ใช้เบราว์เซอร์ของ **โฮสต์** (เข้าสู่ระบบด้วยตนเอง)
- **โพสต์อัปเดต:** ใช้เบราว์เซอร์ของ **โฮสต์** (เข้าสู่ระบบด้วยตนเอง)

## sandboxing+การเข้าถึงเบราว์เซอร์ของโฮสต์

เซสชันเบราว์เซอร์ที่อยู่ในsandboxมีแนวโน้ม **สูงกว่า** ที่จะกระตุ้นการตรวจจับบอต สำหรับX/Twitter(และเว็บไซต์ที่เข้มงวดอื่นๆ) ควรใช้เบราว์เซอร์ของ **โฮสต์** สำหรับ X/Twitter (และเว็บไซต์ที่เข้มงวดอื่น ๆ) ให้เลือกใช้เบราว์เซอร์ **โฮสต์**

หากเอเจนต์ถูกsandboxไว้ เครื่องมือเบราว์เซอร์จะใช้sandboxเป็นค่าเริ่มต้น หากต้องการอนุญาตให้ควบคุมโฮสต์: เพื่ออนุญาตการควบคุมโฮสต์:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

จากนั้นกำหนดเป้าหมายไปยังเบราว์เซอร์ของโฮสต์:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

หรือปิดsandboxingสำหรับเอเจนต์ที่ใช้โพสต์อัปเดต
