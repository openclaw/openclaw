# เครื่องมือสำหรับรีแอ็กชัน

ความหมายของรีแอ็กชันที่ใช้ร่วมกันข้ามช่องทาง:

- `emoji` จำเป็นต้องมีเมื่อเพิ่มรีแอ็กชัน
- `emoji=""` ลบรีแอ็กชันของบอตเมื่อช่องทางรองรับ
- `remove: true` ลบอีโมจิที่ระบุเมื่อช่องทางรองรับ (ต้องใช้ `emoji`)

หมายเหตุของแต่ละช่องทาง:

- **Discord/Slack**: `emoji` ที่ว่างเปล่าจะลบรีแอ็กชันทั้งหมดของบอตบนข้อความนั้น; `remove: true` จะลบเฉพาะอีโมจินั้น
- **Google Chat**: `emoji` ที่ว่างเปล่าจะลบรีแอ็กชันของแอปบนข้อความนั้น; `remove: true` จะลบเฉพาะอีโมจินั้น
- **Telegram**: `emoji` ที่ว่างเปล่าจะลบรีแอ็กชันของบอต; `remove: true` ก็ลบรีแอ็กชันเช่นกันแต่ยังคงต้องมี `emoji` ที่ไม่ว่างสำหรับการตรวจสอบความถูกต้องของเครื่องมือ
- **WhatsApp**: `emoji` ที่ว่างเปล่าจะลบรีแอ็กชันของบอต; `remove: true` จับคู่กับอีโมจิที่ว่างเปล่า (ยังคงต้องใช้ `emoji`)
- **Signal**: การแจ้งเตือนรีแอ็กชันขาเข้าจะปล่อยอีเวนต์ของระบบเมื่อเปิดใช้งาน `channels.signal.reactionNotifications`
