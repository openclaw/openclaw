# การแก้ไขปัญหาช่องทาง

ใช้หน้านี้เมื่อช่องทางเชื่อมต่อได้แต่พฤติกรรมทำงานไม่ถูกต้อง

## ลำดับขั้นคำสั่ง

ให้รันตามลำดับนี้ก่อน:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

Healthy baseline:

- `Runtime: running`
- `RPC probe: ok`
- การตรวจสอบช่องทางแสดงว่าเชื่อมต่อ/พร้อมใช้งาน

## WhatsApp

### ลักษณะอาการเสียของ WhatsApp

| อาการ                             | การตรวจสอบที่เร็วที่สุด                              | วิธีแก้ไข                                                  |
| --------------------------------- | ---------------------------------------------------- | ---------------------------------------------------------- |
| เชื่อมต่อแล้วแต่ไม่มีการตอบ DM    | `openclaw pairing list whatsapp`                     | อนุมัติผู้ส่งหรือเปลี่ยนนโยบาย DM/รายการอนุญาต             |
| ข้อความกลุ่มถูกละเลย              | ตรวจสอบ `requireMention` + รูปแบบการกล่าวถึงในคอนฟิก | กล่าวถึงบอตหรือผ่อนปรนนโยบายการกล่าวถึงสำหรับกลุ่มนั้น     |
| ตัดการเชื่อมต่อ/ล็อกอินซ้ำแบบสุ่ม | `openclaw channels status --probe` + บันทึกล็อก      | ล็อกอินใหม่และยืนยันว่าไดเรกทอรีข้อมูลรับรองอยู่ในสภาพปกติ |

การแก้ไขปัญหาแบบครบถ้วน: [/channels/whatsapp#troubleshooting-quick](/channels/whatsapp#troubleshooting-quick)

## Telegram

### ลักษณะอาการเสียของ Telegram

| อาการ                                    | การตรวจสอบที่เร็วที่สุด                                | วิธีแก้ไข                                                     |
| ---------------------------------------- | ------------------------------------------------------ | ------------------------------------------------------------- |
| `/start` แต่ไม่มีโฟลว์การตอบที่ใช้งานได้ | `openclaw pairing list telegram`                       | อนุมัติการจับคู่หรือเปลี่ยนนโยบาย DM                          |
| บอตออนไลน์แต่กลุ่มเงียบ                  | ตรวจสอบข้อกำหนดการกล่าวถึงและโหมดความเป็นส่วนตัวของบอต | ปิดโหมดความเป็นส่วนตัวเพื่อให้เห็นในกลุ่มหรือกล่าวถึงบอต      |
| ส่งไม่สำเร็จพร้อมข้อผิดพลาดเครือข่าย     | ตรวจสอบล็อกสำหรับความล้มเหลวของการเรียก Telegram API   | แก้ไขการกำหนดเส้นทาง DNS/IPv6/พร็อกซีไปยัง `api.telegram.org` |

การแก้ไขปัญหาแบบครบถ้วน: [/channels/telegram#troubleshooting](/channels/telegram#troubleshooting)

## Discord

### ลักษณะอาการเสียของ Discord

| อาการ                      | การตรวจสอบที่เร็วที่สุด             | วิธีแก้ไข                                                          |
| -------------------------- | ----------------------------------- | ------------------------------------------------------------------ |
| บอตออนไลน์แต่ไม่ตอบในกิลด์ | `openclaw channels status --probe`  | อนุญาตกิลด์/ช่องทางและยืนยัน Intentของเนื้อหาข้อความ               |
| ข้อความกลุ่มถูกละเลย       | Check logs for mention gating drops | กล่าวถึงบอตหรือกำหนดค่า `requireMention: false` ระดับกิลด์/ช่องทาง |
| การตอบ DM หายไป            | `openclaw pairing list discord`     | อนุมัติการจับคู่ DM หรือปรับนโยบาย DM                              |

การแก้ไขปัญหาแบบครบถ้วน: [/channels/discord#troubleshooting](/channels/discord#troubleshooting)

## Slack

### ลักษณะอาการเสียของ Slack

| อาการ                              | การตรวจสอบที่เร็วที่สุด                      | วิธีแก้ไข                                     |
| ---------------------------------- | -------------------------------------------- | --------------------------------------------- |
| Socket mode เชื่อมต่อแล้วแต่ไม่ตอบ | `openclaw channels status --probe`           | ยืนยัน app token + bot token และสโคปที่จำเป็น |
| DMs ถูกบล็อก                       | `openclaw pairing list slack`                | อนุมัติการจับคู่หรือผ่อนปรนนโยบาย DM          |
| ข้อความในช่องทางถูกละเลย           | ตรวจสอบ `groupPolicy` และรายการอนุญาตช่องทาง | อนุญาตช่องทางหรือเปลี่ยนนโยบายเป็น `open`     |

การแก้ไขปัญหาแบบครบถ้วน: [/channels/slack#troubleshooting](/channels/slack#troubleshooting)

### ลักษณะอาการเสียของ iMessage และ BlueBubbles

| อาการ                      | การตรวจสอบที่เร็วที่สุด                                                   | วิธีแก้ไข                                              |
| -------------------------- | ------------------------------------------------------------------------- | ------------------------------------------------------ |
| ไม่มีอีเวนต์ขาเข้า         | ยืนยันการเข้าถึง webhook/เซิร์ฟเวอร์และสิทธิ์ของแอป                       | แก้ไข URL ของ webhook หรือสถานะเซิร์ฟเวอร์ BlueBubbles |

| ผู้ส่ง DM ถูกบล็อก         | `openclaw pairing list imessage` หรือ `openclaw pairing list bluebubbles` | อนุมัติการจับคู่หรืออัปเดตรายการอนุญาต                 |
