# การติดตามการใช้งาน

## คืออะไร

- ดึงข้อมูลการใช้งาน/โควตาของผู้ให้บริการโดยตรงจาก endpoint การใช้งานของพวกเขา
- ไม่มีการประมาณค่าใช้จ่าย; ใช้เฉพาะช่วงเวลาที่ผู้ให้บริการรายงานเท่านั้น

## การใช้งานของผู้ให้บริการจะแสดงสำหรับ **ผู้ให้บริการโมเดลปัจจุบัน** เมื่อพร้อมใช้งาน

- `/status` ในแชต: การ์ดสถานะที่มีอีโมจิพร้อมโทเคนของเซสชัน+ค่าใช้จ่ายโดยประมาณ(เฉพาะ API key) การใช้งานของผู้ให้บริการจะแสดงสำหรับ**ผู้ให้บริการโมเดลปัจจุบัน**เมื่อมีข้อมูล ประทับเวลาของผู้ให้บริการจะถูกเก็บรักษาไว้เพื่อให้เครื่องมือคงความหมายดั้งเดิม (เวลาปัจจุบันสามารถดูได้ผ่าน `session_status`)
- `/usage off|tokens|full` ในแชต: ฟุตเตอร์การใช้งานต่อการตอบหนึ่งครั้ง(OAuth แสดงเฉพาะโทเคน)
- `/usage cost` ในแชต: สรุปค่าใช้จ่ายภายในเครื่องที่ถูกรวมจากบันทึกเซสชันของ OpenClaw
- CLI: `openclaw status --usage` พิมพ์รายละเอียดแยกตามผู้ให้บริการทั้งหมด
- CLI: `openclaw channels list` พิมพ์สแนปช็อตการใช้งานเดียวกันควบคู่กับคอนฟิกของผู้ให้บริการ(ใช้ `--no-usage` เพื่อข้าม)
- แถบเมนู macOS: ส่วน “Usage” ใต้ Context(เฉพาะเมื่อมีข้อมูล)

## ผู้ให้บริการ+ข้อมูลประจำตัว

- **Anthropic (Claude)**: โทเคนOAuthในโปรไฟล์การยืนยันตัวตน
- **GitHub Copilot**: โทเคนOAuthในโปรไฟล์การยืนยันตัวตน
- **Gemini CLI**: โทเคนOAuthในโปรไฟล์การยืนยันตัวตน
- **Antigravity**: โทเคนOAuthในโปรไฟล์การยืนยันตัวตน
- **OpenAI Codex**: โทเคนOAuthในโปรไฟล์การยืนยันตัวตน(ใช้ accountId เมื่อมี)
- **MiniMax**: API key(คีย์แผน coding; `MINIMAX_CODE_PLAN_KEY` หรือ `MINIMAX_API_KEY`); ใช้ช่วงเวลาแผน coding 5ชั่วโมง
- **z.ai**: API key ผ่าน env/config/auth store

การใช้งานจะถูกซ่อนไว้หากไม่มีข้อมูลประจำตัว OAuth/API ที่ตรงกัน
