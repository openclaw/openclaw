# การใช้งานAPIและค่าใช้จ่าย

เอกสารนี้แสดงรายการ**ฟีเจอร์ที่สามารถเรียกใช้คีย์API**และตำแหน่งที่ค่าใช้จ่ายแสดงผล โดยมุ่งเน้นที่
ฟีเจอร์ของOpenClawที่สามารถก่อให้เกิดการใช้งานผู้ให้บริการหรือการเรียกAPIแบบเสียเงิน มุ่งเน้นที่
ฟีเจอร์ของ OpenClaw ที่สามารถก่อให้เกิดการใช้งานผู้ให้บริการหรือการเรียก API แบบมีค่าใช้จ่าย

## ค่าใช้จ่ายแสดงที่ใดบ้าง (แชท + CLI)

**ภาพรวมค่าใช้จ่ายต่อเซสชัน**

- `/status` แสดงโมเดลของเซสชันปัจจุบัน การใช้บริบท และโทเคนของการตอบกลับล่าสุด
- หากโมเดลใช้**การยืนยันตัวตนด้วยคีย์API**, `/status` จะแสดง**ค่าใช้จ่ายโดยประมาณ**สำหรับการตอบกลับล่าสุดด้วย

**ส่วนท้ายค่าใช้จ่ายต่อข้อความ**

- `/usage full` จะเพิ่มส่วนท้ายการใช้งานให้กับทุกการตอบกลับ รวมถึง**ค่าใช้จ่ายโดยประมาณ** (เฉพาะคีย์API)
- `/usage tokens` แสดงเฉพาะโทเคน; โฟลว์OAuthจะซ่อนค่าใช้จ่ายเป็นดอลลาร์

**หน้าต่างการใช้งานCLI (โควตาผู้ให้บริการ)**

- `openclaw status --usage` และ `openclaw channels list` แสดง**หน้าต่างการใช้งาน**ของผู้ให้บริการ
  (ภาพรวมโควตา ไม่ใช่ค่าใช้จ่ายต่อข้อความ)

ดูรายละเอียดและตัวอย่างได้ที่[การใช้โทเคนและค่าใช้จ่าย](/reference/token-use)

## วิธีการค้นหาคีย์

OpenClawสามารถดึงข้อมูลรับรองได้จาก:

- **โปรไฟล์การยืนยันตัวตน** (ต่อเอเจนต์ จัดเก็บใน `auth-profiles.json`)
- **ตัวแปรสภาพแวดล้อม** (เช่น `OPENAI_API_KEY`, `BRAVE_API_KEY`, `FIRECRAWL_API_KEY`)
- **คอนฟิก** (`models.providers.*.apiKey`, `tools.web.search.*`, `tools.web.fetch.firecrawl.*`,
  `memorySearch.*`, `talk.apiKey`)
- **Skills** (`skills.entries.<name>.apiKey`) ซึ่งอาจส่งออกคีย์ไปยังสภาพแวดล้อมของโปรเซสSkill

## ฟีเจอร์ที่สามารถใช้คีย์และก่อค่าใช้จ่าย

### 1. การตอบกลับของโมเดลหลัก (แชท + เครื่องมือ)

ทุกการตอบกลับหรือการเรียกใช้เครื่องมือจะใช้**ผู้ให้บริการโมเดลปัจจุบัน** (OpenAI, Anthropic ฯลฯ) ซึ่งเป็น
แหล่งหลักของการใช้งานและค่าใช้จ่าย นี่คือ
แหล่งหลักของการใช้งานและต้นทุน

ดู[โมเดล](/providers/models)สำหรับการกำหนดค่าราคา และ[การใช้โทเคนและค่าใช้จ่าย](/reference/token-use)สำหรับการแสดงผล

### 2. การทำความเข้าใจสื่อ (เสียง/ภาพ/วิดีโอ)

Inbound media can be summarized/transcribed before the reply runs. สิ่งนี้ใช้ API ของโมเดล/ผู้ให้บริการ

- เสียง: OpenAI / Groq / Deepgram (ขณะนี้**เปิดใช้งานอัตโนมัติ**เมื่อมีคีย์)
- ภาพ: OpenAI / Anthropic / Google
- วิดีโอ: Google

ดู[การทำความเข้าใจสื่อ](/nodes/media-understanding)

### 3. เอ็มเบดดิงหน่วยความจำ + การค้นหาเชิงความหมาย

การค้นหาหน่วยความจำเชิงความหมายจะใช้**APIเอ็มเบดดิง**เมื่อกำหนดค่าให้ใช้ผู้ให้บริการระยะไกล:

- `memorySearch.provider = "openai"` → OpenAI embeddings
- `memorySearch.provider = "gemini"` → Gemini embeddings
- `memorySearch.provider = "voyage"` → Voyage embeddings
- ตัวเลือกสำรองไปยังผู้ให้บริการระยะไกลหากเอ็มเบดดิงภายในเครื่องล้มเหลว

คุณสามารถคงไว้ภายในเครื่องด้วย `memorySearch.provider = "local"` (ไม่มีการใช้API)

ดู[หน่วยความจำ](/concepts/memory)

### 4. เครื่องมือค้นหาเว็บ (Brave / Perplexity ผ่าน OpenRouter)

`web_search` ใช้คีย์APIและอาจมีค่าใช้จ่ายจากการใช้งาน:

- **Brave Search API**: `BRAVE_API_KEY` หรือ `tools.web.search.apiKey`
- **Perplexity** (ผ่าน OpenRouter): `PERPLEXITY_API_KEY` หรือ `OPENROUTER_API_KEY`

**Braveระดับฟรี (ค่อนข้างใจกว้าง):**

- **2,000คำขอ/เดือน**
- **1คำขอ/วินาที**
- **ต้องใช้บัตรเครดิต**เพื่อการยืนยัน (ไม่มีการเรียกเก็บเงินเว้นแต่อัปเกรด)

ดู[เครื่องมือเว็บ](/tools/web)

### 5. เครื่องมือดึงข้อมูลเว็บ (Firecrawl)

`web_fetch` สามารถเรียกใช้**Firecrawl**เมื่อมีคีย์API:

- `FIRECRAWL_API_KEY` หรือ `tools.web.fetch.firecrawl.apiKey`

หากไม่ได้กำหนดค่าFirecrawl เครื่องมือจะย้อนกลับไปใช้การดึงข้อมูลโดยตรง + readability (ไม่มีAPIแบบเสียเงิน)

ดู[เครื่องมือเว็บ](/tools/web)

### 6. ภาพรวมการใช้งานผู้ให้บริการ (สถานะ/สุขภาพระบบ)

คำสั่งสถานะบางรายการจะเรียก**เอ็นด์พอยต์การใช้งานของผู้ให้บริการ**เพื่อแสดงหน้าต่างโควตาหรือสุขภาพการยืนยันตัวตน
โดยทั่วไปเป็นการเรียกปริมาณต่ำ แต่ยังคงกระทบAPIของผู้ให้บริการ:
โดยทั่วไปแล้วเป็นการเรียกที่มีปริมาณต่ำ แต่ยังคงกระทบ API ของผู้ให้บริการ:

- `openclaw status --usage`
- `openclaw models status --json`

ดู[Models CLI](/cli/models)

### 7. การสรุปเพื่อป้องกันการคอมแพคชัน

กลไกป้องกันการคอมแพคชันสามารถสรุปประวัติเซสชันโดยใช้**โมเดลปัจจุบัน** ซึ่งจะเรียกAPIของผู้ให้บริการเมื่อทำงาน

ดู[การจัดการเซสชัน + การคอมแพคชัน](/reference/session-management-compaction)

### 8. การสแกน/โพรบโมเดล

`openclaw models scan` สามารถโพรบโมเดลของOpenRouterและใช้ `OPENROUTER_API_KEY` เมื่อ
เปิดใช้งานการโพรบ

ดู[Models CLI](/cli/models)

### 9. Talk (การพูด)

โหมดTalkสามารถเรียกใช้**ElevenLabs**เมื่อกำหนดค่าแล้ว:

- `ELEVENLABS_API_KEY` หรือ `talk.apiKey`

ดู[โหมดTalk](/nodes/talk)

### 10. Skills (APIของบุคคลที่สาม)

Skillsสามารถจัดเก็บ `apiKey` ใน `skills.entries.<name>.apiKey` .apiKey\` หากSkillใช้คีย์นั้นกับ
APIภายนอก อาจเกิดค่าใช้จ่ายตามผู้ให้บริการของSkillนั้น

ดู[Skills](/tools/skills)
