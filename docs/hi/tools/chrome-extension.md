# Chrome एक्सटेंशन (ब्राउज़र रिले)

OpenClaw Chrome एक्सटेंशन एजेंट को **आपके मौजूदा Chrome टैब्स** (आपकी सामान्य Chrome विंडो) को नियंत्रित करने देता है, बजाय इसके कि एक अलग openclaw-प्रबंधित Chrome प्रोफ़ाइल लॉन्च की जाए।

अटैच/डिटैच **एकल Chrome टूलबार बटन** के माध्यम से होता है।

## यह क्या है (अवधारणा)

तीन हिस्से हैं:

- **ब्राउज़र कंट्रोल सेवा** (Gateway या नोड): वह API जिसे एजेंट/टूल कॉल करता है (Gateway के माध्यम से)
- **लोकल रिले सर्वर** (loopback CDP): कंट्रोल सर्वर और एक्सटेंशन के बीच ब्रिज करता है (डिफ़ॉल्ट रूप से `http://127.0.0.1:18792`)
- **Chrome MV3 एक्सटेंशन**: `chrome.debugger` का उपयोग करके सक्रिय टैब से अटैच होता है और CDP संदेशों को रिले तक पाइप करता है

इसके बाद OpenClaw सामान्य `browser` टूल सरफ़ेस के माध्यम से (सही प्रोफ़ाइल चुनते हुए) अटैच किए गए टैब को नियंत्रित करता है।

## इंस्टॉल / लोड (अनपैक्ड)

1. एक्सटेंशन को एक स्थिर स्थानीय पथ पर इंस्टॉल करें:

```bash
openclaw browser extension install
```

2. इंस्टॉल की गई एक्सटेंशन डायरेक्टरी का पथ प्रिंट करें:

```bash
openclaw browser extension path
```

3. Chrome → `chrome://extensions`

- “Developer mode” सक्षम करें
- “Load unpacked” → ऊपर प्रिंट की गई डायरेक्टरी चुनें

4. एक्सटेंशन को पिन करें।

## अपडेट्स (कोई बिल्ड चरण नहीं)

18. एक्सटेंशन OpenClaw release (npm package) के अंदर static files के रूप में ship होता है। 19. कोई अलग “build” step नहीं है।

OpenClaw अपग्रेड करने के बाद:

- अपने OpenClaw स्टेट डायरेक्टरी के अंतर्गत इंस्टॉल फ़ाइलों को ताज़ा करने के लिए `openclaw browser extension install` फिर से चलाएँ।
- Chrome → `chrome://extensions` → एक्सटेंशन पर “Reload” क्लिक करें।

## उपयोग (कोई अतिरिक्त कॉन्फ़िग नहीं)

OpenClaw एक अंतर्निहित ब्राउज़र प्रोफ़ाइल के साथ आता है जिसका नाम `chrome` है, जो डिफ़ॉल्ट पोर्ट पर एक्सटेंशन रिले को लक्षित करता है।

इसे उपयोग करें:

- CLI: `openclaw browser --browser-profile chrome tabs`
- एजेंट टूल: `browser` के साथ `profile="chrome"`

यदि आप अलग नाम या अलग रिले पोर्ट चाहते हैं, तो अपनी स्वयं की प्रोफ़ाइल बनाएँ:

```bash
openclaw browser create-profile \
  --name my-chrome \
  --driver extension \
  --cdp-url http://127.0.0.1:18792 \
  --color "#00AA00"
```

## अटैच / डिटैच (टूलबार बटन)

- वह टैब खोलें जिसे आप OpenClaw से नियंत्रित करना चाहते हैं।
- एक्सटेंशन आइकन पर क्लिक करें।
  - अटैच होने पर बैज `ON` दिखाता है।
- डिटैच करने के लिए फिर से क्लिक करें।

## यह किस टैब को नियंत्रित करता है?

- यह स्वतः “जिस टैब को आप देख रहे हैं” उसे नियंत्रित नहीं करता।
- यह **केवल उन टैब्स** को नियंत्रित करता है जिन्हें आपने टूलबार बटन पर क्लिक करके स्पष्ट रूप से अटैच किया है।
- स्विच करने के लिए: दूसरे टैब को खोलें और वहाँ एक्सटेंशन आइकन पर क्लिक करें।

## बैज + सामान्य त्रुटियाँ

- `ON`: अटैच्ड; OpenClaw उस टैब को चला सकता है।
- `…`: लोकल रिले से कनेक्ट हो रहा है।
- `!`: रिले पहुँच योग्य नहीं है (सबसे सामान्य: इस मशीन पर ब्राउज़र रिले सर्वर चल नहीं रहा है)।

यदि आपको `!` दिखाई दे:

- सुनिश्चित करें कि Gateway स्थानीय रूप से चल रहा है (डिफ़ॉल्ट सेटअप), या यदि Gateway कहीं और चलता है तो इस मशीन पर नोड होस्ट चलाएँ।
- एक्सटेंशन Options पेज खोलें; यह दिखाता है कि रिले पहुँच योग्य है या नहीं।

## रिमोट Gateway (नोड होस्ट का उपयोग करें)

### लोकल Gateway (Chrome वाली ही मशीन) — सामान्यतः **कोई अतिरिक्त चरण नहीं**

20. यदि Gateway उसी मशीन पर चलता है जिस पर Chrome है, तो यह loopback पर browser control service शुरू करता है
    और relay server को auto-start करता है। 21. एक्सटेंशन local relay से बात करता है; CLI/tool calls Gateway पर जाती हैं।

### रिमोट Gateway (Gateway कहीं और चलता है) — **नोड होस्ट चलाएँ**

22. यदि आपका Gateway किसी अन्य मशीन पर चलता है, तो उस मशीन पर node host शुरू करें जो Chrome चलाती है।
23. Gateway browser actions को उस node पर proxy करेगा; extension + relay ब्राउज़र मशीन पर local रहते हैं।

यदि कई नोड्स जुड़े हों, तो `gateway.nodes.browser.node` के साथ एक को पिन करें या `gateway.nodes.browser.mode` सेट करें।

## Sandboxing (टूल कंटेनर्स)

यदि आपका एजेंट सत्र sandboxed है (`agents.defaults.sandbox.mode != "off"`), तो `browser` टूल सीमित हो सकता है:

- डिफ़ॉल्ट रूप से, sandboxed सत्र अक्सर **sandbox ब्राउज़र** (`target="sandbox"`) को लक्षित करते हैं, न कि आपके होस्ट Chrome को।
- Chrome एक्सटेंशन रिले टेकओवर के लिए **होस्ट** ब्राउज़र कंट्रोल सर्वर को नियंत्रित करना आवश्यक है।

विकल्प:

- सबसे आसान: एक्सटेंशन को **non-sandboxed** सत्र/एजेंट से उपयोग करें।
- या sandboxed सत्रों के लिए होस्ट ब्राउज़र कंट्रोल की अनुमति दें:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

फिर सुनिश्चित करें कि टूल पॉलिसी द्वारा टूल अस्वीकृत न हो, और (यदि आवश्यक हो) `browser` को `target="host"` के साथ कॉल करें।

डिबगिंग: `openclaw sandbox explain`

## रिमोट एक्सेस सुझाव

- Gateway और नोड होस्ट को एक ही tailnet पर रखें; रिले पोर्ट्स को LAN या सार्वजनिक इंटरनेट पर एक्सपोज़ करने से बचें।
- नोड्स को जानबूझकर पेयर करें; यदि आप रिमोट कंट्रोल नहीं चाहते तो ब्राउज़र प्रॉक्सी रूटिंग अक्षम करें (`gateway.nodes.browser.mode="off"`)।

## “एक्सटेंशन पाथ” कैसे काम करता है

`openclaw browser extension path` डिस्क पर मौजूद **इंस्टॉल की गई** डायरेक्टरी प्रिंट करता है जिसमें एक्सटेंशन फ़ाइलें होती हैं।

24. CLI जानबूझकर `node_modules` path print नहीं करता। 25. हमेशा पहले `openclaw browser extension install` चलाएँ ताकि एक्सटेंशन को आपके OpenClaw state directory के अंतर्गत एक stable location पर copy किया जा सके।

यदि आप उस इंस्टॉल डायरेक्टरी को स्थानांतरित या हटा देते हैं, तो Chrome एक्सटेंशन को तब तक टूटा हुआ चिह्नित करेगा जब तक आप उसे किसी वैध पथ से पुनः लोड न करें।

## सुरक्षा प्रभाव (यह पढ़ें)

26. यह शक्तिशाली और जोखिम भरा है। 27. इसे ऐसे समझें जैसे मॉडल को “आपके ब्राउज़र पर हाथ” दे दिए गए हों।

- 28. एक्सटेंशन Chrome के debugger API (`chrome.debugger`) का उपयोग करता है। 29. जब attached होता है, तो मॉडल:
  - उस टैब में क्लिक/टाइप/नेविगेट कर सकता है
  - पेज की सामग्री पढ़ सकता है
  - टैब की लॉग-इन सत्र जिन तक पहुँच है, उन तक पहुँच सकता है
- **यह अलग-थलग नहीं है** जैसे समर्पित openclaw-प्रबंधित प्रोफ़ाइल।
  - यदि आप अपने दैनिक-उपयोग प्रोफ़ाइल/टैब से अटैच करते हैं, तो आप उस खाते की स्थिति तक पहुँच दे रहे हैं।

अनुशंसाएँ:

- एक्सटेंशन रिले उपयोग के लिए एक समर्पित Chrome प्रोफ़ाइल (आपकी व्यक्तिगत ब्राउज़िंग से अलग) को प्राथमिकता दें।
- Gateway और किसी भी नोड होस्ट को केवल tailnet तक सीमित रखें; Gateway प्रमाणीकरण + नोड पेयरिंग पर भरोसा करें।
- रिले पोर्ट्स को LAN (`0.0.0.0`) पर एक्सपोज़ करने से बचें और Funnel (पब्लिक) से बचें।
- रिले गैर-एक्सटेंशन origins को ब्लॉक करता है और CDP क्लाइंट्स के लिए एक आंतरिक ऑथ टोकन की आवश्यकता होती है।
