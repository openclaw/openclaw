# प्लगइन मैनिफ़ेस्ट (openclaw.plugin.json)

हर प्लगइन को **plugin root** में एक `openclaw.plugin.json` फ़ाइल **ज़रूर** शामिल करनी चाहिए।
OpenClaw इस मैनिफ़ेस्ट का उपयोग प्लगइन कोड **चलाए बिना** कॉन्फ़िगरेशन को वैलिडेट करने के लिए करता है। ग़ायब या अमान्य मैनिफ़ेस्ट को प्लगइन त्रुटि माना जाता है और वे कॉन्फ़िग वैलिडेशन को ब्लॉक कर देते हैं।

पूर्ण प्लगइन सिस्टम गाइड देखें: [Plugins](/tools/plugin).

## आवश्यक फ़ील्ड

```json
{
  "id": "voice-call",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {}
  }
}
```

आवश्यक कुंजियाँ:

- `id` (string): कैनॉनिकल प्लगइन आईडी।
- `configSchema` (object): प्लगइन विन्यास के लिए JSON Schema (इनलाइन)।

वैकल्पिक कुंजियाँ:

- `kind` (string): प्लगइन प्रकार (उदाहरण: `"memory"`)।
- `channels` (array): इस प्लगइन द्वारा पंजीकृत चैनल आईडी (उदाहरण: `["matrix"]`)।
- `providers` (array): इस प्लगइन द्वारा पंजीकृत प्रदाता आईडी।
- `skills` (array): लोड किए जाने वाले skill निर्देशिकाएँ (प्लगइन रूट के सापेक्ष)।
- `name` (string): प्लगइन के लिए प्रदर्शित नाम।
- `description` (string): संक्षिप्त प्लगइन सारांश।
- `uiHints` (object): UI रेंडरिंग के लिए विन्यास फ़ील्ड लेबल/प्लेसहोल्डर/संवेदनशील फ़्लैग।
- `version` (string): प्लगइन संस्करण (सूचनात्मक)।

## JSON Schema आवश्यकताएँ

- **हर प्लगइन को एक JSON Schema शामिल करना अनिवार्य है**, भले ही वह कोई विन्यास स्वीकार न करता हो।
- एक खाली स्कीमा स्वीकार्य है (उदाहरण के लिए, `{ "type": "object", "additionalProperties": false }`)।
- स्कीमा का सत्यापन विन्यास पढ़ने/लिखने के समय किया जाता है, रनटाइम पर नहीं।

## सत्यापन व्यवहार

- अज्ञात `channels.*` कुंजियाँ **त्रुटि** होती हैं, जब तक कि चैनल आईडी
  किसी प्लगइन मैनिफ़ेस्ट द्वारा घोषित न की गई हो।
- `plugins.entries.<id>``, `plugins.allow`, `plugins.deny`, और `plugins.slots.\*\` को **discoverable** प्लगइन ids को संदर्भित करना चाहिए। अज्ञात ids **त्रुटियाँ** हैं।
- यदि कोई प्लगइन इंस्टॉल है लेकिन उसका मैनिफ़ेस्ट या स्कीमा टूटा हुआ है या अनुपस्थित है,
  तो सत्यापन विफल हो जाता है और Doctor प्लगइन त्रुटि की रिपोर्ट करता है।
- यदि प्लगइन विन्यास मौजूद है लेकिन प्लगइन **अक्षम** है, तो विन्यास रखा जाता है और
  Doctor + लॉग्स में एक **चेतावनी** प्रदर्शित की जाती है।

## टिप्पणियाँ

- मैनिफ़ेस्ट **सभी प्लगइनों के लिए अनिवार्य** है, जिसमें लोकल फ़ाइलसिस्टम लोड भी शामिल हैं।
- रनटाइम अभी भी प्लगइन मॉड्यूल को अलग से लोड करता है; मैनिफ़ेस्ट केवल
  डिस्कवरी + सत्यापन के लिए है।
- यदि आपका प्लगइन नेटिव मॉड्यूल पर निर्भर करता है, तो बिल्ड चरणों और किसी भी
  पैकेज-मैनेजर allowlist आवश्यकताओं का दस्तावेज़ीकरण करें
  (उदाहरण के लिए, pnpm `allow-build-scripts`
  - `pnpm rebuild <package>`)।
