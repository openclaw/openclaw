    shipping spec नहीं है। 41. वर्तमान व्यवहार के लिए, देखें:

- [Models](/concepts/models)
- [Model failover](/concepts/model-failover)
- [OAuth + profiles](/concepts/oauth)

## प्रेरणा

ऑपरेटर चाहते हैं:

- प्रति प्रदाता कई प्रमाणीकरण प्रोफ़ाइल (व्यक्तिगत बनाम कार्य)।
- पूर्वानुमेय फ़ॉलबैक के साथ सरल `/model` चयन।
- टेक्स्ट मॉडल और छवि-क्षमता वाले मॉडलों के बीच स्पष्ट पृथक्करण।

## संभावित दिशा (उच्च स्तर)

- मॉडल चयन को सरल रखें: वैकल्पिक उपनामों के साथ `provider/model`।
- प्रदाताओं को स्पष्ट क्रम के साथ कई प्रमाणीकरण प्रोफ़ाइल रखने दें।
- एक वैश्विक फ़ॉलबैक सूची का उपयोग करें ताकि सभी सत्रों में समान रूप से फ़ेलओवर हो।
- केवल तभी छवि रूटिंग को ओवरराइड करें जब स्पष्ट रूप से विन्यस्त किया गया हो।

## खुले प्रश्न

- क्या प्रोफ़ाइल रोटेशन प्रति-प्रदाता होना चाहिए या प्रति-मॉडल?
- UI को किसी सत्र के लिए प्रोफ़ाइल चयन कैसे प्रस्तुत करना चाहिए?
- विरासत विन्यास कुंजियों से सबसे सुरक्षित माइग्रेशन पथ क्या है?
