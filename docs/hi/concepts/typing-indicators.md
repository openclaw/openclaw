# टाइपिंग संकेतक

जब कोई रन सक्रिय होता है, तब टाइपिंग संकेतक चैट चैनल में भेजे जाते हैं। **कब** टाइपिंग शुरू हो, इसे नियंत्रित करने के लिए `agents.defaults.typingMode` का उपयोग करें, और **कितनी बार** यह रिफ़्रेश हो, इसे नियंत्रित करने के लिए `typingIntervalSeconds` का उपयोग करें।

## डिफ़ॉल्ट

जब `agents.defaults.typingMode` **सेट नहीं** होता है, तो OpenClaw पुराना व्यवहार बनाए रखता है:

- **डायरेक्ट चैट**: मॉडल लूप शुरू होते ही तुरंत टाइपिंग शुरू होती है।
- **मेंशन के साथ ग्रुप चैट**: टाइपिंग तुरंत शुरू होती है।
- **बिना मेंशन के ग्रुप चैट**: टाइपिंग केवल तब शुरू होती है जब संदेश का पाठ स्ट्रीम होना शुरू होता है।
- **हार्टबीट रन**: टाइपिंग अक्षम रहती है।

## मोड्स

`agents.defaults.typingMode` को इनमें से किसी एक पर सेट करें:

- `never` — कभी भी कोई टाइपिंग संकेतक नहीं।
- `instant` — **जैसे ही मॉडल लूप शुरू होता है**, टाइपिंग शुरू करें, भले ही रन बाद में केवल साइलेंट रिप्लाई टोकन ही लौटाए।
- `thinking` — **पहले रीजनिंग डेल्टा** पर टाइपिंग शुरू करें (रन के लिए `reasoningLevel: "stream"` आवश्यक है)।
- `message` — **पहले नॉन-साइलेंट टेक्स्ट डेल्टा** पर टाइपिंग शुरू करें (`NO_REPLY` साइलेंट टोकन को अनदेखा करता है)।

“कितनी जल्दी ट्रिगर होता है” के क्रम में:
`never` → `message` → `thinking` → `instant`

## विन्यास

```json5
{
  agent: {
    typingMode: "thinking",
    typingIntervalSeconds: 6,
  },
}
```

आप प्रति सत्र मोड या कैडेंस को ओवरराइड कर सकते हैं:

```json5
{
  session: {
    typingMode: "message",
    typingIntervalSeconds: 4,
  },
}
```

## टिप्पणियाँ

- `message` मोड साइलेंट-ओनली रिप्लाई के लिए टाइपिंग नहीं दिखाएगा (उदाहरण के लिए, आउटपुट दबाने के लिए उपयोग किया जाने वाला `NO_REPLY` टोकन)।
- `thinking` केवल तभी ट्रिगर होता है जब रन रीजनिंग स्ट्रीम करता है (`reasoningLevel: "stream"`)। यदि मॉडल रीजनिंग डेल्टा उत्सर्जित नहीं करता, तो टाइपिंग शुरू नहीं होगी।
- मोड चाहे जो भी हो, हार्टबीट कभी भी टाइपिंग नहीं दिखाते।
- `typingIntervalSeconds` **रिफ़्रेश कैडेंस** को नियंत्रित करता है, शुरू होने के समय को नहीं। डिफ़ॉल्ट 6 सेकंड है।
