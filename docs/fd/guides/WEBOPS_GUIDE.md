# WebOps Guide

## What is WebOps?

WebOps keeps your web presence stable and fast. It monitors your websites,
tracks deployments, verifies tracking pixels, audits webhooks, and generates
repair plans when something breaks.

When an issue is found, OpenClaw tells you what's wrong and how to fix it —
then waits for your approval before making changes.

## What can you do here?

- **Check website health** — uptime, speed, errors
- **Review incidents** — what broke and when
- **Run safe fixes** — pre-approved, low-risk repairs
- **Generate repair plans** — step-by-step fix proposals
- **Verify tracking pixels** — make sure analytics are working
- **Audit webhook status** — check integration health

## Common prompts

Try asking OpenClaw:

- "Check website health."
- "Fix what's broken safely."
- "What's wrong with the site?"
- "Show me recent incidents."
- "Are all tracking pixels firing?"

## What requires approval?

- Deploying changes
- Modifying DNS
- Restarting services

Health checks, viewing incidents, and running reports is always safe.

## How to check website health

1. Go to WebOps
2. Ask: "Check website health"
3. Review the health report — uptime, speed, tracking, errors
4. If issues are found, review the suggested repair plan
5. Approve safe fixes or escalate complex ones

## How to handle an incident

1. OpenClaw alerts you when something breaks
2. Go to WebOps to see the incident details
3. Review the auto-generated repair plan
4. Approve the fix or ask for alternatives
5. Monitor the fix after it's applied
