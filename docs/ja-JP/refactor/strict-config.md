# 厳格なコンフィグバリデーション（doctor 専用の移行）

## 目標

- ルート `$schema` メタデータを除き、**すべての場所（ルート + ネスト）で不明なコンフィグキーを拒否する**。
- **スキーマなしのプラグインコンフィグを拒否**。そのプラグインをロードしない。
- **ロード時のレガシー自動移行を削除**。移行は doctor 経由でのみ実行。
- **起動時に doctor（ドライラン）を自動実行**。無効な場合、診断コマンド以外をブロック。

## 非目標

- ロード時の後方互換性（レガシーキーは自動移行しない）。
- 認識されないキーのサイレントドロップ。

## 厳格なバリデーションルール

- コンフィグはすべてのレベルでスキーマと完全に一致する必要があります。
- 不明なキーはバリデーションエラーです（ルートまたはネストでのパススルーなし）。ただし、ルート `$schema` が文字列の場合を除く。
- `plugins.entries.<id>.config` はプラグインのスキーマで検証する必要があります。
  - プラグインにスキーマがない場合、**プラグインロードを拒否**し、明確なエラーを表示します。
- `channels.<id>` の不明なキーは、プラグインマニフェストがチャンネル ID を宣言していない限りエラー。
- プラグインマニフェスト（`openclaw.plugin.json`）はすべてのプラグインに必須。

## プラグインスキーマの強制

- 各プラグインはコンフィグの厳格な JSON スキーマを提供します（マニフェストにインライン）。
- プラグインロードフロー:
  1. プラグインマニフェスト + スキーマを解決（`openclaw.plugin.json`）。
  2. スキーマに対してコンフィグを検証。
  3. スキーマ欠如または無効なコンフィグ: プラグインロードをブロックし、エラーを記録。
- エラーメッセージには以下を含む:
  - プラグイン ID
  - 理由（スキーマ欠如 / 無効なコンフィグ）
  - バリデーションに失敗したパス
- 無効化されたプラグインはコンフィグを維持しますが、Doctor + ログが警告を表示します。

## Doctor フロー

- Doctor はコンフィグがロードされるたびに実行されます（デフォルトではドライラン）。
- コンフィグが無効な場合:
  - サマリー + 実行可能なエラーを表示。
  - `openclaw doctor --fix` を実行するよう指示。
- `openclaw doctor --fix`:
  - 移行を適用。
  - 不明なキーを削除。
  - 更新されたコンフィグを書き込む。

## コマンドゲーティング（コンフィグが無効な場合）

許可（診断のみ）:

- `openclaw doctor`
- `openclaw logs`
- `openclaw health`
- `openclaw help`
- `openclaw status`
- `openclaw gateway status`

それ以外はすべて次のメッセージでハードフェイルする必要があります: 「Config invalid. Run `openclaw doctor --fix`.」

## エラー UX フォーマット

- 単一のサマリーヘッダー。
- グループ化されたセクション:
  - 不明なキー（フルパス）
  - レガシーキー / 必要な移行
  - プラグインロードの失敗（プラグイン ID + 理由 + パス）

## 実装タッチポイント

- `src/config/zod-schema.ts`: ルートのパススルーを削除。すべての場所で厳格なオブジェクト。
- `src/config/zod-schema.providers.ts`: 厳格なチャンネルスキーマを確保。
- `src/config/validation.ts`: 不明なキーで失敗。レガシー移行を適用しない。
- `src/config/io.ts`: レガシー自動移行を削除。常に doctor ドライランを実行。
- `src/config/legacy*.ts`: 使用を doctor のみに移動。
- `src/plugins/*`: スキーマレジストリ + ゲーティングを追加。
- `src/cli` での CLI コマンドゲーティング。

## テスト

- 不明なキーの拒否（ルート + ネスト）。
- プラグインのスキーマ欠如 → 明確なエラーでプラグインロードをブロック。
- 無効なコンフィグ → 診断コマンド以外の Gateway 起動をブロック。
- Doctor ドライランの自動実行。`doctor --fix` で修正されたコンフィグを書き込む。
