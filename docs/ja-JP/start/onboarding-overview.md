# オンボーディング概要

OpenClawには2つのオンボーディングパスがあります。どちらも認証、Gateway ゲートウェイ、およびオプションのチャネルを設定します — 違いはセットアップ時の操作方法だけです。

## どちらのパスを使うべきですか？

|                | CLIオンボーディング                    | macOSアプリオンボーディング |
| -------------- | -------------------------------------- | --------------------------- |
| **プラットフォーム** | macOS、Linux、Windows（ネイティブまたはWSL2） | macOSのみ                   |
| **インターフェース** | ターミナルウィザード                   | アプリ内のガイド付きUI      |
| **最適な用途** | サーバー、ヘッドレス、完全な制御       | デスクトップMac、ビジュアルセットアップ |
| **自動化**     | スクリプト用の`--non-interactive`      | 手動のみ                    |
| **コマンド**   | `openclaw onboard`                     | アプリを起動                |

ほとんどのユーザーは**CLIオンボーディング**から始めるべきです — どこでも動作し、最も細かい制御が可能です。

## オンボーディングで設定されるもの

どちらのパスを選んでも、オンボーディングでは以下が設定されます：

1. **モデルプロバイダーと認証** — 選択したプロバイダーのAPIキー、OAuth、またはセットアップトークン
2. **ワークスペース** — エージェントファイル、ブートストラップテンプレート、メモリ用のディレクトリ
3. **Gateway ゲートウェイ** — ポート、バインドアドレス、認証モード
4. **チャネル**（オプション） — WhatsApp、Telegram、Discord、その他
5. **デーモン**（オプション） — Gateway ゲートウェイが自動的に起動するバックグラウンドサービス

## CLIオンボーディング

任意のターミナルで実行します：

```bash
openclaw onboard
```

`--install-daemon`を追加すると、バックグラウンドサービスのインストールも1ステップで行えます。

完全なリファレンス: [セットアップウィザード（CLI）](/start/wizard)
CLIコマンドドキュメント: [`openclaw onboard`](/cli/onboard)

## macOSアプリオンボーディング

OpenClawアプリを開きます。初回起動時のウィザードが、ビジュアルインターフェースで同じ手順を案内します。

完全なリファレンス: [オンボーディング（macOSアプリ）](/start/onboarding)

## カスタムまたは未掲載のプロバイダー

使用したいプロバイダーがオンボーディングに表示されない場合は、**カスタムプロバイダー**を選択して以下を入力してください：

- API互換モード（OpenAI互換、Anthropic互換、または自動検出）
- ベースURLとAPIキー
- モデルIDとオプションのエイリアス

複数のカスタムエンドポイントを共存させることができます — それぞれに固有のエンドポイントIDが割り当てられます。
