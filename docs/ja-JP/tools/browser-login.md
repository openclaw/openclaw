# ブラウザログイン + X/Twitter投稿

## 手動ログイン（推奨）

サイトがログインを要求する場合は、**ホスト**ブラウザプロファイル（openclawブラウザ）で**手動でサインイン**してください。

モデルに認証情報を渡さないでください。自動ログインはボット対策を引き起こし、アカウントがロックされる可能性があります。

メインのブラウザドキュメントに戻る: [ブラウザ](/tools/browser)。

## どのChromeプロファイルが使用されますか？

OpenClawは**専用のChromeプロファイル**（`openclaw` という名前、オレンジがかったUI）を制御します。これは日常的なブラウザプロファイルとは別です。

アクセスする2つの簡単な方法:

1. **エージェントにブラウザを開くよう依頼して**、自分でログインします。
2. **CLIで開く**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

複数のプロファイルがある場合は、`--browser-profile <name>` を指定してください（デフォルトは `openclaw`）。

## X/Twitter: 推奨フロー

- **閲覧/検索/スレッド:** **ホスト**ブラウザを使用します（手動ログイン）。
- **更新を投稿:** **ホスト**ブラウザを使用します（手動ログイン）。

## サンドボックスとホストブラウザアクセス

サンドボックス化されたブラウザセッションはボット検知を**引き起こしやすく**なります。X/Twitter（および他の厳格なサイト）では、**ホスト**ブラウザを優先してください。

エージェントがサンドボックス化されている場合、ブラウザツールはデフォルトでサンドボックスになります。ホストコントロールを許可するには:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

次にホストブラウザをターゲットにします:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

または、更新を投稿するエージェントのサンドボックス化を無効にします。
