# ブラウザログイン + X/Twitter投稿

## 手動ログイン（推奨）

サイトがログインを必要とする場合、**ホスト**ブラウザプロファイル（openclawブラウザ）で**手動でサインイン**してください。

モデルに認証情報を渡さ**ないで**ください。自動ログインはボット対策の防御を発動させることが多く、アカウントがロックされる可能性があります。

メインのブラウザドキュメントに戻る: [ブラウザ](/tools/browser)。

## どのChromeプロファイルが使用されるか？

OpenClawは**専用のChromeプロファイル**（`openclaw` という名前、オレンジ色のUI）を制御します。これは日常のブラウザプロファイルとは別のものです。

エージェントのブラウザツール呼び出しの場合:

- デフォルトの選択: エージェントは分離された `openclaw` ブラウザを使用すべきです。
- `profile="user"` は既存のログイン済みセッションが重要で、ユーザーがコンピュータの前にいて接続プロンプトのクリック/承認ができる場合にのみ使用してください。
- 複数のユーザーブラウザプロファイルがある場合は、推測せずにプロファイルを明示的に指定してください。

アクセスする2つの簡単な方法:

1. **エージェントにブラウザを開くよう依頼し**、その後自分でログインする。
2. **CLI経由で開く**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

複数のプロファイルがある場合は、`--browser-profile <name>` を渡してください（デフォルトは `openclaw`）。

## X/Twitter: 推奨フロー

- **閲覧/検索/スレッド:** **ホスト**ブラウザを使用（手動ログイン）。
- **更新の投稿:** **ホスト**ブラウザを使用（手動ログイン）。

## サンドボックス化 + ホストブラウザアクセス

サンドボックス化されたブラウザセッションはボット検出を発動する可能性が**より高く**なります。X/Twitter（およびその他の厳格なサイト）では、**ホスト**ブラウザの使用を推奨します。

エージェントがサンドボックス化されている場合、ブラウザツールはデフォルトでサンドボックスを使用します。ホスト制御を許可するには:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

その後、ホストブラウザを指定します:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

または、更新を投稿するエージェントのサンドボックス化を無効にしてください。
