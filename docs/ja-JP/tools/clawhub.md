# ClawHub

ClawHubは**OpenClaw向けのパブリックスキルレジストリ**です。無料サービスであり、すべてのスキルはパブリック、オープンで、共有と再利用のために誰でも閲覧できます。スキルは `SKILL.md` ファイル（および補助テキストファイル）が入ったフォルダに過ぎません。ウェブアプリでスキルを閲覧したり、CLIを使用してスキルを検索、インストール、更新、公開したりできます。

サイト: [clawhub.ai](https://clawhub.ai)

## ClawHubとは

- OpenClawスキル向けのパブリックレジストリ。
- スキルバンドルとメタデータのバージョン管理ストア。
- 検索、タグ、使用シグナルによる発見サーフェス。

## 仕組み

1. ユーザーがスキルバンドル（ファイル + メタデータ）を公開します。
2. ClawHubがバンドルを保存し、メタデータを解析し、バージョンを割り当てます。
3. レジストリがスキルを検索と発見のためにインデックスします。
4. ユーザーがOpenClawでスキルを閲覧、ダウンロード、インストールします。

## できること

- 新しいスキルと既存スキルの新バージョンを公開します。
- 名前、タグ、検索でスキルを発見します。
- スキルバンドルをダウンロードしてファイルを検査します。
- 悪用または安全でないスキルを報告します。
- モデレーターの場合は、非表示、再表示、削除、またはBANができます。

## 対象者（初心者向け）

OpenClawエージェントに新しい機能を追加したい場合、ClawHubはスキルを見つけてインストールする最も簡単な方法です。バックエンドの仕組みを知る必要はありません。できること:

- 自然言語でスキルを検索します。
- スキルをワークスペースにインストールします。
- 1つのコマンドで後からスキルを更新します。
- 自分のスキルを公開することでバックアップします。

## クイックスタート（非技術者向け）

1. CLIをインストールします（次のセクションを参照）。
2. 必要なものを検索します:
   - `clawhub search "calendar"`
3. スキルをインストールします:
   - `clawhub install <skill-slug>`
4. 新しいOpenClawセッションを開始して新しいスキルを取得します。

## CLIのインストール

いずれかを選択してください:

```bash
npm i -g clawhub
```

```bash
pnpm add -g clawhub
```

## OpenClawとの関係

デフォルトでは、CLIはスキルを現在の作業ディレクトリの `./skills` にインストールします。OpenClawワークスペースが設定されている場合、`clawhub` は `--workdir`（または `CLAWHUB_WORKDIR`）をオーバーライドしない限りそのワークスペースにフォールバックします。OpenClawは `<workspace>/skills` からワークスペーススキルをロードし、**次の**セッションで取得します。すでに `~/.openclaw/skills` またはバンドルされたスキルを使用している場合、ワークスペーススキルが優先されます。

スキルのロード、共有、ゲートの詳細については、[スキル](/tools/skills) を参照してください。

## スキルシステムの概要

スキルはOpenClawが特定のタスクを実行する方法を教えるファイルのバージョン管理バンドルです。各公開で新しいバージョンが作成され、レジストリはユーザーが変更を監査できるようにバージョン履歴を保持します。

典型的なスキルには以下が含まれます:

- 主な説明と使用法を含む `SKILL.md` ファイル。
- スキルが使用するオプションの設定、スクリプト、または補助ファイル。
- タグ、サマリー、インストール要件などのメタデータ。

ClawHubはメタデータを使用して発見を促進し、スキル機能を安全に公開します。
レジストリはランキングと可視性を改善するために使用シグナル（スターやダウンロードなど）も追跡します。

## サービスが提供する機能

- スキルとその `SKILL.md` コンテンツの**パブリック閲覧**。
- エンベディング（ベクター検索）を利用した**検索**（キーワードのみではない）。
- semver、変更ログ、タグ（`latest` を含む）による**バージョン管理**。
- バージョンごとのzipとしての**ダウンロード**。
- コミュニティフィードバックのための**スターとコメント**。
- 承認と監査のための**モデレーション**フック。
- 自動化とスクリプティングのための**CLIフレンドリーなAPI**。

## セキュリティとモデレーション

ClawHubはデフォルトでオープンです。誰でもスキルをアップロードできますが、公開するにはGitHubアカウントが少なくとも1週間以上経過している必要があります。これにより、正当な貢献者をブロックすることなく悪用を抑制します。

報告とモデレーション:

- ログインしているユーザーであれば誰でもスキルを報告できます。
- 報告理由は必須で記録されます。
- 各ユーザーは一度に最大20件のアクティブな報告ができます。
- 3件以上のユニークな報告があるスキルはデフォルトで自動的に非表示になります。
- モデレーターは非表示のスキルを表示し、再表示、削除、またはユーザーをBANできます。
- 報告機能の悪用はアカウントBANになる場合があります。

モデレーターに興味がありますか？OpenClaw Discordで質問し、モデレーターまたはメンテナーに連絡してください。

## CLIコマンドとパラメーター

グローバルオプション（すべてのコマンドに適用）:

- `--workdir <dir>`: 作業ディレクトリ（デフォルト: 現在のディレクトリ。OpenClawワークスペースにフォールバック）。
- `--dir <dir>`: workdirからの相対的なスキルディレクトリ（デフォルト: `skills`）。
- `--site <url>`: サイトのベースURL（ブラウザログイン）。
- `--registry <url>`: レジストリAPIのベースURL。
- `--no-input`: プロンプトを無効化します（非対話型）。
- `-V, --cli-version`: CLIバージョンを表示します。

認証:

- `clawhub login`（ブラウザフロー）または `clawhub login --token <token>`
- `clawhub logout`
- `clawhub whoami`

オプション:

- `--token <token>`: APIトークンを貼り付けます。
- `--label <label>`: ブラウザログイントークン用に保存するラベル（デフォルト: `CLI token`）。
- `--no-browser`: ブラウザを開きません（`--token` が必要）。

検索:

- `clawhub search "query"`
- `--limit <n>`: 最大結果数。

インストール:

- `clawhub install <slug>`
- `--version <version>`: 特定バージョンをインストールします。
- `--force`: フォルダがすでに存在する場合に上書きします。

更新:

- `clawhub update <slug>`
- `clawhub update --all`
- `--version <version>`: 特定バージョンに更新します（単一スラッグのみ）。
- `--force`: ローカルファイルが公開済みバージョンと一致しない場合に上書きします。

リスト:

- `clawhub list`（`.clawhub/lock.json` を読みます）

公開:

- `clawhub publish <path>`
- `--slug <slug>`: スキルスラッグ。
- `--name <name>`: 表示名。
- `--version <version>`: semverバージョン。
- `--changelog <text>`: 変更ログテキスト（空でも可）。
- `--tags <tags>`: カンマ区切りのタグ（デフォルト: `latest`）。

削除/削除取り消し（オーナー/管理者のみ）:

- `clawhub delete <slug> --yes`
- `clawhub undelete <slug> --yes`

同期（ローカルスキルをスキャンして新規/更新済みを公開）:

- `clawhub sync`
- `--root <dir...>`: 追加のスキャンルート。
- `--all`: プロンプトなしですべてをアップロードします。
- `--dry-run`: アップロードされる内容を表示します。
- `--bump <type>`: 更新のための `patch|minor|major`（デフォルト: `patch`）。
- `--changelog <text>`: 非対話型更新の変更ログ。
- `--tags <tags>`: カンマ区切りのタグ（デフォルト: `latest`）。
- `--concurrency <n>`: レジストリチェック（デフォルト: 4）。

## エージェント向けの一般的なワークフロー

### スキルを検索する

```bash
clawhub search "postgres backups"
```

### 新しいスキルをダウンロードする

```bash
clawhub install my-skill-pack
```

### インストール済みスキルを更新する

```bash
clawhub update --all
```

### スキルをバックアップする（公開または同期）

単一のスキルフォルダの場合:

```bash
clawhub publish ./my-skill --slug my-skill --name "My Skill" --version 1.0.0 --tags latest
```

一度に多くのスキルをスキャンしてバックアップするには:

```bash
clawhub sync --all
```

## 高度な詳細（技術的）

### バージョン管理とタグ

- 各公開で新しい **semver** `SkillVersion` が作成されます。
- タグ（`latest` など）はバージョンを指します。タグを移動するとロールバックできます。
- 変更ログはバージョンごとに添付され、同期や更新の公開時に空にできます。

### ローカル変更とレジストリバージョン

更新はコンテンツハッシュを使用してローカルスキルコンテンツとレジストリバージョンを比較します。ローカルファイルが公開済みバージョンと一致しない場合、CLIは上書き前に確認を求めます（または非対話型実行では `--force` が必要）。

### 同期スキャンとフォールバックルート

`clawhub sync` は最初に現在のworkdirをスキャンします。スキルが見つからない場合、既知のレガシーロケーション（例: `~/openclaw/skills` と `~/.openclaw/skills`）にフォールバックします。これは追加フラグなしで古いスキルインストールを見つけるために設計されています。

### ストレージとロックファイル

- インストール済みスキルはworkdir下の `.clawhub/lock.json` に記録されます。
- 認証トークンはClawHub CLIコンフィグファイルに保存されます（`CLAWHUB_CONFIG_PATH` でオーバーライド可能）。

### テレメトリー（インストール数）

ログイン中に `clawhub sync` を実行すると、CLIはインストール数を計算するための最小限のスナップショットを送信します。完全に無効化するには:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

## 環境変数

- `CLAWHUB_SITE`: サイトURLをオーバーライドします。
- `CLAWHUB_REGISTRY`: レジストリAPIのURLをオーバーライドします。
- `CLAWHUB_CONFIG_PATH`: CLIがトークン/コンフィグを保存する場所をオーバーライドします。
- `CLAWHUB_WORKDIR`: デフォルトのworkdirをオーバーライドします。
- `CLAWHUB_DISABLE_TELEMETRY=1`: `sync` 時のテレメトリーを無効化します。
