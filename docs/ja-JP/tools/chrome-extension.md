# Chromeエクステンション（ブラウザリレー）

OpenClawのChromeエクステンションを使用すると、エージェントは別のopenclaw管理のChromeプロファイルを起動する代わりに、**既存のChromeタブ**（通常のChromeウィンドウ）を制御できます。

アタッチ/デタッチは**単一のChromeツールバーボタン**で行います。

## 概念（コンセプト）

3つの部分があります:

- **ブラウザコントロールサービス**（Gatewayまたはノード）: エージェント/ツールが呼び出すAPI（Gateway経由）
- **ローカルリレーサーバー**（ループバックCDP）: コントロールサーバーとエクステンションの間を橋渡しします（デフォルト: `http://127.0.0.1:18792`）
- **Chrome MV3エクステンション**: `chrome.debugger` を使用してアクティブなタブにアタッチし、CDPメッセージをリレーにパイプします

OpenClawは通常の `browser` ツールサーフェスを通じてアタッチされたタブを制御します（正しいプロファイルを選択する）。

## インストール/ロード（アンパック）

1. エクステンションを安定したローカルパスにインストールします:

```bash
openclaw browser extension install
```

2. インストール済みのエクステンションディレクトリパスを表示します:

```bash
openclaw browser extension path
```

3. Chrome → `chrome://extensions`

- 「デベロッパーモード」を有効にする
- 「パッケージ化されていない拡張機能を読み込む」→ 上記で表示されたディレクトリを選択

4. エクステンションをピン留めします。

## アップデート（ビルドステップ不要）

エクステンションはOpenClawリリース（npmパッケージ）内に静的ファイルとして同梱されています。別途「ビルド」ステップはありません。

OpenClawをアップグレードした後:

- `openclaw browser extension install` を再実行して、OpenClawの状態ディレクトリ下にインストールされたファイルを更新します。
- Chrome → `chrome://extensions` → エクステンションの「再読み込み」をクリックします。

## 使用方法（Gatewayトークンを一度設定する）

OpenClawにはデフォルトポートのエクステンションリレーをターゲットにする `chrome` という組み込みブラウザプロファイルが付属しています。

最初のアタッチ前に、エクステンションのOptionsを開いて設定してください:

- `Port`（デフォルト: `18792`）
- `Gateway token`（`gateway.auth.token` / `OPENCLAW_GATEWAY_TOKEN` と一致する必要があります）

使用方法:

- CLI: `openclaw browser --browser-profile chrome tabs`
- エージェントツール: `profile="chrome"` で `browser` を使用

別の名前や異なるリレーポートを使用したい場合は、独自のプロファイルを作成してください:

```bash
openclaw browser create-profile \
  --name my-chrome \
  --driver extension \
  --cdp-url http://127.0.0.1:18792 \
  --color "#00AA00"
```

### カスタムGatewayポート

カスタムゲートウェイポートを使用している場合、エクステンションリレーポートは自動的に導出されます:

**エクステンションリレーポート = Gatewayポート + 3**

例: `gateway.port: 19001` の場合:

- エクステンションリレーポート: `19004`（gateway + 3）

エクステンションのOptionsページで派生したリレーポートを使用するよう設定してください。

## アタッチ/デタッチ（ツールバーボタン）

- OpenClawにコントロールさせたいタブを開きます。
- エクステンションアイコンをクリックします。
  - アタッチ時にバッジに `ON` と表示されます。
- 再度クリックしてデタッチします。

## どのタブをコントロールしますか？

- 「現在表示しているタブ」を自動的にコントロールするわけでは**ありません**。
- ツールバーボタンをクリックして**明示的にアタッチしたタブのみ**をコントロールします。
- 切り替えるには: 別のタブを開いてそこでエクステンションアイコンをクリックします。

## バッジとよくあるエラー

- `ON`: アタッチ済み。OpenClawはそのタブを制御できます。
- `…`: ローカルリレーに接続中。
- `!`: リレーに到達できない/認証されていない（最も一般的: リレーサーバーが実行されていない、またはGatewayトークンが欠落/間違っている）。

`!` が表示された場合:

- Gatewayがローカルで実行されていることを確認してください（デフォルトのセットアップ）。Gatewayが他の場所で実行されている場合は、このマシンでノードホストを実行してください。
- エクステンションのOptionsページを開いてください。リレーの到達可能性とゲートウェイトークン認証を検証します。

## リモートGateway（ノードホストを使用する）

### ローカルGateway（Chromeと同じマシン）— 通常は**追加手順不要**

GatewayがChromeと同じマシンで実行されている場合、ループバックでブラウザコントロールサービスを起動し、リレーサーバーを自動起動します。エクステンションはローカルリレーと通信し、CLI/ツール呼び出しはGatewayに送られます。

### リモートGateway（Gatewayが他の場所で実行されている）— **ノードホストを実行する**

GatewayがChromeのマシンとは別のマシンで実行されている場合は、Chromeが実行されているマシンでノードホストを起動してください。Gatewayはブラウザアクションをそのノードにプロキシします。エクステンションとリレーはブラウザマシンのローカルに残ります。

複数のノードが接続されている場合は、`gateway.nodes.browser.node` で1つをピンするか、`gateway.nodes.browser.mode` を設定してください。

## サンドボックス化（ツールコンテナ）

エージェントセッションがサンドボックス化されている場合（`agents.defaults.sandbox.mode != "off"`）、`browser` ツールが制限される場合があります:

- デフォルトでは、サンドボックス化されたセッションは**サンドボックスブラウザ**（`target="sandbox"`）をターゲットにすることが多く、ホストのChromeではありません。
- Chromeエクステンションリレーの引き継ぎには**ホスト**ブラウザコントロールサーバーの制御が必要です。

オプション:

- 最も簡単: **サンドボックス化されていない**セッション/エージェントからエクステンションを使用します。
- またはサンドボックス化されたセッションでホストブラウザコントロールを許可します:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

次に、ツールポリシーによってツールが拒否されていないことを確認し、（必要に応じて）`target="host"` で `browser` を呼び出してください。

デバッグ: `openclaw sandbox explain`

## リモートアクセスのヒント

- GatewayとノードホストをTailnetに置いてください。リレーポートをLANやパブリックインターネットに公開しないでください。
- ノードを意図的にペアリングしてください。リモートコントロールが不要な場合はブラウザプロキシルーティングを無効にしてください（`gateway.nodes.browser.mode="off"`）。

## 「エクステンションパス」の仕組み

`openclaw browser extension path` はエクステンションファイルを含む**インストール済み**のディスク上ディレクトリを表示します。

CLIは意図的に `node_modules` パスを表示しません。常に最初に `openclaw browser extension install` を実行してエクステンションをOpenClaw状態ディレクトリの安定した場所にコピーしてください。

そのインストールディレクトリを移動または削除した場合、有効なパスから再ロードするまでChromeはエクステンションを壊れていると表示します。

## セキュリティの意味（これを読んでください）

これは強力でリスクがあります。モデルに「ブラウザ上の手」を与えるようなものとして扱ってください。

- エクステンションはChromeのデバッガーAPI（`chrome.debugger`）を使用します。アタッチされた場合、モデルは以下ができます:
  - そのタブでクリック/入力/ナビゲート
  - ページコンテンツを読む
  - タブのログイン済みセッションがアクセスできるものにアクセスする
- これは専用のopenclaw管理プロファイルのように**隔離されていません**。
  - 日常的に使うプロファイル/タブにアタッチした場合、そのアカウント状態へのアクセスを許可することになります。

推奨事項:

- エクステンションリレーの使用には専用のChromeプロファイル（個人のブラウジングとは別）を優先してください。
- GatewayとノードホストはTailnetのみに保ってください。Gateway認証とノードペアリングを活用してください。
- リレーポートをLAN（`0.0.0.0`）で公開したり、Funnel（パブリック）を避けてください。
- リレーはエクステンション以外のオリジンをブロックし、`/cdp` と `/extension` 両方にゲートウェイトークン認証を要求します。
