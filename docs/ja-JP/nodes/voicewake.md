# 音声ウェイク（グローバルウェイクワード）

OpenClaw は**ウェイクワードを Gateway が所有する単一のグローバルリスト**として扱います。

- **ノードごとのカスタムウェイクワードはありません**。
- **任意のノード/アプリの UI がリストを編集できます**。変更は Gateway によって保存され、全員にブロードキャストされます。
- 各デバイスは独自の**音声ウェイクの有効/無効**トグルを持ち続けます（ローカルの UX + パーミッションは異なります）。

## ストレージ（Gateway ホスト）

ウェイクワードは Gateway マシンの以下に保存されます:

- `~/.openclaw/settings/voicewake.json`

形状:

```json
{ "triggers": ["openclaw", "claude", "computer"], "updatedAtMs": 1730000000000 }
```

## プロトコル

### メソッド

- `voicewake.get` → `{ triggers: string[] }`
- `voicewake.set` パラメータ `{ triggers: string[] }` → `{ triggers: string[] }`

注意:

- トリガーは正規化されます（トリミング、空のものを削除）。空のリストはデフォルトにフォールバックします。
- 安全のために制限が適用されます（数と長さの上限）。

### イベント

- `voicewake.changed` ペイロード `{ triggers: string[] }`

受信するクライアント:

- すべての WebSocket クライアント（macOS アプリ、WebChat など）
- 接続されているすべてのノード（iOS/Android）、およびノード接続時に「現在の状態」の初期プッシュとして。

## クライアントの動作

### macOS アプリ

- グローバルリストを使用して `VoiceWakeRuntime` のトリガーをゲートします。
- 音声ウェイク設定の「トリガーワード」を編集すると `voicewake.set` を呼び出し、ブロードキャストに依存して他のクライアントを同期させます。

### iOS ノード

- グローバルリストを `VoiceWakeManager` のトリガー検出に使用します。
- 設定でウェイクワードを編集すると `voicewake.set`（Gateway WS 経由）を呼び出し、ローカルのウェイクワード検出を応答性の高い状態に保ちます。

### Android ノード

- 設定にウェイクワードエディターを公開します。
- Gateway WS 経由で `voicewake.set` を呼び出し、編集がどこでも同期されるようにします。
