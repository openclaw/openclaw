# 画像とメディアのサポート — 2025-12-05

WhatsApp チャンネルは **Baileys Web** 経由で動作します。このドキュメントは送信、Gateway、エージェント返信の現在のメディア処理ルールをまとめています。

## 目標

- `openclaw message send --media` でオプションのキャプション付きでメディアを送信する。
- Web 受信トレイからの自動返信にテキストとともにメディアを含められるようにする。
- タイプ別の制限を合理的かつ予測可能に保つ。

## CLI サーフェス

- `openclaw message send --media <path-or-url> [--message <caption>]`
  - `--media` はオプションです。メディアのみの送信ではキャプションを空にできます。
  - `--dry-run` は解決されたペイロードを出力します。`--json` は `{ channel, to, messageId, mediaUrl, caption }` を出力します。

## WhatsApp Web チャンネルの動作

- 入力: ローカルファイルパス**または** HTTP(S) URL。
- フロー: バッファに読み込み、メディアの種類を検出して正しいペイロードを構築します:
  - **画像**: `agents.defaults.mediaMaxMb`（デフォルト 5MB）を目標にしてサイズ 2048px 以下にリサイズして JPEG に再圧縮し、6MB にキャップします。
  - **音声/ボイス/ビデオ**: 16MB まで通過させます。音声はボイスノートとして送信されます（`ptt: true`）。
  - **ドキュメント**: その他すべて、100MB まで、ファイル名が利用可能な場合は保持されます。
- WhatsApp GIF スタイルの再生: `gifPlayback: true`（CLI: `--gif-playback`）で MP4 を送信して、モバイルクライアントがインラインでループするようにします。
- MIME 検出はマジックバイト、次にヘッダー、次にファイル拡張子を優先します。
- キャプションは `--message` または `reply.text` から取得されます。空のキャプションは許可されます。
- ログ: 詳細でない場合は `↩️`/`✅` を表示。詳細表示ではサイズとソースパス/URL を含みます。

## 自動返信パイプライン

- `getReplyFromConfig` は `{ text?, mediaUrl?, mediaUrls? }` を返します。
- メディアが存在する場合、Web 送信者は `openclaw message send` と同じパイプラインを使用してローカルパスまたは URL を解決します。
- 複数のメディアエントリが提供された場合、順次送信されます。

## インバウンドメディアのコマンドへ（Pi）

- インバウンド Web メッセージにメディアが含まれている場合、OpenClaw は一時ファイルにダウンロードしてテンプレート変数を公開します:
  - `{{MediaUrl}}` インバウンドメディアの疑似 URL。
  - `{{MediaPath}}` コマンドを実行する前に書き込まれるローカル一時パス。
- セッションごとの Docker サンドボックスが有効な場合、インバウンドメディアはサンドボックスワークスペースにコピーされ、`MediaPath`/`MediaUrl` は `media/inbound/<filename>` のような相対パスに書き換えられます。
- メディア理解（`tools.media.*` または共有 `tools.media.models` を通じて設定された場合）はテンプレート処理の前に実行され、`Body` に `[Image]`、`[Audio]`、`[Video]` ブロックを挿入できます。
  - 音声は `{{Transcript}}` を設定し、スラッシュコマンドが引き続き動作するようにコマンドパースにトランスクリプトを使用します。
  - ビデオと画像の説明はコマンドパースのためにキャプションテキストを保持します。
- デフォルトでは最初に一致した画像/音声/ビデオの添付ファイルのみが処理されます。複数の添付ファイルを処理するには `tools.media.<cap>.attachments` を設定してください。

## 制限とエラー

**アウトバウンド送信上限（WhatsApp Web 送信）**

- 画像: 再圧縮後約 6MB の上限。
- 音声/ボイス/ビデオ: 16MB の上限。ドキュメント: 100MB の上限。
- サイズ超過または読み取り不能なメディア → ログに明確なエラーが記録されて返信がスキップされます。

**メディア理解の上限（文字起こし/説明）**

- 画像のデフォルト: 10MB（`tools.media.image.maxBytes`）。
- 音声のデフォルト: 20MB（`tools.media.audio.maxBytes`）。
- ビデオのデフォルト: 50MB（`tools.media.video.maxBytes`）。
- サイズ超過のメディアは理解をスキップしますが、元のボディで返信は引き続き送られます。

## テストに関する注意

- 画像/音声/ドキュメントのケースの送信と返信フローをカバーしてください。
- 画像の再圧縮（サイズ制限）と音声のボイスノートフラグを検証してください。
- マルチメディア返信が順次送信に展開されることを確認してください。
