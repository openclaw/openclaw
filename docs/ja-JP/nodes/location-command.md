# 位置情報コマンド（ノード）

## TL;DR

- `location.get` はノードコマンドです（`node.invoke` 経由）。
- デフォルトでオフです。
- 設定はセレクターを使用します: オフ / 使用中のみ / 常に。
- 独立したトグル: 正確な位置情報。

## セレクター（スイッチではなく）の理由

OS のパーミッションはマルチレベルです。アプリ内でセレクターを表示できますが、実際の許可は OS が決定します。

- iOS/macOS: ユーザーはシステムプロンプト/設定で**使用中のみ**または**常に**を選択できます。アプリはアップグレードをリクエストできますが、OS が設定を必要とする場合があります。
- Android: バックグラウンド位置情報は別のパーミッションです。Android 10 以降では設定フローが必要な場合があります。
- 正確な位置情報は別の許可です（iOS 14+「正確」、Android「精細」vs「粗い」）。

UI のセレクターはリクエストするモードを制御します。実際の許可は OS の設定に存在します。

## 設定モデル

ノードデバイスごと:

- `location.enabledMode`: `off | whileUsing | always`
- `location.preciseEnabled`: ブーリアン

UI の動作:

- `whileUsing` を選択するとフォアグラウンドパーミッションをリクエストします。
- `always` を選択すると最初に `whileUsing` を確保してからバックグラウンドをリクエストします（または必要な場合は設定に送ります）。
- OS がリクエストされたレベルを拒否した場合、最も高い許可されたレベルに戻ってステータスを表示します。

## パーミッションマッピング（node.permissions）

オプションです。macOS ノードはパーミッションマップを通じて `location` を報告します。iOS/Android は省略する場合があります。

## コマンド: `location.get`

`node.invoke` を通じて呼び出されます。

パラメータ（推奨）:

```json
{
  "timeoutMs": 10000,
  "maxAgeMs": 15000,
  "desiredAccuracy": "coarse|balanced|precise"
}
```

レスポンスペイロード:

```json
{
  "lat": 48.20849,
  "lon": 16.37208,
  "accuracyMeters": 12.5,
  "altitudeMeters": 182.0,
  "speedMps": 0.0,
  "headingDeg": 270.0,
  "timestamp": "2026-01-03T12:34:56.000Z",
  "isPrecise": true,
  "source": "gps|wifi|cell|unknown"
}
```

エラー（安定したコード）:

- `LOCATION_DISABLED`: セレクターがオフです。
- `LOCATION_PERMISSION_REQUIRED`: リクエストされたモードのパーミッションが欠落しています。
- `LOCATION_BACKGROUND_UNAVAILABLE`: アプリがバックグラウンドにあるが「使用中のみ」のみ許可されています。
- `LOCATION_TIMEOUT`: 時間内に位置情報が取得できませんでした。
- `LOCATION_UNAVAILABLE`: システム障害 / プロバイダーなし。

## バックグラウンド動作（将来）

目標: 以下の条件を満たす場合のみ、ノードがバックグラウンドにあってもモデルが位置情報をリクエストできるようにする:

- ユーザーが**常に**を選択している。
- OS がバックグラウンド位置情報を許可している。
- アプリが位置情報のためにバックグラウンドで実行できる（iOS のバックグラウンドモード / Android のフォアグラウンドサービスまたは特別な許可）。

プッシュトリガーフロー（将来）:

1. Gateway がノードにプッシュを送信します（サイレントプッシュまたは FCM データ）。
2. ノードが一時的に起動してデバイスから位置情報をリクエストします。
3. ノードがペイロードを Gateway に転送します。

注意:

- iOS: 常にパーミッション + バックグラウンド位置情報モードが必要です。サイレントプッシュはスロットルされる場合があります。断続的な失敗を想定してください。
- Android: バックグラウンド位置情報にはフォアグラウンドサービスが必要な場合があります。そうでなければ拒否を想定してください。

## モデル/ツール統合

- ツールサーフェス: `nodes` ツールは `location_get` アクションを追加します（ノードが必要）。
- CLI: `openclaw nodes location get --node <id>`。
- エージェントガイドライン: ユーザーが位置情報を有効にしてスコープを理解している場合のみ呼び出します。

## UX コピー（提案）

- オフ: 「位置情報の共有は無効です。」
- 使用中のみ: 「OpenClaw が開いているときのみ。」
- 常に: 「バックグラウンド位置情報を許可します。システムパーミッションが必要です。」
- 正確: 「正確な GPS 位置情報を使用します。オフにすると大まかな位置情報を共有します。」
