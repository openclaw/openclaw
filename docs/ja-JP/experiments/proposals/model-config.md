# モデルコンフィグ（探索）

このドキュメントは、将来のモデル設定に関する**アイデア**をまとめたものです。出荷仕様ではありません。現在の動作については以下を参照してください:

- [モデル](/concepts/models)
- [モデルフェイルオーバー](/concepts/model-failover)
- [OAuth + プロファイル](/concepts/oauth)

## 動機

オペレーターが求めているもの:

- プロバイダーごとに複数の認証プロファイル（個人用と業務用）。
- 予測可能なフォールバックを持つシンプルな `/model` 選択。
- テキストモデルと画像対応モデルの明確な分離。

## 可能な方向性（高レベル）

- モデル選択をシンプルに保つ: オプションのエイリアスを持つ `provider/model`。
- プロバイダーが複数の認証プロファイルを持てるようにし、明示的な順序を設定する。
- 全セッションが一貫してフェイルオーバーできるよう、グローバルフォールバックリストを使用する。
- 明示的に設定されている場合にのみ、画像ルーティングをオーバーライドする。
