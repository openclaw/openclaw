# OpenClaw 脅威モデル v1.0

## MITRE ATLAS フレームワーク

**バージョン:** 1.0-draft
**最終更新日:** 2026-02-04
**方法論:** MITRE ATLAS + データフロー図
**フレームワーク:** [MITRE ATLAS](https://atlas.mitre.org/)（AI システムに対する敵対的脅威の景観）

### フレームワーク帰属

この脅威モデルは [MITRE ATLAS](https://atlas.mitre.org/) を基に構築されており、AI/ML システムへの敵対的脅威を文書化するための業界標準フレームワークです。ATLAS は AI セキュリティコミュニティと協力して [MITRE](https://www.mitre.org/) によって維持されています。

**主要な ATLAS リソース：**

- [ATLAS テクニック](https://atlas.mitre.org/techniques/)
- [ATLAS タクティクス](https://atlas.mitre.org/tactics/)
- [ATLAS ケーススタディ](https://atlas.mitre.org/studies/)
- [ATLAS GitHub](https://github.com/mitre-atlas/atlas-data)
- [ATLAS への貢献](https://atlas.mitre.org/resources/contribute)

### この脅威モデルへの貢献

これは OpenClaw コミュニティによって維持されている生きたドキュメントです。貢献のガイドラインは [CONTRIBUTING-THREAT-MODEL.md](/security/CONTRIBUTING-THREAT-MODEL) を参照してください。

## 1. はじめに

### 1.1 目的

この脅威モデルは、AI/ML システム専用に設計された MITRE ATLAS フレームワークを使用して、OpenClaw AI エージェントプラットフォームと ClawHub スキルマーケットプレイスへの敵対的脅威を文書化します。

### 1.2 スコープ

| コンポーネント         | 対象 | 備考                                                |
| ---------------------- | ---- | --------------------------------------------------- |
| OpenClaw エージェントランタイム | あり | コアエージェント実行、ツール呼び出し、セッション    |
| Gateway ゲートウェイ  | あり | 認証、ルーティング、チャンネルインテグレーション     |
| チャンネルインテグレーション | あり | WhatsApp、Telegram、Discord、Signal、Slack など     |
| ClawHub マーケットプレイス | あり | スキルの公開、モデレーション、配布                  |
| MCP サーバー           | あり | 外部ツールプロバイダー                               |
| ユーザーデバイス        | 部分的 | モバイルアプリ、デスクトップクライアント            |

### 2.2 データフロー

| フロー | ソース  | 宛先    | データ             | 保護                 |
| ---- | ------- | ------- | ------------------ | -------------------- |
| F1   | チャンネル | Gateway ゲートウェイ | ユーザーメッセージ | TLS、AllowFrom       |
| F2   | Gateway ゲートウェイ | エージェント | ルーティングされたメッセージ | セッション分離      |
| F3   | エージェント | ツール  | ツール呼び出し     | ポリシー強制         |
| F4   | エージェント | 外部    | web_fetch リクエスト | SSRF ブロッキング    |
| F5   | ClawHub | エージェント | スキルコード       | モデレーション、スキャニング |
| F6   | エージェント | チャンネル | レスポンス        | 出力フィルタリング   |

```
T-PERSIST-001 → T-EVADE-001 → T-EXFIL-003
（悪意のあるスキルを公開）→（モデレーションを回避）→（クレデンシャルを収集）

```

```
T-EXEC-002 → T-EXFIL-001 → 外部への持ち出し

```
