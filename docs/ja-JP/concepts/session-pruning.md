# セッションプルーニング

セッションプルーニングは、各LLM呼び出しの前にコンテキストから**古いツール結果**をトリミングします。蓄積されたツール出力（exec結果、ファイル読み取り、検索結果）によるコンテキストの肥大化を、会話メッセージに触れることなく軽減します。

<Info>
プルーニングはメモリ内のみで行われます。ディスク上のセッショントランスクリプトは変更されません。
完全な履歴は常に保持されます。
</Info>

## 重要な理由

長いセッションではツール出力が蓄積され、コンテキストウィンドウが膨張します。これによりコストが増加し、必要以上に早く[コンパクション](/concepts/compaction)が強制される可能性があります。

プルーニングは**Anthropicプロンプトキャッシュ**に対して特に有効です。キャッシュTTLが期限切れになると、次のリクエストでプロンプト全体が再キャッシュされます。プルーニングによりキャッシュ書き込みサイズが削減され、コストが直接的に低下します。

## 仕組み

1. キャッシュTTLの期限切れを待ちます（デフォルト5分）。
2. 古いツール結果を検出します（ユーザーメッセージとアシスタントメッセージは一切触れません）。
3. サイズ超過の結果を**ソフトトリミング**します。先頭と末尾を保持し、`...` を挿入します。
4. 残りを**ハードクリア**します。プレースホルダーに置き換えます。
5. TTLをリセットして、後続のリクエストが新しいキャッシュを再利用できるようにします。

## スマートデフォルト

OpenClawはAnthropicプロファイルに対してプルーニングを自動有効化します:

| プロファイルタイプ       | プルーニング有効 | ハートビート |
| ------------------------ | ---------------- | ------------ |
| OAuthまたはセットアップトークン | はい             | 1時間        |
| APIキー                  | はい             | 30分         |

明示的な値を設定している場合、OpenClawはそれを上書きしません。

## 有効化と無効化

Anthropic以外のプロバイダーではプルーニングはデフォルトで無効です。有効にするには:

```json5
{
  agents: {
    defaults: {
      contextPruning: { mode: "cache-ttl", ttl: "5m" },
    },
  },
}
```

無効にするには: `mode: "off"` を設定します。

## プルーニングとコンパクションの比較

|            | プルーニング       | コンパクション          |
| ---------- | ------------------ | ----------------------- |
| **内容**   | ツール結果をトリミング | 会話を要約              |
| **保存？** | いいえ（リクエストごと） | はい（トランスクリプトに） |
| **スコープ** | ツール結果のみ     | 会話全体                |

両者は補完的です。プルーニングはコンパクションサイクルの間、ツール出力を軽量に保ちます。
