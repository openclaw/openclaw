# コンパクション

すべてのモデルにはコンテキストウィンドウ（処理できるトークンの最大数）があります。
会話がその制限に近づくと、OpenClawは古いメッセージを要約に**コンパクション**し、
チャットを続行できるようにします。

## 仕組み

1. 古い会話のターンがコンパクトなエントリに要約されます。
2. 要約はセッショントランスクリプトに保存されます。
3. 最近のメッセージはそのまま保持されます。

完全な会話履歴はディスク上に残ります。コンパクションが変更するのは、
次のターンでモデルが参照する内容のみです。

## 自動コンパクション

自動コンパクションはデフォルトで有効です。セッションがコンテキスト制限に近づいたとき、
またはモデルがコンテキストオーバーフローエラーを返したときに実行されます
（その場合、OpenClawはコンパクションを行い再試行します）。

<Info>
コンパクション前に、OpenClawはエージェントに対して重要なメモを
[メモリ](/concepts/memory)ファイルに保存するよう自動的にリマインドします。
これによりコンテキストの喪失を防ぎます。
</Info>

## 手動コンパクション

任意のチャットで `/compact` と入力すると、コンパクションを強制実行できます。
要約をガイドするための指示を追加することもできます:

```
/compact Focus on the API design decisions
```

## 別のモデルを使用する

デフォルトでは、コンパクションはエージェントのプライマリモデルを使用します。
より高性能なモデルを使用して、より良い要約を得ることができます:

```json5
{
  agents: {
    defaults: {
      compaction: {
        model: "openrouter/anthropic/claude-sonnet-4-6",
      },
    },
  },
}
```

## コンパクション開始通知

デフォルトでは、コンパクションはサイレントに実行されます。コンパクション開始時に
簡単な通知を表示するには、`notifyUser` を有効にします:

```json5
{
  agents: {
    defaults: {
      compaction: {
        notifyUser: true,
      },
    },
  },
}
```

有効にすると、各コンパクション実行の開始時にユーザーに短いメッセージ
（例:「Compacting context...」）が表示されます。

## コンパクションとプルーニングの比較

|                  | コンパクション                | プルーニング                          |
| ---------------- | ----------------------------- | ------------------------------------- |
| **動作内容**     | 古い会話を要約する            | 古いツール結果をトリミングする        |
| **保存される？** | はい（セッショントランスクリプトに） | いいえ（メモリ内のみ、リクエストごと） |
| **スコープ**     | 会話全体                      | ツール結果のみ                        |

[セッションプルーニング](/concepts/session-pruning)は、要約を行わずに
ツール出力をトリミングする、より軽量な補完機能です。

## トラブルシューティング

**コンパクションが頻繁に発生する場合:** モデルのコンテキストウィンドウが小さいか、
ツール出力が大きい可能性があります。
[セッションプルーニング](/concepts/session-pruning)の有効化を試してください。

**コンパクション後にコンテキストが古く感じる場合:** `/compact Focus on <topic>` を使用して
要約をガイドするか、[メモリフラッシュ](/concepts/memory)を有効にして
メモが残るようにしてください。

**まっさらな状態が必要な場合:** `/new` でコンパクションなしに新しいセッションを開始できます。

高度な設定（予約トークン、識別子の保持、カスタムコンテキストエンジン、
OpenAI サーバーサイドコンパクション）については、
[セッション管理 ディープダイブ](/reference/session-management-compaction)を参照してください。
