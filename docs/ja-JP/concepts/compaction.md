# コンテキストウィンドウとコンパクション

すべてのモデルには**コンテキストウィンドウ**（参照できる最大トークン数）があります。長期間のチャットではメッセージとツール結果が蓄積されます。ウィンドウが逼迫してくると、OpenClaw は制限内に収まるよう古い履歴を**コンパクト**化します。

## コンパクションとは

コンパクションは**古い会話を要約**してコンパクトなサマリーエントリにまとめ、最近のメッセージはそのまま保持します。サマリーはセッション履歴に保存されるため、以降のリクエストでは以下が使用されます。

- コンパクションサマリー
- コンパクションポイント以降の最近のメッセージ

コンパクションはセッションの JSONL 履歴に**永続化**されます。

## 設定

`openclaw.json` の `agents.defaults.compaction` 設定を使用して、コンパクションの動作（モード、ターゲットトークン数など）を設定します。コンパクションの要約はデフォルトで不透明な識別子を保持します（`identifierPolicy: "strict"`）。`identifierPolicy: "off"` でこれをオーバーライドするか、`identifierPolicy: "custom"` と `identifierInstructions` でカスタムテキストを指定できます。

## 自動コンパクション（デフォルトで有効）

セッションがモデルのコンテキストウィンドウに近づくか超過すると、OpenClaw は自動コンパクションをトリガーし、コンパクト化されたコンテキストを使用して元のリクエストをリトライする場合があります。

以下が表示されます。

- verbose モードで `🧹 Auto-compaction complete`
- `/status` に `🧹 Compactions: <count>` が表示される

コンパクションの前に、OpenClaw は**サイレントメモリフラッシュ**ターンを実行して、永続的なメモをディスクに保存できます。詳細と設定については[メモリ](/concepts/memory)を参照してください。

## 手動コンパクション

`/compact`（オプションで指示付き）を使用してコンパクションパスを強制実行します。

```
/compact Focus on decisions and open questions
```

## コンテキストウィンドウのソース

コンテキストウィンドウはモデルによって異なります。OpenClaw は設定されたプロバイダーカタログのモデル定義を使用して制限を決定します。

## コンパクション vs プルーニング

- **コンパクション**: 要約して JSONL に**永続化**します。
- **セッションプルーニング**: 古い**ツール結果**のみをトリミングし、リクエストごとに**メモリ内**で処理します。

プルーニングの詳細については[/concepts/session-pruning](/concepts/session-pruning)を参照してください。

## OpenAI サーバーサイドコンパクション

OpenClaw は互換性のある直接 OpenAI モデル向けに OpenAI Responses のサーバーサイドコンパクションヒントもサポートします。これはローカルの OpenClaw コンパクションとは別のものであり、並行して実行できます。

- ローカルコンパクション: OpenClaw が要約してセッション JSONL に永続化します。
- サーバーサイドコンパクション: `store` + `context_management` が有効な場合、OpenAI がプロバイダー側でコンテキストをコンパクト化します。

モデルのパラメーターとオーバーライドについては[OpenAI プロバイダー](/providers/openai)を参照してください。

## ヒント

- セッションが古くなったりコンテキストが肥大化したりしたときは `/compact` を使用してください。
- 大きなツール出力はすでに切り詰められています。プルーニングによってツール結果の蓄積をさらに削減できます。
- 完全にリセットしたい場合は、`/new` または `/reset` で新しいセッション ID を開始してください。
