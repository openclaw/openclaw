# ビルトインメモリエンジン

ビルトインエンジンはデフォルトのメモリバックエンドです。メモリインデックスをエージェントごとのSQLiteデータベースに保存し、追加の依存関係なしで利用を開始できます。

## 提供する機能

- **キーワード検索** — FTS5全文インデックスによる検索（BM25スコアリング）。
- **ベクトル検索** — サポートされている任意のプロバイダーからの埋め込みによる検索。
- **ハイブリッド検索** — 最良の結果を得るために両方を組み合わせた検索。
- **CJKサポート** — 中国語、日本語、韓国語向けのトライグラムトークン化。
- **sqlite-vecアクセラレーション** — データベース内でのベクトルクエリ（オプション）。

## はじめに

OpenAI、Gemini、Voyage、またはMistralのAPIキーがあれば、ビルトインエンジンが自動検出してベクトル検索を有効にします。設定は不要です。

プロバイダーを明示的に設定するには：

```json5
{
  agents: {
    defaults: {
      memorySearch: {
        provider: "openai",
      },
    },
  },
}
```

埋め込みプロバイダーがない場合、キーワード検索のみ利用可能です。

## サポートされている埋め込みプロバイダー

| プロバイダー | ID        | 自動検出 | 備考                               |
| -------- | --------- | ------------- | ----------------------------------- |
| OpenAI   | `openai`  | あり           | デフォルト: `text-embedding-3-small`   |
| Gemini   | `gemini`  | あり           | マルチモーダル対応（画像 + 音声） |
| Voyage   | `voyage`  | あり           |                                     |
| Mistral  | `mistral` | あり           |                                     |
| Ollama   | `ollama`  | なし            | ローカル、明示的に設定が必要               |
| Local    | `local`   | あり（最優先）   | GGUFモデル、約0.6 GBのダウンロード        |

自動検出は、表示された順序でAPIキーが解決できる最初のプロバイダーを選択します。オーバーライドするには `memorySearch.provider` を設定してください。

## インデックスの仕組み

OpenClawは `MEMORY.md` と `memory/*.md` をチャンク（約400トークン、80トークンのオーバーラップ）に分割してインデックス化し、エージェントごとのSQLiteデータベースに保存します。

- **インデックスの場所：** `~/.openclaw/memory/<agentId>.sqlite`
- **ファイル監視：** メモリファイルへの変更がデバウンスされた再インデックスをトリガーします（1.5秒）。
- **自動再インデックス：** 埋め込みプロバイダー、モデル、またはチャンキング設定が変更されると、インデックス全体が自動的に再構築されます。
- **オンデマンド再インデックス：** `openclaw memory index --force`

<Info>
`memorySearch.extraPaths` を使用して、ワークスペース外のMarkdownファイルもインデックス化できます。
[設定リファレンス](/reference/memory-config#additional-memory-paths)を参照してください。
</Info>

## 使用するタイミング

ビルトインエンジンはほとんどのユーザーに適した選択です：

- 追加の依存関係なしですぐに使える。
- キーワード検索とベクトル検索を適切に処理。
- すべての埋め込みプロバイダーをサポート。
- ハイブリッド検索が両方の検索アプローチの長所を組み合わせる。

リランキング、クエリ拡張、またはワークスペース外のディレクトリのインデックス化が必要な場合は、[QMD](/concepts/memory-qmd)への切り替えを検討してください。

クロスセッションメモリと自動ユーザーモデリングが必要な場合は、[Honcho](/concepts/memory-honcho)を検討してください。

## トラブルシューティング

**メモリ検索が無効？** `openclaw memory status` を確認してください。プロバイダーが検出されない場合は、明示的に設定するかAPIキーを追加してください。

**結果が古い？** `openclaw memory index --force` を実行して再構築してください。まれなエッジケースではウォッチャーが変更を検知できない場合があります。

**sqlite-vecが読み込まれない？** OpenClawはインプロセスのコサイン類似度に自動的にフォールバックします。具体的な読み込みエラーについてはログを確認してください。

## 設定

埋め込みプロバイダーのセットアップ、ハイブリッド検索のチューニング（重み、MMR、時間減衰）、バッチインデックス、マルチモーダルメモリ、sqlite-vec、追加パス、およびその他すべての設定項目については、[メモリ設定リファレンス](/reference/memory-config)を参照してください。
