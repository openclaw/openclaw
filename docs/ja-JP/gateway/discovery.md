# ディスカバリーとトランスポート

OpenClawには、表面的には似ているが異なる2つの問題があります:

1. **オペレーターのリモート制御**: macOSメニューバーアプリが別の場所で動作しているGateway ゲートウェイを制御する。
2. **ノードペアリング**: iOS/Android（および将来のノード）がGateway ゲートウェイを見つけて安全にペアリングする。

設計目標は、すべてのネットワークディスカバリー/アドバタイジングを**ノードGateway ゲートウェイ**（`openclaw gateway`）に集約し、クライアント（macアプリ、iOS）はコンシューマーとして維持することです。

## 用語

- **Gateway ゲートウェイ**: 状態（セッション、ペアリング、ノードレジストリ）を所有し、チャネルを実行する単一の長時間稼働プロセス。ほとんどのセットアップではホストごとに1つ使用しますが、分離されたマルチGateway ゲートウェイセットアップも可能です。
- **Gateway ゲートウェイ WS（コントロールプレーン）**: デフォルトでは `127.0.0.1:18789` のWebSocketエンドポイント。`gateway.bind` でLAN/tailnetにバインドできます。
- **ダイレクトWSトランスポート**: LAN/tailnet向けのGateway ゲートウェイWSエンドポイント（SSHなし）。
- **SSHトランスポート（フォールバック）**: `127.0.0.1:18789` をSSH経由でフォワードするリモート制御。
- **レガシーTCPブリッジ（非推奨/削除済み）**: 古いノードトランスポート（[ブリッジプロトコル](/gateway/bridge-protocol)を参照）。ディスカバリーではアドバタイズされなくなりました。

プロトコルの詳細:

- [Gateway ゲートウェイプロトコル](/gateway/protocol)
- [ブリッジプロトコル（レガシー）](/gateway/bridge-protocol)

## 「ダイレクト」とSSHの両方を維持する理由

- **ダイレクトWS**は同一ネットワークおよびtailnet内で最適なUXを提供します:
  - LAN上でBonjourによる自動ディスカバリー
  - Gateway ゲートウェイが所有するペアリングトークン + ACL
  - シェルアクセス不要。プロトコルサーフェスをタイトで監査可能に保てる
- **SSH**はユニバーサルなフォールバックとして残ります:
  - SSHアクセスがあればどこでも動作する（無関係なネットワーク間でも）
  - マルチキャスト/mDNSの問題があっても機能する
  - SSH以外の新しいインバウンドポートが不要

## ディスカバリー入力（クライアントがGateway ゲートウェイの場所を知る方法）

### 1) Bonjour / mDNS（LANのみ）

Bonjourはベストエフォートであり、ネットワークをまたぎません。「同一LAN」の利便性のためにのみ使用されます。

目標方向:

- **Gateway ゲートウェイ**がBonjourでWSエンドポイントをアドバタイズする。
- クライアントがブラウズして「Gateway ゲートウェイを選択」リストを表示し、選択したエンドポイントを保存する。

トラブルシューティングとビーコンの詳細: [Bonjour](/gateway/bonjour)。

#### サービスビーコンの詳細

- サービスタイプ:
  - `_openclaw-gw._tcp`（Gateway ゲートウェイトランスポートビーコン）
- TXTキー（非シークレット）:
  - `role=gateway`
  - `transport=gateway`
  - `displayName=<friendly name>`（オペレーターが設定した表示名）
  - `lanHost=<hostname>.local`
  - `sshPort=22`（またはアドバタイズされた値）
  - `gatewayPort=18789`（Gateway ゲートウェイ WS + HTTP）
  - `gatewayTls=1`（TLSが有効な場合のみ）
  - `gatewayTlsSha256=<sha256>`（TLSが有効でフィンガープリントが利用可能な場合のみ）
  - `canvasPort=<port>`（canvasホストポート。現在、canvasホストが有効な場合は `gatewayPort` と同じ）
  - `cliPath=<path>`（オプション。実行可能な `openclaw` エントリーポイントまたはバイナリへの絶対パス）
  - `tailnetDns=<magicdns>`（オプションのヒント。Tailscaleが利用可能な場合に自動検出）

セキュリティに関する注意:

- Bonjour/mDNS TXTレコードは**認証されていません**。クライアントはTXT値をUXヒントとしてのみ扱う必要があります。
- ルーティング（ホスト/ポート）は、TXTで提供される `lanHost`、`tailnetDns`、`gatewayPort` よりも**解決済みのサービスエンドポイント**（SRV + A/AAAA）を優先すべきです。
- TLSピンニングでは、アドバタイズされた `gatewayTlsSha256` が以前保存されたピンを上書きすることを許可してはいけません。
- iOS/Androidノードは、ディスカバリーベースのダイレクト接続を**TLSのみ**として扱い、初回ピンを保存する前に明示的な「このフィンガープリントを信頼する」確認を要求する必要があります（帯域外検証）。

無効化/オーバーライド:

- `OPENCLAW_DISABLE_BONJOUR=1` でアドバタイジングを無効にします。
- `~/.openclaw/openclaw.json` の `gateway.bind` でGateway ゲートウェイのバインドモードを制御します。
- `OPENCLAW_SSH_PORT` はTXTでアドバタイズされるSSHポートをオーバーライドします（デフォルトは22）。
- `OPENCLAW_TAILNET_DNS` は `tailnetDns` ヒント（MagicDNS）を公開します。
- `OPENCLAW_CLI_PATH` はアドバタイズされるCLIパスをオーバーライドします。

### 2) Tailnet（クロスネットワーク）

ロンドン/ウィーンのようなセットアップでは、Bonjourは役に立ちません。推奨される「ダイレクト」ターゲットは:

- Tailscale MagicDNS名（推奨）または安定したtailnet IP。

Gateway ゲートウェイがTailscale上で動作していることを検出できる場合、クライアント（広域ビーコンを含む）向けのオプションヒントとして `tailnetDns` を公開します。

macOSアプリは、Gateway ゲートウェイディスカバリーにおいて生のTailscale IPよりもMagicDNS名を優先するようになりました。これにより、tailnet IPが変更された場合（例: ノード再起動やCGNAT再割り当て後）の信頼性が向上します。MagicDNS名は現在のIPに自動的に解決されるためです。

### 3) 手動 / SSHターゲット

ダイレクトルートがない場合（またはダイレクトが無効な場合）、クライアントはloopbackのGateway ゲートウェイポートをフォワードすることで、常にSSH経由で接続できます。

[リモートアクセス](/gateway/remote)を参照してください。

## トランスポート選択（クライアントポリシー）

推奨されるクライアント動作:

1. ペアリング済みのダイレクトエンドポイントが設定されていて到達可能な場合、それを使用する。
2. そうでなければ、BonjourがLAN上でGateway ゲートウェイを見つけた場合、ワンタップの「このGateway ゲートウェイを使用」を提示し、ダイレクトエンドポイントとして保存する。
3. そうでなければ、tailnet DNS/IPが設定されている場合、ダイレクトを試みる。
4. そうでなければ、SSHにフォールバックする。

## ペアリング + 認証（ダイレクトトランスポート）

Gateway ゲートウェイはノード/クライアントの受け入れに関する信頼できる情報源です。

- ペアリングリクエストはGateway ゲートウェイで作成/承認/拒否されます（[Gateway ゲートウェイペアリング](/gateway/pairing)を参照）。
- Gateway ゲートウェイは以下を強制します:
  - 認証（トークン / キーペア）
  - スコープ/ACL（Gateway ゲートウェイはすべてのメソッドへの生のプロキシではない）
  - レート制限

## コンポーネントごとの責務

- **Gateway ゲートウェイ**: ディスカバリービーコンをアドバタイズし、ペアリングの決定を所有し、WSエンドポイントをホストする。
- **macOSアプリ**: Gateway ゲートウェイの選択を支援し、ペアリングプロンプトを表示し、フォールバックとしてのみSSHを使用する。
- **iOS/Androidノード**: 利便性のためにBonjourをブラウズし、ペアリング済みのGateway ゲートウェイWSに接続する。
