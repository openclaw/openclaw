# 設定例

以下の例は現在の設定スキーマに準拠しています。網羅的なリファレンスとフィールドごとの注意事項については、[設定](/gateway/configuration)を参照してください。

## クイックスタート

### 最小限の設定

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: { whatsapp: { allowFrom: ["+15555550123"] } },
}
```

`~/.openclaw/openclaw.json`に保存すれば、その番号からボットにダイレクトメッセージを送信できます。

### 推奨されるスターター設定

```json5
{
  identity: {
    name: "Clawd",
    theme: "helpful assistant",
    emoji: "🦞",
  },
  agent: {
    workspace: "~/.openclaw/workspace",
    model: { primary: "anthropic/claude-sonnet-4-6" },
  },
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"],
      groups: { "*": { requireMention: true } },
    },
  },
}
```

## 一般的なパターン

### マルチプラットフォーム設定

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: {
    whatsapp: { allowFrom: ["+15555550123"] },
    telegram: {
      enabled: true,
      botToken: "YOUR_TOKEN",
      allowFrom: ["123456789"],
    },
    discord: {
      enabled: true,
      token: "YOUR_TOKEN",
      dm: { allowFrom: ["123456789012345678"] },
    },
  },
}
```

### セキュアダイレクトメッセージモード（共有受信箱 / マルチユーザーダイレクトメッセージ）

複数の人がボットにダイレクトメッセージを送信できる場合（`allowFrom`に複数のエントリ、複数人のペアリング承認、または`dmPolicy: "open"`）、異なる送信者からのダイレクトメッセージがデフォルトで1つのコンテキストを共有しないように、**セキュアダイレクトメッセージモード**を有効にしてください:

```json5
{
  // セキュアダイレクトメッセージモード（マルチユーザーまたは機密性の高いダイレクトメッセージエージェントに推奨）
  session: { dmScope: "per-channel-peer" },

  channels: {
    // 例: WhatsAppマルチユーザー受信箱
    whatsapp: {
      dmPolicy: "allowlist",
      allowFrom: ["+15555550123", "+15555550124"],
    },

    // 例: Discordマルチユーザー受信箱
    discord: {
      enabled: true,
      token: "YOUR_DISCORD_BOT_TOKEN",
      dm: { enabled: true, allowFrom: ["123456789012345678", "987654321098765432"] },
    },
  },
}
```

Discord/Slack/Google Chat/Microsoft Teams/Mattermost/IRCでは、送信者認証はデフォルトでID優先です。
各チャネルの`dangerouslyAllowNameMatching: true`で直接の変更可能な名前/メール/ニックネームマッチングを有効にするのは、そのリスクを明示的に受け入れる場合のみにしてください。

### OAuthとAPIキーフェイルオーバー

```json5
{
  auth: {
    profiles: {
      "anthropic:subscription": {
        provider: "anthropic",
        mode: "oauth",
        email: "me@example.com",
      },
      "anthropic:api": {
        provider: "anthropic",
        mode: "api_key",
      },
    },
    order: {
      anthropic: ["anthropic:subscription", "anthropic:api"],
    },
  },
  agent: {
    workspace: "~/.openclaw/workspace",
    model: {
      primary: "anthropic/claude-sonnet-4-6",
      fallbacks: ["anthropic/claude-opus-4-6"],
    },
  },
}
```

## ヒント

- `dmPolicy: "open"`を設定する場合、対応する`allowFrom`リストに`"*"`を含める必要があります。
- プロバイダーIDは異なります（電話番号、ユーザーID、チャネルIDなど）。フォーマットの確認にはプロバイダーのドキュメントを参照してください。
- 後から追加できるオプションセクション: `web`、`browser`、`ui`、`discovery`、`canvasHost`、`talk`、`signal`、`imessage`。
- より詳しいセットアップについては、[プロバイダー](/providers)と[トラブルシューティング](/gateway/troubleshooting)を参照してください。
