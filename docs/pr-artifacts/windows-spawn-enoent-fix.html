Summary

VERIFIED On a host running OpenClaw v2026.6.8 with unpatched dist files, the supervisor fails to spawn unadorned commands like claude, codex, npm, or node with the error Error: spawn <command> ENOENT. After applying the v02 patch to dist/windows-command-*.js, the same spawn calls succeed. The fix adds a system-PATHEXT walk to resolveWindowsCommandShim, preserving the existing cmdCommands allowlist behaviour while extending resolution to all executable extensions the host knows about (.CMD, .BAT, .EXE, etc.).

function resolveWindowsCommandShim(params) {
  if ((params.platform ?? process.platform) !== "win32") return params.command;

FIXED

The patched resolveWindowsCommandShim walks the system PATHEXT environment variable against PATH directories, returning the first match:

import { existsSync } from "node:fs";

function resolveWindowsCommandShim(params) {
  if ((params.platform ?? process.platform) !== "win32") return params.command;
  const basename = (path.basename(params.command)).toLowerCase();
  if (path.extname(basename)) return params.command;
  if (params.cmdCommands.includes(basename)) return `${params.command}.cmd`;
  // BUD PATCH v02 (2026-06-17): walk system PATHEXT for unadorned commands
  const pathext = (process.env.PATHEXT ?? ".EXE;.BAT;.CMD")
    .split(";").map(e => e.trim().toLowerCase()).filter(Boolean);
  const pathEnv = (process.env.PATH ?? "").split(";")
    .map(d => d.trim()).filter(Boolean);
  for (const dir of pathEnv) {
    for (const ext of pathext) {
      const candidate = path.join(dir, `${basename}${ext}`);
      if (existsSync(candidate)) return candidate;
    }
  }
  return params.command;
}
