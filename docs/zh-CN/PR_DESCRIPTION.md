- **保留英文**: API、PR、Issue、CLI、Gateway、Canvas、Skills、WebChat、Node、DM、OAuth 等技术术语

#### 品牌名称
所有品牌和产品名称保持原文，如：OpenClaw、Discord、WhatsApp、Telegram、macOS、iOS、Android 等

#### 格式保持
- ✅ 保持 Markdown 格式
- ✅ 保持所有链接（指向英文原文档）
- ✅ 保持代码块和示例
- ✅ 保持表格和徽章

## 中文翻译说明

### 为什么选择这些翻译？

1. **Gateway → Gateway 网关**
   - 保留 Gateway 因为这是 OpenClaw 的核心概念
   - 添加"网关"帮助中文用户理解

2. **Agent → 智能体**
   - 这是 AI 领域标准的中文翻译
   - 比"代理"更准确表达 AI 助手的含义

3. **Onboarding → 新手引导**
   - 准确表达帮助新用户上手的过程
   - 比"入职"更符合软件使用场景

4. **Skill → skill**
   - 保留原文因为这是 OpenClaw 的专有概念
   - 类似"插件"但有特定含义

5. **Channel → 渠道**
   - 准确表达消息传输通道的含义
   - 在通信领域是标准翻译

### 翻译质量保障

- 使用 AI 辅助翻译提高效率
- 保持技术术语的准确性
- 遵循中文技术文档写作规范
- 与现有中文文档风格保持一致
