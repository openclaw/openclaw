# AI 辅助贡献指南

**使用 AI 工具贡献 OpenClaw**

---

## 🤖 AI 辅助 PR 欢迎

**我们欢迎使用 AI 工具完成的贡献！**

AI 工具（如 Claude Code、GitHub Copilot、ChatGPT）可以帮助你：
- 生成代码
- 编写文档
- 修复 Bug
- 添加测试

---

## 📋 AI 辅助贡献要求

### 1. 透明度要求

**在 PR 描述中声明 AI 使用：**

```markdown
## AI 使用声明

本 PR 由以下 AI 工具辅助完成：
- Claude Code (代码生成)
- GitHub Copilot (代码补全)

**人工审查确认：**
- ✅ 所有代码已人工审查
- ✅ 所有测试已手动运行
- ✅ 所有功能已验证可用
```

### 2. 人工审查

**必须完成：**
- [ ] 所有 AI 生成的代码已人工审查
- [ ] 所有测试已手动运行并通过
- [ ] 所有功能已验证符合预期
- [ ] 没有引入安全问题
- [ ] 没有引入性能问题

### 3. 承担责任

**重要：**
- PR 作者对代码质量负最终责任
- AI 生成的代码也需要符合项目规范
- 如果 AI 生成的代码有问题，PR 作者负责修复

---

## 🎯 最佳实践

### 推荐的 AI 使用方式

**✅ 好的做法：**
- 使用 AI 生成初稿，然后人工审查和修改
- 使用 AI 帮助理解复杂代码
- 使用 AI 生成测试用例
- 使用 AI 检查代码规范

**❌ 不好的做法：**
- 直接提交 AI 生成的代码，不审查
- 依赖 AI 理解业务逻辑
- 让 AI 决定架构设计
- 完全依赖 AI 不人工验证

## 📝 示例 PR 描述

### 示例 1：文档贡献

```markdown
## 变更描述

添加中文安全报告流程文档。

## AI 使用声明

本 PR 由 Claude Code 辅助完成：
- AI 生成了文档初稿
- 人工审查并修改了技术细节

**人工审查确认：**
- ✅ 内容准确性已验证
- ✅ 格式规范性已检查
- ✅ 所有链接已验证
```

### 示例 2：代码贡献

```markdown
## 变更描述

修复 Gateway 模块的内存泄漏问题。

## AI 使用声明

本 PR 由以下 AI 工具辅助：
- GitHub Copilot (代码补全)
- Claude Code (代码审查)

**人工审查确认：**
- ✅ 内存泄漏已修复并验证
- ✅ 所有测试已通过

- ✅ 没有引入新问题
```

## 🔍 审查流程

### AI 辅助 PR 的审查

**审查者会检查：**
1. AI 使用声明是否完整
2. 人工审查是否充分
3. 代码质量是否符合标准
4. 测试是否充分

**可能的结果：**
- ✅ 通过：合并 PR
- ⚠️ 需要修改：补充审查或测试
- ❌ 拒绝：质量不达标

## 📞 常见问题

### Q: 必须声明 AI 使用吗？

**A:** 是的。透明度是我们的核心价值观。

### Q: 使用 AI 会影响 PR 通过吗？

**A:** 不会。只要质量达标，AI 辅助的 PR 同样欢迎。

### Q: 可以使用多个 AI 工具吗？

**A:** 可以！请在声明中列出所有使用的工具。

### Q: AI 生成的代码有问题怎么办？

**A:** 请在 PR 描述中说明，并承诺修复。
