# 任务流（Task Flow）

任务流是位于[后台任务](/zh-CN/automation/tasks)之上的**流程编排底层**。它管理持久化的多步骤流程，拥有自己的状态、修订追踪和同步语义，同时单个任务仍是分离工作的最小单位。

## 何时使用任务流

当工作跨越多个顺序或分支步骤，且需要在 Gateway 重启间保持持久化进度追踪时，使用任务流。对于单次后台操作，普通的[任务](/zh-CN/automation/tasks)即可。

| 场景                            | 方案               |
| ------------------------------- | ------------------ |
| 单个后台作业                    | 普通任务           |
| 多步骤流水线（A 然后 B 然后 C） | 任务流（托管模式） |
| 观察外部创建的任务              | 任务流（镜像模式） |
| 一次性提醒                      | 定时任务           |

## 同步模式

### 托管模式（Managed）

任务流全程拥有生命周期。它将任务创建为流程步骤，驱动至完成，然后自动推进流程状态。

示例：一个周报流程包含三个步骤：（1）收集数据 →（2）生成报告 →（3）投递。任务流将每个步骤创建为后台任务，等待完成后推进到下一步。

```
Flow: weekly-report
  步骤 1: gather-data     → 任务已创建 → succeeded
  步骤 2: generate-report → 任务已创建 → succeeded
  步骤 3: deliver         → 任务已创建 → running
```

### 镜像模式（Mirrored）

任务流观察外部创建的任务，在不获取任务创建所有权的情况下保持流程状态同步。当任务来自定时任务、CLI 命令或其他来源，而你希望将它们作为流程统一追踪进度时，此模式非常有用。

示例：三个独立的定时任务共同构成"早间运维"流程。镜像流程追踪它们的整体进度，但不控制它们的运行时间和方式。

## 持久化状态与修订追踪

每个流程持久化自己的状态并追踪修订，使进度在 Gateway 重启后仍然保留。修订追踪使多个来源同时尝试推进同一流程时能检测到冲突。

## 取消行为

`openclaw tasks flow cancel` 在流程上设置粘性取消意图。流程内的活跃任务被取消，且不会启动新步骤。取消意图在重启间保持，因此已取消的流程即使 Gateway 在所有子任务终止前重启也会保持取消状态。

## CLI 命令

```bash
# 列出活跃和最近的流程
openclaw tasks flow list

# 显示特定流程详情
openclaw tasks flow show <lookup>

# 取消运行中的流程及其活跃任务
openclaw tasks flow cancel <lookup>
```

| 命令                              | 说明                             |
| --------------------------------- | -------------------------------- |
| `openclaw tasks flow list`        | 显示追踪的流程及其状态和同步模式 |
| `openclaw tasks flow show <id>`   | 按流程ID或查找键检查单个流程     |
| `openclaw tasks flow cancel <id>` | 取消运行中的流程及其活跃任务     |

## 流程与任务的关系

流程协调任务，而非替代任务。单个流程在其生命周期内可能驱动多个后台任务。使用 `openclaw tasks` 检查单个任务记录，使用 `openclaw tasks flow` 检查编排流程。
