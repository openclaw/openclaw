# 为 OpenClaw 威胁模型做贡献

感谢您帮助 OpenClaw 更加安全。这个威胁模型是一份活文档，我们欢迎任何人的贡献——您不需要是安全专家。

## 贡献方式

### 添加威胁

发现我们尚未覆盖的攻击向量或风险？在 [openclaw/trust](https://github.com/openclaw/trust/issues) 上开一个 issue，用您自己的话描述它。您不需要了解任何框架或填写每个字段——只需描述场景。

**有帮助的包含内容（但不是必需的）：**

- 攻击场景以及如何被利用
- OpenClaw 的哪些部分受到影响（CLI、Gateway、频道、ClawHub、MCP 服务器等）
- 您认为有多严重（低/中/高/严重）
- 任何相关研究、CVE 或现实世界例子的链接

在审查期间我们会处理 ATLAS 映射、威胁 ID 和风险评估。如果您想包含这些细节，很好——但不是必需的。

> **这是为威胁模型添加内容，不是报告活跃漏洞。** 如果您发现了一个可利用的漏洞，请参阅我们的 [Trust 页面](https://trust.openclaw.ai) 了解负责任的披露说明。

### 建议缓解措施

对如何解决现有威胁有想法？开一个 issue 或 PR 引用该威胁。有用的缓解措施是具体且可操作的——例如，“在 Gateway 处对每分钟 10 条消息进行按发送者速率限制”比“实施速率限制”更好。

### 提出攻击链

攻击链展示多个威胁如何组合成现实世界的攻击场景。如果您发现危险的组合，描述步骤以及攻击者如何将它们链接在一起。关于攻击在实践中如何展开的简短叙述比正式模板更有价值。

### 修复或改进现有内容

错别字、澄清、过时的信息、更好的示例——欢迎 PR，不需要 issue。

## 我们使用的框架

### MITRE ATLAS

这个威胁模型基于 [MITRE ATLAS](https://atlas.mitre.org/)（AI 系统对抗威胁格局），这是一个专为 AI/ML 威胁设计的框架，如提示注入、工具滥用和代理利用。您不需要了解 ATLAS 就能贡献——我们会在审查期间将提交映射到框架。

### 威胁 ID

每个威胁获得一个 ID，如 `T-EXEC-003`。类别有：

| 代码 | 类别 |
| ------- | ------------------------------------------ |
| RECON | 侦察 - 信息收集 |
| ACCESS | 初始访问 - 获得入口 |
| EXEC | 执行 - 运行恶意操作 |
| PERSIST | 持久性 - 维持访问 |
| EVADE | 防御规避 - 避免检测 |
| DISC | 发现 - 了解环境 |
| EXFIL | 窃取 - 窃取数据 |
| IMPACT | 影响 - 损害或中断 |

ID 由维护者在审查期间分配。您不需要选择。

### 风险级别

| 级别 | 含义 |
| ------------ | ----------------------------------------------------------------- |
| **严重** | 完全系统妥协，或高可能性 + 严重影响 |
| **高** | 可能造成重大损害，或中等可能性 + 严重影响 |
| **中** | 中等风险，或低可能性 + 高影响 |
| **低** | 不太可能且影响有限 |

如果您不确定风险级别，只需描述影响，我们会评估。

## 审查流程

1. **分类** - 我们在 48 小时内审查新提交
2. **评估** - 我们验证可行性，分配 ATLAS 映射和威胁 ID，验证风险级别
3. **文档** - 我们确保所有内容格式正确且完整
4. **合并** - 添加到威胁模型和可视化
