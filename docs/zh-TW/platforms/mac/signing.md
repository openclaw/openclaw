# mac 簽署（除錯版本）

此應用程式通常由 [`scripts/package-mac-app.sh`](https://github.com/openclaw/openclaw/blob/main/scripts/package-mac-app.sh) 建置，該腳本目前會：

- 設定穩定的除錯用 bundle identifier：`ai.openclaw.mac.debug`
- 使用該 bundle id 寫入 Info.plist（可透過 `BUNDLE_ID=...` 覆寫）
- 呼叫 [`scripts/codesign-mac-app.sh`](https://github.com/openclaw/openclaw/blob/main/scripts/codesign-mac-app.sh) 來簽署主執行檔與應用程式包，讓 macOS 將每次重建視為相同的簽署包，並保留 TCC 權限（通知、輔助功能、螢幕錄製、麥克風、語音）。為了穩定的權限，請使用真實的簽署身份；ad-hoc 簽署是選擇性且脆弱的（詳見 [macOS 權限](/platforms/mac/permissions)）。
- 預設使用 `CODESIGN_TIMESTAMP=auto`；此選項會啟用 Developer ID 簽署的可信時間戳。設定 `CODESIGN_TIMESTAMP=off` 可跳過時間戳（離線除錯版本）。
- 將建置元資料注入 Info.plist：`OpenClawBuildTimestamp`（UTC）與 `OpenClawGitCommit`（短版 git hash），讓關於視窗能顯示建置、git 與除錯/發行頻道資訊。
- **封裝預設使用 Node 24**：腳本會執行 TS 建置與 Control UI 建置。Node 22 LTS（目前為 `22.16+`）仍維持相容性支援。
- 從環境讀取 `SIGN_IDENTITY`。將 `export SIGN_IDENTITY="Apple Development: Your Name (TEAMID)"`（或你的 Developer ID Application 證書）加入 shell 啟動腳本，讓簽署時永遠使用你的證書。ad-hoc 簽署需明確透過 `ALLOW_ADHOC_SIGNING=1` 或 `SIGN_IDENTITY="-"` 選擇啟用（不建議用於權限測試）。
- 簽署後執行 Team ID 審核，若應用程式包內任一 Mach-O 簽署的 Team ID 不同則失敗。設定 `SKIP_TEAM_ID_CHECK=1` 可跳過此檢查。

## 使用說明

```bash
# from repo root
scripts/package-mac-app.sh               # auto-selects identity; errors if none found
SIGN_IDENTITY="Developer ID Application: Your Name" scripts/package-mac-app.sh   # real cert
ALLOW_ADHOC_SIGNING=1 scripts/package-mac-app.sh    # ad-hoc (permissions will not stick)
SIGN_IDENTITY="-" scripts/package-mac-app.sh        # explicit ad-hoc (same caveat)
DISABLE_LIBRARY_VALIDATION=1 scripts/package-mac-app.sh   # dev-only Sparkle Team ID mismatch workaround
```

### Ad-hoc 簽署注意事項

當使用 `SIGN_IDENTITY="-"`（ad-hoc）簽署時，腳本會自動停用 **Hardened Runtime**（`--options runtime`）。這是為了避免應用程式嘗試載入不共用相同 Team ID 的內嵌框架（如 Sparkle）時發生崩潰。ad-hoc 簽署也會破壞 TCC 權限的持久性；請參考 [macOS 權限](/platforms/mac/permissions) 了解恢復步驟。

## 關於視窗的建置元資料

`package-mac-app.sh` 會在 bundle 中標記：

- `OpenClawBuildTimestamp`：封裝時的 ISO8601 UTC 時間
- `OpenClawGitCommit`：短版 git hash（若無則為 `unknown`）

關於分頁會讀取這些鍵值，顯示版本、建置日期、git commit，以及是否為除錯版本（透過 `#if DEBUG`）。程式碼變更後請執行封裝工具以更新這些值。

## 為什麼要這麼做

TCC 權限綁定於 bundle identifier _與_ 簽署碼。未簽署且 UUID 不斷變動的除錯版本，會導致 macOS 在每次重建後忘記授權。簽署執行檔（預設為 ad-hoc）並保持固定的 bundle id/路徑（`dist/OpenClaw.app`）可在建置間保留授權，與 VibeTunnel 的做法相同。
