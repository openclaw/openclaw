# 語音喚醒與按鍵說話

## 模式

- **喚醒詞模式**（預設）：持續開啟的語音辨識器等待觸發詞 (`swabbleTriggerWords`)。匹配後開始錄音，顯示部分文字的覆蓋層，並在靜音後自動送出。
- **按鍵說話（按住右 Option）**：按住右 Option 鍵立即開始錄音，無需觸發詞。按住時顯示覆蓋層，放開後稍作延遲完成並送出，方便你調整文字。

## 執行時行為（喚醒詞）

- 語音辨識器執行於 `VoiceWakeRuntime`。
- 觸發僅在喚醒詞與下一個詞之間有**明顯停頓**時觸發（約 0.55 秒間隔）。覆蓋層與提示音可在停頓時即開始，甚至在指令開始前。
- 靜音判定時間：語音持續時為 2.0 秒，僅聽到觸發詞時為 5.0 秒。
- 強制停止：120 秒，防止無限持續。
- 會話間防彈跳時間：350 毫秒。
- 覆蓋層由 `VoiceWakeOverlayController` 控制，使用已提交/暫存顏色區分。
- 送出後，辨識器會乾淨重啟，準備聆聽下一個觸發詞。

## 生命週期不變條件

- 若啟用語音喚醒且已授權，喚醒詞辨識器應持續聆聽（明確按鍵說話錄音時除外）。
- 覆蓋層可見性（包含手動點擊 X 關閉）絕不應阻止辨識器恢復運作。

## 覆蓋層卡住失效模式（舊版）

過去若覆蓋層卡住且你手動關閉，語音喚醒可能會「死當」，因為執行時重啟嘗試會被覆蓋層可見性阻擋，且不會排程後續重啟。

強化措施：

- 喚醒執行時重啟不再被覆蓋層可見性阻擋。
- 覆蓋層關閉完成會透過 `VoiceSessionCoordinator` 觸發 `VoiceWakeRuntime.refresh(...)`，確保手動點擊 X 關閉後必定恢復聆聽。

## 按鍵說話細節

- 快捷鍵偵測使用全域 `.flagsChanged` 監聽器監控**右 Option**鍵（`keyCode 61` + `.option`）。僅觀察事件，不攔截。
- 錄音流程在 `VoicePushToTalk`：立即啟動語音辨識，串流部分文字至覆蓋層，放開時呼叫 `VoiceWakeForwarder`。
- 按鍵說話開始時會暫停喚醒詞執行時，避免音訊衝突；放開後自動重啟。
- 權限需求：麥克風 + 語音辨識；監聽事件需輔助使用/輸入監控授權。
- 外接鍵盤：部分可能無法正確辨識右 Option，若用戶回報失效，會提供備用快捷鍵。

## 使用者設定

- **語音喚醒** 開關：啟用喚醒詞執行時。
- **按住 Cmd+Fn 說話**：啟用按鍵說話監聽。macOS 26 以下版本預設關閉。
- 語言與麥克風選擇器、即時音量計、觸發詞列表、測試器（僅本地，不會轉發）。
- 麥克風選擇器會保留最後選擇，若裝置斷線會顯示斷線提示，暫時回退至系統預設，直到裝置回復。
- **音效**：觸發詞偵測與送出時會有提示音，預設為 macOS「Glass」系統音。你可為每個事件選擇任何 `NSSound` 可載入的檔案（如 MP3/WAV/AIFF），或選擇**無聲**。

## 轉發行為

- 啟用語音喚醒時，文字記錄會轉發到目前啟用的閘道/代理（與 mac 應用程式其他部分使用相同的本地或遠端模式）。
- 回覆會傳送到**最後使用的主要提供者**（WhatsApp/Telegram/Discord/WebChat）。若傳送失敗，錯誤會被記錄，且該次執行仍可透過 WebChat/會話記錄查看。

## 轉發負載

- `VoiceWakeForwarder.prefixedTranscript(_:)` 在傳送前會加上機器提示。此功能在喚醒詞和按鍵說話路徑間共用。

## 快速驗證

- 開啟按鍵說話，按住 Cmd+Fn，說話，放開：覆蓋層應顯示部分文字後送出。
- 按住時，選單列的耳朵圖示應保持放大（使用 `triggerVoiceEars(ttl:nil)`）；放開後會縮小。
