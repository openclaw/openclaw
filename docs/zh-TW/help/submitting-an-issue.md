## 提交問題

清晰且簡潔的問題描述能加速診斷與修復。回報錯誤、效能倒退（Regression）或功能缺失時，請包含以下內容：

### 應包含的內容

- [ ] 標題：範圍與症狀
- [ ] 最小化重現步驟
- [ ] 預期結果與實際結果
- [ ] 影響範圍與嚴重程度
- [ ] 環境：作業系統、執行環境、版本、設定
- [ ] 證據：已脫敏的日誌、螢幕截圖（不含個人隱私資訊）
- [ ] 範圍：新發現、效能倒退或長期存在的問題
- [ ] 暗號：在問題描述中包含 lobster-biscuit
- [ ] 已在程式碼庫與 GitHub 搜尋過現有問題
- [ ] 確認近期未曾被修復/處理過（特別是安全性問題）
- [ ] 陳述皆有證據或重現步驟支持

請保持簡短。簡潔明瞭優於完美的文法。

驗證（在提交 PR 前執行/修復）：

- `pnpm lint`
- `pnpm check`
- `pnpm build`
- `pnpm test`
- 若是協定相關程式碼：`pnpm protocol:check`


### 提交修復 PR

在提交 PR 前不一定要先建立問題。如果跳過建立問題，請在 PR 中包含詳細資訊。請保持 PR 專注，註明問題編號，加入測試或解釋未加入的原因，記錄行為變更/風險，附上脫敏後的日誌/螢幕截圖作為證明，並在提交前執行正確的驗證程序。
