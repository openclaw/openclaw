# 貢獻 OpenClaw 威脅模型

感謝您協助 OpenClaw 更加安全。此威脅模型是一份持續更新的文件，我們歡迎任何人的貢獻，您不需要是資安專家。

## 貢獻方式

### 新增威脅

發現了我們尚未涵蓋的攻擊途徑或風險？請在 [openclaw/trust](https://github.com/openclaw/trust/issues) 上開啟一個 issue，並用您自己的話來描述。您不需要了解任何框架或填寫所有欄位，只需描述情境即可。

**有助於包含的資訊（但非必要）：**

- 攻擊情境及其可能被利用的方式
- OpenClaw 的哪些部分受到影響 (CLI, Gateway, 頻道, ClawHub, MCP 伺服器等)
- 您認為其嚴重程度（低 / 中 / 高 / 嚴重）
- 任何相關研究、CVE 或真實世界範例的連結

我們將在審查期間處理 ATLAS 映射、威脅 ID 和風險評估。如果您想包含這些詳細資訊，那很好，但並非預期。

> **這是用於新增至威脅模型，而非回報即時漏洞。** 如果您發現了可利用的漏洞，請參閱我們的 [Trust 頁面](https://trust.openclaw.ai) 以獲取負責任的揭露說明。

### 建議緩解措施

對於如何解決現有威脅有想法嗎？請開啟一個 issue 或 PR，並參考該威脅。有用的緩解措施應具體且可操作，例如，「在 Gateway 對每個發送者實施每分鐘 10 則訊息的速率限制」優於「實施速率限制」。

### 提出攻擊鏈

攻擊鏈顯示多個威脅如何組合為實際的攻擊情境。如果您看到了危險的組合，請描述步驟以及攻擊者如何將它們串聯起來。一個簡短的攻擊實際展開的敘述比正式的範本更有價值。

### 修正或改進現有內容

錯字、澄清、過時資訊、更好的範例，歡迎提交 PR，無需開啟 issue。

## 我們使用的工具

### MITRE ATLAS

此威脅模型建立在 [MITRE ATLAS](https://atlas.mitre.org/) (Adversarial Threat Landscape for AI Systems) 之上，這是一個專為 AI/ML 威脅（如提示注入、工具濫用和智慧代理利用）設計的框架。您無需了解 ATLAS 即可貢獻，我們會在審查期間將提交的內容映射到該框架。

### 威脅 ID

每個威脅都會獲得一個 ID，例如 `T-EXEC-003`。類別如下：

| 代碼    | 類別                                   |
| ------- | ------------------------------------------ |
| RECON   | 偵察 - 資訊收集     |
| ACCESS  | 初始存取 - 獲取入口             |
| EXEC    | 執行 - 執行惡意行為      |
| PERSIST | 持續性 - 維護存取           |
| EVADE   | 防禦規避 - 避免偵測       |
| DISC    | 裝置探索 - 了解環境 |
| EXFIL   | 資料竊取 - 竊取資料       |
| IMPACT  | 影響 - 損害或破壞              |

ID 由維護者在審查期間分配。您無需選擇。
