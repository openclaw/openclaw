# 為 OpenClaw 威脅模型做出貢獻

感謝你協助讓 OpenClaw 更加安全。這份威脅模型是個持續更新的文件，我們歡迎任何人貢獻——你不需要是資安專家。

## 貢獻方式

### 新增威脅

發現我們還沒涵蓋的攻擊向量或風險嗎？請在 [openclaw/trust](https://github.com/openclaw/trust/issues) 開啟一個 issue，並用你自己的話描述它。你不需要了解任何框架或填寫所有欄位——只要描述情境即可。

**建議包含（但非必須）：**

- 攻擊情境以及可能如何被利用
- 受影響的 OpenClaw 部分（CLI、gateway、channels、ClawHub、MCP 伺服器等）
- 你認為的嚴重程度（低 / 中 / 高 / 致命）
- 任何相關研究、CVE 或實際案例的連結

我們會在審查時處理 ATLAS 映射、威脅 ID 和風險評估。如果你想包含這些細節也很好，但不是必要的。

> **這是用來新增威脅模型內容，不是用來回報即時漏洞。** 如果你發現可被利用的漏洞，請參考我們的 [Trust 頁面](https://trust.openclaw.ai) 了解負責任揭露的指引。

### 建議緩解措施

有想法可以解決現有威脅嗎？請開啟一個 issue 或 PR，並引用該威脅。有效的緩解措施應該具體且可執行——例如，「在 gateway 端對每個發送者限制每分鐘 10 則訊息」比「實施速率限制」更好。

### 提出攻擊鏈

攻擊鏈展示多個威脅如何結合成一個真實的攻擊情境。如果你看到危險的組合，請描述步驟以及攻擊者如何串連它們。實際攻擊展開的簡短敘述，比正式的範本更有價值。

### 修正或改進現有內容

拼字錯誤、說明不清、過時資訊、更好的範例——歡迎 PR，不需要先開 issue。

## 我們使用的工具

### MITRE ATLAS

此威脅模型建立於 [MITRE ATLAS](https://atlas.mitre.org/)（針對 AI 系統的對抗性威脅景觀）之上，這是一個專為 AI/ML 威脅設計的框架，涵蓋提示注入、工具誤用及代理利用等威脅。您不需要熟悉 ATLAS 即可參與貢獻——我們會在審核時將提交內容對應到該框架。

### 威脅 ID

每個威脅都會有一個像 `T-EXEC-003` 的 ID。類別如下：

| 程式碼  | 類別                  |
| ------- | --------------------- |
| RECON   | 偵察 - 資訊蒐集       |
| ACCESS  | 初始存取 - 取得入口   |
| EXEC    | 執行 - 執行惡意行為   |
| PERSIST | 持續存在 - 維持存取   |
| EVADE   | 防禦規避 - 避免被偵測 |
| DISC    | 探索 - 瞭解環境       |
| EXFIL   | 外洩 - 竊取資料       |
| IMPACT  | 影響 - 損害或中斷     |

ID 由維護者在審核時分配，您不需要自行選擇。

### 風險等級

| 等級         | 含義                                       |
| ------------ | ------------------------------------------ |
| **Critical** | 完全系統妥協，或高可能性且具關鍵影響       |
| **High**     | 可能造成重大損害，或中等可能性且具關鍵影響 |
| **Medium**   | 中度風險，或低可能性且具高影響             |
| **Low**      | 不太可能且影響有限                         |

如果您不確定風險等級，只要描述影響，我們會協助評估。

## 審核流程

1. **初步篩選** - 我們會在 48 小時內審查新提交
2. **評估** - 驗證可行性、分配 ATLAS 對應及威脅 ID、確認風險等級
3. **文件整理** - 確保格式正確且內容完整
4. **合併** - 加入威脅模型與視覺化圖表
