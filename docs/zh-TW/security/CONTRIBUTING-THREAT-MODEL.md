# 參與 OpenClaw 威脅模型貢獻

感謝你協助讓 OpenClaw 更加安全。此威脅模型是一份持續更新的文件，我們歡迎任何人的貢獻 —— 你不需要是資安專家。

## 貢獻方式

### 新增威脅

發現了我們尚未涵蓋的攻擊向量或風險？請在 [openclaw/trust](https://github.com/openclaw/trust/issues) 提交 Issue 並用你自己的話描述它。你不需要了解任何框架或填寫每個欄位 —— 只要描述該情境即可。

**包含以下資訊會很有幫助（但非必要）：**

- 攻擊情境以及如何被利用
- OpenClaw 的哪些部分受到影響（CLI、Gateway、頻道、ClawHub、MCP 伺服器等）
- 你認為的嚴重程度（低 / 中 / 高 / 緊急）
- 任何相關研究、CVE 或真實案例的連結

我們會在審查期間處理 ATLAS 映射、威脅 ID 和風險評估。如果你想包含這些細節也很好，但這不是預期的要求。

> **這是為了新增至威脅模型，而非回報即時漏洞。** 如果你發現了可被利用的漏洞，請參閱我們的 [Trust 頁面](https://trust.openclaw.ai) 以了解負責任的披露說明。

### 建議緩解措施

對於如何處理現有威脅有想法嗎？請提交引用該威脅的 Issue 或 PR。有效的緩解措施應具體且可執行 —— 例如，「在 Gateway 對每個發送者實施每分鐘 10 則訊息的速率限制」比「實施速率限制」更好。

### 提議攻擊鏈

攻擊鏈展示了多個威脅如何結合成一個現實的攻擊情境。如果你看到危險的組合，請描述其步驟以及攻擊者如何將它們串聯在一起。對攻擊在實作中如何展開的簡短敘述，比正式的範本更有價值。

### 修復或改進現有內容

錯字、說明釐清、過時資訊、更好的範例 —— 歡迎提交 PR，無需先提交 Issue。

## 我們使用的框架

### MITRE ATLAS

此威脅模型建構於 [MITRE ATLAS](https://atlas.mitre.org/) (Adversarial Threat Landscape for AI Systems) 之上，這是一個專為 AI/ML 威脅（如提示注入、工具濫用和智慧代理利用）設計的框架。你不需要了解 ATLAS 也能做出貢獻 —— 我們會在審查期間將提交內容映射到該框架。

### 威脅 ID

每個威脅都會獲得一個如 `T-EXEC-003` 的 ID。類別如下：

| 代碼    | 類別                                       |
| ------- | ------------------------------------------ |
| RECON   | Reconnaissance（偵察）- 資訊收集           |
| ACCESS  | Initial access（初始存取）- 取得進入權限   |
| EXEC    | Execution（執行）- 執行惡意行為            |
| PERSIST | Persistence（持續性）- 維持存取權限        |
| EVADE   | Defense evasion（防禦規避）- 避免被發現    |
| DISC    | Discovery（探索）- 了解環境資訊            |
| EXFIL   | Exfiltration（外洩）- 竊取資料
