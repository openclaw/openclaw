# Telegram 允許列表強化

**日期**: 2026-01-05  
**狀態**: 完成  
**PR**: #216

## 摘要

Telegram 允許列表現在不區分大小寫地接受 `telegram:` 和 `tg:` 前綴，並容忍意外的空白字元。這使得傳入的允許列表檢查與傳出的傳送正規化保持一致。

## 變更內容

- `telegram:` 和 `tg:` 前綴被視為相同（不區分大小寫）。
- 允許列表項目會被修剪；空項目會被忽略。

## 範例

所有這些對於相同的 ID 都會被接受：

- `telegram:123456`
- `TG:123456`
- `tg:123456`

## 重要性

從日誌或聊天 ID 複製/貼上時，通常會包含前綴和空白字元。正規化可以避免在決定是否在私訊或群組中回應時出現假陰性。
