# 模型設定 (探索)

本檔案記錄了未來模型設定的**構想**。這並非正式發布的規格。關於目前的行為，請參閱：

- [模型](/concepts/models)
- [模型容錯移轉](/concepts/model-failover)
- [OAuth + 設定檔](/concepts/oauth)

## 動機

管理者希望：

- 每個供應商可擁有多個身分驗證設定檔（個人 vs 工作）。
- 簡單的 `/model` 選擇方式，並具備可預測的容錯移轉。
- 清晰區分純文字模型與具備圖片處理能力的模型。

## 可能的發展方向（高階概觀）

- 保持模型選擇簡單：`provider/model` 並支援選用別名。
- 讓供應商擁有複數個身分驗證設定檔，並具備明確的優先順序。
- 使用全域容錯移轉清單，讓所有工作階段都能一致地進行容錯移轉。
- 僅在明確設定時才覆蓋圖片路由。

## 待解決問題

- 設定檔輪替應該是以「供應商」為單位還是以「模型」為單位？
- UI 應如何呈現工作階段中的設定檔選擇？
- 從舊版設定鍵名遷移最安全的路徑為何？
