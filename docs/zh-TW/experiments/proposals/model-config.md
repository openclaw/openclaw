# 模型設定 (探索)

這份檔案記錄了未來模型設定的**想法**。這不是最終規格。如需目前行為，請參閱：

- [模型](/concepts/models)
- [模型故障轉移](/concepts/model-failover)
- [OAuth + 設定檔](/concepts/oauth)

## 動機

操作者希望：

- 每個供應商有多個憑證設定檔（個人與工作）。
- 透過 `/model` 進行簡單的模型選擇，並具有可預測的備用機制。
- 文字模型與支援圖片的模型之間有明確的區隔。

## 可能的方向 (高階)

- 保持模型選擇簡單：`供應商/模型`，可選別名。
- 允許供應商擁有多個憑證設定檔，並具備明確的順序。
- 使用全域備用清單，以確保所有工作階段都能一致地故障轉移。
- 僅在明確設定時才覆寫圖片路由。

- UI 應該如何呈現工作階段的設定檔選擇？
- 從舊版設定鍵進行遷移最安全的路徑是什麼？
