# feat: Add exclude patterns and sensible defaults to `openclaw backup create`

## Enhancement Summary

**Deepened:** 2026-03-12  
**Research agents run:** TypeScript review, Security audit, Performance Oracle, Spec/Flow analysis, Architecture review, Data Integrity Guardian, Best practices research, Framework docs research, Code simplicity review

### Key Improvements From Research

1. **Drop `minimatch`, use `ignore` + `picomatch`** — `minimatch` does not implement the gitignore spec; trailing `/` patterns behave differently. Use `ignore` v7.0.5 for pattern files, `picomatch` v4.0.3 for CLI glob flags.
2. **Drop `--include` flag** — YAGNI; it creates 4-level precedence complexity that is hard to implement, test, and document. `--include-all` covers the real-world case.
3. **No schema version bump** — `excluded?: ExcludedEntry[]` as optional field in v1. Reserve version bumps for breaking changes.
4. **Three critical security findings** must be addressed: fail-closed filter (try/catch), symlink traversal prevention, ReDoS protection with pattern length/count limits.
5. **Performance: pre-compile patterns outside the filter closure** — `minimatch()` standalone recompiles regex on every call; `new Minimatch(p, opts)` compiled once before the hot path eliminates O(N×M) recompilation.
6. **Filter must drive excluded[] directly** — populate the excluded array as a side-effect of the filter at runtime, not via post-hoc reconstruction.
7. **Default exclusions = breaking change** — requires a two-phase rollout: opt-in flag first, default-on in the next major version bump.
8. **Five data integrity safeguards** are non-negotiable before shipping: pre-flight file validation, filter-driven excluded[], count cross-check, post-tar archive cross-check, and upgrade behavior gate.

### New Considerations Discovered

- `tar.c({ filter })` receives **relative** paths (to cwd), not absolute — filter logic must use `path.relative()` correctly
- Returning `false` for a **directory** entry prunes the entire subtree automatically — no need to filter each child
- `filter` and `onWriteEntry` are NOT both called — filter gates onWriteEntry; if filter returns false, onWriteEntry never fires
- `.openclawignore` paths passed to `ignore` must be relative with no leading `./`
- Pattern provenance (`source: 'default' | 'cli' | 'config-file'`) belongs in manifest for debuggability
- `excludedBytes` in summary stats is the most actionable field for user awareness

## Current Implementation (Research Findings)

### Architecture

TypeScript compiled to rolldown bundles. Source layout (reconstructed from bundle region comments):

```
src/cli/program/register.backup.ts   ← Commander.js registration, CLI flags
src/cli/program/helpers.ts           ← collectOption() repeatable flag helper

src/commands/backup-verify.ts        ← backupVerifyCommand()
src/commands/backup-shared.ts        ← manifest types, resolveOutputPath()
```

### Archive Creation Flow

1. **`resolveBackupPlanFromDisk()`** — builds asset list; returns `assets[]` and `skipped[]`
2. **`backupCreateCommand()`** — writes `manifest.json` to temp dir, calls `tar.c()` with `gzip: true`, `portable: true`, `preservePaths: true`, **no `filter`** — all files archived unconditionally
3. Publishes archive via hard-link or fallback copy

### Key `tar` Behaviors (Verified from Source)

- **`filter(path, stat)` receives RELATIVE paths** (relative to cwd), not absolute
- **Returning `false` for a directory prunes the entire subtree** — `readdir` is never called on ignored jobs; children are never queued
- **`filter` and `onWriteEntry` are mutually exclusive** — if filter returns false, the WriteEntry is never constructed and `onWriteEntry` never fires
- **`follow: false` (default)** — filter sees symlink stat, not target stat; `follow: true` passes target stat but path is still the symlink path

## Proposed Solution

### Interface Design

```bash
# Opt into smart defaults (Phase 1 — will become default in next major)
openclaw backup create --smart-exclude

# With --smart-exclude, these are excluded: venvs/, models/, logs/, completions/
# Opt back in to everything
openclaw backup create --smart-exclude --include-all

# Add custom excludes (with or without --smart-exclude)
openclaw backup create --exclude "*.log" --exclude "**/__pycache__"

# Load patterns from file (gitignore syntax via `ignore` package)
openclaw backup create --exclude-file ~/.openclaw/.backupignore

# See what would be excluded before committing
openclaw backup create --smart-exclude --dry-run

# JSON output includes excluded[] array
openclaw backup create --smart-exclude --json

# In non-interactive mode, --smart-exclude proceeds without prompts
openclaw backup create --smart-exclude --non-interactive
```

### Two-Phase Rollout for Defaults (Breaking Change Protocol)

**Phase 1 — This PR:** Add `--smart-exclude` as an explicit opt-in flag. When passed, applies default exclusions. Emit a notice on first run:

```
ℹ️  Note: In openclaw v2.0, --smart-exclude will be enabled by default.
   Use --smart-exclude now to opt in early. Use --include-all to restore previous behavior.
```

**Phase 2 — Next major version:** Flip to default-on. `--smart-exclude` becomes a no-op (accepted for compatibility). `--include-all` is the escape hatch. Document in CHANGELOG and migration guide.

> **Rationale:** Changing what gets backed up silently for existing users is a breaking change. Automated backup scripts don't read stderr. The two-phase approach lets users opt in while preserving existing behavior for the major version.

### Smart Exclude Default Patterns

```typescript
export const SMART_EXCLUDE_DEFAULTS = [
  "venvs/", // Python virtual environments (pip/uv) — 8GB typical
  "models/", // Downloaded model weights — 1.7GB typical
  "logs/", // Log files — regenerated
  "completions/", // Shell completions — regenerated by shell setup
] as const;
```

### Flag Summary

| Flag                        | Description                                                        |
| --------------------------- | ------------------------------------------------------------------ |
| `--smart-exclude`           | Apply default exclude patterns (Phase 1: opt-in; Phase 2: default) |
| `--exclude <pattern>`       | Glob pattern to exclude (repeatable, gitignore syntax)             |
| `--exclude-file <path>`     | Load exclude patterns from file (gitignore syntax)                 |
| `--include-all`             | Disable all excludes including defaults                            |
| `--allow-exclude-protected` | Allow excluding protected paths (credentials/, extensions/, cron/) |
| `--non-interactive`         | Skip prompts; fail on ambiguity                                    |

**Precedence (documented, deterministic):**

1. `--include-all` → clears everything (highest)
2. `--exclude <pattern>` → user CLI flags (stacks additively)
3. `--exclude-file <path>` → user file patterns (stacks additively)
4. `.backupignore` auto-detection (if present in stateDir)
5. `--smart-exclude` defaults → `SMART_EXCLUDE_DEFAULTS` (lowest)

All sources are additive (union). No implicit precedence between `--exclude` and `--exclude-file` — both stack. `--include-all` is the only override mechanism.

### `.backupignore` Auto-Detection

> **Note from simplicity review:** Auto-detection is magic behavior. Document it clearly. The path convention is `~/.openclaw/.backupignore`. Users who want a persistent exclude file use `--exclude-file` explicitly; auto-detection is the "drop a file and forget it" convenience path.

If `~/.openclaw/.backupignore` exists, it is loaded automatically as additional patterns on top of defaults. `--include-all` disables auto-detection too. File must be owned by the running user with mode `0600` or `0644` — if group/world writable, skip with a warning (prevents tampering by other local users).

### Manifest Recording

```typescript
interface ExcludedEntry {
  readonly path: string; // relative path
  readonly pattern: string; // the specific pattern that matched
  readonly source: "default" | "cli" | "config-file" | "auto-file";
  readonly bytes: number; // 0 for directory entries (not traversed)
}

// Added as optional field in existing manifest (no version bump needed)
interface BackupManifest {
  // ... existing v1 fields unchanged ...
  excluded?: readonly ExcludedEntry[]; // absent = no exclusions applied
  excludedStats?: {
    totalFiles: number;
    totalBytes: number; // most actionable field for user awareness
    byPattern: Array<{ pattern: string; files: number; bytes: number; source: string }>;
  };
}
```

> **No schema version bump.** An additive optional field does not break existing readers. `manifest.excluded ?? []` everywhere. Reserve version bumps for breaking changes (removed fields, changed types, structural reorganization).

## Technical Approach

### Library Selection

**Use `ignore` v7.0.5 + `picomatch` v4.0.3 instead of `minimatch`.**

| Use case                      | Library            | Reason                                                                                               |
| ----------------------------- | ------------------ | ---------------------------------------------------------------------------------------------------- |
| `.backupignore` file patterns | `ignore` ^7.0.5    | Only library that correctly implements full gitignore spec (trailing `/`, negation, anchoring)       |
| CLI `--exclude` glob flags    | `picomatch` ^4.0.3 | 7× faster than minimatch, zero deps, correct for bash-style globs                                    |
| **Never use**                 | ~~minimatch~~      | Does not implement gitignore trailing-slash semantics; standalone function recompiles regex per call |

```bash
# Verify minimatch warning from ignore package:
# "Pay ATTENTION that minimatch does not follow the gitignore spec"
```

**Critical path convention for `ignore`:** Paths must be `path.relative(baseDir, absolutePath)` — no leading `./`, no absolute paths. The library throws on absolute paths.

#### `src/commands/backup-exclude.ts`

```typescript
import ignore from "ignore"; // ^7.0.5
import picomatch from "picomatch"; // ^4.0.3
import { readFileSync, existsSync, statSync } from "node:fs";
import { resolve, relative, isAbsolute, sep } from "node:path";

// Named types — define before implementation
export type EntryFilter = (entryPath: string, stat: Stats) => boolean;

export interface ExcludeSpec {
  readonly exclude: readonly string[]; // --exclude patterns
  readonly excludeFile?: string; // --exclude-file path
  readonly includeAll: boolean; // --include-all flag
  readonly smartExclude: boolean; // --smart-exclude flag
  readonly allowExcludeProtected: boolean; // --allow-exclude-protected flag
  readonly nonInteractive: boolean; // --non-interactive flag
}

export const SMART_EXCLUDE_DEFAULTS = ["venvs/", "models/", "logs/", "completions/"] as const;

export const PROTECTED_PATHS = ["credentials/", "extensions/", "cron/"] as const;

// Pattern validation limits (ReDoS protection)
const MAX_PATTERN_LENGTH = 256;
const MAX_PATTERN_COUNT = 500;
const MAX_GLOBSTAR_DEPTH = 5;

function validatePattern(pattern: string): void {
  if (pattern.length > MAX_PATTERN_LENGTH) {
    throw new Error(
      `Pattern too long (${pattern.length} > ${MAX_PATTERN_LENGTH}): ${pattern.slice(0, 64)}...`,
    );
  }
  const globstarCount = (pattern.match(/\*\*/g) || []).length;
  if (globstarCount > MAX_GLOBSTAR_DEPTH) {
    throw new Error(`Pattern has too many globstars (${globstarCount}): ${pattern}`);
  }
}

export async function resolveExcludePatterns(
  spec: ExcludeSpec,
  stateDir: string, // operational context, not part of ExcludeSpec
): Promise<readonly string[]> {
  if (spec.includeAll) return [];

  const patterns: string[] = [];

  // Layer 1: smart-exclude defaults (lowest priority)
  if (spec.smartExclude) {
    patterns.push(...SMART_EXCLUDE_DEFAULTS);
  }

  // Layer 2: auto-detect .backupignore in stateDir
  const autoIgnoreFile = resolve(stateDir, ".backupignore");
  if (existsSync(autoIgnoreFile)) {
    const fileStat = statSync(autoIgnoreFile);
    const isGroupOrWorldWritable = (fileStat.mode & 0o022) !== 0;
    if (!isGroupOrWorldWritable) {
      const lines = readFileSync(autoIgnoreFile, "utf-8")
        .split("\n")
        .map((l) => l.trim())
        .filter((l) => l && !l.startsWith("#"));
      patterns.push(...lines);
    } else {
      console.warn("⚠️  .backupignore is group/world writable — skipping for security");
    }
  }

  // Layer 3: --exclude-file (pre-validated before tar starts — fail fast)
  if (spec.excludeFile) {
    const fileStat = statSync(spec.excludeFile); // throws if not found
    if (!fileStat.isFile()) {
      throw new ExcludeFileError(
        spec.excludeFile,
        "must be a regular file (not a device or directory)",
      );
    }
    if (fileStat.size > 1024 * 1024) {
      throw new ExcludeFileError(spec.excludeFile, `too large: ${fileStat.size} bytes (max 1MB)`);
    }
    const lines = readFileSync(spec.excludeFile, "utf-8")
      .split("\n")
      .map((l) => l.trim())
      .filter((l) => l && !l.startsWith("#"));
    patterns.push(...lines);
  }

  // Layer 4: --exclude CLI flags (highest user-level priority)
  patterns.push(...spec.exclude);

  // Validate all patterns (ReDoS protection)
  const deduplicated = [...new Set(patterns)];
  if (deduplicated.length > MAX_PATTERN_COUNT) {
    throw new Error(`Too many exclude patterns: ${deduplicated.length} (max ${MAX_PATTERN_COUNT})`);
  }
  for (const p of spec.exclude) {
    // validate user-supplied only (not defaults)
    validatePattern(p);
  }

  // Warn on protected path matches
  for (const pattern of deduplicated) {
    for (const protected_ of PROTECTED_PATHS) {
      if (pattern === protected_ || pattern === protected_.replace(/\/$/, "")) {
        if (!spec.allowExcludeProtected) {
          const msg = `Pattern "${pattern}" matches protected path. Use --allow-exclude-protected to override.`;
          if (spec.nonInteractive) throw new Error(msg);
          console.warn(`⚠️  ${msg}`);
        }
      }
    }
  }

  return deduplicated;
}

export interface ExcludeFilterResult {
  readonly filter: EntryFilter;
  readonly getExcluded: () => readonly ExcludedEntry[];
}

export function buildExcludeFilter(
  patterns: readonly string[],
  baseDir: string,
): ExcludeFilterResult {
  if (!isAbsolute(baseDir)) {
    throw new TypeError(`baseDir must be absolute, got: ${baseDir}`);
  }

  if (patterns.length === 0) {
    return { filter: () => true, getExcluded: () => [] };
  }

  // Separate gitignore-style patterns (use `ignore`) from bash globs (use picomatch)
  // For simplicity in v1: use `ignore` for all patterns — it's a superset
  const ig = ignore().add(patterns as string[]);

  // Pre-classify for performance: prefix patterns (directory names) get O(1) string check
  const prefixPatterns = patterns
    .filter((p) => !p.includes("*") && !p.includes("?") && !p.includes("{"))
    .map((p) => p.replace(/\/$/, "")); // normalize trailing slash

  const excluded: ExcludedEntry[] = [];

  // CRITICAL: filter populates excluded[] as side-effect — not post-hoc reconstruction
  const filter: EntryFilter = (entryPath, stat) => {
    try {
      // Normalize path to forward slashes (Windows safety)
      const rel = relative(baseDir, entryPath).split(sep).join("/");

      // Fast path: prefix check for directory entries (prunes entire subtree when false)
      for (const prefix of prefixPatterns) {
        if (rel === prefix || rel.startsWith(prefix + "/")) {
          const matchedPattern = patterns.find((p) => p.replace(/\/$/, "") === prefix)!;
          excluded.push({
            path: rel,
            pattern: matchedPattern,
            source: SMART_EXCLUDE_DEFAULTS.includes(matchedPattern as any) ? "default" : "cli",
            bytes: 0, // directory — contents not traversed
          });
          return false; // exclude; prunes subtree if directory
        }
      }

      // Glob path: use ignore package for gitignore-compliant matching
      // `ignore` requires no leading ./, relative path only
      if (rel && ig.ignores(rel)) {
        excluded.push({
          path: rel,
          pattern: "(matched pattern)", // ignore package doesn't expose which rule matched
          source: "cli",
          bytes: stat.size ?? 0,
        });
        return false; // exclude
      }

      return true; // include
    } catch (err) {
      // FAIL-CLOSED: on any filter error, exclude the entry for safety
      console.warn(
        `⚠️  Filter error for "${entryPath}", excluding for safety: ${(err as Error).message}`,
      );
      return false;
    }
  };

  return { filter, getExcluded: () => excluded };
}

export class ExcludeFileError extends Error {
  constructor(
    public readonly filePath: string,
    reason: string,
  ) {
    super(`--exclude-file ${filePath}: ${reason}`);
    this.name = "ExcludeFileError";
  }
}
```

## System-Wide Impact

### Interaction Graph

```
backup create
  → Commander.js parses flags
  → resolveExcludePatterns() [async] (builds pattern list, validates, fails fast)
  → resolveBackupPlanFromDisk() (asset planning)
  → buildExcludeFilter(patterns, stateDir) → { filter, getExcluded }
  → [self-ref output guard]
  → tar.c({ filter, follow: false }) (archives; filter populates excluded[] as side-effect)
  → [post-tar archive cross-check]
  → manifest.excluded = getExcluded()
  → atomic rename (temp → final)
  → optional backup verify (tolerant manifest reader)
```

### Error & Failure Propagation

- `--exclude-file` nonexistent/invalid → `ExcludeFileError` thrown **before** tar starts → no partial archive
- Pattern length/count exceeded → thrown before tar starts → no partial archive
- Protected path conflict in `--non-interactive` → exits 1 before tar starts
- Filter throws on unusual filename → fail-closed (returns false, excludes entry), logs warning
- `tar.c()` throws mid-archive → `finally` block cleans up temp file
- Post-tar cross-check fails → throws, temp file cleaned up, no archive written to final path
- v2 manifest opened by v1 tool → clear error with upgrade instruction

### State Lifecycle Risks

- **Asymmetric filter bugs:** Silently excluding critical files is data loss; failing to exclude recoverable files is just performance. Filter is fail-closed to avoid the dangerous direction.
- **Manifest-archive divergence:** Post-tar cross-check before atomic rename prevents bad archives from reaching users.
- **Upgrade regression:** No silent default exclusion changes in this PR. `--smart-exclude` is opt-in. Phase 2 upgrade path documents the major version bump.
- **Self-referential output:** `--output` inside archive source auto-detected and excluded.

### API Surface Parity

- `backup verify` reads `excluded[]` from manifest; tolerant reader handles v1 (absent field) and v2+ (present field)
- `--json` exposes `excluded[]` and `excludedStats` alongside existing `assets[]` and `skipped[]`
- `--dry-run` enumerates excluded paths with pattern and source
- `backup restore` (future): should warn about excluded paths before restoring

## Fast Follow: Incremental Backups

Incremental backup is intentionally **out of scope** for the following reasons:

1. **Scope isolation** — Two existing PRs for exclude have stalled without maintainer review. Adding incremental would further delay the high-value fix.
2. **Different data model** — Requires checksum tracking, last-backup manifest references, restore-time ordering logic. A meaningful new subsystem.
3. **Restore complexity** — Incremental requires a `restore` command that chains diffs back to a full base. That command doesn't exist yet.

**What an incremental PR would include:**

- `--incremental` flag
- Checksum manifest at `~/.openclaw/.last-backup-manifest.json`
- `tar.c()` filter compares file mtimes/checksums against last manifest
- `openclaw backup restore` command (new) handling full + incremental archives
- Chain restore integration tests
- `backup restore` warning about excluded paths (also needed for the base feature)

**Sequencing:** Land this PR → release → open fast-follow issue for incremental with link to this PR.
