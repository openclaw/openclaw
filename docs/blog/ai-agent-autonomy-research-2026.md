# AI Agent 自主性实测：Anthropic 揭示人机协作的真实图景

## 引言：从理论到实践

AI Agent 正在从科幻走向现实。从邮件分拣到网络间谍活动，Agent 已经被部署在影响程度各异的场景中。然而，我们对人们在真实世界中如何使用 Agent 知之甚少。

2026年2月18日，Anthropic 发布了一项开创性研究——**《在实践中测量 AI Agent 自主性》**。通过分析 Claude Code 和公共 API 上数百万次人机交互，研究团队试图回答一个核心问题：

> 人类究竟给予 Agent 多大的自主权？这种信任如何随经验增长？Agent 在哪些领域运作？它们的行为是否有风险？

本文将深入解读这项研究的关键发现，并探讨其对 AI 产品开发和政策制定的启示。

---

## 一、研究方法：在"野生"环境中观察 Agent

### 1.1 定义 Agent

研究首先解决了一个基础问题：什么是 Agent？

Anthropic 采用了可操作的定义：

> **Agent 是配备工具的 AI 系统，能够执行操作——如运行代码、调用外部 API、向其他 Agent 发送消息。**

这一定义的精妙之处在于：通过研究 Agent 使用的工具，我们可以理解它们在"做什么"。

### 1.2 双数据源策略

研究采用两种互补的数据来源：

| 数据源 | 优势 | 局限 |
|--------|------|------|
| **公共 API** | 覆盖数千客户，视野广阔 | 只能分析单个工具调用，无法重构完整工作流 |
| **Claude Code** | 可追踪完整会话，深度洞察自主性 | 单一产品，多样性不足 |

通过隐私保护工具 Clio，研究团队能够在保护用户隐私的前提下进行大规模分析。

## 二、关键发现：Agent 自主性的真实图景

### 2.1 Agent 工作时间显著延长

**核心数据**：在最长运行的会话中，Claude Code 独立工作的时间在三个月内几乎翻倍——从不到 25 分钟增至超过 45 分钟。

这一增长趋势值得深思：

- **不是单纯能力提升**：增长曲线平滑跨越多个模型版本，说明不完全是模型能力增强的结果
- **潜在自主性大于实际自主性**：现有模型实际上能够比实践中展现更高的自主性

> 这意味着：限制 Agent 自主性的，往往不是技术，而是人类的信任和监督机制。

### 2.2 经验用户的"放手"与"介入"

研究发现了一个有趣的悖论：

**新用户 vs 经验用户的行为对比：**

| 行为特征 | 新用户 | 经验用户 |
|----------|--------|----------|
| 使用全自动批准 | ~20% 会话 | >40% 会话 |
| 介入频率 | 较低 | 较高 |
| 工作模式 | 逐条审查 | 放手运行，需要时介入 |

**解读**：经验用户不再审查每个操作，而是让 Claude 自主运行，只在必要时介入。这是一种更成熟的监督模式——**信任但验证**。

### 2.3 Agent 主动"叫停"：一种被忽视的监督机制

研究发现了一个出人意料的事实：

> 在最复杂的任务中，Claude Code 主动停下来请求澄清的频率，是人类中断它的 **两倍以上**。

这揭示了 Agent 自主性的另一个维度：**Agent 自身可以成为监督的一部分**。

当 Agent 遇到不确定的情况，它选择"暂停"而非"猜测"。这种行为模式对于安全部署至关重要。

### 2.4 风险领域：存在但未规模化

研究对 Agent 操作进行了风险评估：

**Agent 活动领域分布：**

| 领域 | 占比 | 风险等级 |
|------|------|----------|
| 软件工程 | ~50% | 低-中 |
| 医疗健康 | 新兴 | 高 |
| 金融 | 新兴 | 高 |
| 网络安全 | 新兴 | 极高 |

**核心结论**：大多数 Agent 操作是低风险且可逆的。但在高风险领域的使用正在萌芽。

## 三、深度分析：自主性的度量维度

### 3.1 时间维度

研究使用"回合持续时间"（Turn Duration）作为自主性的代理指标：

- **中位数**：约 45 秒（过去几个月波动在 40-55 秒）
- **P99 会话**：从 <25 分钟增长到 >45 分钟

但研究者承认这是不完美的指标：

> 更强的模型可能更快完成任务；子代理允许并行工作——这些都可能缩短回合时间。同时，用户可能尝试更雄心勃勃的任务，这会延长回合时间。

### 3.2 信任维度

"自动批准"（Auto-approve）使用率是信任的直接体现：

- 新用户：20% 会话使用全自动批准
- 经验用户：40%+ 会话使用全自动批准

**问题**：更高的自动批准率是好事还是坏事？

- **乐观视角**：体现了信任和效率
- **悲观视角**：可能意味着监督松懈

### 3.3 主动性维度

Agent 主动请求澄清的频率是一个关键指标：

- 表明 Agent "知道自己的局限"
- 是一种内在安全机制
- 在复杂任务中尤为显著

## 四、对开发者的启示

### 4.1 产品设计：从"全有或全无"到"渐进式自主"

研究表明，用户对自主性的需求随经验变化。产品设计应考虑：

1. **新手模式**：详细审查，频繁确认
2. **进阶模式**：自动批准常见操作，异常时暂停
3. **专家模式**：高度自主，事后审计

### 4.2 监督机制：人机协同的新范式

有效的监督不应只是"人类监控 AI"：

| 监督类型 | 方式 | 示例 |
|----------|------|------|
| 人类发起 | 中断、审查 | 用户主动暂停 |
| Agent 发起 | 请求澄清 | Claude 询问模糊指令 |
| 系统发起 | 安全边界、审计日志 | 敏感操作强制确认 |

### 4.3 风险管理：部署后监控至关重要

研究强调了一个关键观点：

> **Agent 的有效监督需要新型部署后监控基础设施**

这意味着：

- 实时追踪高风险操作
- 分析用户行为变化趋势
- 识别新兴风险领域

## 五、政策建议

### 5.1 对模型开发者

- 提供细粒度的自主性控制选项
- 增强模型的"自我认知"能力——知道何时应该暂停
- 开发隐私保护的监控工具

### 5.2 对产品开发者

- 设计渐进式的自主性曲线
- 建立清晰的风险分级系统
- 提供透明的操作日志

### 5.3 对政策制定者

- 关注高风险领域（医疗、金融、网络安全）的 Agent 使用
- 制定 Agent 行为审计标准
- 平衡创新与安全

---

## 六、未来展望

这项研究只是理解 Agent 的第一步。随着 Agent 能力增强和应用扩展，我们需要：

1. **更好的度量工具**：时间、信任、主动性都需要更精确的量化
2. **跨平台研究**：理解不同 Agent 系统的使用模式
3. **纵向追踪**：观察用户行为如何随时间演变
4. **风险预测**：识别高风险使用模式

---

## 结语

Anthropic 的这项研究具有里程碑意义——它首次系统性地揭示了 AI Agent 在真实世界中的运作图景。

核心启示可以概括为：

1. **自主性是渐进的**：用户随着经验增长逐步放手
2. **监督是双向的**：Agent 可以主动参与自我监督
3. **风险是可控的**：大多数当前使用是低风险的，但高风险领域正在扩展
4. **基础设施是关键的**：需要新的监控工具和人机交互范式

当 AI Agent 变得越来越强大，人类与它们的协作方式将决定这项技术是造福人类还是带来风险。这项研究为我们提供了宝贵的实证基础——但它只是漫长旅程的第一步。
