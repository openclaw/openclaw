🔑 Key Insight: Context vs Memory

Context = what I see RIGHT NOW in a single conversation (ephemeral, per-session, has a token limit).

Memory = files on disk that persist forever (MEMORY.md, daily logs). I read these INTO context when needed.

🔒 Why Sessions Are Separate

Each Discord channel = separate session = separate context window. I can't "see" what was said in #heady-convo from #ren-general.

BUT: I can access the same memory files from any session. That's how I know who you are everywhere.

🔄 Session Reset

Sessions reset daily at 4 AM (configurable). When reset, context is cleared but memory files remain.

You can also manually reset with /new or /reset.

🧠 Memory Search

OpenClaw indexes my memory files with embeddings. When I need to recall something, I can semantically search across all my notes—even if wording differs.
