# ဘရောက်ဇာ လော့ဂ်အင် + X/Twitter ပို့စ်တင်ခြင်း

## လက်ဖြင့် လော့ဂ်အင် (အကြံပြုထားသည်)

ဆိုဒ်တစ်ခုက လော့ဂ်အင် လိုအပ်ပါက **ဟို့စ်** ဘရောက်ဇာ ပရိုဖိုင် (openclaw ဘရောက်ဇာ) တွင် **လက်ဖြင့် လော့ဂ်အင်ဝင်ပါ**။

မော်ဒယ်ထံ သင်၏ အထောက်အထားများကို **မပေးပါနှင့်**။ အလိုအလျောက် လော့ဂ်အင်များသည် အန်တီ-ဘော့ ကာကွယ်မှုများကို မကြာခဏ လှုံ့ဆော်ပြီး အကောင့်ကို ပိတ်သိမ်းစေနိုင်ပါသည်။

အဓိက ဘရောက်ဇာ စာရွက်စာတမ်းများသို့ ပြန်သွားရန်: [Browser](/tools/browser)။

## ဘယ် Chrome ပရိုဖိုင်ကို အသုံးပြုသလဲ?

OpenClaw သည် **သီးသန့် Chrome ပရိုဖိုင်တစ်ခု** (အမည် `openclaw`၊ UI သည် လိမ္မော်ရောင် မျက်နှာပြင်အရိပ်ရှိ) ကို ထိန်းချုပ်ပါသည်။ ၎င်းသည် သင်၏ နေ့စဉ်အသုံးပြုသည့် ဘရောက်ဇာ ပရိုဖိုင်နှင့် ခွဲခြားထားပါသည်။

ဝင်ရောက်ရန် လွယ်ကူသော နည်းလမ်း နှစ်ခုရှိပါသည်—

1. **အေးဂျင့်အား ဘရောက်ဇာကို ဖွင့်ခိုင်းပြီး** ကိုယ်တိုင် လော့ဂ်အင်ဝင်ပါ။
2. **CLI ဖြင့် ဖွင့်ပါ**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

ပရိုဖိုင်များ များစွာရှိပါက `--browser-profile <name>` ကို ပေးပို့ပါ (ပုံမှန်သည် `openclaw` ဖြစ်သည်)။

## X/Twitter: အကြံပြုထားသော လုပ်ငန်းစဉ်

- **ဖတ်ရှုခြင်း/ရှာဖွေခြင်း/threads:** **ဟို့စ်** ဘရောက်ဇာကို အသုံးပြုပါ (လက်ဖြင့် လော့ဂ်အင်)။
- **အပ်ဒိတ်များ ပို့စ်တင်ခြင်း:** **ဟို့စ်** ဘရောက်ဇာကို အသုံးပြုပါ (လက်ဖြင့် လော့ဂ်အင်)။

## Sandboxing + ဟို့စ် ဘရောက်ဇာ ဝင်ရောက်ခွင့်

Sandboxed ဘရောက်ဇာ ဆက်ရှင်များသည် ဘော့ တွေ့ရှိမှုကို **ပိုမို ဖြစ်ပေါ်စေနိုင်ခြေရှိပါသည်**။ X/Twitter (နှင့် အခြား တင်းကျပ်သည့် ဆိုဒ်များ) အတွက် **ဟို့စ်** ဘရောက်ဇာကို ဦးစားပေး အသုံးပြုပါ။

အေးဂျင့်ကို sandboxed လုပ်ထားပါက ဘရောက်ဇာ ကိရိယာသည် ပုံမှန်အားဖြင့် sandbox ကို ရွေးချယ်ပါသည်။ ဟို့စ် ထိန်းချုပ်ခွင့် ပေးရန်—

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

ထို့နောက် ဟို့စ် ဘရောက်ဇာကို ပစ်မှတ်ထားပါ—

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

သို့မဟုတ် အပ်ဒိတ်များ ပို့စ်တင်သော အေးဂျင့်အတွက် sandboxing ကို ပိတ်ထားနိုင်ပါသည်။
