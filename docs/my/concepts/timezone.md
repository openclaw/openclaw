# အချိန်ဇုန်များ

OpenClaw သည် မော်ဒယ်က **ကိုးကားအချိန် တစ်ခုတည်း** ကိုသာ မြင်ရအောင် အချိန်တံဆိပ်များကို စံညွှန်းတကျ ပြောင်းလဲပေးပါသည်။

## မက်ဆေ့ချ် အင်ဗလိုပ်များ (မူလအားဖြင့် local)

ဝင်လာသော မက်ဆေ့ချ်များကို အောက်ပါအတိုင်း အင်ဗလိုပ်တစ်ခုအဖြစ် ထုပ်ပိုးပါသည်–

```
[Provider ... 2026-01-05 16:26 PST] message text
```

အင်ဗလိုပ်အတွင်းရှိ အချိန်တံဆိပ်သည် **မူလအားဖြင့် ဟို့စ်၏ local အချိန်ဇုန်** ကို အသုံးပြုပြီး မိနစ်အဆင့်တိကျမှုရှိပါသည်။

ဤအရာကို အောက်ပါအတိုင်း အစားထိုးသတ်မှတ်နိုင်ပါသည်–

```json5
{
  agents: {
    defaults: {
      envelopeTimezone: "local", // "utc" | "local" | "user" | IANA timezone
      envelopeTimestamp: "on", // "on" | "off"
      envelopeElapsed: "on", // "on" | "off"
    },
  },
}
```

- `envelopeTimezone: "utc"` သည် UTC ကို အသုံးပြုပါသည်။
- `envelopeTimezone: "user"` သည် `agents.defaults.userTimezone` ကို အသုံးပြုပါသည် (ဟို့စ် အချိန်ဇုန်သို့ ပြန်လည်ချိတ်ဆက်သည်)။
- တည်ငြိမ်သော offset အတွက် အတိအကျ IANA အချိန်ဇုန်ကို အသုံးပြုပါ (ဥပမာ၊ `"Europe/Vienna"`)။
- `envelopeTimestamp: "off"` သည် အင်ဗလိုပ် ခေါင်းစဉ်များမှ absolute အချိန်တံဆိပ်များကို ဖယ်ရှားပါသည်။
- `envelopeElapsed: "off"` သည် အချိန်ကာလ ကုန်လွန်မှု အဆုံးသတ်များကို ဖယ်ရှားပါသည် ( `+2m` ပုံစံ)။

### ဥပမာများ

**Local (မူလ):**

```
[Signal Alice +1555 2026-01-18 00:19 PST] hello
```

**တည်ငြိမ်သော အချိန်ဇုန်:**

```
[Signal Alice +1555 2026-01-18 06:19 GMT+1] hello
```

**အချိန်ကာလ ကုန်လွန်မှု:**

```
[Signal Alice +1555 +2m 2026-01-18T05:19Z] follow-up
```

## ကိရိယာ payload များ (ပံ့ပိုးသူ၏ raw ဒေတာ + စံညွှန်းတကျ ပြောင်းထားသော အကွက်များ)

Tool calls (`channels.discord.readMessages`, `channels.slack.readMessages`, စသည်) **raw provider timestamps** ကို ပြန်ပေးပါသည်။
တူညီမှုအတွက် normalized fields များကိုလည်း တွဲဖက်ပေးပါသည်။

- `timestampMs` (UTC epoch milliseconds)
- `timestampUtc` (ISO 8601 UTC စာကြောင်း)

ပံ့ပိုးသူ၏ raw အကွက်များကို မပြောင်းလဲဘဲ ထိန်းသိမ်းထားပါသည်။

## စနစ်ပရောမ့်အတွက် အသုံးပြုသူ အချိန်ဇုန်

မော်ဒယ်အား အသုံးပြုသူ၏ ဒေသခံ time zone ကို သိစေရန် `agents.defaults.userTimezone` ကို သတ်မှတ်ပါ။ မသတ်မှတ်ထားပါက OpenClaw သည် **host timezone ကို runtime အတွင်း ဖြေရှင်းသတ်မှတ်** ပါသည် (config ကို မရေးပါ)။

```json5
{
  agents: { defaults: { userTimezone: "America/Chicago" } },
}
```

စနစ်ပရောမ့်တွင် အောက်ပါအရာများ ပါဝင်ပါသည်–

- local အချိန်နှင့် အချိန်ဇုန် ပါဝင်သော `Current Date & Time` အပိုင်း
- `Time format: 12-hour` သို့မဟုတ် `24-hour`

`agents.defaults.timeFormat` (`auto` | `12` | `24`) ဖြင့် ပရောမ့် ပုံစံကို ထိန်းချုပ်နိုင်ပါသည်။

အပြည့်အစုံ လုပ်ဆောင်ပုံနှင့် ဥပမာများအတွက် [Date & Time](/date-time) ကို ကြည့်ပါ။
