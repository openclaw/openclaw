# 온보딩 개요

OpenClaw 에는 두 가지 온보딩 경로가 있습니다. 둘 다 인증, Gateway, 선택적 채널을 구성하며 — 설정 과정에서의 상호작용 방식만 다릅니다.

## 어떤 경로를 사용해야 하나요?

|                 | CLI 온보딩                                 | macOS 앱 온보딩           |
| --------------- | ------------------------------------------ | ------------------------- |
| **플랫폼**      | macOS, Linux, Windows (네이티브 또는 WSL2) | macOS 전용                |
| **인터페이스**  | 터미널 마법사                              | 앱 내 가이드 UI           |
| **적합한 용도** | 서버, 헤드리스, 완전한 제어                | 데스크톱 Mac, 시각적 설정 |
| **자동화**      | 스크립트용 `--non-interactive`             | 수동만 가능               |
| **명령**        | `openclaw onboard`                         | 앱 실행                   |

대부분의 사용자는 **CLI 온보딩**으로 시작하는 것이 좋습니다 — 어디서든 작동하며 가장 많은 제어를 제공합니다.

## 온보딩이 설정하는 것

어떤 경로를 선택하든 온보딩은 다음을 설정합니다:

1. **모델 프로바이더 및 인증** — 선택한 프로바이더의 API 키, OAuth 또는 설정 토큰
2. **워크스페이스** — 에이전트 파일, 부트스트랩 템플릿, 메모리를 위한 디렉터리
3. **Gateway** — 포트, 바인드 주소, 인증 모드
4. **채널** (선택 사항) — WhatsApp, Telegram, Discord 등
5. **데몬** (선택 사항) — Gateway 가 자동으로 시작되도록 하는 백그라운드 서비스

## CLI 온보딩

아무 터미널에서 실행하세요:

```bash
openclaw onboard
```

`--install-daemon` 을 추가하면 한 번에 백그라운드 서비스도 설치합니다.

전체 참조: [온보딩 (CLI)](/start/wizard)
CLI 명령 문서: [`openclaw onboard`](/cli/onboard)

## macOS 앱 온보딩

OpenClaw 앱을 여세요. 최초 실행 마법사가 시각적 인터페이스로 동일한 단계를 안내합니다.

전체 참조: [온보딩 (macOS 앱)](/start/onboarding)

## 사용자 정의 또는 목록에 없는 프로바이더

온보딩에 프로바이더가 목록에 없는 경우, **Custom Provider** 를 선택하고 다음을 입력하세요:

- API 호환성 모드 (OpenAI 호환, Anthropic 호환 또는 자동 감지)
- 기본 URL 및 API 키
- 모델 ID 및 선택적 별칭

여러 사용자 정의 엔드포인트가 공존할 수 있으며 — 각각 고유한 엔드포인트 ID 를 받습니다.
