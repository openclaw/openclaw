# OpenClaw 위협 모델 기여하기

OpenClaw를 더 안전하게 만드는 데 도움을 주셔서 감사합니다. 이 위협 모델은 살아있는 문서이며, 보안 전문가가 아니더라도 누구나 기여할 수 있습니다.

## 기여 방법

### 위협 추가하기

우리가 다루지 않은 공격 벡터나 위험을 발견하셨나요? [openclaw/trust](https://github.com/openclaw/trust/issues)에 문제를 제기하고 자신의 언어로 설명해 주세요. 특정 프레임워크를 알 필요도 없고 모든 필드를 채울 필요도 없습니다 - 시나리오를 설명만 해주세요.

**포함하면 도움이 되는 내용 (필수 아님):**

- 공격 시나리오와 어떻게 악용될 수 있는지
- OpenClaw의 어느 부분이 영향을 받는지 (CLI, 게이트웨이, 채널, ClawHub, MCP 서버 등)
- 심각도 판단 (낮음 / 중간 / 높음 / 치명적)
- 관련 연구, CVE, 실제 사례에 대한 링크

저희가 검토 과정에서 ATLAS 매핑, 위협 ID, 위험 평가를 처리합니다. 이러한 세부 사항을 포함하고 싶다면 좋은 일이지만, 요구 사항은 아닙니다.

> **이 문서는 위협 모델에 추가하기 위한 것이며, 실시간 취약성을 보고하는 것이 아닙니다.** 취약성을 발견했다면, [신뢰 페이지](https://trust.openclaw.ai)를 참조하여 책임 있는 공개 지침을 확인하세요.

### 완화책 제안

기존 위협을 해결할 방법이 생각나셨나요? 위협을 참조하여 문제나 PR을 열어 제안하세요. 유용한 완화책은 구체적이고 실행 가능해야 합니다 - 예를 들어 "게이트웨이에서 발신자당 10메시지/분 속도 제한"이 "속도 제한 구현"보다 좋습니다.

### 공격 체인 제안

공격 체인은 여러 위협이 결합되어 실제로 어떻게 공격 시나리오를 만들어내는지를 보여줍니다. 위험한 조합을 발견했다면, 단계와 공격자가 어떻게 그것들을 연결할 것인지를 설명하세요. 공격이 실제로 어떻게 전개되는지에 대한 간단한 서술은 공식 템플릿보다 더 가치가 있습니다.

### 기존 내용 수정 또는 개선

오타, 명확성, 오래된 정보, 더 나은 예제 - PR을 환영하며, 문제를 제기할 필요 없습니다.

## 사용 도구

### MITRE ATLAS

이 위협 모델은 AI/ML 위협을 위해 특별히 설계된 프레임워크인 [MITRE ATLAS](https://atlas.mitre.org/) (AI 시스템을 위한 적대적 위협 환경)를 기반으로 구축되었습니다. 기여를 위해 ATLAS를 알 필요는 없습니다 - 검토 과정에서 프레임워크에 제출물을 매핑합니다.

### 위협 ID

각 위협은 `T-EXEC-003`과 같은 ID를 부여받습니다. 카테고리는 다음과 같습니다:

| 코드   | 카테고리                                 |
| ------ | ---------------------------------------- |
| RECON  | 탐색 - 정보 수집                             |
| ACCESS | 초기 접근 - 진입 획득                         |
| EXEC   | 실행 - 악성 행동 실행                         |
| PERSIST| 지속성 - 접근 유지                           |
| EVADE  | 방어 회피 - 탐지 회피                         |
| DISC   | 발견 - 환경 탐색                             |
| EXFIL  | 유출 - 데이터 도난                            |
| IMPACT | 영향 - 손상 또는 중단                          |

ID는 검토 중에 관리자들이 할당합니다. 하나를 선택할 필요는 없습니다.

### 위험 수준

| 수준           | 의미                                                          |
| -------------- | --------------------------------------------------------------|
| **치명적**     | 전체 시스템 손상, 또는 높은 가능성과 치명적 영향                |
| **높음**       | 심각한 손상 가능성, 또는 중간 가능성과 치명적 영향             |
| **중간**       | 보통 위험, 또는 낮은 가능성과 높은 영향                        |
| **낮음**       | 가능성이 낮고 제한된 영향                                      |

위험 수준이 확실하지 않은 경우, 영향을 설명만 해주시면 저희가 평가하겠습니다.

## 검토 과정

1. **분류** - 48시간 이내에 새 제출물을 검토합니다.
2. **평가** - 실현 가능성을 검증하고, ATLAS 매핑과 위협 ID를 할당하며, 위험 수준을 검증합니다.
3. **문서화** - 모든 것이 형식화되고 완성되었는지 확인합니다.
4. **병합** - 위협 모델과 시각화에 추가합니다.
