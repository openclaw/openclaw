# 스크립트

`scripts/` 디렉터리에는 로컬 워크플로와 운영 작업을 위한 보조 스크립트들이 포함되어 있습니다. 작업이 명확하게 스크립트와 연결된 경우에 이들을 사용하고, 그렇지 않다면 CLI 를 선호하십시오.

## 관례

- 문서나 릴리스 체크리스트에 언급되지 않는 한 스크립트는 **선택적**입니다.
- CLI 기능이 존재할 때 이를 선호하십시오 (예: 인증 모니터링에는 `openclaw models status --check` 사용).
- 스크립트는 호스트‑특정이라고 가정하고, 새로운 기기에서 실행하기 전에 읽어보십시오.

## 스크립트 추가 시

- 스크립트는 집중적이고 문서화된 형태로 유지하십시오.
- 관련 문서에 간단한 항목을 추가하십시오 (없을 경우 문서를 새로 만드십시오).
