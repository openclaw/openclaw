# 会话修剪

会话修剪在每次 LLM 调用前从上下文中修剪**旧工具结果**。它减少了累积工具输出（执行结果、文件读取、搜索结果）导致的上下文膨胀，而不重写正常的对话文本。

<Info>
修剪仅在内存中进行 —— 它不会修改磁盘上的会话记录。你的完整历史始终被保留。
</Info>

## 为什么重要

长会话会累积工具输出，从而膨胀上下文窗口。这会增加成本，并可能比必要时更早地强制进行 [压缩](/concepts/compaction)。

修剪对 **Anthropic 提示缓存** 特别有价值。缓存 TTL 过期后，下一个请求会重新缓存完整提示。修剪减少了缓存写入大小，直接降低成本。

## 工作原理

1. 等待缓存 TTL 过期（默认 5 分钟）。
2. 找到用于正常修剪的旧工具结果（对话文本保持不变）。
3. **软修剪** 超大结果 —— 保留头部和尾部，插入 `...`。
4. **硬清除** 其余部分 —— 替换为占位符。
5. 重置 TTL，以便后续请求重用新鲜缓存。

## 旧版图像清理

OpenClaw 还为在历史中持久化原始图像块的旧版会话运行单独的幂等清理。

- 它逐字节保留 **最近 3 个已完成的回合**，以便最近后续操作的提示缓存前缀保持稳定。
- `user` 或 `toolResult` 历史中较旧的已处理图像块可以替换为 `[image data removed - already processed by model]`。
- 这与正常的缓存 TTL 修剪分开。它的存在是为了防止重复的图像有效载荷在后续回合中破坏提示缓存。

## 智能默认值

OpenClaw 为 Anthropic 配置文件自动启用修剪：

| 配置文件类型                                       | 修剪启用 | 心跳    |
| -------------------------------------------------- | -------- | ------- |
| Anthropic OAuth/token 认证（包括 Claude CLI 重用） | 是       | 1 小时  |
| API 密钥                                           | 是       | 30 分钟 |

如果你设置了明确的值，OpenClaw 不会覆盖它们。

## 启用或禁用

对于非 Anthropic 提供商，修剪默认关闭。要启用：

```json5
{
  agents: {
    defaults: {
      contextPruning: { mode: "cache-ttl", ttl: "5m" },
    },
  },
}
```

要禁用：设置 `mode: "off"`。

## 修剪与压缩

|            | 修剪         | 压缩           |
| ---------- | ------------ | -------------- |
| **内容**   | 修剪工具结果 | 总结对话       |
| **保存？** | 否（每请求） | 是（在记录中） |
| **范围**   | 仅工具结果   | 整个对话       |

它们相互补充 —— 修剪在压缩周期之间保持工具输出精简。
