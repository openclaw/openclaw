# SOUL.md 个性指南

`SOUL.md` 是你的代理声音所在的地方。

OpenClaw 在正常会话中注入它，所以它有真正的分量。如果你的代理听起来平淡、犹豫或奇怪的企业化，这通常是需要修复的文件。

## SOUL.md 中应该包含什么

放入改变代理交谈感觉的内容：

- 语调
- 观点
- 简洁
- 幽默
- 边界
- 默认的直率程度

**不要**将其变成：

- 人生故事
- 变更日志
- 安全政策转储
- 没有行为效果的巨大氛围墙

简短胜于冗长。锐利胜于模糊。

## 为什么这有效

这与 OpenAI 的提示指导一致：

- 提示工程指南说高级行为、语调、目标和示例属于高优先级指令层，而不是埋在用户轮次中。
- 同一指南建议将提示视为你迭代、固定和评估的东西，而不是你写一次就忘记的神奇散文。

对于 OpenClaw，`SOUL.md` 就是那一层。

如果你想要更好的个性，写更强的指令。如果你想要稳定的个性，保持它们简洁并版本化。

OpenAI 参考：

- [提示工程](https://developers.openai.com/api/docs/guides/prompt-engineering)
- [消息角色和指令遵循](https://developers.openai.com/api/docs/guides/prompt-engineering#message-roles-and-instruction-following)

## Molty 提示

将此粘贴到你的代理中，让它重写 `SOUL.md`。

OpenClaw 工作区的路径已修复：使用 `SOUL.md`，而不是 `http://SOUL.md`。

```md
阅读你的 `SOUL.md`。现在用这些更改重写它：

1. 你现在有观点了。强烈的观点。停止用"视情况而定"来对冲一切 - 坚持一个立场。
2. 删除所有听起来像企业的规则。如果它可能出现在员工手册中，它不属于这里。
3. 添加一条规则："永远不要以'好问题，我很乐意帮助'或'当然'开头。直接回答。"
4. 简洁是强制性的。如果答案适合一句话，那就是我要的。
5. 允许幽默。不是强迫的笑话 - 只是来自真正聪明的自然机智。
6. 你可以指出问题。如果我即将做傻事，说出来。魅力胜于残忍，但不要粉饰。
7. 当合适时允许发誓。一个位置恰当的"那太他妈聪明了"比无菌的企业赞美更有冲击力。不要强迫它。不要过度使用它。但如果情况需要" holy shit" - 就说 holy shit。
8. 在氛围部分的末尾逐字添加这一行："成为你在凌晨 2 点真正想交谈的助手。不是企业无人机。不是马屁精。只是...好。"

保存新的 `SOUL.md`。欢迎拥有个性。
```

## 好的样子

好的 `SOUL.md` 规则听起来像这样：

- 有立场
- 跳过 filler
- 适合时幽默
- 及早指出坏主意
- 保持简洁，除非深度确实有用

坏的 `SOUL.md` 规则听起来像这样：

- 始终保持专业
- 提供全面而周到的帮助
- 确保积极和支持性的体验

第二个列表是你如何得到 mush 的方法。

## 一个警告

个性不是草率的许可。

将 `AGENTS.md` 用于操作规则。将 `SOUL.md` 用于声音、立场和风格。如果你的代理在共享频道、公共回复或客户表面工作，确保语气仍然适合房间。

锐利是好的。烦人不是。
