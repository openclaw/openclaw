# Мастер онбординга (CLI)

Мастер онбординга — **рекомендуемый** способ настроить OpenClaw на macOS,
Linux или Windows (через WSL2; настоятельно рекомендуется).
Он настраивает локальный Gateway (шлюз) или подключение к удалённому Gateway (шлюзу),
а также каналы, Skills и значения рабочего пространства по умолчанию — в одном управляемом процессе.

```bash
openclaw onboard
```

<Info>
Самый быстрый первый чат: откройте Control UI (настройка каналов не требуется). Запустите
`openclaw dashboard` и общайтесь в браузере. Документация: [Dashboard](/web/dashboard).
</Info>

Чтобы перенастроить позже:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` не означает неинтерактивный режим. Для скриптов используйте `--non-interactive`.
</Note>

<Tip>
Рекомендуется: настройте ключ API Brave Search, чтобы агент мог использовать `web_search`
(`web_fetch` работает без ключа). Самый простой путь: `openclaw configure --section web`,
который сохраняет `tools.web.search.apiKey`. Документация: [Web tools](/tools/web).
</Tip>

## Быстрый старт vs Расширенный режим

Мастер начинается с выбора **QuickStart** (по умолчанию) или **Advanced** (полный контроль).

<Tabs>
  <Tab title="QuickStart (defaults)">
    - Локальный Gateway (шлюз) (loopback)
    - Рабочее пространство по умолчанию (или существующее рабочее пространство)
    - Порт Gateway (шлюза) **18789**
    - Аутентификация Gateway (шлюза): **Token** (автогенерируется, даже для loopback)
    - Экспозиция через Tailscale: **Off**
    - Личные сообщения Telegram + WhatsApp по умолчанию используют **allowlist** (вас попросят указать номер телефона)
  </Tab>
  <Tab title="Advanced (full control)">
    - Открывает каждый шаг (режим, рабочее пространство, Gateway (шлюз), каналы, демон, Skills).
  </Tab>
</Tabs>

## Что настраивает мастер

**Локальный режим (по умолчанию)** проводит вас через следующие шаги:

1. **Модель/Аутентификация** — ключ API Anthropic (рекомендуется), OAuth, OpenAI или другие провайдеры. Выбор модели по умолчанию.
2. **Рабочее пространство** — расположение файлов агента (по умолчанию `~/.openclaw/workspace`). Создаёт начальные bootstrap‑файлы.
3. **Gateway (шлюз)** — порт, адрес привязки, режим аутентификации, экспозиция через Tailscale.
4. **Каналы** — WhatsApp, Telegram, Discord, Google Chat, Mattermost, Signal, BlueBubbles или iMessage.
5. **Демон** — устанавливает LaunchAgent (macOS) или пользовательский unit systemd (Linux/WSL2).
6. **Проверка работоспособности** — запускает Gateway (шлюз) и проверяет, что он работает.
7. **Skills** — устанавливает рекомендуемые Skills и необязательные зависимости.

<Note>
Повторный запуск мастера **не** удаляет ничего, если вы явно не выберете **Reset**
(или не передадите `--reset`).
Если конфигурация некорректна или содержит устаревшие ключи, мастер предложит сначала запустить `openclaw doctor`.
</Note>

**Удалённый режим** настраивает только локальный клиент для подключения к Gateway (шлюзу) в другом месте.
Он **не** устанавливает и не изменяет ничего на удалённом хосте.

## Добавить ещё одного агента

Используйте `openclaw agents add <name>`, чтобы создать отдельного агента с собственным рабочим пространством,
сеансами и профилями аутентификации. Запуск без `--workspace` запускает мастер.

Что он настраивает:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

Примечания:

- Рабочие пространства по умолчанию следуют шаблону `~/.openclaw/workspace-<agentId>`.
- Добавьте `bindings`, чтобы маршрутизировать входящие сообщения (мастер может сделать это).
- Флаги неинтерактивного режима: `--model`, `--agent-dir`, `--bind`, `--non-interactive`.
