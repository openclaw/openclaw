# Инициализация агента

Инициализация — это ритуал **первого запуска**, который подготавливает рабочее
пространство агента и собирает сведения об идентификации. Он выполняется после
онбординга, когда агент запускается в первый раз.

## Что делает bootstrapping

При первом запуске агента OpenClaw инициализирует рабочее пространство (по
умолчанию `~/.openclaw/workspace`):

- Заполняет `AGENTS.md`, `BOOTSTRAP.md`, `IDENTITY.md`, `USER.md`.
- Выполняет короткий ритуал вопросов и ответов (по одному вопросу за раз).
- Записывает идентификацию и предпочтения в `IDENTITY.md`, `USER.md`, `SOUL.md`.
- Удаляет `BOOTSTRAP.md` по завершении, чтобы инициализация выполнялась только один раз.

## Где выполняется

Инициализация всегда выполняется на **хосте шлюза Gateway**. Если приложение для
macOS подключается к удалённому Gateway, рабочее пространство и файлы
инициализации находятся на этой удалённой машине.

<Note>
Когда Gateway запущен на другой машине, редактируйте файлы рабочего
пространства на хосте шлюза Gateway (например, `user@gateway-host:~/.openclaw/workspace`).
</Note>
