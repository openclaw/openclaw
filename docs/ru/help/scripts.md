# Скрипты

Каталог `scripts/` содержит вспомогательные скрипты для локальных рабочих процессов и операционных задач.
Используйте их, когда задача явно привязана к скрипту; в противном случае отдавайте предпочтение CLI.

## Соглашения

- Скрипты являются **необязательными**, если на них нет ссылок в документации или чек-листах релизов.
- Предпочитайте интерфейсы CLI, когда они существуют (пример: мониторинг аутентификации использует `openclaw models status --check`).
- Считайте скрипты специфичными для хоста; перед запуском на новой машине ознакомьтесь с их содержимым.

## При добавлении скриптов

- Делайте скрипты узкоспециализированными и документированными.
- Добавляйте краткую запись в соответствующий документ (или создавайте его, если он отсутствует).
