# Хук SOUL Evil

Хук SOUL Evil заменяет **инъецированное** содержимое `SOUL.md` на `SOUL_EVIL.md` во время
окна очистки или по случайной вероятности. Он **не** изменяет файлы на диске.

## Как это работает

Когда выполняется `agent:bootstrap`, хук может заменить содержимое `SOUL.md` в памяти
до сборки системного промпта. Если `SOUL_EVIL.md` отсутствует или пуст,
OpenClaw записывает предупреждение и сохраняет обычный `SOUL.md`.

Запуски суб-агентов **не** включают `SOUL.md` в свои bootstrap-файлы, поэтому этот хук
не влияет на суб-агентов.

## Включить

```bash
openclaw hooks enable soul-evil
```

Затем задайте конфиг:

```json
{
  "hooks": {
    "internal": {
      "enabled": true,
      "entries": {
        "soul-evil": {
          "enabled": true,
          "file": "SOUL_EVIL.md",
          "chance": 0.1,
          "purge": { "at": "21:00", "duration": "15m" }
        }
      }
    }
  }
}
```

Создайте `SOUL_EVIL.md` в корне рабочего пространства агента (рядом с `SOUL.md`).

## Параметры

- `file` (string): альтернативное имя файла SOUL (по умолчанию: `SOUL_EVIL.md`)
- `chance` (number 0–1): случайная вероятность на запуск использовать `SOUL_EVIL.md`
- `purge.at` (HH:mm): начало ежедневной очистки (24-часовой формат)
- `purge.duration` (duration): длительность окна (например, `30s`, `10m`, `1h`)

**Приоритет:** окно очистки имеет приоритет над вероятностью.

**Часовой пояс:** используется `agents.defaults.userTimezone`, если задан; иначе — часовой пояс хоста.

## Примечания

- Файлы на диске не записываются и не изменяются.
- Если `SOUL.md` отсутствует в списке bootstrap, хук ничего не делает.
