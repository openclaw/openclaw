# Конфигурация моделей (исследование)

Этот документ фиксирует **идеи** для будущей конфигурации моделей. Это не
спецификация для поставки. Текущее поведение см. здесь:

- [Модели](/concepts/models)
- [Отказоустойчивость моделей](/concepts/model-failover)
- [OAuth + профили](/concepts/oauth)

## Мотивация

Операторам требуется:

- Несколько профилей аутентификации на провайдера (личный vs рабочий).
- Простой выбор `/model` с предсказуемыми откатами.
- Чёткое разделение между текстовыми моделями и моделями с поддержкой изображений.

## Возможное направление (в общих чертах)

- Сохранить простой выбор моделей: `provider/model` с необязательными алиасами.
- Разрешить провайдерам иметь несколько профилей аутентификации с явным порядком.
- Использовать глобальный список отката, чтобы все сеансы переключались согласованно.
- Переопределять маршрутизацию изображений только при явной конфигурации.

- Как UI должен отображать выбор профиля для сеанса?
