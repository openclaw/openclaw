# Вход в браузер + публикации в X/Twitter

## Ручной вход (рекомендуется)

Когда сайт требует входа, **войдите вручную** в браузерном профиле **хоста** (браузер OpenClaw).

**Не** передавайте модели свои учётные данные. Автоматизированные входы часто срабатывают как антибот‑защита и могут привести к блокировке аккаунта.

Назад к основной документации по браузеру: [Browser](/tools/browser).

## Какой профиль Chrome используется?

OpenClaw управляет **выделенным профилем Chrome** (с именем `openclaw`, интерфейс с оранжевым оттенком). Он отделён от вашего повседневного браузерного профиля.

Два простых способа получить к нему доступ:

1. **Попросите агента открыть браузер**, затем войдите самостоятельно.
2. **Откройте его через CLI**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

Если у вас несколько профилей, передайте `--browser-profile <name>` (по умолчанию используется `openclaw`).

## X/Twitter: рекомендуемый поток работы

- **Чтение/поиск/треды:** используйте браузер **хоста** (ручной вход).
- **Публикация обновлений:** используйте браузер **хоста** (ручной вход).

## Sandboxing + доступ к браузеру хоста

Изолированные (sandboxed) браузерные сеансы **чаще** вызывают срабатывание обнаружения ботов. Для X/Twitter (и других строгих сайтов) предпочтителен браузер **хоста**.

Если агент работает в sandbox, инструмент браузера по умолчанию использует sandbox. Чтобы разрешить управление хостом:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

Затем укажите браузер хоста:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

Или отключите sandboxing для агента, который публикует обновления.
