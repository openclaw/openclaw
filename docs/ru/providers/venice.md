# Venice AI (выделенный вариант Venice)

**Venice** — это наш рекомендуемый вариант настройки Venice для приватной инференции с возможностью анонимизированного доступа к проприетарным моделям.

Venice AI предоставляет ориентированную на приватность AI‑инференцию с поддержкой нецензурируемых моделей и доступом к ведущим проприетарным моделям через анонимизированный прокси. Вся инференция по умолчанию приватна — без обучения на ваших данных и без логирования.

## Зачем Venice в OpenClaw

- **Приватная инференция** для open‑source моделей (без логирования).
- **Нецензурируемые модели**, когда это необходимо.
- **Анонимизированный доступ** к проприетарным моделям (Opus/GPT/Gemini), когда важнее качество.
- Совместимые с OpenAI `/v1` эндпоинты.

## Режимы приватности

Venice предлагает два уровня приватности — понимание различий важно для выбора модели:

| Режим          | Описание                                                                                                                                                                     | Модели                                                         |
| -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------- |
| **Private**    | Полностью приватно. Запросы и ответы **никогда не сохраняются и не логируются**. Эфемерно.                                   | Llama, Qwen, DeepSeek, Venice Uncensored и др. |
| **Anonymized** | Проксирование через Venice с удалением метаданных. Базовый провайдер (OpenAI, Anthropic) видит анонимизированные запросы. | Claude, GPT, Gemini, Grok, Kimi, MiniMax                       |

## Возможности

- **Ориентация на приватность**: выбор между режимами «private» (полностью приватно) и «anonymized» (через прокси)
- **Нецензурируемые модели**: доступ к моделям без контентных ограничений
- **Доступ к ведущим моделям**: Claude, GPT-5.2, Gemini, Grok через анонимизированный прокси Venice
- **API, совместимый с OpenAI**: стандартные `/v1` эндпоинты для простой интеграции
- **Стриминг**: ✅ поддерживается на всех моделях
- **Вызов функций**: ✅ поддерживается на отдельных моделях (проверьте возможности модели)
- **Vision**: ✅ поддерживается на моделях с возможностями зрения
- **Без жёстких лимитов**: может применяться fair‑use‑троттлинг при экстремальном использовании

## Настройка

### 1. Получите API‑ключ

1. Зарегистрируйтесь на [venice.ai](https://venice.ai)
2. Перейдите в **Settings → API Keys → Create new key**
3. Скопируйте ваш API‑ключ (формат: `vapi_xxxxxxxxxxxx`)

### 2) Настройте OpenClaw

**Вариант A: Переменная окружения**

```bash
export VENICE_API_KEY="vapi_xxxxxxxxxxxx"
```

**Вариант B: Интерактивная настройка (рекомендуется)**

```bash
openclaw onboard --auth-choice venice-api-key
```

Это выполнит:

1. Запрос вашего API‑ключа (или использование существующего `VENICE_API_KEY`)
2. Отображение всех доступных моделей Venice
3. Выбор модели по умолчанию
4. Автоматическую настройку провайдера

**Вариант C: Неинтерактивный режим**

```bash
openclaw onboard --non-interactive \
  --auth-choice venice-api-key \
  --venice-api-key "vapi_xxxxxxxxxxxx"
```

### 3. Проверьте настройку

```bash
openclaw chat --model venice/llama-3.3-70b "Hello, are you working?"
```

## Выбор модели

После настройки OpenClaw отображает все доступные модели Venice. Выбирайте в зависимости от задач:

- **По умолчанию (наш выбор)**: `venice/llama-3.3-70b` — приватно, сбалансированная производительность.
- **Лучшее общее качество**: `venice/claude-opus-45` для сложных задач (Opus остаётся самым сильным).
- **Приватность**: выбирайте модели «private» для полностью приватной инференции.
- **Возможности**: выбирайте модели «anonymized» для доступа к Claude, GPT, Gemini через прокси Venice.

Изменить модель по умолчанию можно в любой момент:

```bash
openclaw models set venice/claude-opus-45
openclaw models set venice/llama-3.3-70b
```

Список всех доступных моделей:

```bash
openclaw models list | grep venice
```

## Настройка через `openclaw configure`

1. Запустите `openclaw configure`
2. Выберите **Model/auth**
3. Выберите **Venice AI**

## Обнаружение моделей

OpenClaw автоматически обнаруживает модели из API Venice, когда установлен `VENICE_API_KEY`. Если API недоступен, используется статический каталог.

Эндпоинт `/models` является публичным (аутентификация не требуется для получения списка), но для инференции нужен действительный API‑ключ.

## Стриминг и поддержка инструментов

| Функция                | Поддержка                                                                            |
| ---------------------- | ------------------------------------------------------------------------------------ |
| **Стриминг**           | ✅ Все модели                                                                         |
| **Вызов функций**      | ✅ Большинство моделей (проверьте `supportsFunctionCalling` в API) |
| **Vision/Изображения** | ✅ Модели с пометкой «Vision»                                                         |
| **JSON‑режим**         | ✅ Поддерживается через `response_format`                                             |

## Цены

Venice использует систему кредитов. Актуальные тарифы смотрите на [venice.ai/pricing](https://venice.ai/pricing):

- **Private‑модели**: как правило, дешевле
- **Anonymized‑модели**: сопоставимы с прямым API + небольшая комиссия Venice

## Сравнение: Venice vs прямой API

| Аспект          | Venice (Anonymized) | Прямой API                 |
| --------------- | -------------------------------------- | -------------------------- |
| **Приватность** | Метаданные удалены, анонимно           | Привязка к вашему аккаунту |
| **Задержка**    | +10–50 мс (прокси)  | Прямое подключение         |
| **Возможности** | Большинство возможностей               | Полные возможности         |
| **Биллинг**     | Кредиты Venice                         | Биллинг провайдера         |

## Примеры использования

```bash
# Use default private model
openclaw chat --model venice/llama-3.3-70b

# Use Claude via Venice (anonymized)
openclaw chat --model venice/claude-opus-45

# Use uncensored model
openclaw chat --model venice/venice-uncensored

openclaw chat --model venice/qwen3-vl-235b-a22b

# Use coding model
openclaw chat --model venice/qwen3-coder-480b-a35b-instruct
```

## Устранение неполадок

### API‑ключ не распознан

```bash
echo $VENICE_API_KEY
openclaw models list | grep venice
```

Убедитесь, что ключ начинается с `vapi_`.

### Модель недоступна

Каталог моделей Venice обновляется динамически. Запустите `openclaw models list`, чтобы увидеть доступные в данный момент модели. Некоторые модели могут быть временно недоступны.

### Проблемы с подключением

API Venice доступен по адресу `https://api.venice.ai/api/v1`. Убедитесь, что ваша сеть разрешает HTTPS‑подключения.

```json5
  agents: { defaults: { model: { primary: "venice/llama-3.3-70b" } } },

    mode: "merge",
    providers: {
      venice: {
        baseUrl: "https://api.venice.ai/api/v1",
        apiKey: "${VENICE_API_KEY}",
        api: "openai-completions",
        models: [
          {
            id: "llama-3.3-70b",
            name: "Llama 3.3 70B",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 131072,
            maxTokens: 8192,
          },
        ],
      },
    },
  },
}
```
