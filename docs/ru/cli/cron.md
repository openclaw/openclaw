# `openclaw cron`

Управление заданиями cron для планировщика Gateway (шлюз).

Связанное:

- Задания cron: [Задания cron](/automation/cron-jobs)

Совет: выполните `openclaw cron --help` для просмотра полного набора команд.

Примечание: изолированные задания `cron add` по умолчанию используют доставку `--announce`. Используйте `--no-deliver`, чтобы оставить
вывод внутренним. `--deliver` остаётся устаревшим псевдонимом для `--announce`.

Примечание: одноразовые задания (`--at`) по умолчанию удаляются после успешного выполнения. Используйте `--keep-after-run`, чтобы сохранить их.

Примечание: повторяющиеся задания теперь используют экспоненциальную задержку повторных попыток после последовательных ошибок (30с → 1м → 5м → 15м → 60м), затем возвращаются к обычному расписанию после следующего успешного запуска.

## Общие правки

Обновить параметры доставки без изменения сообщения:

```bash
openclaw cron edit <job-id> --announce --channel telegram --to "123456789"
```

Отключить доставку для изолированного задания:

```bash
openclaw cron edit <job-id> --no-deliver
```

Объявить в конкретный канал:

```bash
openclaw cron edit <job-id> --announce --channel slack --to "channel:C1234567890"
```
