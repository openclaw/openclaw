# معالج التهيئة الأولية (CLI)

يُعد معالج التهيئة الأولية الطريقة **الموصى بها** لإعداد OpenClaw على macOS،
أو Linux، أو Windows (عبر WSL2؛ موصى به بشدة).
يقوم بتهيئة Gateway محلية أو اتصال Gateway عن بُعد، إضافةً إلى القنوات وSkills
وإعدادات مساحة العمل الافتراضية ضمن مسار إرشادي واحد.

```bash
openclaw onboard
```

<Info>
أسرع محادثة أولى: افتح واجهة التحكم (لا حاجة لإعداد القنوات). شغّل
`openclaw dashboard` وتحادث في المتصفح. التوثيق: [لوحة التحكم](/web/dashboard).
</Info>

لإعادة التهيئة لاحقًا:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` لا يعني الوضع غير التفاعلي. للبرامج النصية، استخدم `--non-interactive`.
</Note>

<Tip>
موصى به: إعداد مفتاح API لـ Brave Search لكي يتمكن الوكيل من استخدام `web_search`
(`web_fetch` يعمل بدون مفتاح). أسهل مسار: `openclaw configure --section web`
الذي يخزّن `tools.web.search.apiKey`. التوثيق: [أدوات الويب](/tools/web).
</Tip>

## البدء السريع مقابل المتقدم

يبدأ المعالج بخيار **البدء السريع** (الإعدادات الافتراضية) مقابل **المتقدم** (تحكم كامل).

<Tabs>
  <Tab title="QuickStart (defaults)">
    - Gateway محلية (loopback)
    - مساحة عمل افتراضية (أو مساحة عمل موجودة)
    - منفذ Gateway **18789**
    - مصادقة Gateway **Token** (يتم إنشاؤه تلقائيًا، حتى على loopback)
    - تعريض Tailscale **مُعطّل**
    - الرسائل الخاصة عبر Telegram وWhatsApp افتراضيًا على **allowlist** (سيُطلب منك رقم هاتفك)
  </Tab>
  <Tab title="Advanced (full control)">
    - يعرض كل خطوة (الوضع، مساحة العمل، Gateway، القنوات، الخادم، Skills).
  </Tab>
</Tabs>

## ما هي تشكيلات المعالج

**الوضع المحلي (الافتراضي)** يرشدك عبر الخطوات التالية:

1. **النموذج/المصادقة** — مفتاح API لـ Anthropic (موصى به)، OAuth، OpenAI، أو موفّرين آخرين. اختر نموذجًا افتراضيًا.
2. **مساحة العمل** — موقع ملفات الوكيل (الافتراضي `~/.openclaw/workspace`). يقوم بتهيئة ملفات البدء.
3. **Gateway** — المنفذ، عنوان الربط، وضع المصادقة، وتعريض Tailscale.
4. **القنوات** — WhatsApp، Telegram، Discord، Google Chat، Mattermost، Signal، BlueBubbles، أو iMessage.
5. **الخادم** — تثبيت LaunchAgent (على macOS) أو وحدة systemd للمستخدم (على Linux/WSL2).
6. **فحص السلامة** — تشغيل Gateway والتحقق من أنها تعمل.
7. **Skills** — تثبيت Skills الموصى بها والاعتمادات الاختيارية.

<Note>
إعادة تشغيل المعالج **لا** تمسح أي شيء ما لم تختر صراحةً **Reset** (أو تمرّر `--reset`).
إذا كانت التهيئة غير صالحة أو تحتوي على مفاتيح قديمة، سيطلب منك المعالج تشغيل `openclaw doctor` أولًا.
</Note>

**الوضع البعيد** يقوم فقط بتهيئة العميل المحلي للاتصال بـ Gateway موجودة في مكان آخر.
ولا يقوم **بتثبيت** أو **تغيير** أي شيء على المضيف البعيد.

## إضافة وكيل آخر

استخدم `openclaw agents add <name>` لإنشاء وكيل منفصل بمساحة عمل خاصة به،
وجلسات، وملفات تعريف المصادقة. التشغيل بدون `--workspace` سيُطلق المعالج.

ما الذي يقوم بإعداده:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

ملاحظات:

- تتبع مساحات العمل الافتراضية النمط `~/.openclaw/workspace-<agentId>`.
- أضف `bindings` لتوجيه الرسائل الواردة (يمكن للمعالج القيام بذلك).
- أعلام الوضع غير التفاعلي: `--model`، `--agent-dir`، `--bind`، `--non-interactive`.
