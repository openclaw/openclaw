# تهيئة النموذج (استكشاف)

يوثّق هذا المستند **أفكارًا** لتهيئة النماذج مستقبلًا. وهو ليس
مواصفة جاهزة للشحن. للسلوك الحالي، راجع:

- [النماذج](/concepts/models)
- [التجاوز عند فشل النموذج](/concepts/model-failover)
- [OAuth + الملفات التعريفية](/concepts/oauth)

## الدوافع

يرغب المشغّلون في:

- وجود عدة ملفات تعريف مصادقة لكل موفّر (شخصي مقابل عمل).
- اختيار `/model` بسيط مع تراجعات متوقعة.
- فصل واضح بين نماذج النصوص والنماذج القادرة على معالجة الصور.

## اتجاه محتمل (على مستوى عالٍ)

- الإبقاء على اختيار النموذج بسيطًا: `provider/model` مع أسماء مستعارة اختيارية.
- السماح للموفّرين بامتلاك عدة ملفات تعريف مصادقة، مع ترتيب صريح.
- استخدام قائمة تراجع عامة بحيث تتراجع جميع الجلسات بشكل متّسق.
- عدم تجاوز توجيه الصور إلا عند التهيئة الصريحة.

## أسئلة مفتوحة

- هل ينبغي أن يكون دوران الملف الشخصي لكل موفر أو لكل نموذج؟
- كيف ينبغي لواجهة المستخدم إظهار اختيار الملف التعريفي لجلسة ما؟
- ما هو مسار الترحيل الأكثر أمانًا من مفاتيح التهيئة القديمة؟
