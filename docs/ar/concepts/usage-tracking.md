# تتبّع الاستخدام

## ما هو

- يسحب استخدام/حصص الموفّر مباشرةً من نقاط نهاية الاستخدام الخاصة بهم.
- لا توجد تكاليف مُقدَّرة؛ فقط النوافذ التي يبلّغ عنها الموفّر.

## أين يظهر

- `/status` في الدردشات: بطاقة حالة غنيّة بالرموز التعبيرية تتضمن رموز الجلسة + التكلفة المُقدَّرة (مفتاح API فقط). يظهر استخدام الموفّر **لموفّر النموذج الحالي** عند توفره.
- `/usage off|tokens|full` في الدردشات: تذييل استخدام لكل استجابة (يعرض OAuth الرموز فقط).
- `/usage cost` في الدردشات: ملخص تكلفة محلي مُجمَّع من سجلات جلسات OpenClaw.
- CLI: `openclaw status --usage` يطبع تفصيلاً كاملاً لكل موفّر.
- CLI: `openclaw channels list` يطبع لقطة الاستخدام نفسها إلى جانب تهيئة الموفّر (استخدم `--no-usage` للتخطي).
- شريط قوائم macOS: قسم «الاستخدام» ضمن «السياق» (فقط عند توفره).

## الموفّرون + بيانات الاعتماد

- **Anthropic (Claude)**: رموز OAuth في ملفات تعريف المصادقة.
- **GitHub Copilot**: رموز OAuth في ملفات تعريف المصادقة.
- **Gemini CLI**: رموز OAuth في ملفات تعريف المصادقة.
- **Antigravity**: رموز OAuth في ملفات تعريف المصادقة.
- **OpenAI Codex**: رموز OAuth في ملفات تعريف المصادقة (يُستخدم accountId عند توفره).
- **MiniMax**: مفتاح API (مفتاح خطة البرمجة؛ `MINIMAX_CODE_PLAN_KEY` أو `MINIMAX_API_KEY`)؛ يستخدم نافذة خطة البرمجة لمدة 5 ساعات.
- **z.ai**: مفتاح API عبر env/التهيئة/مخزن المصادقة.

يتم إخفاء الاستخدام إذا لم تتوفر بيانات اعتماد OAuth/API مطابقة.
