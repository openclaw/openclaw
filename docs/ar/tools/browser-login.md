# تسجيل دخول المتصفح + النشر على X/Twitter

## تسجيل الدخول اليدوي (موصى به)

عندما يتطلّب موقع ما تسجيل الدخول، **سجّل الدخول يدويًا** في متصفح **المضيف** (متصفح OpenClaw).

**لا** تمنح النموذج بيانات اعتمادك. غالبًا ما تُفعِّل تسجيلات الدخول المؤتمتة آليات مكافحة الروبوتات وقد تؤدي إلى قفل الحساب.

العودة إلى توثيق المتصفح الرئيسي: [المتصفح](/tools/browser).

## أي ملف تعريف Chrome يُستخدم؟

يتحكّم OpenClaw في **ملف تعريف Chrome مخصّص** (باسم `openclaw`، بواجهة مائلة للون البرتقالي). وهو منفصل عن ملف تعريف المتصفح اليومي لديك.

طريقتان سهلتان للوصول إليه:

1. **اطلب من الوكيل فتح المتصفح** ثم سجّل الدخول بنفسك.
2. **افتحه عبر CLI**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

إذا كان لديك عدة ملفات تعريف، مرّر `--browser-profile <name>` (الإعداد الافتراضي هو `openclaw`).

## X/Twitter: التدفق الموصى به

- **القراءة/البحث/السلاسل:** استخدم متصفح **المضيف** (تسجيل دخول يدوي).
- **نشر التحديثات:** استخدم متصفح **المضيف** (تسجيل دخول يدوي).

## sandboxing + الوصول إلى متصفح المضيف

تكون جلسات المتصفح داخل sandbox **أكثر عرضة** لتفعيل كشف الروبوتات. بالنسبة إلى X/Twitter (وغيره من المواقع الصارمة)، يُفضَّل استخدام متصفح **المضيف**.

إذا كان الوكيل يعمل داخل sandbox، فإن أداة المتصفح تستخدم sandbox افتراضيًا. للسماح بالتحكّم في المضيف:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

ثم استهدف متصفح المضيف:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

أو عطّل sandboxing للوكيل الذي ينشر التحديثات.
