# استكشاف أخطاء القنوات

استخدم هذه الصفحة عندما تتصل القناة لكن يكون السلوك غير صحيح.

## سُلَّم الأوامر

شغّل ما يلي بالترتيب أولًا:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

خط أساس سليم:

- `Runtime: running`
- `RPC probe: ok`
- يُظهر فحص القناة أنها متصلة/جاهزة

## WhatsApp

### بصمات فشل WhatsApp

| العَرَض                                   | أسرع فحص                                             | الإصلاح                                                          |
| ----------------------------------------- | ---------------------------------------------------- | ---------------------------------------------------------------- |
| متصل لكن لا توجد ردود على الرسائل الخاصة  | `openclaw pairing list whatsapp`                     | الموافقة على المرسِل أو تبديل سياسة الرسائل الخاصة/قائمة السماح. |
| تجاهل رسائل المجموعات                     | تحقّق من `requireMention` + أنماط الذِّكر في التهيئة | ذِكر البوت أو تخفيف سياسة الذِّكر لتلك المجموعة.                 |
| انقطاعات عشوائية/حلقات إعادة تسجيل الدخول | `openclaw channels status --probe` + السجلات         | أعد تسجيل الدخول وتحقق من سلامة دليل بيانات الاعتماد.            |

استكشاف الأخطاء الكامل: [/channels/whatsapp#troubleshooting-quick](/channels/whatsapp#troubleshooting-quick)
