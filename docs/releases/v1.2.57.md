# Zorg MemoryDB v1.2.57

## LAN chat and full public rule seed

This release fixes two install gaps in the public Zorg MemoryDB overlay.

## Changed

- `zorg/db/public_canonical_rules_update_2026_06_02.sql` is now the single
  packaged rule file for the add-on and seeds the full public-safe canonical
  rule set: 104 active rules.
- The public canonical rule SQL now verifies the 104-rule count after applying
  the seed and fails if that full public set is not present.
- `scripts/first_run.sh` and `scripts/upgrade_existing_openclaw.sh` apply the
  public canonical rule SQL after the base schema.
- `scripts/upgrade_existing_openclaw.sh` now installs the built-in LAN command
  chat automatically after DB-memory verification. Set
  `ZORG_SKIP_LAN_CHAT_INSTALL=1` only when intentionally skipping that surface.

## Verification

- Confirmed the packaged add-on rule surface now has one canonical rule file
  with 104 unique public-safe rules.
- Regenerated the public-safe SQL seed from the public-scoped source rows and
  folded in the package-only DB/runtime rules.
- Rechecked GitHub documentation for pull request checks and Actions log
  inspection before preparing this repository update.
