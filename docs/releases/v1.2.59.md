# Zorg MemoryDB v1.2.59

## PostgreSQL scheduler and ANN schema package update

This release publishes the public-safe structure for the current Zorg MemoryDB
PostgreSQL add-ons without shipping private live memory rows or scheduled job
payloads.

## Changed

- Added packaged schema support for pgvector-backed ANN recall tables, query
  embedding cache rows, HNSW cosine indexes, and ANN recall helper functions.
- Added DB-owned LLM scheduler tables and queue functions for schedule import,
  enqueue, claim, finish, and `pg_notify` worker wakeups.
- Added installer support for PostgreSQL pgvector and pg_cron packages on local
  PostgreSQL installs when root or passwordless sudo is available.
- Documented pg_cron requirements: `shared_preload_libraries`,
  `cron.database_name`, `cron.timezone`, a PostgreSQL restart when first
  enabled, and extension creation in the target MemoryDB database.
- Documented that PostgreSQL only owns scheduling, queueing, claiming, and
  bounded result recording; LLM execution remains in the OpenClaw runtime worker.

## Public safety

- The public package includes only structure, functions, indexes, and generic
  installation guidance.
- It does not include live scheduled job rows, job prompt payloads, private
  transcripts, contacts, credentials, backup dumps, or operator-only context.
