# Zorg MemoryDB v1.2.58

## Canonical public rule update

This release publishes the current public-safe MemoryDB rule structure on top of
the latest OpenClaw/Zorg main branch.

## Changed

- Added `zorg/db/public_canonical_rules_update_2026_06_02.sql`.
- Added `zorg_logic_rule_dynamic_weights` and
  `zorg_logic_rule_dynamic_ranking_v` to the public schema.
- Seeded public-safe rules that make `zorg_logic_rules` the active rule surface.
- Documented that older `zorg_rules` and `zorg_rule_catalog` compatibility
  tables must not remain active recall sources after canonical migration.
- Raised existing operator-visible chat timing dynamic weights through the
  dynamic-weight table without creating replacement timing rules.
- Updated the packaged SQL recall tools to rank rule matches with dynamic
  weights when those weights are present.
- Superseded older private-GitHub DB-dump guidance with the current
  temporary-local-only rollback backup rule.
