# Zorg MemoryDB v1.2.53

## Scripted PostgreSQL memory recovery

This release adds a public-safe recovery script and documents that backups are not useful unless they can be restored through a tested process.

New script:

- `scripts/postgres_memory_recovery.sh`

Supported modes:

- `list` — list local full database backups newest first.
- `drill [backup.sql.gz]` — restore a dump into an isolated temporary database, verify `public.zorg_memory` exists and has rows, then drop the temporary database.
- `restore-active backup.sql.gz` — after explicit approval, run the drill, rename the live database aside as a safety database, restore the selected dump into the live database name, and verify it.

Live restore requires:

```bash
CONFIRM_RESTORE_ACTIVE=YES bash scripts/postgres_memory_recovery.sh restore-active /path/to/zorgdb-YYYY-MM-DD_HHMMSS.sql.gz
```

The backup manifest now points to the scripted drill and gated live restore process instead of a one-line manual restore hint.
