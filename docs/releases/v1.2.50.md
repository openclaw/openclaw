# Zorg MemoryDB v1.2.50

## 100% memory working definition

This release documents the operator correction that "memory working" must mean end-to-end ingestion and recall, not partial database health.

Required behavior:

- PostgreSQL connectivity alone is not enough to claim memory is working.
- Recent Telegram/chat turns and assistant replies must be written to PostgreSQL memory.
- Generated instructions, code/documentation changes, GitHub publication records, installer or upgrade decisions, meaningful process logs, and other durable operational facts must be written to PostgreSQL memory or explicitly reported as blocked/degraded.
- Retired markdown memory surfaces such as `memory/` daily notes, project notes, JSON state, or ad hoc logging files must not be recreated as durable memory.
- Memory health checks must verify recent rows, ingestion bridge status, natural-language recall of new records, and absence of retired markdown memory output.

This is an operating rule and a verification requirement for MemoryDB installs and repairs.
