# 配置示例

以下示例与当前配置架构保持一致。有关详尽的参考和每个字段的说明，请参阅 [配置](/gateway/configuration)。

## 快速开始

### 绝对最小值

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: { whatsapp: { allowFrom: ["+15555550123"] } },
}
```

保存到 `~/.openclaw/openclaw.json`，您可以从该号码向机器人发送 DM。

### 推荐的初学者配置

```json5
{
  identity: {
    name: "Clawd",
    theme: "helpful assistant",
    emoji: "🦞",
  },
  agent: {
    workspace: "~/.openclaw/workspace",
    model: { primary: "anthropic/claude-sonnet-4-6" },
  },
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"],
      groups: { "*": { requireMention: true } },
    },
  },
}
```

## 常见模式

### 共享技能基线与一个覆盖

```json5
{
  agents: {
    defaults: {
      workspace: "~/.openclaw/workspace",
      skills: ["github", "weather"],
    },
    list: [
      { id: "main", default: true },
      { id: "docs", workspace: "~/.openclaw/workspace-docs", skills: ["docs-search"] },
    ],
  },
}
```

- `agents.defaults.skills` 是共享基线。
- `agents.list[].skills` 为一个代理替换该基线。
- 当代理不应看到任何技能时使用 `skills: []`。

### 多平台设置

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: {
    whatsapp: { allowFrom: ["+15555550123"] },
    telegram: {
      enabled: true,
      botToken: "YOUR_TOKEN",
      allowFrom: ["123456789"],
    },
    discord: {
      enabled: true,
      token: "YOUR_TOKEN",
      dm: { allowFrom: ["123456789012345678"] },
    },
  },
}
```

### 安全 DM 模式（共享收件箱 / 多用户 DMs）

如果多人可以向您的机器人发送 DM（`allowFrom` 中有多个条目、多个人的配对批准或 `dmPolicy: "open"`），请启用 **安全 DM 模式**，以便不同发送者的 DM 默认不会共享一个上下文：

```json5
{
  // 安全 DM 模式（推荐用于多用户或敏感 DM 代理）
  session: { dmScope: "per-channel-peer" },

  channels: {
    // 示例：WhatsApp 多用户收件箱
    whatsapp: {
      dmPolicy: "allowlist",
      allowFrom: ["+15555550123", "+15555550124"],
    },

    // 示例：Discord 多用户收件箱
    discord: {
      enabled: true,
      token: "YOUR_DISCORD_BOT_TOKEN",
      dm: { enabled: true, allowFrom: ["123456789012345678", "987654321098765432"] },
    },
  },
}
```

对于 Discord/Slack/Google Chat/Microsoft Teams/Mattermost/IRC，发送者授权默认优先使用 ID。
只有当您明确接受该风险时，才启用每个频道的 `dangerouslyAllowNameMatching: true` 进行直接可变的名称/电子邮件/昵称匹配。

### Anthropic API 密钥 + MiniMax 回退

```json5
{
  auth: {
    profiles: {
      "anthropic:api": {
        provider: "anthropic",
        mode: "api_key",
      },
    },
    order: {
      anthropic: ["anthropic:api"],
    },
  },
  models: {
    providers: {
      minimax: {
        baseUrl: "https://api.minimax.io/anthropic",
        api: "anthropic-messages",
        apiKey: "${MINIMAX_API_KEY}",
      },
    },
  },
  agent: {
    workspace: "~/.openclaw/workspace",
    model: {
      primary: "anthropic/claude-opus-4-6",
      fallbacks: ["minimax/MiniMax-M2.7"],
    },
  },
}
```

## 提示

- 如果您设置 `dmPolicy: "open"`，匹配的 `allowFrom` 列表必须包含 `"*"`。
- 提供者 ID 不同（电话号码、用户 ID、频道 ID）。使用提供者文档确认格式。
- 稍后添加的可选部分：`web`、`browser`、`ui`、`discovery`、`canvasHost`、`talk`、`signal`、`imessage`。
- 有关更深入的设置说明，请参阅 [提供者](/providers) 和 [故障排除](/gateway/troubleshooting)。
