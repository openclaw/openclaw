# Task: Fix ENOENT from ffprobe crashing agent WebSocket (Issue #40382)

## Problem

When ffprobe is not installed and an audio/voice message is received via Telegram,
the unhandled ENOENT error kills the agent WebSocket with close code 1006 instead of
returning a human-readable error to the conversation.

## Key files

- `src/media-understanding/audio-transcription-runner.ts` — runs transcription
- `src/media-understanding/audio-preflight.ts` — already has try/catch (good pattern)
- `src/media/audio.ts` — may call ffprobe directly

## What the fix should do

1. Find where ffprobe/ffmpeg ENOENT propagates unhandled
2. Catch ENOENT errors from ffprobe/ffmpeg spawn
3. Return a graceful error like: "Cannot process audio: ffprobe is not installed. Install ffmpeg to enable audio transcription."
4. Do NOT crash the WebSocket — log internally and continue
