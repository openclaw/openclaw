## PR #55778 - fix(feishu): clear session bindings on /new command

### Issue #55254 (Beta Release Blocker)
**Problem**: `/new` command creates duplicate `agent:main:main` session that hijacks Feishu message routing

**Root Cause**: 
- `/new` triggers `emitResetCommandHooks()` but does NOT clear session bindings
- Subsequent messages routed via stale bindings to `agent:main:main` instead of correct channel session
- User sees no replies in Feishu (replies go to control console)

**Fix**:
```typescript
// src/auto-reply/reply/commands-core.ts
if (params.action === "new" && params.sessionKey) {
  const bindingService = getSessionBindingService();
  await bindingService.unbind({
    targetSessionKey: params.sessionKey,
    reason: "reset_command_new",
  });
}
```

**Key Design Decisions**:
1. Only clear on `/new`, NOT on `/reset` (preserves session affinity)
2. Guard against undefined sessionKey (Greptile P1 review fix)
3. Fire-and-forget async pattern with try/catch

**Tests Added** (6 tests total):
- `clears session bindings on /new command`
- `does not clear bindings on /reset command`
- `does not clear bindings when sessionKey is undefined`

## PR #55859 - fix(whatsapp): fix DM self-chat echo loop

### Issue #55426
**Problem**: WhatsApp DM self-chat echo loop when using personal number (bot and owner same number)

**Root Cause**:
- `send.ts` returns: `{ messageId, toJid }`
- `monitor.ts` expected: `result.key?.id` ← result has no 'key' property!
- Result: messageId = "" → outbound not recorded → fromMe not filtered → echo loop

**Fix**:
```typescript
// extensions/whatsapp/src/inbound/monitor.ts
const rememberOutboundMessage = (remoteJid: string, result: unknown) => {
  let messageId = "";

  // Support send.ts return format: { messageId, toJid }
  if (typeof result === "object" && result && "messageId" in result) {
    messageId = String((result as { messageId?: string }).messageId ?? "");
  }
  // Fallback to Baileys format: { key: { id } }
  else if (typeof result === "object" && result && "key" in result) {
    messageId = String((result as { key?: { id?: string } }).key?.id ?? "");
  }

  if (!messageId || messageId === "unknown") {
    return;
  }
  rememberRecentOutboundMessage({...});
};
```
