## 🚀 完整部署步骤

### 第一步: 生成部署包

```bash
cd D:\OpenClaw\Develop\openclaw

# 清理旧的缓存
pnpm store prune

# 安装依赖（使用HTTPS而不是SSH）
git config --global url."https://github.com/".insteadOf "git+ssh://git@github.com/"
pnpm install --no-frozen-lockfile

# 构建UI
pnpm ui:build

# 构建Docker镜像
pnpm build:docker

# 生成部署包
pnpm pack --pack-destination D:\OpenClaw\deploy --config.ignore-scripts=true
```

**预期输出**:
```
✓ openclaw-2026.3.14.tgz 已生成
```

---

### 第二步: 部署到运行目录

```bash
cd D:\OpenClaw\deploy

# 解压到运行目录
tar -xf openclaw-2026.3.14.tgz -C openclaw-runtime-next

# 进入包目录
cd openclaw-runtime-next\package

# 安装生产依赖
pnpm install --prod --ignore-scripts

# 复制UI资源
xcopy D:\OpenClaw\Develop\openclaw\dist\control-ui dist\control-ui /E /I /Y
```

**预期结果**:
```
✓ openclaw.mjs 存在
✓ dist/control-ui/ 存在
✓ node_modules/ 已安装
```

---

### 第三步: 启动服务

cd D:\OpenClaw\deploy\openclaw-runtime-next\package

## 🔍 故障排除

### 问题1: 无法访问Web界面

**症状**: 浏览器显示"无法连接"

**解决方案**:
1. 检查服务是否运行
   ```bash
   Get-Process node | Where-Object { $_.CommandLine -match "openclaw" }
   ```

2. 检查端口是否被占用
   ```bash
   Get-NetTCPConnection -LocalPort 18789 -ErrorAction SilentlyContinue
   ```

3. 检查UI资源是否存在
   ```bash
   Test-Path "D:\OpenClaw\deploy\openclaw-runtime-next\package\dist\control-ui\index.html"
   ```

4. 查看日志
   ```bash
   Get-Content "D:\OpenClaw\deploy\gateway.log" -Tail 50
   ```

---

### 问题2: 插件未加载

**症状**: 聊天中无法使用abb_robot工具

**解决方案**:
1. 检查插件配置
   ```json
   {
     "plugins": {
       "entries": {
         "abb-robot-control": {
           "enabled": true
         }
       }
     }
   }
   ```

2. 检查插件文件
   ```bash
   Test-Path "D:\OpenClaw\Develop\openclaw\extensions\abb-robot-control\index.ts"
   ```

3. 重启服务

---

### 问题3: 命令返回错误

**症状**: 命令执行失败

**解决方案**:
1. 检查参数是否正确
2. 查看错误消息
3. 检查日志文件
4. 验证配置文件
