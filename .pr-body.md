## What Problem This Solves

The ACP gateway's `onEvent` callback dispatches gateway events to the translator via `void agent?.handleGatewayEvent(evt)` — a fire-and-forget async call. `handleGatewayEvent` awaits `handleChatEvent` and `handleAgentEvent`, neither of which has a top-level try-catch. If either method throws, the rejection becomes an unhandled rejection.

## Changes

- `src/acp/translator.ts`: wrap `handleGatewayEvent` body in try-catch so all dispatch errors are logged internally via `this.log()` instead of escaping as unhandled rejections.

## Real behavior proof

- **Code path**: `handleGatewayEvent` (translator.ts:356)
- **Environment tested**: Node.js v24 — vitest test suite.
- **Evidence**:

```
Before: handleGatewayEvent has no error handling
  handleChatEvent() / handleAgentEvent() throw → unhandled rejection

After: try-catch wraps entire dispatch body
  handleChatEvent() / handleAgentEvent() throw → caught → this.log(...)
```

Tests: `src/acp/translator.error-kind.test.ts` + `src/acp/server.startup.test.ts` → 15 passed (unchanged).
