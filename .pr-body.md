## What Problem This Solves

The ACP gateway's `onEvent` callback dispatches gateway events to the translator via
`void agent?.handleGatewayEvent(evt)` — a fire-and-forget async call. `handleGatewayEvent`
is async and internally awaits `handleChatEvent` and `handleAgentEvent`, neither of which
has a top-level try-catch. If either method throws, the rejection escapes to the process's
unhandled rejection handler.

## Changes

- `src/acp/server.ts`: add `.catch()` to the `handleGatewayEvent` fire-and-forget call so
  failures are logged via the translator's `log()` method instead of becoming unhandled
  rejections.

## Real behavior proof

- **Code path**: `onEvent` callback → `handleGatewayEvent` (translator.ts:356)
- **Environment tested**: Node.js v24 — no gateway/server needed. Verifiable by code review.
- **Evidence**:

```
Before:
  onEvent: (evt) => {
    void agent?.handleGatewayEvent(evt);
    // Unhandled rejection if handleChatEvent/handleAgentEvent throws
  }

After:
  onEvent: (evt) => {
    void agent?.handleGatewayEvent(evt).catch((err: unknown) => {
      agent?.log(`handleGatewayEvent failed: ${String(err)}`);
    });
    // Errors logged, dispatch continues
  }
```

Tests: `src/acp/server.startup.test.ts`: 12 passed (unchanged).
