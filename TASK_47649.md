# Task: Fix Discord large PDF attachment crashing message listener (Issue #47649)

## Problem

When a user with Discord Nitro uploads a PDF larger than 8MB, OpenClaw's message
listener crashes and stops receiving inbound messages in that channel.

The PDF tool fetches the file from Discord CDN. For large Nitro files, CDN returns
HTTP 404, which crashes/deadlocks the message listener instead of handling gracefully.

## Expected behavior

1. Fetch failure for large attachment should be caught gracefully
2. Log a warning identifying the failed attachment and reason
3. Continue processing the message (with attachment marked as unavailable)
4. Message listener must NOT crash — channel should keep receiving messages

## Key files to investigate

- `extensions/discord/src/bot-message-dispatch.ts` or similar — message handling
- `extensions/discord/src/monitor/message-handler.ts` — inbound message pipeline
- Look for where attachment/media fetch errors propagate unhandled

## Instructions

1. Read key files to trace the crash path
2. Plan — wrap attachment fetch in try/catch, continue on failure
3. Implement with minimal diff
4. Run formatter: `npx oxfmt` on changed files
5. Run tests: `npx vitest run --reporter=verbose` on changed test files
6. Show output — only commit if ALL tests pass
7. Do NOT use `String(err)` — use `err instanceof Error ? err.message : "unknown error"`
8. Do NOT run npm/pnpm install
