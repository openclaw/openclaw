## 3) Analytics Spec (Minimum)

### 3.1 Event envelope (required on every event)

| Field                                      | Description                                               |
| ------------------------------------------ | --------------------------------------------------------- |
| `event_name`                               | Stable snake_case event name.                             |
| `event_time`                               | ISO timestamp in UTC.                                     |
| `user_id`                                  | Authenticated ID when available; else null.               |
| `anon_id`                                  | Anonymous session identifier.                             |
| `session_id`                               | Session token for funnel stitching.                       |
| `flow_id`                                  | One of `F01..F12` once inferred/selected.                 |
| `route_id`                                 | Current route/page identifier.                            |
| `wizard_signature`                         | Compact hash of wizard choices for replay and routing QA. |
| `utm_source`, `utm_medium`, `utm_campaign` | Entry attribution fields.                                 |

### 3.2 Flow event chains and acceptance criteria

| Flow                                | Required event chain                                                                                                                                                                                | Acceptance criteria                                                                                                 |
| ----------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------- |

| F02 Book buyer                      | `book_bridge_viewed -> book_bridge_wizard_clicked -> wizard_completed -> triad_viewed -> return_review_completed -> conversation_submitted or probe_checkout_completed`                             | Book path explicitly bridges method learning to governed practice and yields at least one concrete next-step CTA.   |

## 5) Definition of Done (Flow-Level)

1. Every flow `F01..F12` has a validated event chain in analytics with no missing required events.
2. Every key transition has a visible CTA and a tracked outcome (`continue`, `buy`, `request`, `defer`).
3. Wizard result page always renders 3 recommendations with explanation for each slot.
4. Trust/procurement docs are reachable without completing the wizard.
5. Return review is reachable from Free, Probe, and Project states.
6. Add-ons are attachable from pricing, probe, and project continuation surfaces.
