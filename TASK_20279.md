# Task: Fix media fetch errors leaking PII to channels (Issue #20279)

## STEP 0 — Run /init first

Run /init to generate/update CLAUDE.md for this project before anything else.

## STEP 1 — git pull

Confirm you are on main and it is up to date with origin/main.

## Problem

When a media fetch operation fails, the raw error message posted to the channel
includes serialized JSON with:

- Private group IDs (@g.us)
- Real names and phone numbers (+90...)
- Internal system paths and error stack traces
- Full conversation context (untrusted metadata)

## Expected behavior

Channel receives only a generic safe message like:
`⚠️ Media failed: unable to fetch attachment.`
Raw error details stay in internal logs only.

## Key files to investigate

- `src/media/fetch.ts` — MediaFetchError class, fetchRemoteMedia()
- `src/media/web-media.ts` — calls fetchRemoteMedia, where errors propagate
- `src/media-understanding/attachments.cache.ts` — another fetch call site
- `src/infra/outbound/` — where messages are sent to channels (look for media error handling)
- Extensions (telegram, discord, matrix, whatsapp) — each may have its own media error path

## What the fix should do

1. Find all places where MediaFetchError (or generic fetch errors for media)
   propagate to become channel-visible messages
2. Catch these errors before they reach the channel delivery layer
3. Replace raw error text with a safe generic message
4. Keep full error in internal logs (logVerbose / debugLogger)
5. The fix should be consistent across channels (not just one)
