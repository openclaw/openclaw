# OpenClaw Deployment Guide

This guide covers the deployment of OpenClaw, specifically tailored for server environments (e.g., Shanghai server).

## Prerequisites

- **Docker** and **Docker Compose** installed on the target server.
- **Node.js** (optional, but useful for local scripts).
- Access to the server via SSH.

## 1. Setup

Clone the repository and run the setup script:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
./docker-setup.sh
```

This script will:

1.  Check dependencies.
2.  Generate a secure `OPENCLAW_GATEWAY_TOKEN`.
3.  Build the Docker image.
4.  Run the onboarding wizard (you can skip daemon install for server deployments).
5.  Start the gateway.

## 2. Configuration

Key configuration files:

- `.env`: Environment variables (generated by setup script).
- `docker-compose.yml`: Service definition.
- `docker-compose.extra.yml`: Additional mounts (if configured).

Ensure `OPENCLAW_GATEWAY_BIND` is set appropriately in `.env` (e.g., `0.0.0.0` or `lan` if accessing externally, or keep `loopback` and use a reverse proxy).

## 3. Validation

After checking that the containers are running, execute the validation script to ensure system health:

```bash
./scripts/deploy-validation.sh
```

This script performs the following checks:

- Container uptime.
- Port accessibility.
- Application-level health check (using the internal health endpoint).
- Log error scanning.

## 4. Maintenance

- **Logs**: `docker compose logs -f openclaw-gateway`
- **Restart**: `docker compose restart openclaw-gateway`
- **Update**: `git pull && ./docker-setup.sh` (rebuilds image)

## Troubleshooting

If `deploy-validation.sh` fails:

1.  Check detailed logs: `docker compose logs --tail 100 openclaw-gateway`
2.  Verify firewall settings for port 18789.
3.  Ensure the token in `.env` matches the running container.
