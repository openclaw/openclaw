# Task: Fix gateway crash on bad hook entries (Issue #51266)

## STEP 0 — Run /init first

Before anything else, run /init to generate CLAUDE.md for this project.
This creates project context that optimizes future sessions.

## Problem

If a user adds a hook entry to openclaw.json that points to a non-existent path,
the gateway crashes on startup with an unhandled MODULE_NOT_FOUND error.
The error message doesn't indicate which hook entry is bad or how to fix it.

## Expected behavior

1. Gateway wraps hook require() calls in try/catch
2. Bad entries log a clear warning:
   `⚠️ Hook "session:compact:after" failed to load: ./path.js (MODULE_NOT_FOUND). Skipping.`
3. Gateway continues startup with the remaining valid hooks

## Instructions

1. Run /init to generate CLAUDE.md
2. Find where hooks are loaded at startup (search for require() or import() related to hooks config)
3. Wrap the hook loading in try/catch
4. Log a clear warning identifying which hook failed and why
5. Continue loading remaining hooks instead of crashing
6. Add tests verifying graceful degradation on bad hook entries
7. Do NOT run npm install or any package manager commands

## Quality standards

- Follow existing code style (TypeScript)
- Tests must verify gateway continues with valid hooks when one is bad
- Warning message must include the hook name and error reason
