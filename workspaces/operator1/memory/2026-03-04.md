### ACP Code 5 Debug Notes (2026-03-04)

**Root cause identified:**

- Exit code 5 = `PERMISSION_DENIED` (not a crash)
- acpx runs in non-interactive mode → can't prompt for write permissions → exits with code 5
- Default: `permissionMode: "approve-reads"` + `nonInteractivePermissions: "fail"`

**Fix attempted:**

- Added `permissionMode: "approve-all"` to `hooks.internal.entries.acpx` in openclaw.json
- Gateway detected hot-reload: "config hot reload applied (hooks.internal.entries.acpx.permissionMode)"
- But acpx still fails with code 5

**Issue:**

- Config is set correctly in openclaw.json
- Gateway hot-reloads the config
- But the `permissionMode` is NOT being passed to acpx at runtime
- Need to investigate: `/Users/rohits/dev/operator1/extensions/acpx/src/service.ts` and gateway plugin initialization

**Workaround:**

- Run acpx directly with `--approve-all` flag: `acpx --approve-all claude "task"`
- Or run Claude CLI directly: `claude --dangerously-skip-permissions -p "task"`

**Verified working:**

```
cd ~/dev/operator1 && claude --dangerously-skip-permissions -p "Create a file..."
# ✅ Creates file successfully
```
