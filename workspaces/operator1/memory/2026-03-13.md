## Session Memory Isolation (Phase 8.5)

### Problem Identified

- Parallel sessions (web chat, Telegram topics, WhatsApp) share same workspace + MEMORY.md
- Session A working on Project X writes to shared MEMORY.md
- Session B working on Project Y writes to same file
- Session C reads mixed context → confused about "current task"
- User wants bulletproof isolation like Claude Code's separate terminal sessions

### Solution: Four-Layer Approach

Added **Phase 8.5** to `/Project-tasks/operator1-config-sqlite.md`:

| Phase | Description                                           | Effort   |
| ----- | ----------------------------------------------------- | -------- |
| 8.5   | Project/Workspace/Channel unification (SQLite schema) | 2 days   |
| 8A    | Session tagging                                       | 0 days   |
| 8B    | Project-scoped memory                                 | 1 day    |
| 8C    | Per-project workspace                                 | 2-3 days |

### New SQLite Tables (Phase 8.5)

```sql
-- Projects (replaces PROJECTS.md)
CREATE TABLE op1_projects (
  id TEXT PRIMARY KEY,
  name, path, type, tech, status, is_default, keywords_json, links_json, ...
);

-- Telegram topic → project binding
CREATE TABLE op1_telegram_topic_bindings (
  chat_id, topic_id, project_id, group_name, topic_name, bound_at, bound_by, ...
);

-- Extended tables
ALTER TABLE workspace_state ADD COLUMN project_id REFERENCES op1_projects(id);
ALTER TABLE session_entries ADD COLUMN project_id REFERENCES op1_projects(id);
```

### New RPCs

- `projects.bindTelegramTopic({ projectId, chatId, topicId })`
- `projects.unbindTelegramTopic({ chatId, topicId })`
- `projects.getTelegramBindings({ projectId })`

### Unified Chain

```
Telegram Group/Topic → op1_telegram_topic_bindings → op1_projects → workspace_state → session_entries
```

## Lessons Learned

1. **Telegram images:** Can't send local paths directly — must copy to workspace (allowed directory) first, then use `message` tool with `media` parameter
2. **Forum topics:** Use `threadId` parameter when sending to Telegram forum topics
3. **NotebookLM artifacts:** Mind maps are JSON only; use infographics for visual diagrams
4. **Session isolation:** Current OpenClaw shares workspace across all sessions; SQLite unification needed for proper project-scoped memory
