# Titanium Claws - Project Provenance

## Origins

**Titanium Claws** originated from work developed from the [OpenClaw](https://github.com/openclaw/openclaw) project, an open-source AI agent framework released under the MIT License.

## Timeline

### Pre-1.0.0: Foundation Phase

During the foundation phase, Titanium Claws was developed as work derived from OpenClaw. This included:

- Initial architecture exploration based on OpenClaw patterns
- Adaptation of core concepts for the Titanium Claws use case
- Development of foundational components with compatibility in mind
- Integration testing with OpenClaw-based workflows

### Version 1.0.0: Independence Declaration

Starting with version 1.0.0, Titanium Claws development proceeded independently under its own:

- **Roadmap**: Independent feature prioritization and planning
- **Architecture**: Original design decisions and system structure
- **Governance**: Independent contribution and maintenance policies
- **Release Process**: Independent versioning and release cadence
- **Community**: Separate contributor base and user community

While maintaining compliance with applicable open-source license obligations from the original project.

## What "Independent" Means

Independence for Titanium Claws means:

### Architectural Independence

- **Original Design**: Core architecture decisions made for Titanium Claws use cases
- **Independent Evolution**: System evolves based on Titanium Claws requirements
- **Separate Roadmap**: Features prioritized for Titanium Claws ecosystem
- **Original Innovation**: Novel solutions developed for Titanium Claws challenges

### Governance Independence

- **Independent Maintenance**: Maintained by Titanium Claws contributors
- **Separate Contribution Process**: Own contribution guidelines and review process
- **Independent Releases**: Own release schedule and versioning policy
- **Separate Community**: Own user community and support channels

### Technical Independence

- **Original Implementation**: Code written for Titanium Claws architecture
- **Independent Testing**: Own test suite and quality standards
- **Separate Documentation**: Own documentation and user guides
- **Independent Infrastructure**: Own CI/CD, hosting, and deployment

## Compatibility Commitment

Despite independence, Titanium Claws maintains compatibility with OpenClaw where practical:

### Preserved Compatibility

- **API Compatibility**: Maintains compatibility with OpenClaw API patterns
- **Configuration Compatibility**: Supports OpenClaw configuration formats
- **Environment Variables**: Dual-support for OpenClaw and Titanium Claws environment variables
- **Migration Path**: Clear migration guide from OpenClaw to Titanium Claws

### Deprecation Policy

Titanium Claws follows a structured deprecation policy:

1. **Deprecation**: Feature marked as deprecated in a major release
2. **Warning Phase**: Deprecation warnings shown throughout that release cycle
3. **Removal**: Feature removed in the next major release

This provides users ample time to migrate while allowing the project to evolve.
