# Music Creator V1 Validation Checklist

- [ ] `create` writes a project manifest.
- [ ] `plan-generation` writes OpenClaw `music_generate` calls.
- [ ] `provider-setup` reports provider readiness without printing secrets.
- [ ] `generate-live --dry-run` records an executable tool call.
- [ ] `generate-live` records `blocked_missing_credentials` when no provider
      key is configured.
- [ ] `sync-live-output` reads OpenClaw task JSON and ingests returned local
      `MEDIA:` audio paths.
- [ ] `ingest-candidate` hashes and records local audio files.
- [ ] `qa` records technical and creative readiness.
- [ ] `select` only accepts draft-ready candidates.
- [ ] `publish-package` remains blocked until release gates are satisfied.
- [ ] `set-release-gate` records explicit release approvals.
- [ ] `publish-package` can become ready only after candidate selection, QA pass,
      and all release gates are true.
- [ ] `bridge-init` creates the shared GarageBand bridge folders and MacBook
      helper commands.
- [ ] `macbook-pair-openclaw-node.command` exists in the bridge folder, is
      executable, targets the Mac Studio Gateway, avoids writing Gateway tokens
      into iCloud status files, and writes
      `from-macbook/macbook-node-status.json`.
- [ ] `macstudio-open-node-enrollment.command` opens a short-lived tokenless
      enrollment window, restores the previous Gateway auth mode on exit, and
      writes `from-macbook/node-enrollment-window.json`.
- [ ] `macbook-pair-openclaw-node-window.command` exists in the bridge folder,
      is executable, uses the tokenless enrollment window, and writes
      `from-macbook/macbook-node-window-status.json`.
- [ ] `macbook-enable-remote-exec.command` is not generated by default; Remote
      Login/SSH is deprecated for this bridge unless explicitly requested with
      `bridge-init --include-remote-exec`.
- [ ] `macbook-disable-remote-exec.command` exists in the bridge folder, is
      executable, removes the old OpenClaw SSH authorized-key line when present,
      and reminds the user to keep Remote Login off.
- [ ] `macstudio-bridge-signing.pub.pem` is synced to the bridge while the
      matching private signing key remains only under `music-creator-v1/state/`.
- [ ] `bridge-sync-probe` writes a Mac Studio sync probe,
      `macbook-sync-check.command` writes the matching MacBook reply, and
      `bridge-sync-status` reports bidirectional sync only after the reply
      returns.
- [ ] `macbook-start-safe-bridge.command` blocks when Remote Login is on, writes
      the sync reply when safe, runs exactly one pull-agent cycle, and records
      `from-macbook/macbook-safe-bridge-status.json`.
- [ ] `00-RUN-ME-MACBOOK-SAFE-BRIDGE.command` exists as the obvious MacBook
      entry point and delegates only to `macbook-start-safe-bridge.command`.
- [ ] `macbook-pull-agent.command` exists in the bridge folder, is executable,
      verifies signed request files, rejects expired/unsigned requests, and only
      allows whitelisted actions without arbitrary shell commands or free-form
      file paths.
- [ ] `bridge-queue-job` writes signed requests to `to-macbook/requests/` and
      `bridge-status` reports queued, processed, rejected, and pull-agent status.
- [ ] `bridge-transfer-kit` creates a portable safe MacBook kit for cases where
      no editable two-way shared folder exists, excludes Remote Login/SSH
      helpers, includes a fresh sync probe plus signed health-check request, and
      keeps the private signing key out of the kit.
- [ ] `bridge-import-transfer-return` imports only returned
      `from-macbook/`, `sync/macbook/`, and `logs/` data and never executes
      returned files or imports MacBook-supplied request queues.
- [ ] `macbook-finish-setup.command` exists in the bridge folder, is executable,
      opens GarageBand/Valhalla setup when needed, and writes
      `from-macbook/macbook-prereq-status.json`.
- [ ] `bridge-export` copies a selected/candidate audio file into a MacBook job
      packet with `job.json`, README, AppleScript, and executable command.
- [ ] `bridge-status` reports GarageBand, Valhalla AU, `auval`, bridge queue,
      synced MacBook setup status, and synced MacBook node status without
      requiring secrets.
- [ ] `bridge-ingest` imports returned GarageBand bounces from
      `from-macbook/<job-id>/` as new Music Creator V1 candidates.
- [ ] `bridge-import-garageband` imports MacBook-originated songs, stems,
      references, or vocal ideas as project source assets.
- [ ] `vocal-plan` creates an OpenClaw/cloud vocal plan with prompt, lyrics,
      cloud brief, rights warnings, and source lineage.
- [ ] `vocal-generate-live --dry-run` records the exact `music_generate` vocal
      call without requiring provider credentials.
- [ ] `vocal-ingest` imports OpenClaw or cloud-generated vocal audio and records
      hash, plan id, source label, and project lineage.
- [ ] `bridge-export --vocal` sends the ingested vocal layer back to the MacBook
      GarageBand queue.
- [ ] `provider-setup` includes `KITS_API_KEY` readiness without printing the
      secret.
- [ ] `kits-list-voices` lists Kits voice model ids and titles when
      `KITS_API_KEY` is configured.
- [ ] `kits-convert` submits a Kits voice-conversion job from a guide vocal,
      records the job id, and preserves vocal-plan compliance blockers.
- [ ] `kits-sync` fetches a completed Kits job, downloads the signed output URL,
      ingests it as a vocal layer, and keeps the token out of state files.
- [ ] `validate --rebuild-catalog` indexes all project manifests.

Public publishing, monetization, distribution, Content ID, and copyright claims
remain manual approval gates.
