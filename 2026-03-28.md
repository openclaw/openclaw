### 🔧 PR #55892 创建

**Issue:** #55816, #55760

**问题：** pi-coding-agent 0.63.0 升级后，自定义 provider 和 OAuth 认证的 provider 报错 "No API key for provider"

**根因：**

- createAgentSession() 包装 streamFn 来注入 API key
- OpenClaw 的 attempt.ts 用 raw streamSimple 覆盖了它
- 导致 auth 注入 wrapper 丢失

**修复：** 在 streamFn 赋值后恢复 getApiKey 回调

```typescript
// Restore getApiKey callback after streamFn override for custom/OAuth providers.
if (!(activeSession.agent as typeof activeSession.agent & { getApiKey?: unknown }).getApiKey) {
  (
    activeSession.agent as typeof activeSession.agent & {
      getApiKey: (provider: string) => Promise<string | undefined>;
    }
  ).getApiKey = (provider: string) => params.authStorage.getApiKey(provider);
}
```

## 📚 工作流程规则固化 (2026-03-28 下午)

### 🚨 P0 事故教训 (#56186)

**问题：** 修复 UI config save error 时，不小心删除了 `saveConfig(host)` 调用

**后果：**

- 用户点击 Save 后配置**不会保存**，数据丢失
- `lastError` 在 save 前读取，永远是 null，修复无效

**正确修复：**

```typescript
export async function handleChannelConfigSave(host: OpenClawApp) {
  await saveConfig(host); // ✅ 先保存
  const saveError = host.lastError; // ✅ 保存后读取错误
  await loadConfig(host);
  await loadChannels(host, true);
  if (saveError) {
    host.channelsError = saveError;
  }
}
```

**Greptile 发现：** P0 regression, Confidence Score 1/5, Not safe to merge

**教训：**

- 修改关键路径代码后必须验证核心功能（save 是否真的调用）
- 不能只关注修复逻辑，忽略原有功能
- Greptile 审查非常重要，能发现严重回归

### ⚙️ GitHub PR 工作流规则 (永久固化)

**分支创建铁律：**

```bash
# ✅ 正确：从 upstream/main 创建
git fetch upstream main
git checkout upstream/main
git checkout -b <新分支> upstream/main
git cherry-pick <修复提交>
git push -f fork <新分支>

# ❌ 错误：从 fork/main 创建（可能落后 upstream 100+ commits）
git checkout -b <新分支> origin/main
```

**PR 维护规则：**

- 长生命周期 PR（>24h）必须每 2 天 rebase 到 `upstream/main`
- 如果 PR 落后 main >100 commits，关闭旧 PR，从最新 `upstream/main` 重新创建
- 创建 PR 前验证修复是否已在 `upstream/main` 中
