# Gemmaclaw

Gemmaclaw is a Gemma-focused, volunteer-driven project built on top of the upstream OpenClaw personal AI assistant framework.

If you are looking for the upstream OpenClaw project and its full README, docs, and onboarding instructions, start here:

- OpenClaw README: https://github.com/openclaw/openclaw#readme
- OpenClaw docs: https://docs.openclaw.ai

## What this project is

Gemmaclaw aims to empower people to build and run helpful AI assistants and agents using the Gemma family of models.

In practice, this repo is intended to be a place where we:

- Make the OpenClaw experience smoother when running Gemma locally or on common hosted setups.
- Improve templates, tool use, and structured output reliability for Gemma-based assistants.
- Publish reference configurations, skills, and examples that are Gemma-first.
- Build community knowledge and reusable components around agentic workflows with Gemma.

## Scope and principles

We try to keep changes practical and user-facing:

- Focus on real workflows and measurable improvements.
- Prefer small, reviewable changes over large refactors.
- Keep compatibility and portability in mind, users should be able to run this on their own devices.

## Getting started

Until Gemmaclaw publishes its own end-to-end onboarding guide, the recommended path is:

1. Follow the upstream OpenClaw getting started guide: https://docs.openclaw.ai/start/getting-started
2. Then follow any Gemmaclaw-specific notes in this repo (docs, examples, configs) to run with Gemma.

## Contributing

We welcome issues and pull requests. If you are adding new skills, examples, or documentation, please keep them small and easy to reproduce.
