# MEMORY (Long-Term Memory)

## Architecture Guidelines

- **Primary inference:** OpenRouter API (cloud multi-model) with tier-based routing
- **Orchestration:** MAS (Multi-Agent System) — agents run autonomously via AgentOrchestrator
- **Memory stack:** SuperMemory = RAG (ChromaDB) + TieredMemory (hot/warm/cold) + EpisodicMemory
- **Brigades:** Dmarket (11 roles), OpenClaw (9 roles) — each with unique system prompts
- **Safety chain:** PromptInjectionDefender → HallucinationDetector → CodeValidator → AutoRollback
- Zero-VRAM safety: all guardrails are regex/heuristic, no LLM calls

## Compliance Rules

- Never expose API keys in code or logs (use environment variables)
- All user inputs pass through 5-layer PromptInjectionDefender before processing
- Code generated by LLM must pass CodeValidator (semgrep + bandit + ruff) before execution
- AutoRollback creates HMAC-signed checkpoints; any validation failure triggers git reset
- Rust sandbox enforces deny-list for dangerous Python operations (os, subprocess, eval, exec)

## Core Lessons

- OpenRouter free-tier models have rate limits — always implement retry + model fallback logic
- Multi-agent orchestration requires clear task isolation to prevent context contamination
- RAG indexing must use content hashing to avoid re-indexing unchanged documents
- Tiered memory (hot/warm/cold) reduces token usage while preserving critical context
- Parallel executor roles require merged context briefing for downstream Auditor
- Always test with mocked LLM calls in unit tests — never hardcode API keys
