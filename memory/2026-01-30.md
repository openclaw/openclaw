## Tailwind CSS v4 Cascade Bug â€” LESSON LEARNED

**Problem:** Mission Control dashboard had no padding on mobile despite `px-4` classes being applied.

**Root Cause:** In Tailwind v4, all utilities are in `@layer utilities`. I had a CSS reset outside any layer:
```css
* {
  margin: 0;
  padding: 0;
}
```

**Unlayered CSS has HIGHER cascade priority than layered CSS** â€” so this reset was overriding every Tailwind padding/margin class, even though `*` has lower specificity.

**Fix:** 
1. Remove the `* { margin: 0; padding: 0 }` reset (Tailwind's preflight handles this)
2. Put ALL custom CSS inside proper `@layer` directives:
   - `@layer base` â€” theme variables, html/body, scrollbar
   - `@layer components` â€” custom component classes
   - `@layer utilities` â€” animations, custom utilities

**Rule for future:** When using Tailwind v4, NEVER put CSS outside of `@layer` blocks â€” it will override Tailwind utilities.
