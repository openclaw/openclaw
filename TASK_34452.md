# Task: Fix maskApiKey exposing too many key characters (Issue #34452)

## Problem

`maskApiKey()` in `src/utils/mask-api-key.ts` exposes up to 16 characters of API keys:

- Keys > 16 chars: shows first 8 + last 8 = 16 chars exposed (e.g. `sk-ant-a_bcd...wxyz1234`)
- Keys <= 16 chars: shows first 2 + last 2 = 4 chars, but for short keys this is most of the key
- Keys <= 6 chars: shows first 1 + last 1 = effectively the whole key

This masked output is sent as plaintext to chat channels (Telegram, Discord, etc.).

## Current code (src/utils/mask-api-key.ts)

```typescript
export const maskApiKey = (value: string): string => {
  const trimmed = value.trim();
  if (!trimmed) {
    return "missing";
  }
  if (trimmed.length <= 6) {
    return `${trimmed.slice(0, 1)}...${trimmed.slice(-1)}`;
  }
  if (trimmed.length <= 16) {
    return `${trimmed.slice(0, 2)}...${trimmed.slice(-2)}`;
  }
  return `${trimmed.slice(0, 8)}...${trimmed.slice(-8)}`; // 16 chars exposed!
};
```

## Fix

Reduce exposed characters. Safe masking standard (used by GitHub, npm, etc.):

- Show only first 4 chars + `...` — never expose the end of the key
- For very short keys (<= 4 chars): show only first 1 char + `...`
- Empty/missing: keep returning `"missing"`

New behavior:

- `"sk-ant-api03-abc...xyz1234"` → `"sk-a..."`
- `"abcdefghijklmnop"` → `"abcd..."`
- `"short"` → `"s..."`
- `"ab"` → `"a..."`
- `""` → `"missing"`
