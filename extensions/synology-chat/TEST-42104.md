# Test documentation for #42104 – Synology-Chat optional keyword filtering

## Issue summary

[#42104](https://github.com/openclaw/openclaw/issues/42104): Synology-Chat plugin required a mandatory keyword for message parsing. Users could not send full message content without the plugin stripping the trigger word.

## Fix summary

Added `keywordFilterEnabled` config option (default: `true` for backward compatibility). When set to `false`, the full message including the trigger word is sent to the agent.

### Manual test steps

1. Configure Synology Chat with `keywordFilterEnabled: false`:

```json5
{
  channels: {
    "synology-chat": {
      enabled: true,
      token: "your-token",
      incomingUrl: "https://nas.example.com/...",
      dmPolicy: "allowlist",
      allowedUserIds: ["123456"],
      keywordFilterEnabled: false,
    },
  },
}
```

2. In Synology Chat, post a message with a trigger word (e.g. `ask Explain contextWindow configuration`).
3. Verify the agent receives the full message including `ask`, not just `Explain contextWindow configuration`.
4. Set `keywordFilterEnabled: true` (or omit it) and verify the trigger word is stripped again.
