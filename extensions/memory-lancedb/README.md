# OpenClaw Memory (LanceDB) Extension\n\nLong-term memory with vector search for AI conversations using LanceDB for storage and OpenAI for embeddings.\n\n## Overview\n\nThis extension provides seamless auto-recall and auto-capture capabilities via lifecycle hooks, enabling your OpenClaw assistant to remember and recall information from past conversations.\n\n## Features\n\n- **Vector Search**: Semantic search through conversation history using embeddings\n- **Auto-Capture**: Automatically stores important conversation moments\n- **Auto-Recall**: Intelligently recalls relevant context when needed\n- **LanceDB Storage**: Fast, scalable vector database backend\n- **OpenAI Embeddings**: High-quality semantic embeddings for search\n- **Lifecycle Integration**: Seamless integration with OpenClaw's agent lifecycle\n\n## Configuration\n\nAdd the following to your OpenClaw config:\n\n```json\n{\n  \"plugins\": {\n    \"entries\": {\n      \"memory-lancedb\": {\n        \"config\": {\n          \"databasePath\": \"./memory/lancedb\",\n          \"openaiApiKey\": \"your-openai-api-key\",\n          \"embeddingModel\": \"text-embedding-3-small\",\n          \"maxRecallItems\": 5,\n          \"similarityThreshold\": 0.7\n        }\n      }\n    }\n  }\n}\n```\n\n## Configuration Options\n\n| Option | Type | Default | Description |\n|--------|------|---------|-------------|\n| `databasePath` | string | `\"./memory/lancedb\"` | Path to LanceDB storage directory |\n| `openaiApiKey` | string | - | OpenAI API key for embeddings (required) |\n| `embeddingModel` | string | `\"text-embedding-3-small\"` | OpenAI embedding model to use |\n| `maxRecallItems` | number | `5` | Maximum number of items to recall per query |\n| `similarityThreshold` | number | `0.7` | Minimum similarity score for recall |\n| `captureThreshold` | number | `0.8` | Minimum importance score for auto-capture |\n| `enableAutoCapture` | boolean | `true` | Enable automatic capture of important moments |\n| `enableAutoRecall` | boolean | `true` | Enable automatic recall during conversations |\n\n## Usage\n\n### Manual Memory Operations\n\n```javascript\n// Store a memory manually\nconst memoryId = await ctx.memory.store({\n  content: \"User prefers morning meetings\",\n  metadata: { type: \"preference\", user: \"john\" },\n  importance: 0.9\n});\n\n// Search memories\nconst memories = await ctx.memory.search({\n  query: \"meeting preferences\",\n  limit: 3,\n  threshold: 0.7\n});\n\n// Retrieve specific memory\nconst memory = await ctx.memory.get(memoryId);\n```\n\n### Automatic Behavior\n\nWhen enabled, the extension automatically:\n\n1. **Captures** important conversation moments based on:\n   - User preferences and personal information\n   - Task completions and decisions\n   - Error resolutions and learnings\n   - Important facts and insights\n\n2. **Recalls** relevant context when:\n   - Similar topics are discussed\n   - User references past conversations\n   - Context would be helpful for current task\n\n## Memory Schema\n\nEach memory entry contains:\n\n```typescript\ninterface Memory {\n  id: string;              // Unique memory identifier\n  content: string;         // The actual memory content\n  embedding: number[];     // Vector embedding for search\n  timestamp: Date;         // When the memory was created\n  sessionId?: string;      // Associated session ID\n  importance: number;      // Importance score (0-1)\n  metadata: {\n    type?: string;         // Memory type (preference, fact, task, etc.)\n    tags?: string[];       // Searchable tags\n    user?: string;         // Associated user\n    [key: string]: any;    // Additional metadata\n  };\n}\n```\n\n## Performance\n\n- **Fast Retrieval**: LanceDB provides sub-millisecond vector search\n- **Scalable**: Handles millions of memory entries efficiently\n- **Low Overhead**: Minimal impact on conversation latency\n- **Disk Efficient**: Compressed storage with automatic cleanup\n\n## Privacy & Security\n\n- **Local Storage**: All memories stored locally in your LanceDB instance\n- **No Cloud Dependency**: Only embeddings API calls to OpenAI\n- **Configurable Retention**: Set automatic cleanup policies\n- **Encrypted Storage**: Optional encryption at rest (configure LanceDB)\n\n## Troubleshooting\n\n### Common Issues\n\n**\"No OpenAI API key configured\"**\n- Ensure `openaiApiKey` is set in plugin config\n- Verify API key has embedding permissions\n\n**\"Database connection failed\"**\n- Check `databasePath` directory permissions\n- Ensure sufficient disk space available\n\n**\"Poor recall quality\"**\n- Adjust `similarityThreshold` (lower = more permissive)\n- Try different embedding model (`text-embedding-3-large`)\n- Check memory content quality and importance scores\n\n### Debug Mode\n\nEnable debug logging:\n\n```json\n{\n  \"config\": {\n    \"debug\": true,\n    \"logLevel\": \"debug\"\n  }\n}\n```\n\n## Development\n\n### Building\n\n```bash\nnpm run build\n```\n\n### Testing\n\n```bash\nnpm test\n```\n\n### Contributing\n\nSee the main [OpenClaw
