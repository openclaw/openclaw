# SaaS Plan Templates

## PT-SS-001: Feature Development Cycle

**Applicable Goals:** Ship new feature, product improvement
**Success Rate:** 72%

### Steps

1. Requirements gathering — user research, stakeholder input (compound)
2. Technical design — architecture, API design, data model (primitive)
3. Sprint planning — break into tasks, assign (primitive)
4. Development — implementation sprints (compound)
5. QA and testing — unit, integration, UAT (compound)
6. Staging deployment and review (primitive)
7. Production deployment with feature flag (primitive)
8. Monitor adoption and performance metrics (primitive)
9. Iterate based on user feedback (primitive)

## PT-SS-002: Customer Onboarding Flow

**Applicable Goals:** Improve activation rate, reduce time-to-value
**Success Rate:** 79%

### Steps

1. Map current onboarding journey (primitive)
2. Identify drop-off points and friction (primitive)
3. Design improved flow — in-app guides, email drip (compound)
4. Implement changes (compound)
5. A/B test old vs new flow (primitive)
6. Measure activation rate change (primitive)
7. Document and share findings (primitive)

## PT-SS-003: Churn Reduction

**Applicable Goals:** Reduce monthly churn, improve retention
**Success Rate:** 65%

### Steps

1. Analyze churn data — cohorts, reasons, timing (primitive)
2. Segment at-risk customers — usage patterns, engagement (primitive)
3. Design intervention strategies per segment (primitive)
4. Implement automated health scoring (compound)
5. Create re-engagement campaigns (compound)
6. Monitor churn rate weekly (primitive)
7. Store outcome as case for CBR (primitive)

## PT-SS-004: Pricing Model Optimization

**Applicable Goals:** Increase ARPU, improve conversion
**Success Rate:** 61%

### Steps

1. Analyze current pricing vs value metrics (primitive)
2. Research competitor pricing (primitive)
3. Model scenarios — tier changes, usage-based, freemium (primitive)
4. Survey existing customers on willingness to pay (compound)
5. Design new pricing tiers (primitive → decision_request)
6. Grandfather existing customers plan (primitive)
7. Launch and monitor conversion impact (primitive)
