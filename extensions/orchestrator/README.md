# @openclaw/orchestrator

Deterministic routing layer + file-backed task store for Phase B of the Fleet Command Center. Receives a high-level task description, picks a specialist agent via regex rules in `~/.openclaw/extensions/orchestrator/routing.json`, persists the task record under `~/.openclaw/tasks/orchestrator/<id>.json`, and (in `live` mode) hands the work off to the existing `sessions_spawn` machinery via the Fleet Orchestrator agent's prompt loop.

> **Status:** in progress. The pieces shipped in this iteration are the schema (Unit 2), the routing engine (Unit 3), and the task store (Unit 4). Cross-repo HTTP routes, the orchestrator agent runtime, the synthetic-task observability gate, and the Mission Control Pipeline / Approvals tabs land in subsequent PRs. Recon notes for the design are at [`./RECON-NOTES.md`](./RECON-NOTES.md); the full plan lives at `(MissionControl) docs/plans/2026-04-25-005-feat-orchestrator-routing-layer-plan.md`.

## Configuration

| Key                               | Type                                | Default       | Notes                                                                   |
| --------------------------------- | ----------------------------------- | ------------- | ----------------------------------------------------------------------- |
| `extensions.orchestrator.enabled` | boolean                             | `true`        | Master off-switch. `false` makes the extension contribute nothing.      |
| `extensions.orchestrator.mode`    | `"synthetic" \| "shadow" \| "live"` | `"synthetic"` | Dispatch mode. New installs default to `synthetic` and never auto-flip. |

## File layout

| Path                                                               | Owner            | Purpose                                                                   |
| ------------------------------------------------------------------ | ---------------- | ------------------------------------------------------------------------- |
| `~/.openclaw/extensions/orchestrator/routing.json`                 | operator         | Routing rules (created on first load with sensible defaults).             |
| `~/.openclaw/credentials/orchestrator-bearer.json`                 | extension (init) | Bearer token for the Mission Control client. Generated by `init` verb.    |
| `~/.openclaw/tasks/orchestrator/<id>.json`                         | extension        | Live task records (atomic-rename; one writer).                            |
| `~/.openclaw/tasks/orchestrator/synthetic/<id>.json`               | extension        | Synthetic-mode tasks. Excluded from default lists.                        |
| `~/.openclaw/tasks/orchestrator/shadow/<id>.json`                  | extension        | Shadow-mode archive (real spawn, bypassed approval).                      |
| `~/.openclaw/agents/fleet-orchestrator/sessions/<sid>.tasks.jsonl` | extension        | `task.*` trajectory events (sidecar — distinct from `.trajectory.jsonl`). |

## Boundaries

The extension imports only from `openclaw/plugin-sdk/*` and Node built-ins. It does not import `src/**`, channel internals, or other extensions. Trajectory events use the public `openclaw-trajectory schemaVersion:1` envelope; the JSONL sidecar avoids contention with the in-process `QueuedFileWriter` for the agent's main trajectory file.

## Why a separate task store?

`~/.openclaw/tasks/runs.sqlite` (under `src/tasks/`) already exists as the single-writer store for OpenClaw's internal task registry. The orchestrator's namespace (`tasks/orchestrator/`) sits alongside but does not interact — different writers, different shape, different retention policy. No `readdir()` consumer in core treats unknown JSON entries under `~/.openclaw/tasks/` as work.
