# PRD: Phase 5 — Agent Features & Memory Integration

**Version:** 1.0  
**Date:** January 29, 2026  
**Status:** ✅ Complete (All Phases)

---

## Overview

Implement agent-focused features that leverage Clawd IDE's unique advantage: DNA memory integration. Skip cloud-centric features that conflict with our local-first philosophy.

---

## Goals

1. Surface existing DNA capabilities in IDE UI
2. Add quality-of-life improvements for agent workflows
3. Build memory-aware features no competitor has
4. Keep architecture simple — don't duplicate DNA functionality

## Feature Specifications

### 1. Dedicated Agent Terminal (P0)

**What:** Isolated terminal panel for agent command execution

**Why:** Current terminal mixes agent commands with user commands. Confusing.

**Requirements:**
- [ ] Separate terminal panel labeled "Agent Terminal"
- [ ] Auto-scrolls to show agent output
- [ ] Clear visual distinction (different background/border)
- [ ] Shows running/idle status
- [ ] Can be minimized/expanded
- [ ] Command history specific to agent

**UI Location:** Bottom panel, tab alongside main terminal

**Implementation:**
- New terminal instance in IDE
- Agent commands route here via `agentExec()` function
- WebSocket connection to show real-time output

---

### 2. Auto-Lint Fix (P0)

**What:** Automatically fix lint errors in AI-generated code before showing user

**Why:** Agent generates code → ESLint finds issues → user sees broken code. Bad UX.

**Requirements:**
- [ ] After agent generates code, run ESLint --fix
- [ ] Show "Auto-fixed X lint errors" toast
- [ ] Option to see what was fixed (expandable)
- [ ] Configurable: on/off in settings
- [ ] Only fix auto-fixable issues, report others

**Implementation:**
- Hook into code insertion flow
- Run `eslint --fix` on generated code
- Parse output, show toast with count
- Store diff of fixes for "show details"

---

### 3. AI Attribution / Blame (P1)

**What:** Track which lines were AI-generated vs human-written

**Why:** Know what to trust, what to review carefully, audit trail

**Requirements:**
- [ ] Mark AI-generated code in git commits (metadata)
- [ ] Gutter indicator showing AI vs human lines
- [ ] Hover to see: "Generated by Clawd on 2026-01-29, conversation: [link]"
- [ ] Filter: "Show only AI-generated lines"
- [ ] Integration with memory (link to conversation in daily file)

**Data Model:**
```json
{
  "file": "src/app.js",
  "ranges": [
    { "start": 10, "end": 25, "type": "ai", "date": "2026-01-29", "session": "abc123" }
  ]
}
```

**Storage:** `.clawd/attribution.json` per project

**Implementation:**
- Track insertions during agent edits
- Store attribution data
- Gutter renderer reads attribution
- Git hook to include in commit message

---

### 4. Multi-Agent UI (P1)

**What:** Dashboard to spawn, monitor, and manage DNA sub-agents

**Why:** `sessions_spawn` exists but no visibility in IDE

**Requirements:**
- [ ] "Agents" panel in sidebar
- [ ] List active sessions (from sessions_list)
- [ ] Spawn new agent: task input + model selector
- [ ] View agent status: running/completed/error
- [ ] Click to see agent output/history
- [ ] Kill agent button

**UI Design:**
```
┌─────────────────────────┐
│ 🤖 Agents               │
├─────────────────────────┤
│ ▶ research-keywords     │
│   Running • gpt5 • 2m   │
├─────────────────────────┤
│ ✓ analyze-competitors   │
│   Done • kimi • 5m ago  │
├─────────────────────────┤
│ [+ New Agent]           │
└─────────────────────────┘
```

**Implementation:**
- Call DNA `sessions_list` API
- Poll for updates (or WebSocket if available)
- `sessions_spawn` for new agents
- `sessions_history` for viewing output

---

### 5. Memory Search in IDE (P1)

**What:** Search DNA memory files directly from IDE

**Why:** "What did we decide about the auth flow?" — answer without leaving IDE

**Requirements:**
- [ ] Cmd+Shift+M opens memory search
- [ ] Search across MEMORY.md + memory/*.md
- [ ] Show results with file + line number
- [ ] Click to open in editor
- [ ] Recent searches history

**UI:** Command palette style overlay

**Implementation:**
- Read memory files from workspace
- Simple text search (or leverage memory_search if API available)
- Results panel with snippets
- Open file on click

---

### 6. Session History Panel (P2)

**What:** View past conversations about current file/project

**Why:** Context about why code looks this way

**Requirements:**
- [ ] "History" tab in sidebar
- [ ] Filter by current file
- [ ] Show conversation snippets
- [ ] Link to full daily memory file
- [ ] Timeline view

**Implementation:**
- Parse memory/*.md files
- Extract file references
- Build index of file → conversations
- Display filtered by active file

## Technical Architecture

### New Files

```
ide/
├── public/
│   ├── modules/

│   └── dna-bridge.js       # API calls to DNA
└── .clawd/
```

### API Integration

Need to call DNA APIs:
- `sessions_list` — List agents
- `sessions_spawn` — Create agent
- `sessions_history` — Get agent output
- `memory_search` — Search memory (if exposed)

**Bridge approach:** IDE server proxies requests to DNA gateway
