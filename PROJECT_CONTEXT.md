## 2. OpenClaw Bot — Мета-система

### Миссия

OpenClaw — автономный ИИ-инженер, который проектирует, пишет, тестирует и улучшает код **самостоятельно**. Это не просто бот — это **система, создающая и контролирующая другие системы**.

### Ключевые функции

- **Архитектурное проектирование** — LATS / Tree-of-Thought / Graph-RAG пайплайны
- **Безопасность и Guardrails** — L1-L4 песочница, аудиторский протокол REFLEXION, kill-list запрещённых паттернов
- **Самообучение** — KnowledgeStore (Python 3.14 / Rust 2024 / TS 5.8), семантический граф зависимостей
- **Разработка внешних ботов** — генерация, рефакторинг и аудит кода Dmarket Bot
- **Мульти-агентный пайплайн** — Planner → Architect → Coder → Test_Writer → Auditor → Archivist

### Технологический стек

- **Язык**: Python 3.14 (primary), Rust (core performance), TypeScript (web/CLI)
- **LLM Backbone**: OpenRouter API (cloud, multi-model routing via SmartModelRouter)
- **Фреймворк**: Собственный пайплайн (pipeline_utils.py), Brigade Policy, MCP Tools
- **Хранение знаний**: KnowledgeStore + GraphEngine (graph-RAG)
- **Безопасность**: Sandbox L1-L4, OWASP-aligned Auditor

### Сильные стороны

1. Полная автономность — от идеи до коммита без вмешательства человека
2. Встроенный аудит — каждый результат проходит через Auditor с REFLEXION-протоколом
3. Живая база знаний — актуальные стандарты Python 3.14, Rust 2024, TypeScript 5.8
4. Параллельное выполнение — Brigade Policy позволяет масштабировать агентов
5. Многоязычность — бот работает на трёх языках программирования с единой базой знаний

### Слабые стороны / Ограничения

1. Зависимость от LLM — качество результата привязано к возможностям модели
2. Холодный старт — построение KnowledgeStore + GraphEngine при каждом запуске
3. Сложность отладки — мульти-агентный пайплайн трудно дебажить пошагово
4. Потребление памяти — полный граф зависимостей для больших проектов дорог по RAM

## 3. Dmarket Bot — Торговая система

### Миссия

Максимизация прибыли на платформе DMarket через автоматизированный высокоскоростной трейдинг скинами CS2/Dota 2/Rust.

### Ключевые функции

- **Анализ цен** — мониторинг 100k+ предметов в реальном времени, детекция аномалий
- **Торговые стратегии** — арбитраж, Buy-the-Dip, Trend Following, Spread Trading
- **Исполнение ордеров** — автоматическое размещение/отмена ордеров через DMarket API
- **Управление рисками** — лимиты позиций, stop-loss, максимальная просадка
- **Отчётность** — PnL-трекинг, аналитика сделок, алерты в Telegram

### Технологический стек

- **Язык**: Python 3.14
- **API**: DMarket REST API v2, WebSocket для стримов цен
- **Хранение**: SQLite (сделки), Redis (кеш цен), JSON (конфигурация)
- **Инфраструктура**: Локальный сервер (низкая латентность), планировщик задач

### Сильные стороны

1. Скорость реакции — обработка ценовых событий за миллисекунды
2. Охват — мониторинг всех доступных предметов параллельно
3. Дисциплина — машинное исполнение без эмоциональных решений
4. Бэктестинг — стратегии проверяются на исторических данных перед деплоем

### Слабые стороны / Ограничения

1. API Rate Limits — DMarket ограничивает количество запросов
2. Ликвидность — на редких предметах спред слишком высок для прибыльного арбитража
3. Изменения API — DMarket может менять эндпойнты без предупреждения
4. Рыночный риск — резкие обвалы цен до исполнения стоп-лоссов

## 4. Взаимодействие проектов

```
┌──────────────────────────────────────────────────────┐
│                   OpenClaw Bot                       │
│              (Мета-система / ИИ-инженер)             │
│                                                      │
│  Planner → Architect → Coder → Test → Auditor       │
│                        │                             │
│                        ▼                             │
│   ┌─────────────────────────────────────────────┐    │
│   │  Dmarket Bot (Генерируемый/Управляемый)     │    │
│   │  - Стратегии / Исполнение / Риск-контроль   │    │
│   └─────────────────────────────────────────────┘    │
└──────────────────────────────────────────────────────┘
```

### Модель "Родитель — Потомок"

- **OpenClaw = Родитель** — генерирует код, проводит рефакторинг, аудит безопасности
- **Dmarket Bot = Потомок** — исполняет торговую логику, сгенерированную родителем
- **Критический путь**: вся торговая логика проходит через `Auditor` OpenClaw перед деплоем
- **Разделение ответственности**: OpenClaw не торгует напрямую; Dmarket Bot не изменяет себя

### Правила безопасности взаимодействия

1. Никогда не деплоить торговый код без прохождения Auditor с флагом `approved: true`
2. Финансовые операции требуют двойную верификацию (Auditor + человек для сумм > порога)
3. Dmarket Bot не имеет доступа к модификации собственного исходного кода
4. Логи всех торговых операций хранятся отдельно от логов OpenClaw

При получении задачи агент **обязан** определить контекст:

```

  ├── ДА → Используй KnowledgeStore + GraphEngine + полный пайплайн
  │         Стек: Python 3.14 / Rust 2024 / TypeScript 5.8
  │
  └── НЕТ → Касается Dmarket Bot?
       ├── ДА → Используй KnowledgeStore (Python 3.14)
       │         Auditor: ОБЯЗАТЕЛЕН для финансовой логики
       │         Рабочая директория: D:\Dmarket_bot
       │
       └── НЕТ → Общая задача / ответ пользователю
                  Используй контекст из BRAIN.md + IDENTITY.md
```
