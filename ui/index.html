OpenClaw Control

Connect to your gateway with less guessing.

This page is for operators connecting to a running OpenClaw gateway. Start the gateway on the host machine, generate a tokenized dashboard link, then connect here or open that tokenized URL directly.

Gateway token is usually required
127.0.0.1 only works on the host machine
Password is only needed if your gateway uses one

Before you start

  1. Run openclaw gateway run on the machine hosting OpenClaw.
  2. Run openclaw dashboard to get a tokenized dashboard URL.
  3. Use that URL directly, or paste its WebSocket URL and token into the form below.
Connection hints

Use the right address

The default ws://127.0.0.1:18789 only works when your browser is on the same machine as the gateway. If the gateway runs elsewhere, replace 127.0.0.1 with that host’s reachable address.

Common issue

“Gateway token missing” is a setup hint, not a dead end.

  • Run openclaw dashboard on the host machine.
  • Open the generated tokenized URL directly, or paste the token into the Gateway Token field.
  • If a password was configured on the gateway, enter it too.
Operator note

What happens after connect

Once connected, this control app shows sessions, channels, logs, and other operator tools. The sign-in form below is just the front door to the live gateway.

Need a tokenized link? Run openclaw dashboard on the host machine.
