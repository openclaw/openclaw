# 面試話術手冊

_博弈圈面試不看學歷，看你懂不懂。以下是 Cruz 的真實經歷轉化為各崗位的面試話術。_

---

## 通用開場（適用所有崗位）

> 「我在博弈行業做了快 8 年，從 Python 開發一路做到技術經理，管過 300 多人的跨國團隊。後來回台灣自己做，現在想找一個穩定的遠端坑，專心寫代碼。」

---

## 按崗位的重點話術

### Python 後端 / 數據工程

**Q: 你做過什麼系統？**
> 「主要是數據監控和自動化。我從零搭過完整的數據環境 — Linux、Docker、Jupyter、MySQL，一個月交付。後面做了自動化報表系統，每天跑 1000 多條數據，從採集到清洗到出報表全自動，最後推到 Telegram 群。」

**Q: 你熟悉什麼技術棧？**
> 「Python 十年了，Pandas、NumPy 是日常。資料庫 PostgreSQL 和 MySQL 都做過，ORM 和原生 SQL 都行。部署用 Docker + Linux，監控用 Telegram Bot 做實時告警。」

**Q: 有沒有做過高併發 / 大量數據？**
> 「之前做的平台日均打碼量 9400 萬人民幣，5000 個活躍用戶。我的數據系統需要實時採集這些交易數據做異常檢測和報表。不是純技術上的高併發，但數據量和實時性要求很高。」

### 前端開發

**Q: 你有前端經驗嗎？**
> 「你可以看我的履歷網站（dufu-resume.vercel.app），那就是我自己寫的，純 CSS 動畫、零外部 JS、ESPN API 即時數據。我也做過 LINE LIFF 應用，48 小時從需求到上線。React 和 Vue 都能寫。」

**Q: 你主要用什麼框架？**
> 「看專案需求。快速原型用 Vanilla JS + Tailwind，正式項目 React 或 Vue 都行。部署走 Vercel 或 Docker。我比較強的是快速交付能力 — 給我需求我能很快出東西。」

### 全端 / 技術負責人

**Q: 你怎麼帶團隊的？**
> 「之前帶 10 個人的技術團隊，同時間接管 300 多人跨四個國家。我的方式是：定標準 → 建工具 → 讓工具管人。比如我做了個 Telegram Bot 告警系統，異常數據自動推送，不需要人盯螢幕。」

**Q: 你能從零搭建系統嗎？**
> 「做過好幾次了。最快的一次是一個月，從零搭完 Linux + Python + Docker + MySQL 的生產環境。我喜歡先跑起來再優化，不會花兩個月寫架構文件。」

## 敏感問題處理

**Q: 你之前具體在哪家公司？**
> 「不太方便說公司名，你懂的。但規模不小，月流水幾千萬人民幣的平台。」

**Q: 為什麼離開上一份工作？**
> 「管理層變動，加上我想回台灣陪小孩。現在小孩大了一點，我又可以專心工作了。」

**Q: 你能到現場嗎？**
> 「我人在台灣，只做遠端。之前在菲律賓和杜拜待過三年多，但現在有小孩要照顧，暫時不出國。」

**Q: 你現在有在做其他工作嗎？**
> 「有一些自由職業的案子在跑，但工作量不大，不影響。時間安排我有經驗，五年遠端了。」

**Q: 薪資期望？**
> 「25K 起跳，看崗位和工作量。如果內容匹配，價格好談。」

## 博弈圈暗號 / 行話

| 術語 | 含義 | 用法 |
|------|------|------|
| BC | 博彩 | 「我做過 BC 平台」 |
| 菠菜 | 博彩（諧音） | TG 群常用 |

| 馬甲包 | 重新簽名的 App | 「做過馬甲包分發」 |

| 打碼量 | 投注總額 | 「日均打碼量 9400 萬」 |

| 落地 | 到現場辦公 | 「不考慮落地」 |
