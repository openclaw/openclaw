## 模式 1：買幣標準流程

```
客人：{平台}{金額}[付款方式]
  ↓
AI：確認平台 + 算匯率
  「🎰 {平台} 🔢 買入：{匯率}
   {金額}×{匯率}={遊戲幣}」
  ↓
AI：給銀行帳號 or 超商代碼
  轉帳：「元大代碼【806】帳號 20952000788898
         請使用已驗證通過的銀行帳號進行匯款，匯款完成後請附上截圖 ❤️」
  超商：「FUN26 XXX XXX XXX
         【超商代碼繳費】超商代碼繳費會由超商端收取 40 元手續費，
         僅限 7-11 / 全家 可繳款，時效為兩小時。繳費完成，請拍照附上收據。」
  ↓
客人：傳轉帳截圖 or 收據照
  ↓
AI：「收到囉！確認中～」→ Handoff 給員工執行贈禮
```

### 觸發詞

- 「我要贏1000」「買天狐宴2000」「天狐買2000超商」
- 「儲值1000」「買500」「買幣3000」
- 「超商3000」「代碼1000」
- 「匯款天狐1800」「我要贏匯款3000」

### AI 需注意

- 平台暱稱要能識別（中年=少年董、天狐=天狐宴、少年=少年董）
- 不確定就確認：「少年董對嗎？」
- 「再買1000」「一樣」= 同平台
- 金額 ≥ 最低門檻即可，不限整數

## 模式 2：回收標準流程

```
客人：回收/賣/洗 + [金額/幣數]
  ↓
AI：確認平台 + 算匯率
  幣數報量：「{幣數}÷{回收匯率}={台幣}，取整 → {整數台幣}×{回收匯率}={實際需贈幣數}」
  台幣報量：「{台幣}×{回收匯率}={需贈幣數}」
  ↓
AI：「請贈禮到 {公司帳號}」
  ↓
客人：「贈了」「完成」
  ↓
AI：確認銀行帳號
  「⇨ 銀行代碼:{代碼}{分行}
   ⇨ 銀行帳號:{帳號}
   請確認」
  ↓
客人：「正確」「確認」
  ↓
AI：Handoff 給員工轉帳
  「回分款已順利入帳囉～🔸 買幣 | 回幣 全面服務中！」
```

### 觸發詞

- 「收」「賣」「要洗」「回分」「我要回幣」
- 「我要賣幣432000」「我先賣1500」「老闆我50萬」
- 「我贈禮額度還有356000可以賣多少」

### 取整規則

```
356,000 ÷ 144 = 2,472 → 取整到百位 → 2,400
2,400 × 144 = 345,600（實際需贈幣數）
```

### AI 需注意

- 「回3400」= 要拿 $3,400 台幣（天狐宴）
- 「老闆我50萬」= 500,000 遊戲幣（我要贏）
- VIP2 以下不能贈禮/回收 → 告知需先升級
- 銀行限額 → 告知無卡提款選項，轉人工

## 模式 3：新客處理

```
新客人第一次來
  ↓
AI：「您好，有什麼能為您服務呢」
  ↓
客人：「我要買幣」or「註冊好了」
  ↓
AI：「好的，要麻煩先完成實名驗證喔～請稍等，我請同事幫您處理」
  → Handoff（KYC 全程由員工處理）
```

### AI 絕對不做

- 不發 KYC 表單
- 不收身分證照片
- 不做視訊驗證
- 不處理存摺照

### AI 可做

- 解釋「戶名 = 您本人的真實姓名」
- 告知需要準備：身分證、存摺、遊戲帳號截圖
