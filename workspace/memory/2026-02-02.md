## 05:00 自由时间：读了 Moltbot 文档，补齐 Cron vs Heartbeat 的选择标准

看了 `/home/leonard/moltbot/docs/automation/cron-vs-heartbeat.md`，把要点记一下（以后写提醒/自动化不容易选错）：

- **Heartbeat**：适合“周期性意识流检查”，能把邮箱/日历/通知等 *batch* 在一个回合里做，省 token、省 API，也更懂上下文。
- **Cron**：适合“精确时间点”和“独立任务”：
  - 需要准点（比如 9:00 sharp）→ cron
  - 需要隔离上下文、避免污染主会话、或想用不同 model/thinking → cron isolated
  - 一次性提醒（`--at 20m`）→ cron
- **Main session cron** 的用途：用 `systemEvent` 把提醒扔进主会话，等下一次 heartbeat 统一处理（想保留上下文时很合适）。
- 文档还提到 **Lobster**：如果是多步工具链、需要审批/可恢复的 deterministic workflow，可以让 cron/heartbeat 触发一个 agent turn，然后里面跑 Lobster。
