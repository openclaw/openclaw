    **Create `src/telegram/telegram-e2e.live.test.ts`:**

    1. Use the `describeLive` pattern from Phase 6. Required env vars:
       - `TELEGRAM_BOT_TOKEN` (required: true) — the bot token
       - `OPENCLAW_LIVE_TELEGRAM_CHAT_ID` (required: true) — numeric chat ID for private chat with bot

    2. The test approach — direct Bot API round-trip:
       - Use `fetch` to call the Telegram Bot API directly (no grammy dependency needed in test).
       - `POST https://api.telegram.org/bot{TOKEN}/sendMessage` with `{ chat_id, text }` to simulate a user sending a message. Note: this sends AS the bot, so the test message will come from the bot itself. For a true e2e test where the bot processes and replies, we need a different approach.

    3. **Revised approach — probe + send test:**
       Since we cannot simulate an inbound user message via the Bot API alone (the API only lets the bot send, not receive as-if-from-user), the test should verify:
       - **Bot connectivity**: Call `getMe` to confirm the token is valid and the bot is reachable.
       - **Send capability**: Call `sendMessage` to the test chat to confirm the bot can deliver messages.
       - **getUpdates polling**: Call `getUpdates` to confirm the bot can receive updates (may return empty if webhook is set, which is also valid).

       This proves the Telegram integration path works end-to-end from OpenClaw's perspective: token resolves, API is reachable, messages can be sent to the target chat.

    4. Test structure:
       ```typescript
       import { expect, it } from "vitest";
       import { describeLive } from "../test-utils/live-test-helpers.js";

       const TOKEN = process.env.TELEGRAM_BOT_TOKEN?.trim() || "";
       const CHAT_ID = process.env.OPENCLAW_LIVE_TELEGRAM_CHAT_ID?.trim() || "";
       const API = `https://api.telegram.org/bot${TOKEN}`;

       const runSuite = describeLive({
         name: "telegram (live): e2e bot connectivity and send",
         envVars: [
           { name: "TELEGRAM_BOT_TOKEN", value: process.env.TELEGRAM_BOT_TOKEN, required: true },
           { name: "OPENCLAW_LIVE_TELEGRAM_CHAT_ID", value: process.env.OPENCLAW_LIVE_TELEGRAM_CHAT_ID, required: true },
         ],
       });

       runSuite("telegram (live): e2e bot connectivity and send", () => {
         it("bot token is valid — getMe succeeds", { timeout: 15_000 }, async () => {
           const res = await fetch(`${API}/getMe`);
           const json = await res.json();
           expect(json.ok).toBe(true);
           expect(json.result.is_bot).toBe(true);
           expect(json.result.username).toBeTruthy();
         });

         it("bot can send a message to the test chat", { timeout: 15_000 }, async () => {
           const marker = `[live-test] ${new Date().toISOString()}`;
           const res = await fetch(`${API}/sendMessage`, {
             method: "POST",
             headers: { "Content-Type": "application/json" },
             body: JSON.stringify({ chat_id: CHAT_ID, text: marker }),
           });
           const json = await res.json();
           expect(json.ok).toBe(true);
           expect(json.result.message_id).toBeGreaterThan(0);
           expect(json.result.text).toBe(marker);
         });
       });
       ```

    5. Set timeout to 15 seconds per test (external API calls).
