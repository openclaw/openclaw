  <name>Task 1: Create live test helper module with skip/classify/retry utilities</name>
  <files>src/test-utils/live-test-helpers.ts, src/test-utils/live-test-helpers.test.ts</files>
  <action>
Create `src/test-utils/live-test-helpers.ts` with these exports:

1. **`requireLiveKey(envVar: string, opts?: { hint?: string }): string`**
   - Reads `process.env[envVar]`. If empty/undefined, calls `it.skip()` (via Vitest) with message:
     `Skipped: ${envVar} not set — export it or add to .env`
   - If `opts.hint` is provided, append it: `Skipped: ${envVar} not set — ${hint}`
   - Returns the key value if present.
   - Implementation note: This can't literally call `it.skip()` from outside a test. Instead, return a sentinel or throw a custom `LiveSkipError` that the `describeLive` wrapper catches. Alternatively, make it a function that returns `{ key: string } | { skip: string }` and let the caller handle it.
   - Best approach: Export a `skipUnless(condition: boolean, reason: string)` that calls `describe.skip` at the suite level. For per-test key checks, throw a `LiveTestSkip` class that extends Error with a `reason` property — the custom reporter (Plan 06-02) will display these.

2. **`describeLive(opts: { name: string; envVars: Array<{ name: string; value: string | undefined; required?: boolean }> })`**
   - Returns `describe` if all required env vars are set AND the LIVE flag (`OPENCLAW_LIVE_TEST` or `LIVE`) is truthy.
   - Returns `describe.skip` otherwise.
   - Logs to console which keys are missing when skipping: `[live-skip] ${name}: missing ${missingKeys.join(', ')} — export or add to .env`
   - The skip message is yellow using `\x1b[33m...\x1b[0m` ANSI escape.

3. **`classifyLiveError(err: unknown): { type: 'auth' | 'rate-limit' | 'unavailable' | 'network' | 'logic'; message: string }`**
   - `auth`: HTTP 401, 403, "unauthorized", "forbidden", "invalid.\*key", "billing"
   - `rate-limit`: HTTP 429, "rate.limit", "too many requests", "quota"
   - `unavailable`: HTTP 502, 503, "service unavailable", "ECONNREFUSED", "ETIMEDOUT" on connect
   - `network`: "ECONNRESET", "EPIPE", "fetch failed", "abort"
   - `logic`: everything else (real test failure)
   - Return `message` as the error string (no stack trace — `String(err)` but strip anything after first newline that starts with whitespace/at).

4. **`withLiveRetry<T>(fn: () => Promise<T>, opts?: { retries?: number; baseDelayMs?: number }): Promise<T>`**
   - Default: 2 retries, 1000ms base delay, exponential backoff (1s, 2s).
   - On each catch, call `classifyLiveError`. If `rate-limit`, retry. If `auth` or `unavailable`, throw immediately (no retry). Otherwise throw.
   - Log retries: `[live-retry] attempt ${n}/${max}: ${error.message}`

Create `src/test-utils/live-test-helpers.test.ts` with unit tests:

- `classifyLiveError` correctly classifies auth (401), rate-limit (429), unavailable (503), network (ECONNREFUSED), and logic errors
- `classifyLiveError` strips stack traces from messages
- `withLiveRetry` retries on rate-limit errors up to max
- `withLiveRetry` does NOT retry on auth errors
- `describeLive` returns `describe.skip` when key is missing (mock env)
- `describeLive` logs yellow skip message when keys are missing

Use `import { describe, expect, it, vi } from "vitest"` for tests. Follow existing test patterns in the codebase.
