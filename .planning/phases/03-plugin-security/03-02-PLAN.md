<objective>
Add capability declarations to plugin manifests and enforce them via a Proxy-based API wrapper that blocks undeclared API surface access at runtime.

Purpose: Plugins only see the API surfaces they declare, preventing a compromised or malicious plugin from accessing config, media, channels, or other APIs it doesn't need (PLUG-02).
Output: `src/plugins/capabilities.ts` with capability map and proxy factory, extended manifest type, loader/registry integration.
</objective>

3. Create `src/plugins/capabilities.ts` with:

   a. **Capability map** (`PLUGIN_CAPABILITIES`): maps capability names to arrays of allowed API property names.
   ```typescript
   export const PLUGIN_CAPABILITIES = {
     // Registration capabilities
     tools: ["registerTool"],
     hooks: ["registerHook", "on"],
     channels: ["registerChannel"],
     providers: ["registerProvider"],
     http: ["registerHttpHandler", "registerHttpRoute"],
     gateway_methods: ["registerGatewayMethod"],
     cli: ["registerCli"],
     services: ["registerService"],
     commands: ["registerCommand"],
     // Access capabilities
     config_read: ["config"],
     media: ["runtime.media"],
     runtime_channel: ["runtime.channel"],
     runtime_config: ["runtime.config"],
     runtime_system: ["runtime.system"],
     runtime_state: ["runtime.state"],
     runtime_tts: ["runtime.tts"],
     runtime_tools: ["runtime.tools"],
     runtime_logging: ["runtime.logging"],
   } as const;

   export type PluginCapability = keyof typeof PLUGIN_CAPABILITIES;
   ```

   b. **`resolveEffectiveCapabilities`**: Given a manifest (with optional `capabilities`, `channels`, `providers` fields), returns a `Set<PluginCapability>`. If `capabilities` is present and non-empty, use it. Also auto-infer: if `channels` has entries, add `"channels"` capability; if `providers` has entries, add `"providers"` capability. If no `capabilities` field at all, return `null` (signals legacy/full-access mode).

   c. **`createCapabilityScopedApi`**: Given the full `OpenClawPluginApi`, a `Set<PluginCapability>`, and a `pluginId`, return a Proxy that:
   - Always allows: `id`, `name`, `version`, `description`, `source`, `logger`, `resolvePath`, `pluginConfig`
   - Allows top-level methods/properties if their name appears in the allowed set derived from capabilities
   - For `runtime` property access: return a nested Proxy that gates second-level properties (e.g., `runtime.media` requires a `runtime.media` entry in allowed set)
   - For `config`: allow only if `config` is in the allowed set (from `config_read` capability)
   - For undeclared registration methods (functions): `throw new Error(...)` with a message naming the plugin, the method, and the required capability. Also call `emitSecurityEvent` with `eventType: "plugin.capability.denied"`.
   - For undeclared property access (non-functions like `config`, `runtime` sub-properties): return `undefined` and emit security event.
   - Initially stub the function body to always throw (for RED phase).

4. Create `src/plugins/capabilities.test.ts` with test cases:

   - `config` returns `undefined` when `config_read` capability is NOT declared
   - `runtime.media` is accessible when `media` capability is declared
   - `runtime.media` returns `undefined` when `media` capability is NOT declared
   - `runtime.channel` is accessible when `runtime_channel` capability is declared
   - Security event is emitted (mock `emitSecurityEvent`) when an undeclared method is called
   - Security event is emitted when an undeclared property is accessed

   Run tests: all should FAIL initially.

**GREEN phase:**

5. Implement `resolveEffectiveCapabilities`:
   - If `manifest.capabilities` is defined and non-empty, start with `new Set(manifest.capabilities)` (cast/validate against `PluginCapability` keys; skip unknown entries with a warning)
   - If `manifest.channels` has entries, add `"channels"`
   - If `manifest.providers` has entries, add `"providers"`
   - If `manifest.capabilities` is undefined/null, return `null`
   - Return the `Set<PluginCapability>`

6. Implement `createCapabilityScopedApi`:
   - Build `allowedMethods: Set<string>` from the capabilities set by iterating `PLUGIN_CAPABILITIES`
   - Define `ALWAYS_ALLOWED` set: `["id", "name", "version", "description", "source", "logger", "resolvePath", "pluginConfig"]`
   - Build `allowedRuntimeProps: Set<string>` -- for each capability starting with `runtime_`, extract the second segment (e.g., `"runtime.media"` -> `"media"`)
   - Return `new Proxy(fullApi, { get(target, prop, receiver) { ... } })`:
     - If `prop` is symbol or `typeof prop !== "string"`, pass through
     - If in `ALWAYS_ALLOWED`, pass through
     - If `prop === "config"` and `"config"` is in `allowedMethods`, pass through; else emit event + return `undefined`
     - If `prop === "runtime"`, return a runtime sub-proxy (see below)
     - If `prop` is in `allowedMethods`, pass through
     - If target[prop] is a function: emit security event, throw error
     - Otherwise: emit security event, return `undefined`
   - Runtime sub-proxy: `new Proxy(target.runtime, { get(rtTarget, rtProp) { ... } })`:
     - If `rtProp` is symbol, pass through
     - If `String(rtProp)` is in `allowedRuntimeProps`, pass through
     - Else: emit security event, return `undefined`

7. Run tests: all should PASS.

**REFACTOR:** Clean up, run `bun run check`.

**Important implementation notes:**
- Use `Reflect.get(target, prop, receiver)` for pass-through to maintain correct `this` binding
- The Proxy must preserve the `OpenClawPluginApi` TypeScript type -- callers see the full type, enforcement is at runtime
- The `runtime` sub-proxy should handle common properties like `toString`, `valueOf`, `Symbol.toPrimitive` by passing through
