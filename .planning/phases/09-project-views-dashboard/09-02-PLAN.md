# Plan 09-02: Project Views, Dashboard Widgets & App Integration

<objective>
Build the project list view, dashboard view with 4 configurable widgets, sub-project navigation, and integrate everything into the app shell via app-render.ts and app.ts. This plan depends on Plan 09-01 which provides the navigation tab, CSS, controller, i18n, and gateway event wiring.
</objective>

<must_haves>
- Project list view shows all projects with name, status badge, task counts by column, active agent count, and owner (UI-02)
- Project dashboard renders 4 configurable widgets: Project Status, Task Counts (stacked bar), Active Agents (with pulsing dots), Recent Activity (UI-03)
- Dashboard widget list from `PROJECT.md` frontmatter `dashboard.widgets` controls which widgets render and in what order; defaults to all 4 when unconfigured (UI-04)
- Sub-projects appear as clickable mini cards at bottom of parent dashboard; clicking navigates to sub-project dashboard at `/projects/:parent/sub/:child` (UI-09)
- Views lazy-load via `createLazy` in `app-render.ts` following existing pattern
- URL routing: `/projects` (list), `/projects/:name` (dashboard), `/projects/:parent/sub/:child` (sub-project dashboard)
</must_haves>

### 3. `renderActiveAgentsWidget(queue: QueueIndex | null)`
Widget with `.card.projects-widget.stagger-3`:
- Header: `.card-title` "Active Agents"

  - `.statusDot.ok` (green pulsing dot indicator)
  - `.projects-agent-name`: agent name from `entry.metadata.agent` or `entry.metadata.claimed_by` or "Unknown"
  - `.projects-agent-task`: `"${entry.taskId}"` (task ID reference)
  - `.projects-agent-time`: relative time from `entry.metadata.claimed_at` using `formatRelativeTimestamp()`

### 4. `renderRecentActivityWidget(board: BoardIndex | null, queue: QueueIndex | null)`
Widget with `.card.projects-widget.stagger-4`:
- Header: `.card-title` "Recent Activity"

- **D-11 scope note:** The gateway RPC responses (BoardIndex, QueueIndex) do not include task file log entries. This widget shows task state changes derived from board column positions as a pragmatic approximation. Full log-entry activity requires a future gateway endpoint.
- Build activity entries from board tasks across all columns. For each task, create an entry showing its current column placement. Sort by column priority (Done last, In Progress first), take up to 10 entries:
