<objective>
Create the ProjectSyncService class that watches project markdown files for changes, debounces updates per-project, triggers incremental index regeneration, and performs full reindex on startup. Wire all new exports into the barrel.

Purpose: This is the runtime sync engine. It connects the index generator (Plan 01) to file system changes, making .index/ automatically reflect the current state of project markdown files. The service lifecycle and typed events enable Gateway (Phase 7) to subscribe and broadcast changes over WebSocket.
Output: sync-service.ts (ProjectSyncService class), sync-service.test.ts (integration tests), updated index.ts barrel
</objective>

```typescript
import { EventEmitter } from "node:events";
import fs from "node:fs/promises";
import path from "node:path";
import chokidar from "chokidar";
```

**Class: `ProjectSyncService extends EventEmitter`**

Constructor takes `projectsRoot: string` (the `~/.openclaw/projects/` path).

Private state:

- `watcher: chokidar.FSWatcher | null = null`
- `debounceTimers: Map<string, ReturnType<typeof setTimeout>>` — keyed by project name (per D-06)

**`async start(): Promise<void>`**

1. Discover all project directories by scanning `projectsRoot` for directories containing `PROJECT.md`. Also check one level deeper for sub-projects at `<project>/sub-projects/*/PROJECT.md`.
2. Full reindex: for each discovered project dir, call `generateAllIndexes(projectDir)`. Collect all returned SyncEvents and emit them. This satisfies SYNC-06 (startup reindex) and SYNC-07 (regeneratable).
3. Start chokidar watcher on `projectsRoot` with config (per D-07, following memory manager pattern):

   ```
   chokidar.watch(projectsRoot, {
     ignoreInitial: true,
     ignored: [/(^|[\/\\])\.index/, /(^|[\/\\])\.lock/, /(^|[\/\\])node_modules/],
     awaitWriteFinish: { stabilityThreshold: 200, pollInterval: 100 },
     depth: 5,
   })
   ```

   - Ignore `.index/` directory (prevents infinite loops), `.lock` files, `node_modules`.
   - `awaitWriteFinish` with 200ms stabilityThreshold (SYNC-02).
   - Bind `"add"`, `"change"` to `this.handleFileChange(filePath)`.
   - Bind `"unlink"` to `this.handleFileDelete(filePath)`.

**`async stop(): Promise<void>`**

1. Close watcher if it exists: `await this.watcher?.close()`.
2. Clear all debounce timers from the Map.
3. Set watcher to null.

**`private handleFileChange(filePath: string): void`**

1. Only process `.md` files. Ignore others.
2. Determine which project this file belongs to by resolving relative to `projectsRoot`.
3. Call `this.scheduleUpdate(projectName, filePath, "change")`.

**`private handleFileDelete(filePath: string): void`**

1. Only process `.md` files.
2. Determine project name.
3. Call `this.scheduleUpdate(projectName, filePath, "delete")`.

**`private scheduleUpdate(projectName: string, filePath: string, action: "change" | "delete"): void`**

1. Per-project debounce (D-06): clear existing timer for this project if any.
2. Set new timer at 300ms (SYNC-03):
   ```
   this.debounceTimers.set(projectName, setTimeout(() => {
     this.debounceTimers.delete(projectName);
     void this.processUpdate(projectName, filePath, action).catch(err => {
       this.emit("error", err);
     });
   }, 300));
   ```

**`private async processUpdate(projectName: string, filePath: string, action: "change" | "delete"): Promise<void>`**
Incremental update per D-03:

1. Determine file type from filename:
   - `PROJECT.md` → regenerate `project.json` only. Emit `{ type: "project:changed", project: projectName }`.
   - `queue.md` → regenerate `queue.json` only. Emit `{ type: "queue:changed", project: projectName }`.
   - `tasks/TASK-NNN.md` (change) → regenerate that task's JSON + `board.json`. Emit `{ type: "task:changed", project: projectName, taskId }`.
   - `tasks/TASK-NNN.md` (delete) → delete that task's JSON + regenerate `board.json`. Emit `{ type: "task:deleted", project: projectName, taskId }`.
2. For each regeneration, read the source markdown file, parse with appropriate parser, call the matching `generate*` function, write with `writeIndexFile`.
3. For board.json regeneration: read ALL task files in the project, parse all, get columns from project.json (read .index/project.json or re-parse PROJECT.md), call `generateBoardIndex`.
4. Wrap in try/catch. On parse failure, log warning and skip (per D-09).

**`private resolveProjectDir(filePath: string): { projectName: string; projectDir: string } | null`**
Helper to determine the project directory and name from an absolute file path. Handle both top-level projects and sub-projects under `sub-projects/`.

**`async discoverProjects(): Promise<string[]>`**
Public helper that returns all project directory paths. Useful for CLI reindex command (Phase 8).
