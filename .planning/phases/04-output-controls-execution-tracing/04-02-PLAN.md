1. Create `src/security/trace-context.ts`:

- Import `crypto` from `node:crypto`.

- Define types:
  ```typescript
  export type TraceContext = {
    traceId: string;       // 32 hex chars (W3C trace-id)
    spanId: string;        // 16 hex chars (W3C parent-id)
    parentSpanId?: string; // 16 hex chars, undefined for root spans
  };
  ```

- ID generation functions:
  - `createTraceId(): string` â€” `crypto.randomBytes(16).toString("hex")` (32 hex chars)
  - `createSpanId(): string` â€” `crypto.randomBytes(8).toString("hex")` (16 hex chars)

- Trace context factory functions:
  - `createRootTrace(): TraceContext` â€” new traceId + new spanId, no parentSpanId
  - `createChildSpan(parent: TraceContext): TraceContext` â€” same traceId, new spanId, parentSpanId = parent.spanId

- `formatTraceparent(ctx: TraceContext): string` â€” returns `"00-{traceId}-{spanId}-01"` (W3C format: version-traceId-parentId-traceFlags)

- Run-keyed storage (parallel to `runContextById` in agent-events.ts, but for trace context â€” kept separate to avoid circular imports and keep the trace module self-contained):
  ```typescript
  const traceByRunId = new Map<string, TraceContext>();

  export function setTraceContextForRun(runId: string, ctx: TraceContext): void
  export function getTraceContextForRun(runId: string): TraceContext | undefined
  export function clearTraceContextForRun(runId: string): void
  export function resetTraceContextForTests(): void
  ```

2. In `src/infra/agent-events.ts`, extend `AgentRunContext` type with an optional trace context field:
  ```typescript
  traceContext?: { traceId: string; spanId: string; parentSpanId?: string };
  ```
  This is a plain object shape (not importing TraceContext type) to avoid a circular dependency from agent-events -> security/trace-context. The trace-context module stores the canonical trace data in its own Map; this field allows callers who register run context to also pass trace info for convenience.

  In `registerAgentRunContext()`, if `context.traceContext` is provided, merge it into the stored context (same pattern as existing sessionKey/verboseLevel/isHeartbeat merging).

- `clearTraceContextForRun` removes the entry
